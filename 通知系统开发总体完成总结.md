## 五、技术亮点

### 5.1 统一地址管理

```typescript
// 所有通知中的链接都使用配置地址
export const SITE_CONFIG = {
  CRM_URL: 'https://crm.yunkes.com',
  WEBSITE_URL: 'https://yunkes.com',
  API_URL: 'https://api.yunkes.com',
  ADMIN_URL: 'https://admin.yunkes.com'
};

// 模板中自动注入
{{crmUrl}}/login → https://crm.yunkes.com/login
```

### 5.2 变量化模板

```typescript
// 模板内容
"欢迎注册{{systemName}}，您的账号是{{adminUsername}}"

// 渲染后
"欢迎注册云客CRM，您的账号是admin"
```

### 5.3 多渠道支持

```typescript
// 一次调用,自动发送到多个渠道
await notificationTemplateService.sendByTemplate('template_code', variables);
// 根据模板配置自动发送: 邮件 + 短信
```

### 5.4 防重复机制

```typescript
// 检查今天是否已发送
const logs = await AppDataSource.query(`
  SELECT id FROM notification_logs
  WHERE template_code = ? AND related_id = ? AND DATE(created_at) = ?
`, [templateCode, tenantId, today]);

if (logs.length > 0) {
  return false; // 跳过重复发送
}
```

### 5.5 错误隔离

```typescript
// 通知发送失败不影响主业务
try {
  await notificationTemplateService.sendByTemplate(...)
} catch (error) {
  console.error('发送通知失败:', error)
  // 不抛出错误,继续执行主流程
}
```

### 8.2 监控SQL

```sql
-- 今日发送统计
SELECT 
  template_code,
  COUNT(*) as total,
  SUM(CASE WHEN status = 'success' THEN 1 ELSE 0 END) as success,
  ROUND(SUM(CASE WHEN status = 'success' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) as success_rate
FROM notification_logs
WHERE DATE(created_at) = CURDATE()
GROUP BY template_code;

-- 最近失败记录
SELECT * FROM notification_logs
WHERE status = 'failed'
ORDER BY created_at DESC
LIMIT 20;
```
