## 🚀 快速开始

### 发送通知(代码中调用)

```typescript
import { notificationTemplateService } from '@/services/NotificationTemplateService';

// 发送通知
await notificationTemplateService.sendByTemplate('模板代码', {
  变量1: '值1',
  变量2: '值2'
}, {
  to: 'email@example.com',  // 可选
  priority: 'high'           // 可选
});
```

## ⏰ 定时任务

| 任务 | 执行时间 | 功能 |
|------|---------|------|
| license-expiration-check | 每天9:00 | 授权到期检查 |
| payment-reminder-check | 每天10:00 | 支付提醒检查 |

### 手动触发任务

```bash
# 授权到期检查
curl -X POST http://localhost:3000/api/v1/admin/scheduler/tasks/license-expiration-check/trigger \
  -H "Authorization: Bearer YOUR_TOKEN"

# 支付提醒检查
curl -X POST http://localhost:3000/api/v1/admin/scheduler/tasks/payment-reminder-check/trigger \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## 🔧 常用操作

### 1. 查看通知发送记录

```sql
SELECT * FROM notification_logs
ORDER BY created_at DESC
LIMIT 100;
```

### 2. 查看失败的通知

```sql
SELECT * FROM notification_logs
WHERE status = 'failed'
ORDER BY created_at DESC;
```

### 3. 统计发送成功率

```sql
SELECT 
  template_code,
  COUNT(*) as total,
  SUM(CASE WHEN status = 'success' THEN 1 ELSE 0 END) as success,
  ROUND(SUM(CASE WHEN status = 'success' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) as success_rate
FROM notification_logs
GROUP BY template_code;
```

### 4. 修改模板内容

Admin后台 → 系统设置 → 通知模板 → 选择模板 → 编辑

### 5. 测试模板

```bash
# 测试通知模板
cd backend
node test-notification-templates.js

# 测试授权提醒
node test-license-expiration-reminder.js

# 测试支付提醒
node test-payment-reminder.js
```

## 📊 监控指标

### 今日发送统计

```sql
SELECT 
  template_code,
  COUNT(*) as total,
  SUM(CASE WHEN status = 'success' THEN 1 ELSE 0 END) as success
FROM notification_logs
WHERE DATE(created_at) = CURDATE()
GROUP BY template_code;
```

### 待支付订单统计

```sql
SELECT 
  COUNT(*) as total,
  COUNT(CASE WHEN created_at < DATE_SUB(NOW(), INTERVAL 24 HOUR) THEN 1 END) as over_24h
FROM payment_orders
WHERE status = 'pending';
```

### 即将到期授权统计

```sql
SELECT 
  COUNT(*) as total,
  COUNT(CASE WHEN expire_date <= DATE_ADD(CURDATE(), INTERVAL 7 DAY) THEN 1 END) as within_7days
FROM tenants
WHERE status = 'active';
```

## 🐛 故障排查

### 问题1: 没有收到通知

检查清单:
1. 邮件/短信服务是否配置 (Admin后台 → 系统设置 → 基础配置)
2. 模板是否启用 (Admin后台 → 系统设置 → 通知模板)
3. 查看 `notification_logs` 表的错误信息
4. 检查邮件是否进入垃圾箱

### 问题2: 定时任务未执行

检查清单:
1. 后端服务是否正常运行: `pm2 status`
2. 查看后端日志: `pm2 logs backend`
3. 手动触发任务测试
4. 检查服务器时间是否正确

### 问题3: 发送失败率高

检查清单:
1. 邮件服务器配置是否正确
2. 短信服务余额是否充足
3. 查看具体错误信息
4. 检查网络连接
