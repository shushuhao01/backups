# 服务器回退 - 最简单步骤

## 问题
服务器上的代码没有正确回退，导致构建失败。

## 解决方案（复制粘贴执行）

登录服务器，复制粘贴以下命令：

```bash
# 1. 进入目录并停止服务
cd /www/wwwroot/abc789.cn/backend
pm2 stop crm-backend-api

# 2. 彻底清理并重置（这是关键！）
git clean -fdx
git fetch origin
git reset --hard origin/main

# 3. 验证版本（应该显示 c2e9b1a）
git log --oneline -1

# 4. 检查问题文件是否还存在（应该显示 "No such file"）
ls -la src/routes/public/payment.ts
ls -la src/controllers/admin/PaymentController.ts

# 5. 清除缓存并构建
rm -rf dist
rm -rf node_modules/.cache
npm run build

# 6. 如果构建成功，启动服务
pm2 start crm-backend-api
pm2 save
pm2 logs crm-backend-api --lines 50
```

## 关键命令说明

### `git clean -fdx`
- 删除所有未跟踪的文件和目录
- 包括被 .gitignore 忽略的文件
- 这会彻底清理工作区

### `git reset --hard origin/main`
- 强制重置到远程 main 分支的状态
- 丢弃所有本地修改

## 如果还是失败

说明 GitHub 仓库本身有问题，需要在本地重新处理。

请提供服务器上的输出：
```bash
cd /www/wwwroot/abc789.cn/backend
git log --oneline -3
git status
git ls-tree -r HEAD --name-only | grep payment
```
