# 客户管理模块优化 - 实施方案

## 需求概述

### 需求1: 手机号重复验证应该在租户级别
**当前问题**: 客户手机号重复验证是全局的，不同租户不能有相同的客户号码
**期望行为**: 手机号重复验证应该只在当前租户内检查，不同租户可以有相同的客户号码

### 需求2: 支持境外号码和固定电话
**当前问题**: 手机号输入框强制要求11位数字验证
**期望行为**: 
- 在手机号输入框右侧添加图标按钮
- 点击后支持选择"境外国家"和"固定电话"
- 境外号码和固定电话不受11位数限制

## 技术分析

### 1. 手机号重复验证的租户隔离

**当前实现**:
- 后端路由: `backend/src/routes/customers.ts` 的 `/check-exists` 端点
- 使用 `getTenantRepo(Customer)` 获取仓库
- `getTenantRepo` 已经自动添加了 `tenant_id` 过滤

**验证结果**: ✅ 后端已经实现租户级别的隔离
- `getTenantRepo` 函数会自动在查询中添加 `tenant_id` 过滤条件
- 不同租户的客户数据已经完全隔离
- 手机号重复验证已经是租户级别的

**结论**: 需求1已经满足，无需修改！

### 2. 境外号码和固定电话支持

**需要修改的文件**:
1. `src/views/Customer/Add.vue` - 新增客户表单
2. `src/views/Customer/Edit.vue` - 编辑客户表单（如果存在）
3. 后端验证逻辑（如果有）

**实现方案**:
1. 在手机号输入框右侧添加下拉选择按钮
2. 选项包括:
   - 中国大陆 (默认，11位验证)
   - 境外号码 (不限制位数)
   - 固定电话 (不限制位数)
3. 根据选择动态调整验证规则
4. 保存时记录号码类型
