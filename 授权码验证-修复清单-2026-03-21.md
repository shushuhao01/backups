### 2. 私有部署首次激活流程缺失
**问题**: 私有授权码无法创建本地租户记录  
**修复**:
- 验证 `licenses` 表后自动在 `tenants` 表创建记录
- 生成 P 前缀租户编码 (如 P260321ABCD)
- 关联私有客户信息

**影响文件**:
- `backend/src/routes/tenantLicense.ts` (L60-L95)

**代码片段**:
```typescript
if (!tenant) {
  // 首次激活：创建本地租户记录
  const tenantCode = `P${year}${month}${day}${random}`
  await AppDataSource.query(
    `INSERT INTO tenants (id, name, code, license_key, ...)
     VALUES (?, ?, ?, ?, ...)`,
    [tenantId, companyName, tenantCode, licenseKey, ...]
  )
}
```

### 3. 部署模式需要手动配置环境变量
**问题**: 必须在 `.env` 中配置 `DEPLOY_MODE`  
**修复**:
- 激活时后端自动返回 `deployType`
- 前端自动保存到 localStorage
- 无需手动配置环境变量

**优先级**:
```
1. localStorage.crm_deploy_mode (激活时自动写入)
2. VITE_DEPLOY_MODE 环境变量 (可选)
3. 默认 'saas'
```

### 4. 私有部署无法自动识别已激活租户

**影响文件**:

### 5. 租户编码格式不统一
**问题**: 私有和SaaS租户编码无法区分  
**修复**:
- 私有部署: P 前缀 (如 P260321ABCD)
- SaaS租户: T 前缀 (如 T20260XXXXX)
- 便于快速识别租户类型

### 6. 授权码验证逻辑分散
**问题**: 私有和SaaS验证逻辑混在一起  
**修复**:
- 统一在 `/verify` API 中处理
- 根据前缀自动路由到不同逻辑
- 代码结构更清晰

**代码结构**:
```typescript
router.post('/verify', async (req, res) => {
  if (licenseKey.startsWith('PRIVATE-')) {

```
