## 一、系统架构总览

| 层级 | 技术栈 |
|------|--------|
| **前端** | Vue 3.4 + TypeScript 5.3 + Element Plus 2.4 + Pinia 2.1 + Vite 5 |
| **后端** | Node.js + Express + TypeORM + 原生SQL混合 |
| **数据库** | MySQL (`crm_local`) |
| **端口** | 前端 5174，后端 3000 (共用CRM后端) |
| **认证** | JWT (`admin_token` in localStorage) + bcrypt 密码加密 |
| **API前缀** | `/api/v1/admin/` (前端通过Vite代理到localhost:3000) |

## 四、全局性问题汇总

### A. API响应格式不统一 🔴

| 路由文件 | 响应格式 | 备注 |
|----------|----------|------|
| auth, dashboard, licenses, tenants, versions, payment, admin-users | `{success: true/false, data, message}` | 标准格式 |
| **packages.ts** | `{code: 0, data, message}` | ⚠️ 不一致 |
| modules (Controller) | 需确认 | 通过Controller返回 |

**影响**: 前端 `api/packages.ts` 处理了 `res.code === 0`，但如果通过 `adminApi` 统一调用可能出问题。

### B. 前端API调用方式不统一

| 页面 | 调用方式 |
|------|----------|
| private-customers, Dashboard, AdminUsers, Api | `adminApi.xxx()` (通过admin.ts) |
| tenant-customers, modules, settings/Basic | `request.get/post()` (直接用axios) |
| Packages | `getPackages()` (独立packages.ts) |

**建议**: 统一使用 `adminApi` 封装层。

### C. 文件上传未实现 🔴

以下位置都是 `action="#" :auto-upload="false"`，只做了本地预览：

| 页面 | 上传内容 |
|------|----------|
| `modules/Config.vue` | 系统Logo、联系二维码 |
| `settings/Basic.vue` | 后台Logo、登录背景图 |
| `versions/Upload.vue` | 版本发布文件(.zip/.tar.gz) |
| `payment/Config.vue` | 微信支付证书(.pem) |

### D. 侧边栏菜单与路由不匹配

| 问题 | 详情 |
|------|------|
| **通知模板未在菜单显示** | 路由 `/settings/notification-templates` 已注册，但 `MainLayout.vue` 侧边栏settings菜单下只有"管理员账号"和"基础配置" |
| **支付报表未注册路由** | `payment/Reports.vue` 文件存在但路由中payment下只有list和config |

### E. 详情页编辑功能缺失

`private-customers/Detail.vue` 和 `tenant-customers/Detail.vue` 的"编辑"按钮都只是跳转到列表页，没有在详情页实现编辑。

## 六、认证与安全机制

### 登录流程
1. 前端 `Login.vue` 提交 username/password
2. 后端 `auth.ts` → bcrypt比较密码 → 生成JWT token（含adminId/username/role）
3. Token存储在 `localStorage('admin_token')`
4. 所有API请求通过 `request.ts` 拦截器自动附加 `Authorization: Bearer <token>`

### 中间件
- `adminAuthMiddleware`: 验证JWT token，跳过 `/auth/login` 路径
- 公开接口: `/api/v1/admin/verify/*` 和 `/api/v1/admin/public/*` 不需认证

### 权限
- 角色: `super_admin` / `admin` / `operator`
- 当前仅有1个super_admin用户
- 前端未实现基于角色的菜单/按钮权限控制
