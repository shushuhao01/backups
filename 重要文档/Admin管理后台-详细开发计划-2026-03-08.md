#### 任务 1.1：版本列表页面 - 对接真实 API

**涉及文件**：`admin/src/views/versions/List.vue`

**后端已有 API**：
- `GET /api/v1/admin/versions` → 返回 `{success, data: {list, total, page, pageSize}}`
- `DELETE /api/v1/admin/versions/:id` → 删除版本
- `POST /api/v1/admin/versions/:id/publish` → 发布版本
- `POST /api/v1/admin/versions/:id/deprecate` → 废弃版本

**具体改动**：
```
1. 引入 adminApi（或 request）
2. 删除 tableData 中的 3 条硬编码静态数据
3. 重写 fetchData() 方法：
   - 调用 request.get('/versions', { params: { page, pageSize, status } })
   - 解析 res.data.list 赋值给 tableData
   - 解析 res.data.total 赋值给 total
4. 重写 handleDelete：
   - 调用 request.delete(`/versions/${row.id}`)
   - 成功后 fetchData() 刷新列表
5. 新增 handlePublish 方法：
   - 调用 request.post(`/versions/${row.id}/publish`)
6. 新增 handleDeprecate 方法：
   - 调用 request.post(`/versions/${row.id}/deprecate`)
7. 操作列新增"发布"、"废弃"按钮（根据 status 条件显示）
8. 修复 handleView：跳转到详情或弹出详情对话框
9. 修复 handleDownload：使用 row.downloadUrl 实际下载
```

**表格字段映射**（后端 → 前端）：
| 后端字段 | 前端显示 | 说明 |
|---------|---------|------|
| version | 版本号 | v前缀显示 |
| releaseType | 类型 | major/minor/patch → 正式版/功能更新/补丁 |
| fileSize | 文件大小 | 格式化为 MB |
| downloadCount (需新增) | 下载次数 | 后端暂无此字段，先显示 0 |
| status | 状态 | draft/published/deprecated |
| createdAt | 创建时间 | |
| publishedAt | 发布时间 | |

#### 任务 1.6：支付报表 - 后端新增 reports 和 export 接口

**涉及文件**：`backend/src/routes/admin/payment.ts`

**前端 Reports.vue 已调用但后端缺失的接口**：
- `GET /payment/reports` → 时间序列数据 + 按支付方式/套餐/客户类型分组
- `GET /payment/export` → CSV 导出

**需新增接口**：

```typescript
// GET /payment/reports
// 参数：startDate, endDate, groupBy(day/month/year), payType, customerType
// 返回：{
//   success: true,
//   data: {
//     timeSeriesData: [{date, amount, count}],
//     byPayType: [{payType, amount, count}],
//     byPackage: [{packageName, amount, count}],
//     byCustomerType: [{customerType, amount, count}]
//   }
// }

// GET /payment/export
// 参数：startDate, endDate, payType, customerType
// 返回：CSV 文件流
```

**实现要点**：
```sql
-- timeSeriesData (按日/月/年分组)
SELECT DATE_FORMAT(created_at, '%Y-%m-%d') as date,
       SUM(amount) as amount, COUNT(*) as count
FROM payment_orders WHERE status = 'paid' ${dateWhere}
GROUP BY date ORDER BY date

-- byPayType
SELECT pay_type as payType, SUM(amount) as amount, COUNT(*) as count
FROM payment_orders WHERE status = 'paid' ${dateWhere}
GROUP BY pay_type

-- byPackage
SELECT package_name as packageName, SUM(amount) as amount, COUNT(*) as count
FROM payment_orders WHERE status = 'paid' ${dateWhere}
GROUP BY package_name ORDER BY amount DESC

-- byCustomerType（根据 tenant_name 判断）
-- 需要 LEFT JOIN tenants 表或根据业务逻辑判断
```

#### 任务 2.1：套餐管理 API 响应格式统一

**涉及文件**：`backend/src/routes/admin/packages.ts`

**问题**：所有其他后端 API 返回 `{success: true, data: ...}`，唯独 packages 返回 `{code: 0, data: ...}`，导致前端需要特殊处理。

**改动范围**：
```
将 packages.ts 中所有响应格式从：
  res.json({ code: 0, data: result })
  res.json({ code: 0, message: '创建成功' })
  res.status(400).json({ code: 400, message: '...' })
  res.status(500).json({ code: 500, message: '...' })
  
统一修改为：
  res.json({ success: true, data: result })
  res.json({ success: true, message: '创建成功' })
  res.status(400).json({ success: false, message: '...' })
  res.status(500).json({ success: false, message: '...' })
```

**同时检查前端**：`admin/src/api/packages.ts` 和 `admin/src/views/tenant-customers/Packages.vue` 中的响应判断条件是否需要同步修改。

## 五、技术注意事项

### 🚫 开发红线（务必遵守）

1. **不动 CRM 前端代码**：`src/` 目录下的文件不能修改
2. **不动 Website 代码**：`website/` 目录下的文件不能修改
3. **不动 crmAPP 代码**：`crmAPP/` 目录下的文件不能修改
4. **后端改动需向后兼容**：不能破坏现有 CRM 前端调用的 API
5. **避免代码错误**：每次改动后必须编译检查，不允许出现：
   - TypeScript 类型错误
   - 重复声明
   - 未闭合标签
   - import 缺失
   - 图标组件不存在

### 📋 编码规范

1. **API 响应格式**：统一使用 `{success: boolean, data?: any, message?: string}`
2. **前端 API 调用**：优先使用 `admin/src/api/request.ts` 封装的 request 实例
3. **错误处理**：所有 API 调用必须 try-catch
4. **加载状态**：所有列表页必须有 loading 状态
5. **空状态**：列表无数据时显示空状态提示

### 🗄️ 数据库相关

**可能需要新增/修改的表**：

| 表名 | 操作 | 说明 |
|------|------|------|
| versions | 已存在 | 确认字段完整性 |
| api_configs | 修改 | 可能需加 call_count 等统计字段 |
| admin_operation_logs | 新建 | 操作日志表（P3） |
| payment_orders | 已存在 | reports 查询使用 |

### 🔑 环境变量

确保以下环境配置正确：
```
ENABLE_ADMIN=true
数据库：crm_local
后端端口：3000
Admin前端端口：5174
Admin前端代理：/api/v1/admin → http://localhost:3000/api/v1/admin
```

## 七、风险评估
