# 模块禁用缓存清除指南

## 当前状态

数据库显示已禁用2个模块：
- ❌ 物流管理 (logistics_management)
- ❌ 资料管理 (data_management)

但CRM前端还能看到这些模块，原因是**5分钟缓存**。

## 最简单的解决方法（推荐）

### 方法1：强制刷新（最快）
在CRM前端页面（http://localhost:5173）：
1. 按 `Ctrl + Shift + R`（Windows）
2. 或 `Ctrl + F5`

这会清除浏览器缓存并重新加载页面。

### 方法2：清除缓存（最彻底）
1. 打开CRM前端（http://localhost:5173）
2. 按 `F12` 打开开发者工具
3. 在控制台（Console）执行：
```javascript
// 清除缓存并刷新
localStorage.clear();
sessionStorage.clear();
location.reload();
```

### 方法3：等待5分钟
什么都不做，等待5分钟后缓存自动过期，然后刷新页面。

## 验证结果

清除缓存后，CRM前端左侧菜单应该：

✅ **显示的模块（8个）**：
- 数据看板
- 订单管理
- 客户管理
- 财务管理
- 售后管理
- 通话管理
- 系统管理
- 业绩统计
- 商品管理

❌ **不显示的模块（2个）**：
- 物流管理
- 资料管理

## 如何查看控制台日志

1. 按 `F12` 打开开发者工具
2. 点击 `Console`（控制台）标签
3. 刷新页面
4. 查找以下日志：
```
[ModuleStatus] 成功更新模块状态: ['dashboard', 'customer', 'order', 'finance', 'service', 'serviceManagement', 'system', 'performance', 'product']
```

注意：列表中不应该包含 `logistics` 和 `data`。

## 缓存机制说明

- **缓存时间**：5分钟
- **缓存位置**：前端内存（不是localStorage）
- **更新时机**：
  - 首次加载页面
  - 缓存过期后（5分钟）
  - 强制刷新（Ctrl+Shift+R）

## 如果还是不生效

### 1. 检查API响应
在控制台查看：
```
[ModuleStatus] API响应完整对象: ...
```

### 2. 手动调用API
在控制台执行：
```javascript
fetch('http://localhost:3000/api/v1/system/modules/status', {
  headers: {
    'Authorization': 'Bearer ' + localStorage.getItem('token')
  }
})
.then(r => r.json())
.then(d => console.log('API返回:', d))
```

### 3. 检查后端服务
确认后端服务正在运行：
```bash
# 检查后端进程
netstat -ano | findstr :3000
```

## 注意事项

1. 每次在Admin后台修改模块状态后，需要在CRM前端清除缓存
2. 或者等待5分钟让缓存自动过期
3. 模块禁用后，所有用户（包括管理员）都看不到该模块
4. 只有"系统管理"模块不能禁用（is_system=1）
