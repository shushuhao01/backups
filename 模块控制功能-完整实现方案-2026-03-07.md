# 模块控制功能 - 完整实现方案

## 完成时间
2026-03-07

## 设计原则

### 1. 防御性编程
- 所有可能失败的地方都有容错处理
- API调用失败时有降级策略
- 数据验证完整

### 2. 降级策略
- API失败 → 使用默认模块列表
- 数据异常 → 使用缓存或默认值
- 保证系统始终可用

### 3. 清晰的日志
- 每个关键步骤都有日志输出
- 方便排查问题
- 便于调试

### 4. 缓存机制
- 5分钟缓存时间
- 减少API调用
- 提升性能

## 核心逻辑

### 访问控制流程

```
用户访问功能
    ↓
┌─────────────────────────────────┐
│  第一关: 模块检查(功能级)        │
│  该模块是否启用?                 │
│  - YES → 继续                    │
│  - NO  → 拒绝(所有用户)          │
└──────────────┬──────────────────┘
               ↓
┌─────────────────────────────────┐
│  第二关: 权限检查(用户级)        │
│  该用户是否有权限?               │
│  - YES → 允许访问                │
│  - NO  → 拒绝(该用户)            │
└─────────────────────────────────┘
```

### 模块控制 vs 权限控制

| 维度 | 模块控制 | 权限控制 |
|------|---------|---------|
| 作用范围 | 全局(所有用户) | 用户级(特定角色/用户) |
| 控制对象 | 功能模块 | 功能权限 |
| 控制者 | Admin后台管理员 | 系统管理员 |
| 优先级 | 最高 | 次之 |
| 示例 | "财务模块被禁用" | "销售员无财务权限" |

## 技术实现

### 1. 后端API

**接口**: `GET /api/v1/system/modules/status`

**功能**: 返回启用的模块列表

**实现位置**: `backend/src/routes/system.ts`

```typescript
router.get('/modules/status', authenticateToken, async (_req: Request, res: Response) => {
  try {
    const moduleRepository = AppDataSource.getRepository(Module);

    // 获取所有启用的模块
    const enabledModules = await moduleRepository.find({
      where: { status: 'enabled' },
      select: ['code']
    });

    // 模块代码映射
    const moduleMapping: Record<string, string> = {
      'order_management': 'order',
      'customer_management': 'customer',
      'finance_management': 'finance',
      'logistics_management': 'logistics',
      'aftersales_management': 'service',
      'call_management': 'serviceManagement',
      'data_management': 'data',
      'performance_management': 'performance',
      'product_management': 'product',
      'system_management': 'system'
    };

    // 转换为前端菜单ID
    const menuIds = enabledModules
      .map(m => moduleMapping[m.code])
      .filter(id => id !== undefined);

    // 始终包含dashboard
    if (!menuIds.includes('dashboard')) {
      menuIds.unshift('dashboard');
    }

    res.json({
      success: true,
      data: {
        enabledModules: menuIds
      }
    });
  } catch (error: any) {
    console.error('[System] 获取模块状态失败:', error);
    res.status(500).json({
      success: false,
      message: '获取模块状态失败',
      error: error.message
    });
  }
});
```

### 2. 前端服务

**服务**: `moduleStatusService`

**功能**: 获取和缓存模块状态

**实现位置**: `src/services/moduleStatusService.ts`

**关键特性**:
- 5分钟缓存
- 防止并发请求
- API失败时使用默认列表
- 详细日志输出

```typescript
class ModuleStatusService {
  private enabledModules: Set<string> = new Set()
  private lastFetchTime = 0
  private readonly CACHE_DURATION = 5 * 60 * 1000
  private isInitialized = false
  private isFetching = false

  async getEnabledModules(): Promise<string[]> {
    // 缓存有效,直接返回
    if (this.isInitialized && (now - this.lastFetchTime) < this.CACHE_DURATION) {
      return Array.from(this.enabledModules)
    }

    // 防止并发请求
    if (this.isFetching) {
      await this.waitForFetch()
      return Array.from(this.enabledModules)
    }

    try {
      // 调用API
      const response = await request.get('/system/modules/status')
      
      if (response && response.success && response.data?.enabledModules) {
        this.enabledModules = new Set(response.data.enabledModules)
        this.lastFetchTime = now
        this.isInitialized = true
        return Array.from(this.enabledModules)
      }
    } catch (error) {
      console.error('获取模块状态失败:', error)
      
      // 使用旧缓存或默认列表
      if (this.isInitialized && this.enabledModules.size > 0) {
        return Array.from(this.enabledModules)
      }
      
      return this.useDefaultModules()
    }
  }

  private useDefaultModules(): string[] {
    // 返回所有模块,保证系统可用
    const defaultModules = [
      'dashboard', 'customer', 'order', 'finance', 'logistics',
      'service', 'data', 'performance', 'product', 'serviceManagement', 'system'
    ]
    this.enabledModules = new Set(defaultModules)
    this.isInitialized = true
    return defaultModules
  }
}
```

### 3. 菜单过滤

**功能**: 根据模块状态和用户权限过滤菜单

**实现位置**: `src/utils/menu.ts`

```typescript
export async function getUserAccessibleMenus(menuItems: MenuItem[]): Promise<MenuItem[]> {
  const userStore = useUserStore()

  if (!userStore.currentUser) {
    return []
  }

  // 第一步: 模块过滤
  let enabledModules: string[] = []
  try {
    enabledModules = await moduleStatusService.getEnabledModules()
  } catch (error) {
    // 失败时使用默认列表
    enabledModules = ['dashboard', 'customer', 'order', ...]
  }

  const filterByModuleStatus = (items: MenuItem[]): MenuItem[] => {
    return items.filter(item => {
      // dashboard始终显示
      if (item.id === 'dashboard') {
        return true
      }

      // 检查模块是否启用
      if (item.id && !enabledModules.includes(item.id)) {
        return false
      }

      // 递归过滤子菜单
      if (item.children && item.children.length > 0) {
        item.children = filterByModuleStatus(item.children)
        return item.children.length > 0
      }

      return true
    })
  }

  let filteredByModule = filterByModuleStatus([...menuItems])

  // 第二步: 权限过滤
  const userRole = userStore.currentUser.role
  const userPermissions = userStore.permissions

  if (userRole === 'super_admin' || userRole === 'admin' || userPermissions.includes('*')) {
    return filteredByModule.filter(item => !item.hidden)
  }

  return filterMenuItems(filteredByModule, userRole, userPermissions)
}
```

## 容错机制

### 1. API调用失败

**场景**: 后端服务未启动、网络问题、权限问题

**处理**:
```typescript
try {
  const response = await request.get('/system/modules/status')
  // 处理响应
} catch (error) {
  console.error('API调用失败:', error)
  
  // 策略1: 使用旧缓存
  if (this.enabledModules.size > 0) {
    return Array.from(this.enabledModules)
  }
  
  // 策略2: 使用默认列表
  return DEFAULT_MODULES
}
```

### 2. 数据格式异常

**场景**: API返回数据格式不正确

**处理**:
```typescript
if (response && response.success && response.data?.enabledModules) {
  const modules = response.data.enabledModules
  
  // 验证数据
  if (Array.isArray(modules) && modules.length > 0) {
    return modules
  } else {
    console.warn('模块列表为空,使用默认配置')
    return DEFAULT_MODULES
  }
} else {
  console.warn('响应格式不正确,使用默认配置')
  return DEFAULT_MODULES
}
```

### 3. 并发请求

**场景**: 多个组件同时请求模块状态

**处理**:
```typescript
if (this.isFetching) {
```

## 故障排查

### 问题1: 菜单全部消失

**原因**: API调用失败且没有使用默认列表

**解决**: 检查`moduleStatusService.ts`中的`useDefaultModules`方法

### 问题2: 禁用模块后菜单仍显示

**原因**: 缓存未刷新

**解决**: 
- 等待5分钟缓存过期
- 或手动刷新: `moduleStatusService.refresh()`
- 或清除缓存: `moduleStatusService.clearCache()`

### 问题3: 系统管理模块可以禁用

**原因**: `is_system`字段设置错误

**解决**: 
```bash
cd backend
node fix-modules-system-flag.js
```
