### 3. 配置数据结构

**TenantSettings实体字段**:
- `id`: 配置ID（UUID）
- `tenantId`: 租户ID
- `settingKey`: 配置键
- `settingValue`: 配置值（文本存储）
- `settingType`: 配置类型（string/number/boolean/json/array）
- `description`: 配置描述
- `createdAt`: 创建时间
- `updatedAt`: 更新时间

**支持的配置类型**:
- `string`: 字符串
- `number`: 数字
- `boolean`: 布尔值
- `json`: JSON对象
- `array`: 数组

## 技术要点

### 1. 配置值的序列化和反序列化

```typescript
// TenantSettings实体提供了getValue()和setValue()方法
// 自动根据settingType进行序列化和反序列化

// 获取配置值
const value = setting.getValue(); // 自动解析JSON

// 设置配置值
setting.setValue({ key: 'value' }); // 自动序列化为JSON
```

### 2. 配置类型自动检测

```typescript
// 根据值的类型自动设置settingType
if (typeof value === 'object') {
  setting.settingType = 'json';
}
```

### 3. 批量配置更新

```typescript
// 支持一次更新多个配置项
{
  "settings": {
    "theme": {
      "value": { "primaryColor": "#1890ff" },
      "description": "主题配置"
    },
    "features": {
      "value": { "enableCustomerManagement": true },
      "description": "功能开关"
    }
  }
}
```

### 4. 路由合并

```typescript
// 将配置路由合并到租户路由中
// 避免Express路由参数传递问题
router.get('/:id/settings', getTenantSettings);
router.put('/:id/settings', updateTenantSettings);
```

## API使用示例

### 1. 获取租户配置

```bash
GET /api/v1/admin/tenants/:id/settings
Authorization: Bearer <admin_token>
```

**响应**:
```json
{
  "success": true,
  "data": {
    "tenantId": "tenant-id",
    "tenantName": "租户名称",
    "settings": {
      "theme": {
        "value": {
          "primaryColor": "#1890ff",
          "mode": "light"
        },
        "description": "主题配置"
      },
      "features": {
        "value": {
          "enableCustomerManagement": true,
          "enableOrderManagement": true
        },
        "description": "功能开关"
      }
    }
  }
}
```

### 2. 更新租户配置

```bash
PUT /api/v1/admin/tenants/:id/settings
Authorization: Bearer <admin_token>
Content-Type: application/json

{
  "settings": {
    "theme": {
      "value": {
        "primaryColor": "#ff0000",
        "mode": "dark"
      },
      "description": "主题配置"
    }
  }
}
```

### 3. 重置租户配置

```bash
POST /api/v1/admin/tenants/:id/settings/reset
Authorization: Bearer <admin_token>
```

**响应**:
```json
{
  "success": true,
  "message": "租户配置已重置为默认值",
  "data": {
    "tenantId": "tenant-id",
    "settingsCount": 3
  }
}
```

## 遇到的问题和解决方案

### 问题1: 路由参数传递

**问题描述**: 
使用`router.use('/tenants/:id/settings', tenantSettingsRouter)`时，`:id`参数无法传递到子路由。

**解决方案**:
将配置路由直接合并到租户路由文件中，避免路由嵌套问题。

### 问题2: 字段名称不匹配

**问题描述**:
控制器使用`configKey`和`configValue`，但实体使用`settingKey`和`settingValue`。

**解决方案**:
统一使用实体定义的字段名称，修改控制器代码。
