### 2. 修改UserController的login方法

**文件**：`backend/src/controllers/UserController.ts`

**修改流程**：

#### 步骤1：提取请求参数
```typescript
const { username, password, license_key } = req.body;
```

#### 步骤2：根据部署模式处理
```typescript
import { deployConfig } from '../config/deploy';
import { Tenant } from '../entities/Tenant';

let tenantId: string | undefined;
let tenantInfo: any;

if (deployConfig.isSaaS()) {
  // SaaS模式：必须提供license_key
  if (!license_key) {
    throw new BusinessError('SaaS模式下必须提供授权码', 'LICENSE_KEY_REQUIRED');
  }

  // 验证license_key，查询租户
  const tenantRepository = AppDataSource.getRepository(Tenant);
  const tenant = await tenantRepository.findOne({
    where: { licenseKey: license_key }
  });

  if (!tenant) {
    throw new BusinessError('授权码无效', 'INVALID_LICENSE_KEY');
  }

  // 检查租户状态
  if (tenant.status !== 'active') {
    throw new BusinessError('租户已被禁用，请联系管理员', 'TENANT_DISABLED');
  }

  // 检查授权状态
  if (tenant.licenseStatus !== 'active') {
    const statusMessages: Record<string, string> = {
      'pending': '租户授权待激活，请联系管理员',
      'suspended': '租户授权已暂停，请联系管理员',
      'expired': '租户授权已过期，请联系管理员续费'
    };
    throw new BusinessError(
      statusMessages[tenant.licenseStatus] || '租户授权状态异常',
      'TENANT_LICENSE_INVALID'
    );
  }

  // 检查过期时间
  if (tenant.expireDate && new Date(tenant.expireDate) < new Date()) {
    throw new BusinessError('租户已过期，请联系管理员续费', 'TENANT_EXPIRED');
  }

  tenantId = tenant.id;
  tenantInfo = tenant;
} else {
  // 私有部署模式：不需要license_key
  tenantId = undefined;
  tenantInfo = null;
}
```

#### 步骤3：查找用户（添加租户过滤）

## 🔍 关键点

### 1. 部署模式判断

使用`deployConfig.isSaaS()`判断当前部署模式：
- SaaS模式：必须提供license_key，验证租户
- 私有模式：不需要license_key，直接验证用户

### 2. 租户验证

SaaS模式下需要验证：
- ✅ license_key是否有效
- ✅ 租户状态是否为active
- ✅ 授权状态是否为active
- ✅ 是否已过期

### 3. 用户查询

查询用户时需要添加租户过滤：
- SaaS模式：`WHERE username = ? AND tenant_id = ?`
- 私有模式：`WHERE username = ? AND tenant_id IS NULL`

### 4. JWT Token

JWT Token需要包含tenantId（SaaS模式）：
```typescript
{
  userId: 'xxx',
  username: 'xxx',
  role: 'xxx',
  departmentId: 'xxx',
  tenantId: 'xxx'  // SaaS模式有值，私有模式为undefined
}
```

### 5. 错误处理

需要处理的错误场景：
- license_key缺失（SaaS模式）
- license_key无效
- 租户已禁用
- 租户授权状态异常
- 租户已过期
- 用户名或密码错误
- 账户已锁定
- 账户已禁用
- IP未授权

## 📊 测试场景

### SaaS模式测试

1. **正常登录**
   - 提供有效的license_key
   - 提供正确的用户名和密码
   - 预期：登录成功，返回tenantId

2. **缺少license_key**
   - 不提供license_key
   - 预期：返回错误"SaaS模式下必须提供授权码"

3. **无效license_key**
   - 提供无效的license_key
   - 预期：返回错误"授权码无效"

4. **租户已禁用**
   - 提供已禁用租户的license_key
   - 预期：返回错误"租户已被禁用"

5. **租户已过期**
   - 提供已过期租户的license_key
   - 预期：返回错误"租户已过期"

6. **用户不属于该租户**
   - 提供租户A的license_key
   - 提供租户B的用户名
   - 预期：返回错误"用户名或密码错误"

### 私有模式测试

1. **正常登录**
   - 不提供license_key
   - 提供正确的用户名和密码
   - 预期：登录成功，tenantId为null

2. **提供license_key（应被忽略）**
   - 提供license_key
   - 提供正确的用户名和密码
   - 预期：登录成功，license_key被忽略
