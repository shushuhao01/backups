## 📊 API 使用示例

### 1. 查询租户操作日志
```bash
curl -X GET \
  "http://localhost:3000/api/v1/admin/tenants/TENANT_ID/logs?page=1&pageSize=20" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

**响应示例**：
```json
{
  "success": true,
  "data": {
    "logs": [
      {
        "id": "log-001",
        "tenantId": "tenant-001",
        "action": "create",
        "operator": "admin",
        "operatorId": "admin-001",
        "details": {
          "name": "测试租户",
          "code": "TEST001",
          "maxUsers": 20
        },
        "ipAddress": "127.0.0.1",
        "userAgent": "Mozilla/5.0...",
        "createdAt": "2024-03-03T10:00:00.000Z"
      }
    ],
    "pagination": {
      "total": 10,
      "page": 1,
      "pageSize": 20,
      "totalPages": 1
    }
  }
}
```

```bash

  "http://localhost:3000/api/v1/admin/logs?action=create&page=1&pageSize=20" \
```

```bash

  "http://localhost:3000/api/v1/admin/logs/stats?tenantId=TENANT_ID" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

**响应示例**：
```json

  "data": [
    { "action": "create", "count": 5 },
    { "action": "update", "count": 12 },
    { "action": "suspend", "count": 2 },
```

## ⚠️ 注意事项

### 1. 日志存储
- 日志会持续增长，建议定期归档
- 可以考虑添加日志清理策略
- 重要日志建议永久保留

### 2. 性能考虑
- 日志记录是异步的，不影响主流程
- 使用索引优化查询性能
- 大量日志查询时注意分页

### 3. 隐私保护
- 敏感信息（如授权码）只记录部分内容
- IP 地址和 User-Agent 可能包含隐私信息
- 遵守数据保护法规

### 4. 审计要求
- 日志不可修改和删除
- 记录完整的操作链
- 支持审计追溯
