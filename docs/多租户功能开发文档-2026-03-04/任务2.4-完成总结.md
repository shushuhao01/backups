## 🔍 BaseRepository的优势

### 1. 自动租户过滤

所有查询自动添加tenant_id过滤：
```typescript
// 你写的代码
await messageRepo.find({ where: { isRead: 0 } });

// 实际执行的SQL（SaaS模式）
SELECT * FROM system_messages 
WHERE is_read = 0 AND tenant_id = 'current-tenant-id';

// 实际执行的SQL（私有模式）
SELECT * FROM system_messages 
WHERE is_read = 0 AND tenant_id IS NULL;
```

### 2. 自动设置tenant_id

所有保存操作自动设置tenant_id：
```typescript
// 你写的代码
await messageRepo.save({ title: '新消息', content: '内容' });

// BaseRepository自动添加
{
  title: '新消息',
  content: '内容',
  tenantId: 'current-tenant-id'  // 自动添加
}
```

私有部署模式完全兼容：
- 查询：WHERE tenant_id IS NULL

## 🎓 技术要点

### 1. BaseRepository的save方法

BaseRepository的save方法支持单个和批量保存：
```typescript
// 单个保存
async save(entity: DeepPartial<Entity>): Promise<Entity>;

// 批量保存
async save(entities: DeepPartial<Entity>[]): Promise<Entity[]>;
```

使用方式：
```typescript
// 单个
const message = await messageRepo.save({ title: '消息' });

// 批量
const messages = await messageRepo.save([
  { title: '消息1' },
  { title: '消息2' }
]);
```

### 2. 不需要create方法

BaseRepository的save方法可以直接接受普通对象：
```typescript
// ❌ 旧方式（需要create）
const entity = repo.create(data);
await repo.save(entity);

// ✅ 新方式（直接save）
await repo.save(data);
```

### 3. 租户ID自动注入

BaseRepository在save时自动注入tenantId：
```typescript
protected setTenantId(entity: DeepPartial<Entity>): void {
  if (deployConfig.isSaaS()) {
    const tenantId = this.getTenantId();
    if (tenantId) {
      entity.tenantId = tenantId;
    }
  } else {
    entity.tenantId = null;
  }
}
```

### 4. 函数式Service的改造

messageService使用函数式编程：
```typescript
// 每个函数内部创建Repository实例
export async function sendSystemMessage(options) {
  const messageRepo = new BaseRepository(SystemMessage);
  // 使用messageRepo...
}
```

优点：
- 简单直接
- 每次调用都获取最新的租户上下文
- 适合无状态的Service

## ⚠️ 重要注意事项

### 1. 其他Service的改造

本任务只改造了messageService作为示例。其他Service的改造策略：

**需要改造的Service**：
- 涉及业务数据的Service
- 实体有tenant_id字段的Service

**不需要改造的Service**：
- 第三方API调用Service（如AliyunSmsService）
- 不涉及数据库操作的Service（如WebSocketService）
- 定时任务Service（如MessageCleanupService）

### 2. Controller层的改造

Controller层大部分直接使用Repository,也需要改造：
```typescript
// Controller中的改造
// 旧代码
const customerRepo = AppDataSource.getRepository(Customer);

// 新代码
const customerRepo = new BaseRepository(Customer);
```

建议：
- 在Service层完成改造后
- 再逐步改造Controller层
- 优先改造核心业务Controller

**TenantService**：
- 管理租户本身,不应该被租户过滤
- 应该直接使用TypeORM Repository
- 不使用BaseRepository
