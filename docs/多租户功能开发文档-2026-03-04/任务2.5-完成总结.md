## BaseRepository工作原理

### 1. 保存数据时自动设置tenant_id

```typescript
protected setTenantId(entity: DeepPartial<Entity>): void {
  if (!this.hasTenantIdField()) return;
  
  const tenantFieldName = this.getTenantFieldName();
  const entityAny = entity as any;
  
  // 如果已经设置了 tenant_id，不覆盖
  if (entityAny[tenantFieldName] !== undefined) return;
  
  // 根据部署模式设置 tenant_id
  if (deployConfig.isSaaS()) {
    const tenantId = this.getTenantId();
    if (tenantId) {
      entityAny[tenantFieldName] = tenantId;  // SaaS模式：设置租户ID
    }
  } else {
    entityAny[tenantFieldName] = null;  // 私有模式：设置为NULL
  }
}
```

### 2. 查询数据时自动添加过滤

```typescript
protected addTenantFilter<T extends FindOptionsWhere<Entity>>(where?: T | T[]): T | T[] | undefined {
  if (!this.hasTenantIdField()) return where;
  
  const tenantFieldName = this.getTenantFieldName();
  let tenantValue: string | null | undefined;
  
  // 根据部署模式获取租户ID
  if (deployConfig.isSaaS()) {
    tenantValue = this.getTenantId();  // SaaS模式：获取当前租户ID
  } else {
    tenantValue = null;  // 私有模式：过滤 tenant_id = NULL
  }
  
  const tenantFilter = { [tenantFieldName]: tenantValue } as any;
  
  // 合并租户过滤条件
  return { ...where, ...tenantFilter } as T;
}
```

### 3. 更新/删除时自动添加过滤

```typescript
async update(criteria: string | number | FindOptionsWhere<Entity>, partialEntity: DeepPartial<Entity>): Promise<void> {
  let where: FindOptionsWhere<Entity>;
  
  if (typeof criteria === 'string' || typeof criteria === 'number') {
    where = { id: criteria } as unknown as FindOptionsWhere<Entity>;
  } else {
    where = criteria;
  }
  
  // 添加租户过滤
  where = this.addTenantFilter(where) as FindOptionsWhere<Entity>;
  
  await this.repository.update(where, partialEntity as any);
}
```

## 关键发现

### 1. 实体字段映射的重要性

- 数据库使用下划线命名：`tenant_id`
- 实体使用驼峰命名：`tenantId`
- 通过`@Column({ name: 'tenant_id' })`映射

### 2. 部署模式的影响

- SaaS模式：需要设置`DEPLOY_MODE=saas`
- 私有模式：默认`DEPLOY_MODE=private`
- BaseRepository根据模式自动处理tenant_id

### 3. 测试环境配置

- 测试脚本需要显式设置环境变量
- 在require('dotenv')之前设置process.env
