## 二、关于 user_roles 和 authorized_ips 表

### ❓ user_roles 表

**状态**: 不存在  
**是否需要**: ❌ 不需要

**原因**:
- 当前系统采用**一对一关系**设计（一个用户只能有一个角色）
- 使用 `users.role` 和 `users.role_id` 字段直接关联
- 不是多对多关系，因此不需要中间表
- 前端和后端代码都基于这种设计实现

**验证**:
- ✓ User 实体使用 `role` 和 `roleId` 字段
- ✓ Role 实体使用 `permissions` JSON 字段存储权限
- ✓ 前端使用 `user.role` 和 `user.roleId` 访问角色
- ✓ 没有 UserRole 实体类

### ❓ authorized_ips 表

**状态**: 不存在  
**是否需要**: ❌ 不需要

**原因**:
- IP白名单功能已集成在 `users` 表中
- 使用 `users.authorized_ips` JSON 字段存储
- User 实体已包含此字段定义

**验证**:
