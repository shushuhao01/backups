# 租户状态控制 - 完整说明

## 📋 概述

租户系统有**两个独立的状态控制维度**，分别控制租户账号和授权码的生命周期。

---

## 🎯 两个状态字段

### 1. 租户账号状态 (status)
**用途**: 管理员控制租户账号本身的启用/禁用

**可选值**:
- `active`: 启用（正常）
- `inactive`: 禁用（软删除/封禁）

**操作接口**:
```
POST /api/v1/admin/tenants/:id/enable   - 启用租户
POST /api/v1/admin/tenants/:id/disable  - 禁用租户
```

**使用场景**:
- 封禁违规租户
- 软删除租户（保留数据但禁止访问）
- 临时关闭租户账号

---

### 2. 授权码状态 (licenseStatus)
**用途**: 管理授权码的生命周期和有效性

**可选值**:
- `pending`: 待激活（新创建，未使用）
- `active`: 已激活（正常使用中）
- `suspended`: 已暂停（临时暂停，可恢复）
- `expired`: 已过期（到期失效）

**操作接口**:
```
POST /api/v1/admin/tenants/:id/suspend  - 暂停授权
POST /api/v1/admin/tenants/:id/resume   - 恢复授权
POST /api/v1/admin/tenants/:id/renew    - 续期租户
```

**使用场景**:
- 暂停：欠费、维护、临时限制访问
- 恢复：缴费后恢复、维护完成
- 续期：延长授权有效期

---

## ✅ 租户可用条件

租户必须**同时满足**以下条件才能正常使用：

```typescript
isAvailable() {
  return this.status === 'active' &&           // 租户账号已启用
         this.licenseStatus === 'active' &&    // 授权码已激活
         !this.isExpired()                     // 未过期
}
```

---

## 🔄 状态组合说明

| status | licenseStatus | 是否可用 | 说明 |
|--------|---------------|---------|------|
| active | active | ✅ 是 | 正常状态 |
| active | suspended | ❌ 否 | 授权暂停（可恢复） |
| active | pending | ❌ 否 | 待激活 |
| active | expired | ❌ 否 | 已过期 |
| inactive | active | ❌ 否 | 租户禁用 |
| inactive | suspended | ❌ 否 | 双重禁用 |
| inactive | pending | ❌ 否 | 禁用且未激活 |
| inactive | expired | ❌ 否 | 禁用且过期 |

## 🔧 API接口详解

### 1. 启用租户
```http
POST /api/v1/admin/tenants/:id/enable
Authorization: Bearer {adminToken}
```

**功能**: 将 `status` 从 `inactive` 改为 `active`

**响应**:
```json
{
  "success": true,
  "message": "租户已启用",
  "data": { ...tenant }
}
```

**错误情况**:
- 租户不存在 → 404
- 已经是启用状态 → 400

---

### 2. 禁用租户
```http
POST /api/v1/admin/tenants/:id/disable
Authorization: Bearer {adminToken}
```

**功能**: 将 `status` 从 `active` 改为 `inactive`

**响应**:
```json
{
  "success": true,
  "message": "租户已禁用",
  "data": { ...tenant }
}
```

**错误情况**:
- 租户不存在 → 404
- 已经是禁用状态 → 400

---

### 3. 暂停授权
```http
POST /api/v1/admin/tenants/:id/suspend
Authorization: Bearer {adminToken}
```

**功能**: 将 `licenseStatus` 从 `active` 改为 `suspended`

**响应**:
```json
{
  "success": true,
  "message": "租户授权已暂停",
  "data": { ...tenant }
}
```

**错误情况**:
- 租户不存在 → 404
- 已经是暂停状态 → 400
- 当前状态不是active → 400

---

### 4. 恢复授权
```http
POST /api/v1/admin/tenants/:id/resume
Authorization: Bearer {adminToken}
```

**功能**: 将 `licenseStatus` 从 `suspended` 改为 `active`

**响应**:
```json
{
  "success": true,
  "message": "租户授权已恢复",
  "data": { ...tenant }
}
```

**错误情况**:
- 租户不存在 → 404
- 已经是激活状态 → 400
- 当前状态不是suspended → 400
- 租户已过期 → 400（需要先续期）

---

### 5. 续期租户
```http
POST /api/v1/admin/tenants/:id/renew
Authorization: Bearer {adminToken}
Content-Type: application/json

{
  "months": 12  // 续期月数，默认12个月
}
```

**功能**: 延长 `expireDate`，如果之前是 `expired` 状态则恢复为 `active`

**响应**:
```json
{
  "success": true,
  "message": "租户已续期12个月",
  "data": {
    ...tenant,
    "renewMonths": 12
  }
}
```

**计算逻辑**:
- 如果当前过期时间 > 现在：从当前过期时间延长
- 如果当前过期时间 ≤ 现在：从现在开始延长

---

## 🎭 使用场景示例

### 场景1: 租户欠费
```
1. 暂停授权: POST /tenants/:id/suspend
   → licenseStatus: active -> suspended
   → 租户无法登录，提示"授权已暂停"

2. 租户缴费后恢复: POST /tenants/:id/resume
   → licenseStatus: suspended -> active
   → 租户恢复正常使用
```

### 场景2: 租户违规
```
1. 禁用租户: POST /tenants/:id/disable
   → status: active -> inactive
   → 租户无法登录，提示"租户已被禁用"

2. 整改完成后启用: POST /tenants/:id/enable
   → status: inactive -> active
   → 租户恢复正常使用
```

### 场景3: 租户到期
```
1. 系统自动检测过期
   → expireDate < now
   → isExpired() = true
   → 租户无法登录，提示"租户已过期"

2. 续期: POST /tenants/:id/renew
   → expireDate 延长12个月
   → licenseStatus: expired -> active (如果之前是expired)
   → 租户恢复正常使用
```

### 场景4: 系统维护
```
1. 暂停授权: POST /tenants/:id/suspend
   → 临时禁止访问

2. 维护完成后恢复: POST /tenants/:id/resume
   → 恢复正常访问
```

## 🔒 中间件验证逻辑

在 `tenantAuth` 中间件中，会依次检查：

```typescript
// 1. 检查租户是否存在
if (!tenant) {
  return 403 "租户不存在"
}

// 2. 检查租户账号状态
if (tenant.status !== 'active') {
  return 403 "租户已被禁用，请联系管理员"
}

// 3. 检查授权状态
if (tenant.licenseStatus !== 'active') {
  const messages = {
    'pending': '租户授权待激活，请联系管理员',
    'suspended': '租户授权已暂停，请联系管理员',
    'expired': '租户授权已过期，请联系管理员续费'
  }
  return 403 messages[tenant.licenseStatus]
}

// 4. 检查过期时间
if (tenant.expireDate && new Date(tenant.expireDate) < new Date()) {
  return 403 "租户已过期，请联系管理员续费"
}

// 全部通过，允许访问
next()
```

## ✅ 总结

### 核心概念
- **status**: 控制租户账号本身（启用/禁用）
- **licenseStatus**: 控制授权码状态（激活/暂停/过期）
- **expireDate**: 授权有效期

### 操作分类
- **账号管理**: enable / disable
- **授权管理**: suspend / resume / renew

### 可用条件
```
status === 'active' && 
licenseStatus === 'active' && 
!isExpired()
```

### 灵活性
- 可以单独控制账号状态
- 可以单独控制授权状态
- 两者组合提供精细化管理
