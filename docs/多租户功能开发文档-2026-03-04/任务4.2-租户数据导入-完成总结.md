## 🎓 使用示例

### 1. 使用 curl 导入数据
```bash
# 创建导入任务
curl -X POST \
  http://localhost:3000/api/v1/admin/tenants/tenant-001/import \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@export_data.json" \
  -F "conflictStrategy=skip"

# 查询导入进度
curl -X GET \
  http://localhost:3000/api/v1/admin/tenants/tenant-001/import/JOB_ID \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 2. 使用 JavaScript/Axios
```javascript
// 创建导入任务
const formData = new FormData();
formData.append('file', fileBlob);
formData.append('conflictStrategy', 'skip');

const response = await axios.post(
  `/api/admin/tenants/${tenantId}/import`,
  formData,
  {
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'multipart/form-data'
    }
  }
);

const jobId = response.data.data.jobId;

// 轮询查询进度
const checkProgress = async () => {
  const res = await axios.get(
    `/api/admin/tenants/${tenantId}/import/${jobId}`,
    {
      headers: { 'Authorization': `Bearer ${token}` }
    }
  );
  
  const job = res.data.data;
  console.log(`进度: ${job.progress}%`);
  
  if (job.status === 'completed') {
    console.log('导入完成！');
  } else if (job.status === 'failed') {
    console.error('导入失败:', job.errors);
  } else {
    setTimeout(checkProgress, 1000);
  }
};

checkProgress();
```

## ⚠️ 注意事项

### 1. 文件大小限制
- 当前限制：100MB
- 可在 `tenant-import.ts` 中调整 `limits.fileSize`

### 2. 冲突策略选择
- **skip**：适合首次导入或增量导入
- **overwrite**：适合数据同步或更新
- **error**：适合严格的数据一致性要求

### 3. 性能考虑
- 大数据量导入建议使用 skip 策略
- 导入过程中避免重复提交
- 定期清理过期的导入任务

### 4. 错误处理
- 导入失败不会回滚已导入的数据
- 建议在测试环境先验证
- 保留原始导出文件作为备份

### 5. 安全性
- 只有 Admin 用户可以执行导入
- 上传文件会进行类型验证
- 导入数据会验证租户归属
