---

### 步骤2: 执行数据库迁移脚本 ⭐ 核心步骤

**目的**: 给32个业务表添加tenant_id字段，创建租户相关表

**操作方式A - 使用MySQL命令行（推荐）**:
```bash
# 1. 打开PowerShell或CMD
# 2. 执行迁移脚本
mysql -u root -p crm_dev < backend/database-migrations/add-tenant-support.sql

# 3. 输入MySQL密码
# 4. 等待执行完成（约10-30秒）
```

**操作方式B - 使用Navicat/phpMyAdmin**:
```
1. 打开Navicat或phpMyAdmin
2. 连接到crm_dev数据库
3. 打开SQL查询窗口
4. 复制 backend/database-migrations/add-tenant-support.sql 的全部内容
5. 粘贴到查询窗口
6. 点击"运行"或"执行"
7. 等待执行完成
```

**可能遇到的问题**:

**问题1**: 唯一索引冲突
```
错误: Duplicate entry 'xxx' for key 'uk_tenant_username'
解决: 这是正常的，因为修改唯一索引时可能有重复数据
     先删除旧索引，再添加新索引
```

**问题2**: 表不存在
```
错误: Table 'crm_dev.xxx' doesn't exist
解决: 检查表名是否正确，或者该表在你的数据库中不存在
     可以注释掉该表的ALTER语句
```

**预计时间**: 1-3分钟

### 步骤3: 验证迁移结果 ⭐ 必须验证

**目的**: 确认所有表都已正确添加tenant_id字段

**操作方式A - 使用SQL查询**:
```sql
-- 在MySQL命令行或Navicat中执行：
SELECT 
  TABLE_NAME,
  COLUMN_NAME,
  COLUMN_TYPE,
  IS_NULLABLE,
  COLUMN_COMMENT
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_SCHEMA = 'crm_dev'
  AND COLUMN_NAME = 'tenant_id'
ORDER BY TABLE_NAME;
```

**预期结果**:
- 应该看到32个表都有tenant_id字段
- 字段类型: VARCHAR(36)
- 允许NULL: YES
- 注释: 租户ID（NULL表示私有部署）

**操作方式B - 手动检查几个关键表**:
```sql
-- 检查users表
DESC users;

-- 检查customers表
DESC customers;

-- 检查orders表
DESC orders;

-- 应该都能看到tenant_id字段
```

**验证清单**:
- [ ] users表已添加tenant_id
- [ ] customers表已添加tenant_id
- [ ] orders表已添加tenant_id
- [ ] tenants表已创建
- [ ] tenant_settings表已创建
- [ ] 唯一索引已修改（uk_tenant_username等）

**预计时间**: 2-3分钟

### 步骤4: 测试现有功能是否正常 ⭐ 重要

**目的**: 确认添加tenant_id后，现有功能不受影响

**操作**:
```bash
# 1. 启动后端服务
cd backend
npm run dev

# 2. 启动前端服务（新开一个终端）
npm run dev

# 3. 打开浏览器访问 http://localhost:5173
```

**测试项目**:
- [ ] 登录功能正常
- [ ] 客户列表可以正常显示
- [ ] 订单列表可以正常显示
- [ ] 可以创建新客户
- [ ] 可以创建新订单
- [ ] 没有报错

**为什么现有功能还能正常工作？**
- tenant_id字段允许NULL
- 现有数据的tenant_id都是NULL
- 现有代码没有tenant_id过滤，所以能查询到所有数据
- 这就是私有部署模式的工作方式

**如果出现问题**:
- 查看浏览器控制台错误
- 查看后端终端错误日志
- 如果是数据库错误，可能需要回滚

**预计时间**: 5-10分钟

## 📋 第二阶段：后端租户隔离（下周完成）

### 步骤5: 创建BaseRepository

**目的**: 实现自动添加tenant_id过滤的基础Repository

**我会创建的文件**:
- `backend/src/repositories/BaseRepository.ts`

**功能**:
- 自动在查询时添加tenant_id过滤
- 自动在插入时添加tenant_id
- 支持私有部署模式（tenant_id=NULL）
- 支持SaaS模式（tenant_id=UUID）

**预计时间**: 30分钟

---

### 步骤6: 修改现有Repository

**目的**: 让所有Repository继承BaseRepository

**需要修改的文件**:
- `backend/src/repositories/CustomerRepository.ts`
- `backend/src/repositories/OrderRepository.ts`
- `backend/src/repositories/ProductRepository.ts`
- `backend/src/repositories/UserRepository.ts`
- 其他所有Repository

**修改方式**:
```typescript
// 修改前
export class CustomerRepository {
  async find() {
    return Customer.find()
  }
}

// 修改后
export class CustomerRepository extends BaseRepository<Customer> {
  constructor(req: TenantRequest) {
    super(Customer, req)
  }
}
```

**预计时间**: 1-2小时

## ⚠️ 重要提醒

### 如果迁移失败怎么办？

**方法1: 从备份恢复数据库**
```bash
# 1. 删除当前数据库
mysql -u root -p -e "DROP DATABASE crm_dev;"

# 2. 重新创建数据库
mysql -u root -p -e "CREATE DATABASE crm_dev CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 3. 恢复备份
mysql -u root -p crm_dev < D:\kaifa\backup\crm_dev_before_migration.sql
```

**方法2: 从Git恢复代码**
```bash
git reset --hard v1.8.0-before-tenant-migration
```
