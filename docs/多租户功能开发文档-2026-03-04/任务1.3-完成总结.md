## 🔍 核心代码片段

### 租户验证逻辑

```typescript
// SaaS模式：提取租户ID并验证
if (deployConfig.isSaaS()) {
  req.tenantId = decoded.tenantId
  req.userId = decoded.userId

  if (!req.tenantId) {
    return res.status(403).json({
      success: false,
      message: '租户信息缺失'
    })
  }

  // 验证租户状态
  const tenant = await tenantRepository.findOne({
    where: { id: req.tenantId }
  })

  if (!tenant) {
    return res.status(403).json({
      success: false,
      message: '租户不存在'
    })
  }

  // 检查租户状态
  if (tenant.status !== 'active') {
    return res.status(403).json({
      success: false,
      message: '租户已被禁用，请联系管理员'
    })
  }

  // 检查授权状态
  if (tenant.licenseStatus !== 'active') {
    const statusMessages: Record<string, string> = {
      'pending': '租户授权待激活，请联系管理员',
      'suspended': '租户授权已暂停，请联系管理员',
      'expired': '租户授权已过期，请联系管理员续费'
    }
    return res.status(403).json({
      success: false,
      message: statusMessages[tenant.licenseStatus] || '租户授权状态异常'
    })
  }

  // 检查过期时间
  if (tenant.expireDate && new Date(tenant.expireDate) < new Date()) {
    return res.status(403).json({
      success: false,
      message: '租户已过期，请联系管理员续费'
    })
  }

  // 将租户信息注入到请求对象
  req.tenantInfo = tenant
}
```

### 资源限制检查

```typescript
// 检查用户数限制（创建用户时）
if (req.path.includes('/users') && req.method === 'POST') {
  if (!tenant.canAddUser()) {
    return res.status(403).json({
      success: false,
      message: `用户数已达上限(${tenant.maxUsers}个)`
    })
  }
}

// 检查存储空间限制（上传文件时）
if (req.path.includes('/upload') && req.method === 'POST') {
  const fileSize = req.headers['content-length']
  if (fileSize) {
    const fileSizeMb = parseInt(fileSize) / (1024 * 1024)
    if (!tenant.hasEnoughStorage(fileSizeMb)) {
      return res.status(403).json({
        success: false,
        message: `存储空间不足，已使用${tenant.usedStorageMb}MB，最大${tenant.maxStorageGb}GB`
      })
    }
  }
}
```

## 🎓 技术要点

### 1. 中间件设计模式
- 使用Express中间件模式，支持链式调用
- 扩展Request接口，添加 `tenantId` 和 `tenantInfo` 属性
- 错误处理统一返回JSON格式

### 2. 部署模式兼容
- 使用 `deployConfig.isSaaS()` 判断部署模式
- SaaS模式：强制验证租户
- 私有模式：跳过租户验证

### 3. 数据库查询优化
- 使用TypeORM的 `findOne` 方法查询租户
- 只查询必要的字段，减少数据传输

### 4. 安全性考虑
- 所有租户验证失败都返回403状态码
- 错误消息清晰明确，便于用户理解
- 不暴露系统内部实现细节
