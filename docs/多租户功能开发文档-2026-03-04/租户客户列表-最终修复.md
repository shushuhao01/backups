## 修复的问题

### 1. ✅ 启用/禁用API端点不存在
**问题**：点击启用/禁用开关时，API请求失败

**原因**：代码逻辑有冗余的三元运算符

**修复前**：
```typescript
const res = isActive
  ? await request.post(`/tenants/${row.id}/disable`)
  : await request.post(`/tenants/${row.id}/enable`)
```

**修复后**：
```typescript
const endpoint = isActive ? 'disable' : 'enable'
const res = await request.post(`/tenants/${row.id}/${endpoint}`)
```

**后端路由**（已存在）：
- `POST /api/v1/admin/tenants/:id/enable` - 启用租户
- `POST /api/v1/admin/tenants/:id/disable` - 禁用租户

### 2. ✅ 列表没有铺满右侧，留白
**问题**：表格右侧有大量空白

**解决方案**：
- 授权码列从固定宽度改为 `min-width: 200`，允许自适应扩展
- 优化其他列宽，减少固定宽度列

### 3. ✅ 授权码显示不全

**解决方案**：
- 授权码列使用 `min-width: 200`（最小宽度）
- 允许根据内容自动扩展
- 字体大小调整为 11px，更紧凑

## API端点说明

### 租户状态管理
```
POST /api/v1/admin/tenants/:id/enable   - 启用租户（status: inactive -> active）
POST /api/v1/admin/tenants/:id/disable  - 禁用租户（status: active -> inactive）
POST /api/v1/admin/tenants/:id/suspend  - 暂停授权（licenseStatus: active -> suspended）
POST /api/v1/admin/tenants/:id/resume   - 恢复授权（licenseStatus: suspended -> active）
POST /api/v1/admin/tenants/:id/renew    - 续期租户
```

### 双状态设计
1. **status**（租户账号状态）
   - `active` - 启用
   - `inactive` - 禁用

2. **licenseStatus**（授权码状态）
   - `pending` - 待激活
   - `active` - 已激活
   - `suspended` - 已暂停
   - `expired` - 已过期
