## ⚠️ 注意事项

### 1. 表不存在的情况
如果你的数据库中某些表不存在（比如 `customer_tags`、`logistics` 等），迁移脚本会报错：
```
ERROR 1146 (42S02): Table 'crm_local.customer_tags' doesn't exist
```

**解决方案**：
- 使用 `IF NOT EXISTS` 语法（已在完整版脚本中使用）
- 或者手动注释掉不存在的表

### 2. 索引已存在的情况
如果某些索引已经存在，会报错：
```
ERROR 1061 (42000): Duplicate key name 'idx_tenant_id'
```

**解决方案**：
- 使用 `IF NOT EXISTS` 语法（已在完整版脚本中使用）
- 或者使用 `DROP INDEX IF EXISTS` 先删除

### 3. MySQL版本兼容性
`IF NOT EXISTS` 和 `IF EXISTS` 语法需要 MySQL 5.7+ 版本。

如果你的MySQL版本较低，需要手动检查表和索引是否存在。

## 🚀 推荐执行流程

### 第1步：检查数据库中有哪些表
```sql
SHOW TABLES FROM crm_local;
```

### 第2步：确认要迁移的表
对比数据库中的表和迁移脚本中的表，确认哪些表需要迁移。

### 第3步：选择合适的迁移脚本
- 如果所有32个表都存在 → 使用完整版脚本
- 如果只有17个核心表存在 → 使用原始版脚本
- 如果部分表存在 → 手动修改脚本，注释掉不存在的表

### 第4步：执行迁移
```bash
mysql -u root -p crm_local < backend/database-migrations/add-tenant-support-complete.sql
```

```bash

node scripts/migrate-crm-local.js
```
