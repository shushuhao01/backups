## ⚠️ 注意事项

### 1. 关于错误提示

如果执行过程中出现以下错误，可以忽略：
```
Duplicate column name 'tenant_id'
Duplicate key name 'idx_tenant_id'
```

这表示字段或索引已经存在，不影响其他表的执行。

### 2. 关于现有数据

- 所有 tenant_id 字段初始值为 NULL
- 不会影响现有数据的正常使用
- 后续需要通过应用程序为数据分配租户ID

### 3. 关于性能影响

- 添加字段和索引的操作很快（通常几秒内完成）
- 对现有业务无影响
- 建议在业务低峰期执行

### 4. 回滚方案

如果需要回滚，可以执行以下SQL：

```sql
-- 删除所有 tenant_id 字段（谨慎使用！）
ALTER TABLE users DROP COLUMN tenant_id;
ALTER TABLE roles DROP COLUMN tenant_id;
-- ... 依次删除其他表的字段
```

或者直接恢复之前的数据库备份。
