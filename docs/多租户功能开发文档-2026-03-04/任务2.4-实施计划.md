## 🔧 改造模式

### 改造前（直接使用TypeORM Repository）

```typescript
import { AppDataSource } from '../config/database';
import { Customer } from '../entities/Customer';

class CustomerService {
  private customerRepository = AppDataSource.getRepository(Customer);

  async getCustomers(status: string) {
    // ❌ 没有租户过滤
    return await this.customerRepository.find({
      where: { status }
    });
  }

  async createCustomer(data: any) {
    const customer = this.customerRepository.create(data);
    // ❌ 没有设置tenant_id
    return await this.customerRepository.save(customer);
  }
}
```

### 改造后（使用BaseRepository）

```typescript
import { BaseRepository } from '../repositories/BaseRepository';
import { Customer } from '../entities/Customer';

class CustomerService {
  private customerRepository: BaseRepository<Customer>;

  constructor() {
    this.customerRepository = new BaseRepository(Customer);
  }

  async getCustomers(status: string) {
    // ✅ 自动添加tenant_id过滤
    return await this.customerRepository.find({
      where: { status }
    });
  }

  async createCustomer(data: any) {
    // ✅ 自动设置tenant_id
    return await this.customerRepository.save(data);
  }
}
```

## ⚠️ 重要注意事项

### 1. TenantService特殊处理

TenantService管理租户本身,不应该被租户过滤:
```typescript
// TenantService应该直接使用TypeORM Repository
// 不使用BaseRepository
private tenantRepository = AppDataSource.getRepository(Tenant);
```

### 2. 向后兼容

所有改造必须保证私有部署模式正常工作:
- BaseRepository会自动处理
- 私有模式：WHERE tenant_id IS NULL
- SaaS模式：WHERE tenant_id = ?

### 3. 测试验证

每改造一个Service,立即测试:
1. 编译检查
2. 功能测试
3. 数据隔离测试

### 4. 渐进式改造

不要一次性改造所有Service:
- 先改造核心Service
- 逐个测试验证
- 确保稳定后再继续
