## 🎓 技术要点总结

### 1. 实体命名规范

**严格遵守**：
- 数据库字段：`tenant_id`（下划线命名）
- 实体属性：`tenantId`（驼峰命名）
- 映射方式：`@Column({ name: 'tenant_id' })`

### 2. AsyncLocalStorage的优势

- 不需要在每个函数中传递tenant_id参数
- 支持异步调用链中传递上下文
- 线程安全，不会混淆不同请求的租户信息

### 3. TypeORM元数据的使用

- 通过`repository.metadata.columns`获取实体字段信息
- 动态检测字段名，支持不同命名风格
- 获取数据库列名用于QueryBuilder

### 4. 部署模式兼容

- SaaS模式：`WHERE tenant_id = 'xxx'`
- 私有模式：`WHERE tenant_id IS NULL`
- 通过`deployConfig.isSaaS()`动态判断

### 5. 类型安全

- 使用TypeScript泛型保持类型安全
- 方法签名与TypeORM Repository一致
- 支持方法重载（如save方法）

## 📝 重要注意事项

### 1. 字段命名规范必须严格遵守

```typescript
// ✅ 正确
@Column('varchar', { length: 36, name: 'tenant_id' })
tenantId: string;

// ❌ 错误
@Column('varchar', { length: 36 })
tenant_id: string;
```

### 2. 使用BaseRepository的注意事项

```typescript
// ✅ 安全：使用BaseRepository
const customerRepo = new BaseRepository(Customer);
const customers = await customerRepo.find(); // 自动过滤

// ⚠️ 危险：使用getRawRepository
const rawRepo = customerRepo.getRawRepository();
const customers = await rawRepo.find(); // 不会自动过滤！
```

### 3. 没有tenant_id字段的表

BaseRepository会自动检测，如果表没有tenant_id字段，不会添加过滤：
```typescript
// tenants表本身没有tenant_id字段
const tenantRepo = new BaseRepository(Tenant);
const tenants = await tenantRepo.find(); // 不会添加tenant_id过滤
```

### 4. 在中间件中设置上下文

```typescript
// 在tenantAuth中间件之后添加
app.use(tenantAuth);
app.use(tenantContextMiddleware);

// 所有后续的路由都可以使用BaseRepository
app.use('/api/customers', customerRoutes);
```
