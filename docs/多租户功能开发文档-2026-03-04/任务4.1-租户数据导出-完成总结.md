## 🔧 技术实现

### 导出数据结构
```json
{
  "version": "1.0",
  "tenant": {
    "id": "租户ID",
    "name": "租户名称",
    "code": "租户代码"
  },
  "exportTime": "2026-03-03T12:00:00Z",
  "dataRange": {
    "startDate": "开始日期",
    "endDate": "结束日期"
  },
  "data": {
    "customers": [...],
    "orders": [...],
    "products": [...]
  }
}
```

### 异步导出流程
1. 创建导出任务，返回 jobId
2. 后台异步执行导出
3. 客户端轮询查询进度
4. 导出完成后下载文件

## 📝 使用说明

### 1. 创建导出任务
```bash
POST /api/admin/tenants/:id/export
Authorization: Bearer {admin_token}

{
  "tables": ["customers", "orders", "products"],  # 可选
  "startDate": "2026-01-01",  # 可选
  "endDate": "2026-03-03"     # 可选
}
```

### 2. 查询导出进度
```bash
GET /api/admin/tenants/:id/export/:jobId
Authorization: Bearer {admin_token}
```

### 3. 下载导出文件
```bash
GET /api/admin/tenants/:id/export/:jobId/download
Authorization: Bearer {admin_token}
```

## 📌 注意事项

1. **文件存储**：
   - 导出文件存储在 `backend/exports/` 目录
   - 需要定期清理过期文件

2. **性能考虑**：
   - 大数据量导出可能需要较长时间
   - 建议使用异步导出

3. **安全性**：
   - 只有 Admin 用户可以导出数据
   - 导出文件包含敏感信息，需要妥善保管

4. **后续优化**：
