## 第2步：执行数据库迁移脚本 ⭐ （5分钟）

### 操作命令
在项目根目录下执行：

```bash
mysql -u root -p crm_dev < backend/database-migrations/add-tenant-support.sql
```

### 说明
- 输入 MySQL 密码
- 等待执行完成（约10-30秒）
- 如果没有报错，说明执行成功

### 这个脚本会做什么？
1. ✅ 创建 `tenants` 表（租户表）
2. ✅ 创建 `tenant_settings` 表（租户配置表）
3. ✅ 给32个业务表添加 `tenant_id` 字段
4. ✅ 修改8个唯一索引（支持租户隔离）
5. ✅ 创建复合索引（优化查询性能）

### 可能遇到的问题

#### 问题1：索引已存在
```
错误: Duplicate key name 'idx_tenant_id'
解决: 这是正常的，说明索引已经存在，可以忽略
```

#### 问题2：表不存在
```
错误: Table 'crm_dev.xxx' doesn't exist
解决: 检查你的数据库中是否有这个表，如果没有可以忽略
```

### ✅ 完成标记
- [ ] 脚本执行完成
- [ ] 没有严重错误

## 第3步：验证迁移结果 ⭐ （5分钟）

### 方法A：使用验证脚本（推荐）

在项目根目录下执行：

```bash
cd backend
node scripts/tenant-migration-helper.js
```

这个脚本会自动检查：
- ✅ tenants 和 tenant_settings 表是否创建
- ✅ 有多少个表添加了 tenant_id 字段
- ✅ 索引是否创建成功
- ✅ 查询功能是否正常

### 方法B：手动SQL验证

在 MySQL 命令行或 Navicat 中执行：

```sql
-- 检查有多少表添加了 tenant_id
SELECT 
  TABLE_NAME,
  COLUMN_NAME,
  COLUMN_TYPE,
  IS_NULLABLE,
  COLUMN_COMMENT
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_SCHEMA = 'crm_dev'
  AND COLUMN_NAME = 'tenant_id'
ORDER BY TABLE_NAME;
```

### 预期结果
- 应该看到至少32个表都有 tenant_id 字段
- 字段类型: VARCHAR(36)
- 允许NULL: YES
- 注释: 租户ID（NULL表示私有部署）

## ⚠️ 如果迁移失败怎么办？

### 快速恢复方法

```bash
# 1. 删除当前数据库
mysql -u root -p -e "DROP DATABASE crm_dev;"

# 2. 重新创建数据库
mysql -u root -p -e "CREATE DATABASE crm_dev CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 3. 恢复备份
mysql -u root -p crm_dev < D:\kaifa\backup\crm_dev_before_migration.sql
```
