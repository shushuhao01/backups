<div class="button-group">

#### 调整用户数对话框
```vue
<el-dialog v-model="showAdjustUsersDialog" title="调整用户数" width="400px">
  <el-form label-width="100px">
    <el-form-item label="客户名称">{{ detail.name }}</el-form-item>
    <el-form-item label="当前用户数">{{ detail.userCount }} / {{ detail.maxUsers }}</el-form-item>
    <el-form-item label="新最大用户数" required>
      <el-input-number v-model="newMaxUsers" :min="detail.userCount || 1" :max="9999" style="width: 100%" />
      <div style="color: #909399; font-size: 12px; margin-top: 4px;">
        最小值为当前已使用用户数（{{ detail.userCount || 0 }}）
      </div>
    </el-form-item>
  </el-form>
  <template #footer>
    <el-button @click="showAdjustUsersDialog = false">取消</el-button>
    <el-button type="primary" @click="confirmAdjustUsers" :loading="submitting">确定</el-button>
  </template>
</el-dialog>
```

**功能实现**:
```typescript
const handleAdjustUsers = () => {
  newMaxUsers.value = detail.value.maxUsers || 10
  showAdjustUsersDialog.value = true
}

const confirmAdjustUsers = async () => {
  if (!newMaxUsers.value || newMaxUsers.value < (detail.value.userCount || 0)) {
    ElMessage.warning('新的最大用户数不能小于当前已使用用户数')
    return
  }
  
  submitting.value = true
  try {
    const res = await request.put(`/tenants/${route.params.id}`, {
      maxUsers: newMaxUsers.value
    })
    if (res.success) {
      ElMessage.success('调整成功')
      showAdjustUsersDialog.value = false
      fetchDetail()
    }
  } catch (e: any) {
    ElMessage.error(e.message || '调整失败')
  } finally {
    submitting.value = false
  }
}
```

**验证规则**:
- 新的最大用户数不能小于当前已使用用户数
- 最小值：当前已使用用户数
- 最大值：9999

### 3. 编辑功能改进
**问题**: 点击编辑按钮直接返回租户列表，无法在详情页编辑

**解决方案**: 实现详情页内编辑对话框

#### 编辑对话框
```vue
<el-dialog v-model="showEditDialog" title="编辑租户信息" width="600px">
  <el-form ref="editFormRef" :model="editForm" :rules="editRules" label-width="100px">
    <el-form-item label="客户名称" prop="name">
      <el-input v-model="editForm.name" placeholder="请输入客户名称" />
    </el-form-item>
    <el-form-item label="租户编码" prop="code">
      <el-input v-model="editForm.code" disabled placeholder="租户编码不可修改" />
    </el-form-item>
    <el-form-item label="联系人" prop="contact">
      <el-input v-model="editForm.contact" placeholder="请输入联系人" />
    </el-form-item>
    <el-form-item label="联系电话" prop="phone">
      <el-input v-model="editForm.phone" placeholder="请输入联系电话" />
    </el-form-item>
    <el-form-item label="邮箱" prop="email">
      <el-input v-model="editForm.email" placeholder="请输入邮箱" />
    </el-form-item>
    <el-form-item label="备注">
      <el-input v-model="editForm.remark" type="textarea" :rows="3" placeholder="备注信息" />
    </el-form-item>
  </el-form>
  <template #footer>
    <el-button @click="showEditDialog = false">取消</el-button>
    <el-button type="primary" @click="confirmEdit" :loading="submitting">确定</el-button>
  </template>
</el-dialog>
```

**功能实现**:
```typescript
const editForm = reactive({
  name: '',
  code: '',
  contact: '',
  phone: '',
  email: '',
  remark: ''
})

const editRules: FormRules = {
  name: [{ required: true, message: '请输入客户名称', trigger: 'blur' }],
  phone: [{ pattern: /^1[3-9]\d{9}$/, message: '请输入正确的手机号', trigger: 'blur' }],
  email: [{ type: 'email', message: '请输入正确的邮箱', trigger: 'blur' }]
}

const handleEdit = () => {
  // 填充编辑表单
  Object.assign(editForm, {
    name: detail.value.name,
    code: detail.value.code,
    contact: detail.value.contact || '',
    phone: detail.value.phone || '',
    email: detail.value.email || '',
    remark: detail.value.remark || ''
  })
  showEditDialog.value = true
}

const confirmEdit = async () => {
  await editFormRef.value?.validate()
  submitting.value = true
  try {
    const res = await request.put(`/tenants/${route.params.id}`, {
      name: editForm.name,
      contact: editForm.contact,
      phone: editForm.phone,
      email: editForm.email,
      remark: editForm.remark
    })
    if (res.success) {
      ElMessage.success('更新成功')
      showEditDialog.value = false
      fetchDetail()
    }
  } catch (e: any) {
    ElMessage.error(e.message || '更新失败')
  } finally {
    submitting.value = false
  }
}
```

**特点**:
- 租户编码不可修改（disabled）
- 表单验证：
  - 客户名称必填
  - 手机号格式验证
  - 邮箱格式验证
- 编辑成功后自动刷新详情

## 业务规则

### 调整用户数
- 新的最大用户数 ≥ 当前已使用用户数
- 范围：[当前已使用用户数, 9999]

### 调整存储空间
- 新的最大存储空间 ≥ 当前已使用空间
- 范围：[当前已使用空间(GB), 10000GB]
- 自动将MB转换为GB（向上取整）

### 编辑租户信息
- 客户名称：必填
- 租户编码：不可修改
- 手机号：格式验证（1开头的11位数字）
- 邮箱：格式验证
