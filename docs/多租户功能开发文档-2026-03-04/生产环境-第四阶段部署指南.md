# 生产环境 - 第四阶段部署指南

## 📋 部署概述

**部署内容**：第四阶段数据库迁移  
**部署时间**：建议在业务低峰期执行  
**预计耗时**：5-10分钟  
**影响范围**：无业务影响（仅添加新表和索引）

---

## 🎯 部署内容

### 1. 新增表
- `tenant_logs` - 租户操作日志表

### 2. 新增索引
- `tenants` 表：3个索引
- `users` 表：1个索引
- `customers` 表：1个索引
- `orders` 表：2个索引
- `products` 表：1个索引

## 📝 部署步骤

### 方式一：使用宝塔面板（推荐）

#### 步骤 1：登录宝塔面板
1. 打开浏览器，访问宝塔面板
2. 登录管理员账号

#### 步骤 2：打开数据库管理
1. 点击左侧菜单 "数据库"
2. 找到生产环境数据库（通常是 `crm` 或 `crm_production`）
3. 点击 "管理" 按钮

#### 步骤 3：执行SQL脚本
1. 点击 "SQL" 标签页
2. 复制下面的SQL代码
3. 粘贴到SQL输入框
4. 点击 "执行" 按钮

## 📄 生产环境SQL脚本

```sql
-- ========================================
-- 第四阶段数据库迁移 - 生产环境
-- ========================================
-- 执行时间：2026-03-03
-- 说明：创建租户日志表和性能优化索引
-- ========================================

-- 1. 创建租户操作日志表
-- ========================================

CREATE TABLE IF NOT EXISTS `tenant_logs` (
  `id` VARCHAR(36) PRIMARY KEY,
  `tenant_id` VARCHAR(255) NOT NULL COMMENT '租户ID',
  `action` VARCHAR(50) NOT NULL COMMENT '操作类型',
  `operator` VARCHAR(100) NOT NULL COMMENT '操作人名称',
  `operator_id` VARCHAR(255) NOT NULL COMMENT '操作人ID',
  `details` TEXT COMMENT '操作详情（JSON格式）',
  `ip_address` VARCHAR(45) COMMENT 'IP地址',
  `user_agent` VARCHAR(500) COMMENT '用户代理',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  
  INDEX `idx_tenant_id` (`tenant_id`),
  INDEX `idx_tenant_created` (`tenant_id`, `created_at`),
  INDEX `idx_action` (`action`),
  INDEX `idx_operator_id` (`operator_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='租户操作日志表';

-- ========================================
-- 2. 性能优化索引
-- ========================================

-- 2.1 优化租户表查询
ALTER TABLE `tenants` 
ADD INDEX `idx_status_license` (`status`, `license_status`);

ALTER TABLE `tenants` 
ADD INDEX `idx_expire_date` (`expire_date`);

ALTER TABLE `tenants` 
ADD INDEX `idx_created_at` (`created_at`);

-- 2.2 优化用户表查询（按租户）
ALTER TABLE `users` 
ADD INDEX `idx_tenant_status` (`tenant_id`, `status`);

-- 2.3 优化客户表查询（按租户）
ALTER TABLE `customers` 
ADD INDEX `idx_tenant_created` (`tenant_id`, `created_at`);

-- 2.4 优化订单表查询（按租户）
ALTER TABLE `orders` 
ADD INDEX `idx_tenant_status` (`tenant_id`, `status`);

ALTER TABLE `orders` 
ADD INDEX `idx_tenant_created` (`tenant_id`, `created_at`);

-- 2.5 优化商品表查询（按租户）
ALTER TABLE `products` 
ADD INDEX `idx_tenant_status` (`tenant_id`, `status`);

-- ========================================
-- 3. 分析表以更新统计信息
-- ========================================

ANALYZE TABLE `tenants`;
ANALYZE TABLE `users`;
ANALYZE TABLE `customers`;
ANALYZE TABLE `orders`;
ANALYZE TABLE `products`;
ANALYZE TABLE `tenant_settings`;
ANALYZE TABLE `tenant_logs`;

-- ========================================
-- 4. 验证
-- ========================================

-- 检查 tenant_logs 表是否创建成功
SELECT 
  TABLE_NAME,
  TABLE_ROWS,
  CREATE_TIME
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = DATABASE()
AND TABLE_NAME = 'tenant_logs';

-- 检查索引
SHOW INDEX FROM tenant_logs;

-- ========================================
-- 完成！
-- ========================================

SELECT '✅ 第四阶段数据库迁移完成' AS message;
```

## ⚠️ 注意事项

### 1. 索引已存在的处理
如果执行时提示 "Duplicate key name"（索引已存在），这是正常的，可以忽略该错误，继续执行后续SQL。

### 2. 备份建议
虽然本次迁移只是添加新表和索引，不会修改现有数据，但仍建议：
- 在执行前备份数据库
- 或者在宝塔面板中创建数据库快照

### 3. 执行时间
- 创建表：< 1秒
- 添加索引：根据数据量，可能需要几秒到几分钟
- 分析表：< 10秒

### 4. 业务影响
- 无需停止服务
- 不影响现有功能
- 添加索引时可能会短暂锁表（通常 < 1秒）
