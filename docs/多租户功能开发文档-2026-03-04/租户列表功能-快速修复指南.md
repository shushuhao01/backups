## 当前问题

❌ 启用/禁用功能报404错误

## 快速解决

### 方法1：使用重启脚本（推荐）
```bash
.\restart-backend.bat
```

### 方法2：手动重启
1. 在后端终端按 `Ctrl+C` 停止服务
2. 运行 `npm run dev` 重新启动

## 为什么需要重启？

后端代码已经更新，包括：
- ✅ 启用/禁用路由已添加
- ✅ 控制器函数已实现
- ✅ 路由已正确注册

但是**Node.js服务需要重启才能加载新代码**。

## 如果重启后还有问题

### 1. 检查后端日志
查看终端输出，确认：
- 服务启动成功
- 端口3000已监听
- 没有错误信息

### 2. 检查浏览器控制台
按F12打开开发者工具，查看：
- Network标签中的请求
- 请求URL是否正确
- 响应状态码

### 3. 验证API端点
运行测试脚本：
```bash
cd backend
node test-admin-tenant-routes.js
```

应该看到所有路由都是 ✅
