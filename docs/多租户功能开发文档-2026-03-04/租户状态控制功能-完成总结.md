## 🎯 问题分析

### 发现的问题
1. **概念混淆**: 之前的suspend/resume接口没有实现
2. **业务逻辑不清晰**: status和licenseStatus的职责不明确
3. **缺少操作接口**: 只有通用的updateStatus接口，没有专门的操作接口

### 明确的设计
采用**方案B：双状态设计**，明确两个独立的控制维度：

**1. 租户账号状态 (status)**
- 用途：管理员控制租户账号本身
- 值：`active` (启用) / `inactive` (禁用)
- 操作：启用/禁用租户

**2. 授权码状态 (licenseStatus)**
- 用途：管理授权码的生命周期
- 值：`pending` (待激活) / `active` (已激活) / `suspended` (已暂停) / `expired` (已过期)
- 操作：暂停/恢复授权、续期

### 3. 业务逻辑完善

#### 状态检查
每个操作都会检查当前状态，避免重复操作：
```typescript
// 暂停授权
if (tenant.licenseStatus === 'suspended') {
  return 400 "租户授权已处于暂停状态"
}

// 恢复授权
if (tenant.licenseStatus !== 'suspended') {
  return 400 "无法恢复，当前授权状态为: xxx"
}
```

#### 恢复授权时检查过期
```typescript
if (tenant.isExpired()) {
  return 400 "租户已过期，请先续期"
}
```

#### 续期逻辑
```typescript
// 从当前过期时间或现在开始延长
const baseDate = currentExpireDate > new Date() 
  ? currentExpireDate 
  : new Date();
newExpireDate.setMonth(baseDate.getMonth() + months);

// 如果之前是过期状态，恢复为active
if (tenant.licenseStatus === 'expired') {
  tenant.licenseStatus = 'active';
}
```

## 📊 状态组合说明

| status | licenseStatus | 是否可用 | 说明 |

| active | active | ✅ 是 | 正常状态 |
| active | suspended | ❌ 否 | 授权暂停（可恢复） |

| active | expired | ❌ 否 | 已过期 |

## 🎭 使用场景

### 场景1: 租户欠费
```
1. 暂停授权: POST /tenants/:id/suspend
   → licenseStatus: active -> suspended
   → 租户无法登录

2. 缴费后恢复: POST /tenants/:id/resume
   → licenseStatus: suspended -> active
   → 租户恢复使用
```

### 场景2: 租户违规
```
1. 禁用租户: POST /tenants/:id/disable
   → status: active -> inactive
   → 租户无法登录

2. 整改后启用: POST /tenants/:id/enable
   → status: inactive -> active
   → 租户恢复使用
```

### 场景3: 租户到期
```
1. 系统自动检测过期
   → expireDate < now
   → 租户无法登录

2. 续期: POST /tenants/:id/renew
   → expireDate 延长12个月
   → licenseStatus: expired -> active
   → 租户恢复使用
```

## 💡 技术亮点

### 1. 双状态设计
- 清晰分离账号状态和授权状态
- 提供精细化的控制能力
- 灵活应对各种业务场景

### 2. 状态检查
- 每个操作都检查当前状态
- 避免重复操作
- 提供友好的错误提示

### 3. 恢复授权时的过期检查
```typescript
if (tenant.isExpired()) {
  return 400 "租户已过期，请先续期"
}
```
防止恢复已过期的授权。

### 4. 续期时的状态恢复
```typescript
if (tenant.licenseStatus === 'expired') {
  tenant.licenseStatus = 'active';
}
```
续期时自动恢复过期状态。

### 5. 灵活的续期计算
```typescript
const baseDate = currentExpireDate > new Date() 
  ? currentExpireDate  // 从当前过期时间延长
  : new Date();        // 从现在开始延长
```
