## 🎓 使用指南

### 1. 导出租户数据
```bash
# 创建导出任务
curl -X POST \
  http://localhost:3000/api/v1/admin/tenants/TENANT_ID/export \
  -H "Authorization: Bearer TOKEN" \
  -d '{"tables": ["customers", "orders", "products"]}'

# 查询导出进度
curl -X GET \
  http://localhost:3000/api/v1/admin/tenants/TENANT_ID/export/JOB_ID \
  -H "Authorization: Bearer TOKEN"

# 下载导出文件
curl -X GET \
  http://localhost:3000/api/v1/admin/tenants/TENANT_ID/export/JOB_ID/download \
  -H "Authorization: Bearer TOKEN" \
  -o export_data.json
```

### 2. 导入租户数据
```bash
# 创建导入任务
curl -X POST \
  http://localhost:3000/api/v1/admin/tenants/TENANT_ID/import \
  -H "Authorization: Bearer TOKEN" \
  -F "file=@export_data.json" \
  -F "conflictStrategy=skip"

# 查询导入进度
curl -X GET \
  http://localhost:3000/api/v1/admin/tenants/TENANT_ID/import/JOB_ID \
  -H "Authorization: Bearer TOKEN"
```

### 3. 查询操作日志
```bash
# 查询租户日志
curl -X GET \
  "http://localhost:3000/api/v1/admin/tenants/TENANT_ID/logs?page=1&pageSize=20" \
  -H "Authorization: Bearer TOKEN"

# 查询所有日志
curl -X GET \
  "http://localhost:3000/api/v1/admin/logs?page=1&pageSize=20" \
  -H "Authorization: Bearer TOKEN"

# 获取操作统计
curl -X GET \
  "http://localhost:3000/api/v1/admin/logs/stats?tenantId=TENANT_ID" \
  -H "Authorization: Bearer TOKEN"
```

### 4. 执行性能优化
```bash
# 在数据库中执行
mysql -u root -p crm_local < backend/database-migrations/optimize-tenant-performance.sql

# 运行性能测试
node backend/test-tenant-performance.js
```

## ⚠️ 注意事项

### 1. 数据导出/导入
- 大数据量导出可能需要较长时间
- 导入前建议先备份数据
- 注意选择合适的冲突处理策略

### 2. 操作日志
- 日志会持续增长，建议定期归档
- 敏感信息已脱敏处理
- 日志不可修改和删除

### 3. 性能优化
- 索引优化后需要重启应用
- 定期执行 ANALYZE TABLE 更新统计信息
- 监控慢查询日志

### 4. 测试环境
- 测试脚本需要 Admin 权限
- 确保后端服务正常运行
- 测试数据会写入数据库
