- 选择要导出的表(用户、客户、订单、产品)

- 创建导出任务
- 自动轮询检查导出进度
- 导出完成后自动下载

**API调用**:
```javascript
POST /api/admin/tenants/:id/export
GET /api/admin/tenants/:id/export/:jobId
GET /api/admin/tenants/:id/export/:jobId/download
```

#### 4. 导入数据对话框

**功能**:

  - 遇到冲突时报错
- 创建导入任务
- 自动轮询检查导入进度
- 导入完成后刷新页面数据和日志

**API调用**:
```javascript
POST /api/admin/tenants/:id/import
GET /api/admin/tenants/:id/import/:jobId
```

### 4. 进度轮询机制

**导出进度轮询**:
```javascript
const checkProgress = setInterval(async () => {
  const progressRes = await request.get(`/tenants/${id}/export/${jobId}`)
  if (status === 'completed') {
    clearInterval(checkProgress)
    // 下载文件
    window.open(`/api/admin/tenants/${id}/export/${jobId}/download`, '_blank')
  } else if (status === 'failed') {
    clearInterval(checkProgress)
    ElMessage.error('导出失败')
  }
}, 2000) // 每2秒检查一次
```

**导入进度轮询**:

const checkProgress = setInterval(async () => {

## 用户交互流程

### 导出数据流程

1. 用户点击"导出数据"按钮
2. 弹出导出对话框
3. 用户选择要导出的表和时间范围
4. 点击"开始导出"
5. 系统创建导出任务,显示"导出任务已创建,正在处理..."
6. 后台自动轮询检查进度
7. 导出完成后显示"导出完成!正在下载..."
8. 自动触发文件下载

### 导入数据流程

1. 用户点击"导入数据"按钮
2. 弹出导入对话框
3. 用户选择JSON文件和冲突处理策略
4. 点击"开始导入"
5. 系统上传文件并创建导入任务,显示"导入任务已创建,正在处理..."
6. 后台自动轮询检查进度
7. 导入完成后显示"导入完成!共处理 X/Y 条记录"
8. 自动刷新页面数据和操作日志

### 查看日志流程

**租户详情页面**:
1. 页面加载时自动获取操作日志
2. 显示最近20条日志记录
3. 用户可以点击"刷新"按钮更新日志
4. 支持分页浏览历史日志

**租户列表页面**:
1. 用户点击操作列的"更多" → "查看日志"
2. 跳转到租户详情页面
3. 在详情页面查看完整的操作日志

## 已知限制

1. **导出/导入进度轮询**: 使用2秒间隔轮询,对于大数据量可能需要较长时间
2. **文件大小限制**: 导入文件最大100MB
3. **并发限制**: 未实现并发导出/导入的限制,可能导致服务器压力
4. **错误恢复**: 导出/导入失败后需要重新操作,不支持断点续传
