# 第四阶段 - 快速验收指南

## 🚀 快速开始

### 步骤 1：准备数据库

执行以下 SQL 脚本创建必要的表：

```bash
# 1. 创建租户日志表
mysql -u root -p crm_local < backend/database-migrations/create-tenant-logs-table.sql

# 2. 执行性能优化
mysql -u root -p crm_local < backend/database-migrations/optimize-tenant-performance.sql
```

或者在 MySQL 客户端中直接执行：

```sql
-- 使用数据库
USE crm_local;

-- 创建租户日志表
SOURCE backend/database-migrations/create-tenant-logs-table.sql;

-- 执行性能优化
SOURCE backend/database-migrations/optimize-tenant-performance.sql;
```

### 步骤 2：重启后端服务

**重要**：必须重启后端服务以加载新的实体类（TenantLog）

```bash
# 停止当前后端服务（Ctrl+C）
# 然后重新启动
cd backend
npm run dev
```

或者使用快捷脚本：
```bash
restart-backend.bat
```

### 步骤 3：运行验收测试

```bash
cd backend
node test-phase4-complete.js
```

## 🔍 故障排查

### 问题 1：测试脚本报 500 错误
**原因**：后端服务未重启，TenantLog 实体未加载  
**解决**：重启后端服务

### 问题 2：测试脚本报 404 错误
**原因**：路由未正确注册  
**解决**：检查 `backend/src/routes/admin/index.ts` 和 `backend/src/routes/admin/tenants.ts`

### 问题 3：数据库表不存在
**原因**：未执行数据库迁移脚本  
**解决**：执行步骤 1 中的 SQL 脚本

### 问题 4：登录失败
**原因**：Admin 用户不存在  
**解决**：
```bash
cd backend
node create-admin-user.js
```
