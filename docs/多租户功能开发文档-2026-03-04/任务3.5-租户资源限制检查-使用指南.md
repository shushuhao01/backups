# 任务3.5 - 租户资源限制检查 - 使用指南

## 📋 功能概述

租户资源限制检查中间件用于在SaaS多租户模式下，限制每个租户的资源使用，包括：
1. **用户数限制**: 限制租户可创建的最大用户数
2. **存储空间限制**: 限制租户可使用的最大存储空间

## 🎯 核心功能

### 1. 用户数限制检查 (`checkUserLimit`)
- 在创建用户前检查是否超过租户的 `max_users` 限制
- 返回友好的错误提示，包含当前用户数和使用率
- 当使用率超过90%时记录警告日志

### 2. 存储空间限制检查 (`checkStorageLimit`)
- 在上传文件前检查是否超过租户的 `max_storage_gb` 限制
- 支持多种文件大小获取方式（Content-Length、multer等）
- 返回友好的错误提示，包含剩余空间和使用率
- 当使用率超过90%时记录警告日志

### 3. 资源统计更新
- `updateTenantUserCount`: 增加租户用户数统计
- `decrementTenantUserCount`: 减少租户用户数统计
- `updateTenantStorage`: 增加租户存储空间统计
- `decrementTenantStorage`: 减少租户存储空间统计

### 4. 资源使用情况查询
- `getTenantResourceUsage`: 获取租户资源使用详情
- 返回用户数和存储空间的使用情况、使用率、剩余量等

## 📝 使用方法

### 1. 在用户创建路由中使用

```typescript
// backend/src/routes/users.ts
import { checkUserLimit, updateTenantUserCount } from '../middleware/checkTenantLimits';
import { tenantAuth } from '../middleware/tenantAuth';

// 创建用户
router.post('/', 
  tenantAuth,           // 租户认证
  checkUserLimit,       // 检查用户数限制
  async (req, res) => {
    try {
      // 创建用户逻辑
      const user = await createUser(req.body);
      
      // 更新租户用户数统计
      if (req.tenantId) {
        await updateTenantUserCount(req.tenantId, 1);
      }
      
      res.json({ success: true, data: user });
    } catch (error) {
      res.status(500).json({ success: false, message: error.message });
    }
  }
);
```

### 2. 在文件上传路由中使用

```typescript
// backend/src/routes/upload.ts
import { checkStorageLimit, updateTenantStorage } from '../middleware/checkTenantLimits';
import { tenantAuth } from '../middleware/tenantAuth';
import multer from 'multer';

const upload = multer({ dest: 'uploads/' });

// 上传文件
router.post('/upload', 
  tenantAuth,           // 租户认证
  checkStorageLimit,    // 检查存储空间限制
  upload.single('file'),
  async (req, res) => {
    try {
      // 文件上传逻辑
      const file = req.file;
      
      // 更新租户存储空间统计
      if (req.tenantId && file) {
        const fileSizeMb = file.size / (1024 * 1024);
        await updateTenantStorage(req.tenantId, fileSizeMb);
      }
      
      res.json({ success: true, data: { filename: file.filename } });
    } catch (error) {
      res.status(500).json({ success: false, message: error.message });
    }
  }
);
```

### 3. 在删除用户时更新统计

```typescript
// backend/src/routes/users.ts
import { decrementTenantUserCount } from '../middleware/checkTenantLimits';

// 删除用户
router.delete('/:id', 
  tenantAuth,
  async (req, res) => {
    try {
      const user = await findUserById(req.params.id);
      
      // 删除用户
      await deleteUser(req.params.id);
      
      // 更新租户用户数统计
      if (user.tenantId) {
        await decrementTenantUserCount(user.tenantId, 1);
      }
      
      res.json({ success: true });
    } catch (error) {
      res.status(500).json({ success: false, message: error.message });
    }
  }
);
```

### 4. 在删除文件时更新统计

```typescript
// backend/src/routes/upload.ts
import { decrementTenantStorage } from '../middleware/checkTenantLimits';

// 删除文件
router.delete('/files/:id', 
  tenantAuth,
  async (req, res) => {
    try {
      const file = await findFileById(req.params.id);
      
      // 删除文件
      await deleteFile(req.params.id);
      
      // 更新租户存储空间统计
      if (file.tenantId) {
        const fileSizeMb = file.size / (1024 * 1024);
        await decrementTenantStorage(file.tenantId, fileSizeMb);
      }
      
      res.json({ success: true });
    } catch (error) {
      res.status(500).json({ success: false, message: error.message });
    }
  }
);
```

### 5. 查询资源使用情况

```typescript
// backend/src/routes/dashboard.ts
import { getTenantResourceUsage } from '../middleware/checkTenantLimits';

// 获取资源使用情况
router.get('/resource-usage', 
  tenantAuth,
  async (req, res) => {
    try {
      const usage = await getTenantResourceUsage(req.tenantId);
      res.json({ success: true, data: usage });
    } catch (error) {
      res.status(500).json({ success: false, message: error.message });
    }
  }
);
```

## 📌 注意事项

### 1. 私有部署模式兼容
- 资源限制检查只在SaaS模式下生效
- 私有部署模式下会自动跳过检查

### 2. 统计数据准确性
- 创建/删除用户后必须调用统计更新函数
- 上传/删除文件后必须调用统计更新函数
- 建议定期运行数据校准脚本

### 3. 性能考虑
- 资源限制检查会增加一次数据库查询
- 建议在租户认证中间件中缓存租户信息
- 统计更新是异步操作，不会阻塞主流程

### 4. 错误处理
- 资源限制检查失败返回403状态码
- 统计更新失败只记录日志，不影响主流程
- 建议在前端显示友好的错误提示

### 5. 监控和告警
- 当资源使用率超过90%时会记录警告日志
- 建议配置日志监控和告警
- 定期检查租户资源使用情况

## 🔧 后续优化建议

1. **缓存优化**: 在租户认证中间件中缓存租户信息，减少数据库查询
2. **批量更新**: 支持批量创建用户时的统计更新
3. **异步统计**: 使用消息队列异步更新统计数据
4. **数据校准**: 定期运行脚本校准统计数据
5. **资源预警**: 当资源使用率达到阈值时发送通知
6. **资源配额**: 支持更细粒度的资源配额管理（如API调用次数、并发连接数等）
