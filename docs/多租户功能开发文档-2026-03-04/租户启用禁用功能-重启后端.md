# 租户启用/禁用功能 - 需要重启后端

## 问题
点击启用/禁用开关时，提示"Request failed with status code 404"

## 原因
后端路由已经正确配置，但**后端服务没有重启**，新的路由还没有生效。

## 解决方案

### 1. 重启后端服务

**Windows (PowerShell)**:
```powershell
# 停止后端
cd backend
# 按 Ctrl+C 停止当前运行的服务

# 重新启动
npm run dev
```

**或者使用重启脚本**:
```powershell
.\restart-backend.bat
```

### 2. 验证路由

后端启动后，应该能看到类似的日志：
```
[Admin] Admin routes loaded
[Admin] Tenant routes registered
Server running on port 3000
```

### 3. 测试API

可以使用测试脚本验证：
```bash
cd backend
node test-admin-tenant-routes.js
```

## 已配置的API端点

所有端点都已正确配置：

### 租户状态管理
- `POST /api/v1/admin/tenants/:id/enable` - 启用租户 ✅
- `POST /api/v1/admin/tenants/:id/disable` - 禁用租户 ✅

### 授权管理
- `POST /api/v1/admin/tenants/:id/suspend` - 暂停授权 ✅
- `POST /api/v1/admin/tenants/:id/resume` - 恢复授权 ✅

### 其他操作
- `POST /api/v1/admin/tenants/:id/renew` - 续期 ✅
- `POST /api/v1/admin/tenants/:id/regenerate-license` - 重新生成授权码 ✅
- `DELETE /api/v1/admin/tenants/:id` - 删除租户 ✅

## 前端配置

前端配置也是正确的：

**API Base URL**: `/api/v1/admin` ✅

**请求示例**:
```typescript
// 启用
await request.post(`/tenants/${row.id}/enable`)

// 禁用
await request.post(`/tenants/${row.id}/disable`)
```
