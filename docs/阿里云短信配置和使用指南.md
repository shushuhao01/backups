## 1️⃣ 申请模板时如何填写变量

### 在阿里云控制台申请模板时的操作步骤：

#### 步骤1：登录阿里云短信服务控制台
- 访问：https://dysms.console.aliyun.com/
- 进入"国内消息" -> "模板管理"

#### 步骤2：点击"添加模板"

#### 步骤3：填写模板信息

**以"注册成功通知"模板为例：**

| 字段 | 填写内容 | 说明 |
|------|---------|------|
| 模板类型 | 短信通知 | 验证码选"验证码"，其他选"短信通知" |
| 模板名称 | 租户注册成功通知 | 自定义，方便识别 |
| 模板内容 | 见下方 | 按照阿里云格式填写 |

**模板内容填写格式：**
```
尊敬的${tenantName}，您的云客CRM账号已开通！租户编码：${tenantCode}，授权码：${licenseKey}，管理员账号为您的手机号，初始密码：${adminPassword}，套餐：${packageName}，到期日期：${expireDate}。请妥善保管账号信息，登录后请立即修改密码。
```

**⚠️ 重要说明：**
1. 不要在模板内容开头添加【】签名符号，阿里云会自动添加签名
2. 不要将URL作为变量（如 `${loginUrl}`），阿里云不允许
3. 不要将手机号作为变量（如 `${adminUsername}`），改用固定文字"管理员账号为您的手机号"

#### 步骤4：添加变量属性

**重要：在阿里云控制台的"变量属性"部分，需要为每个变量选择类型**

根据你提供的截图，阿里云提供了以下变量类型选项：
- **仅数字**
- **数字+字母组合或仅字母**
- **仅数字，1-2位**

**变量类型选择对照表：**

根据阿里云控制台提供的变量属性选项，正确选择如下：

| 变量名 | 阿里云变量属性选择 | 说明 | 示例值 |
|--------|------------------|------|--------|
| `tenantName` | 企业/组织名称 | 企业名称，可能包含中文 | 某某科技有限公司 |
| `tenantCode` | 其他（姓名、账号、地址等） | 租户编码，6位字母数字 | ABC123 |
| `licenseKey` | 其他（姓名、账号、地址等） | 授权码，格式：TENANT-XXXX | TENANT-A1B2-C3D4-E5F6 |
| `adminPassword` | 其他（姓名、账号、地址等） | 初始密码，8位随机字符 | Abc12345 |
| `packageName` | 其他（姓名、账号、地址等） | 套餐名称 | 专业版 |
| `expireDate` | 时间 | 到期日期，格式：2026-12-31 | 2026-12-31 |
| `code` | 仅数字 | 验证码（6位数字） | 123456 |
| `orderNo` | 其他（姓名、账号、地址等） | 订单号 | PAY20260328120000ABC123 |
| `amount` | 仅数字 | 金额（不带小数点） | 299 |
| `months` | 仅数字 | 月数 | 12 |
| `maxUsers` | 仅数字 | 用户数 | 50 |
| `maxStorageGb` | 仅数字 | 存储空间GB | 50 |
| `remainDays` | 仅数字 | 剩余天数 | 7 |
| `reason` | 其他（姓名、账号、地址等） | 原因说明 | 账号欠费 |
| `servicePhone` | 仅数字 | 客服电话 | 4001234567 |
| `refundAmount` | 仅数字 | 退款金额 | 299 |
| `refundDays` | 仅数字 | 到账工作日 | 3 |
| `retentionDays` | 仅数字 | 数据保留天数 | 30 |
| `newPackageName` | 其他（姓名、账号、地址等） | 新套餐名称 | 企业版 |

**⚠️ 重要提示：**
1. 阿里云的变量属性选项包括：
   - 时间
   - 仅数字
   - 企业/组织名称
   - 其他（姓名、账号、地址等）
2. 对于包含中文的企业名称，选择"企业/组织名称"
3. 对于纯数字的变量（如验证码、金额、天数），选择"仅数字"
4. 对于混合内容的变量（如授权码、订单号、密码），选择"其他（姓名、账号、地址等）"
5. 对于日期格式，选择"时间"

#### 步骤5：填写申请说明

在"申请说明"中简要描述模板用途，例如：
```
用于用户注册成功后，发送账号开通通知，包含登录所需的全部信息。
```

#### 步骤6：提交审核

提交后等待1-2个工作日审核，审核通过后会获得一个模板CODE，格式如：`SMS_123456789`

## 2️⃣ 审核通过后如何配置到系统

### 方式一：通过Admin后台配置（推荐）

#### 步骤1：登录Admin后台
访问：http://localhost:5174 （开发环境）或你的Admin后台地址

#### 步骤2：进入系统设置
导航：系统设置 -> 基础设置 -> 通知服务配置

#### 步骤3：配置阿里云短信参数

填写以下信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| AccessKey ID | 阿里云访问密钥ID | LTAI5t... |
| AccessKey Secret | 阿里云访问密钥Secret | 保密信息 |
| 短信签名 | 申请的短信签名 | 云客CRM |
| 是否启用 | 开启短信功能 | ✅ 启用 |

#### 步骤4：配置短信模板CODE

在"短信模板配置"部分，填写每个场景对应的模板CODE：

```json
{
  "VERIFY_CODE": "SMS_123456789",           // 验证码模板CODE
  "REGISTER_SUCCESS": "SMS_234567890",      // 注册成功模板CODE
  "PAYMENT_SUCCESS": "SMS_345678901",       // 支付成功模板CODE
  "RENEW_SUCCESS": "SMS_456789012",         // 续期成功模板CODE
  "PACKAGE_CHANGE": "SMS_567890123",        // 套餐变更模板CODE
  "QUOTA_CHANGE": "SMS_678901234",          // 配额变更模板CODE
  "ACCOUNT_SUSPEND": "SMS_789012345",       // 账号暂停模板CODE
  "ACCOUNT_RESUME": "SMS_890123456",        // 账号激活模板CODE
  "ACCOUNT_CANCEL": "SMS_901234567",        // 账号注销模板CODE
  "REFUND_SUCCESS": "SMS_012345678",        // 退款成功模板CODE
  "EXPIRE_REMIND": "SMS_123450000",         // 到期提醒模板CODE
  "EXPIRED_NOTICE": "SMS_234560000"         // 过期通知模板CODE
}
```

#### 步骤5：保存配置

点击"保存"按钮，系统会将配置加密存储到数据库的 `system_config` 表中。

---

### 方式二：通过环境变量配置

如果你不想通过Admin后台配置，也可以直接在环境变量中配置：

#### 编辑 `backend/.env` 文件：

```env
# 阿里云短信配置
ALIYUN_SMS_ACCESS_KEY_ID=你的AccessKey_ID
ALIYUN_SMS_ACCESS_KEY_SECRET=你的AccessKey_Secret
ALIYUN_SMS_SIGN_NAME=云客CRM

# 短信模板CODE（验证码）
ALIYUN_SMS_TEMPLATE_CODE=SMS_123456789

# 其他模板CODE（可选，如果不配置则使用默认模板）
ALIYUN_SMS_TEMPLATE_CODE_REGISTER=SMS_234567890
ALIYUN_SMS_TEMPLATE_CODE_PAYMENT=SMS_345678901
ALIYUN_SMS_TEMPLATE_CODE_RENEW=SMS_456789012
# ... 其他模板CODE
```

**⚠️ 注意：**
- 环境变量配置优先级低于Admin后台配置
- 如果Admin后台已配置，环境变量配置将被忽略
- 生产环境建议使用Admin后台配置，更安全

---

### 方式三：直接修改数据库（不推荐）

如果前两种方式都不可用，可以直接在数据库中配置：

```sql
-- 插入或更新短信配置
INSERT INTO system_config (config_key, config_value, config_type, description)
VALUES (
  'sms_config',
  '{
    "enabled": true,
    "accessKeyId": "你的AccessKey_ID",
    "accessKeySecret": "你的AccessKey_Secret",
    "signName": "云客CRM",
    "templateCode": "SMS_123456789",
    "templates": {
      "VERIFY_CODE": "SMS_123456789",
      "REGISTER_SUCCESS": "SMS_234567890",
      "PAYMENT_SUCCESS": "SMS_345678901"
    }
  }',
  'json',
  '阿里云短信服务配置'
)
ON DUPLICATE KEY UPDATE
  config_value = VALUES(config_value),
  updated_at = NOW();
```

## 3️⃣ 系统如何自动调用发送

### 系统会在以下场景自动发送短信：

#### 场景1：用户注册时发送验证码

**触发时机：** 用户在官网注册页面点击"获取验证码"

**代码位置：** `backend/src/controllers/public/RegisterController.ts`

**发送流程：**
```typescript
// 1. 用户点击"获取验证码"
// 2. 前端调用 POST /api/v1/public/register/send-code
// 3. 后端生成6位随机验证码
// 4. 调用 AliyunSmsService.sendVerificationCode(phone, code)
// 5. 阿里云短信服务发送短信
// 6. 用户收到验证码短信
```

**自动传递的变量：**
- `code`: 6位数字验证码（如：123456）

---

#### 场景2：注册成功后发送账号信息

**触发时机：** 用户完成注册并创建租户成功

**代码位置：** `backend/src/controllers/public/RegisterController.ts`

**发送流程：**
```typescript
// 1. 用户提交注册表单
// 2. 后端创建租户和管理员账号
// 3. 生成授权码和初始密码
// 4. 调用 NotificationTemplateService.sendByTemplate('REGISTER_SUCCESS', variables)
// 5. 系统自动填充变量并发送短信
```

**自动传递的变量：**
```javascript
{
  tenantName: "某某科技有限公司",      // 从注册表单获取
  tenantCode: "ABC123",                // 系统自动生成
  licenseKey: "TENANT-A1B2-C3D4-...",  // 系统自动生成
  adminPassword: "Abc12345",           // 系统自动生成
  packageName: "专业版",               // 从套餐信息获取
  expireDate: "2026-12-31"             // 根据套餐计算
}
```

**注意：** 已移除 `adminUsername` 和 `loginUrl` 变量，因为阿里云不允许手机号和URL作为变量传入。

---

#### 场景3：支付成功后发送通知

**触发时机：** 微信/支付宝支付成功回调，或管理员确认对公转账到账

**代码位置：** `backend/src/services/PaymentService.ts`

**发送流程：**
```typescript
// 1. 收到支付成功回调
// 2. 更新订单状态为"已支付"
// 3. 激活租户并生成授权码
// 4. 调用 NotificationTemplateService.sendByTemplate('PAYMENT_SUCCESS', variables)
// 5. 发送短信通知
```

**自动传递的变量：**
```javascript
{
  tenantName: "某某科技有限公司",
  orderNo: "PAY20260328120000ABC123",
  amount: "299",
  packageName: "专业版",
  expireDate: "2027-03-28",
  licenseKey: "TENANT-A1B2-C3D4-..."
}
```

**注意：** 已移除 `loginUrl` 变量。

---

#### 场景4：管理员操作触发的短信

**触发时机：** 管理员在Admin后台进行以下操作：
- 续期租户
- 变更套餐
- 暂停/激活账号
- 注销账号
- 处理退款

**代码位置：** `backend/src/controllers/admin/TenantController.ts`

**示例：续期操作**
```typescript
// 管理员点击"续期"按钮
// -> renewTenant() 方法
// -> 更新过期时间
// -> 调用 NotificationTemplateService.sendByTemplate('RENEW_SUCCESS', variables)
// -> 自动发送短信
```

**自动传递的变量：**
```javascript
{
  tenantName: "某某科技有限公司",
  months: "12",
  expireDate: "2027-03-28"
}
```

---

#### 场景5：定时任务触发的短信

**触发时机：** 系统定时检查租户到期情况

**代码位置：** `backend/src/services/LicenseExpirationReminderService.ts`

**定时任务配置：**
- 每天凌晨2点检查所有租户
- 到期前7天、3天、1天发送提醒
- 过期当天发送过期通知

**自动传递的变量：**
```javascript
{
  tenantName: "某某科技有限公司",
  expireDate: "2026-04-04",
  remainDays: "7",
  servicePhone: "4001234567"
}
```

### 系统自动发送短信的核心流程

```
┌─────────────────┐
│  业务触发事件    │ (注册、支付、续期等)
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 调用通知服务     │ NotificationTemplateService.sendByTemplate()
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 1. 查询模板配置  │ 从数据库读取模板CODE
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 2. 替换变量      │ 将业务数据填充到模板变量
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 3. 调用阿里云API │ AliyunSmsService.sendVerificationCode()
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 4. 阿里云发送    │ 阿里云短信服务
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 5. 用户收到短信  │ 📱
└─────────────────┘
```

## 4️⃣ 完整配置流程（从零开始）

### 第一步：申请阿里云短信服务

1. 登录阿里云控制台
2. 开通"短信服务"
3. 创建AccessKey（保存好ID和Secret）
4. 申请短信签名（如：【云客CRM】）
5. 等待签名审核通过（1-2个工作日）

### 第二步：申请短信模板

按照《阿里云短信模板申请清单.md》中的12个模板，逐条申请：

**优先申请的模板（高优先级）：**
1. ✅ 注册验证码（验证码类型）
2. ✅ 注册成功通知
3. ✅ 支付成功通知
4. ✅ 账号到期提醒
5. ✅ 账号过期通知

**后续申请的模板（中低优先级）：**
6. 续期成功通知
7. 套餐变更通知
8. 账号暂停通知
9. 账号激活通知
10. 配额变更通知
11. 账号注销通知
12. 退款成功通知

### 第三步：配置到系统

**方式A：通过Admin后台配置（推荐）**
1. 登录Admin后台
2. 进入"系统设置" -> "基础设置"
3. 填写AccessKey ID、Secret、签名
4. 填写各个模板的CODE
5. 保存配置

**方式B：通过环境变量配置**
1. 编辑 `backend/.env` 文件
2. 添加阿里云短信配置
3. 重启后端服务

### 第四步：测试短信发送

#### 测试1：验证码短信
```bash
# 在官网注册页面测试
# 访问：http://localhost:8080/register
# 输入手机号，点击"获取验证码"
# 检查是否收到短信
```

#### 测试2：注册成功短信
```bash
# 完成注册流程
# 检查是否收到包含授权码的短信
```

#### 测试3：支付成功短信
```bash
# 模拟支付成功
node backend/simulate-payment-success.js
# 检查是否收到支付成功短信
```

### 第五步：监控和日志

#### 查看短信发送日志
```sql
-- 查看最近的短信发送记录
SELECT * FROM notification_logs 
WHERE channel = 'sms'

```

#### 查看阿里云短信控制台
- 登录阿里云短信服务控制台
- 查看"发送记录"
- 检查发送状态和失败原因

## 🔧 常见问题

### Q1: 短信发送失败，提示"签名不合法"
**A:** 检查签名是否审核通过，签名名称是否正确（不要带【】符号）

### Q2: 短信发送失败，提示"模板不存在"
**A:** 检查模板CODE是否正确，模板是否审核通过

### Q3: 短信发送失败，提示"余额不足"
**A:** 登录阿里云控制台充值

### Q4: 开发环境如何测试短信？
**A:** 开发环境默认使用固定验证码123456，不会真实发送短信。如需测试真实发送，需要配置真实的阿里云参数。

### Q5: 如何查看短信发送是否成功？
**A:** 
1. 查看后端日志：`backend/backend.log`
2. 查看数据库日志：`notification_logs` 表
3. 查看阿里云控制台：短信服务 -> 发送记录

### Q6: 变量值太长怎么办？
**A:** 阿里云短信单条最多500字符，如果变量值太长，需要简化模板内容或拆分成多条短信。

### Q7: 如何修改短信模板内容？
**A:** 需要在阿里云控制台重新申请模板，审核通过后更新模板CODE到系统配置。
