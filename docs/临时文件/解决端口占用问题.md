# 解决端口5173占用问题

## 问题
端口5173已经被占用,无法启动CRM前端服务

## 解决方案

### 方法1:查找并关闭占用端口的进程(推荐)

```powershell
# 1. 查找占用5173端口的进程
netstat -ano | findstr :5173

# 你会看到类似这样的输出:
# TCP    0.0.0.0:5173    0.0.0.0:0    LISTENING    12345
# 最后一列的数字(12345)就是进程ID(PID)

# 2. 结束该进程(将12345替换为实际的PID)
taskkill /PID 12345 /F

# 3. 重新启动CRM前端
npm run dev
```

### 方法2:一键解决(推荐使用)

直接执行以下命令:

```powershell
# 查找并结束占用5173端口的进程
FOR /F "tokens=5" %P IN ('netstat -ano ^| findstr :5173') DO taskkill /PID %P /F

# 然后启动服务
npm run dev
```

### 方法3:修改端口(临时方案)

如果不想关闭现有进程,可以临时使用其他端口:

```powershell
# 使用5174端口启动(但Admin已经用了5174)
npm run dev -- --port 5175

# 然后访问 http://localhost:5175
```

## 完整操作步骤

在PowerShell中执行:

```powershell
# 进入CRM目录
cd D:\kaifa\CRM` - 1.8.0

# 查找占用5173的进程
netstat -ano | findstr :5173

# 假设输出显示PID是12345,结束它
taskkill /PID 12345 /F

# 启动服务
npm run dev
```

## 如果还是不行

可能有多个进程占用,执行:

```powershell
# 结束所有node进程
taskkill /F /IM node.exe

# 然后重新启动
npm run dev
```

⚠️ 注意:这会关闭所有Node.js进程,包括后端和Admin前端,需要重新启动它们
