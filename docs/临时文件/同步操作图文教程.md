## 🎬 完整操作流程（手把手）

### 场景: 你修改了订单功能

#### 第1步: 修改代码
```
你在IDE里打开并修改了:
src/views/Order/List.vue
```

#### 第2步: 提交到主仓库

在IDE底部的终端，依次输入这3条命令:

```powershell
# 命令1: 添加所有修改的文件
git add .

# 命令2: 提交并写说明（这里的说明很重要！）
git commit -m "优化订单列表显示"

# 命令3: 推送到GitHub
git push origin main
```

**重点**: 看到 `"优化订单列表显示"` 了吗？这就是你的提交信息!

#### 第3步: 同步到子仓库

还是在同一个终端，输入:

```powershell
.\sync-repos-v2.ps1 -message "优化订单列表显示"
```

**重点**: 这里的 `"优化订单列表显示"` 和第2步的一模一样!

## 💡 提交信息怎么写？

提交信息就是描述你做了什么，随便写，比如:

### 好的例子:
```powershell
git commit -m "修复订单bug"
.\sync-repos-v2.ps1 -message "修复订单bug"

git commit -m "添加客户搜索功能"
.\sync-repos-v2.ps1 -message "添加客户搜索功能"

git commit -m "优化界面显示"
.\sync-repos-v2.ps1 -message "优化界面显示"

git commit -m "更新"
.\sync-repos-v2.ps1 -message "更新"
```

### 简单粗暴的例子:
```powershell
git commit -m "修改"
.\sync-repos-v2.ps1 -message "修改"

git commit -m "更新代码"
.\sync-repos-v2.ps1 -message "更新代码"

git commit -m "fix"
.\sync-repos-v2.ps1 -message "fix"
```

**都可以!** 只要两个地方写的一样就行。

## 🎯 最简单的方法

如果你觉得每次都要写两遍很麻烦，可以这样:

### 方法1: 复制粘贴

```powershell
# 第2步: 提交（写一次）
git commit -m "优化订单列表"

# 第3步: 同步（复制上面那句话）
.\sync-repos-v2.ps1 -message "优化订单列表"
```

### 方法2: 用简单的词

```powershell
# 第2步
git commit -m "更新"

# 第3步
.\sync-repos-v2.ps1 -message "更新"
```

### 方法3: 一次性执行（高级）

把3条命令写在一起:

```powershell
git add . ; git commit -m "更新功能" ; git push origin main ; .\sync-repos-v2.ps1 -message "更新功能"
```

这样只需要输入一次，全部自动执行!

## ❓ 常见问题

### Q: 我不知道写什么提交信息怎么办？
A: 随便写!比如:
- "更新"
- "修改"
- "fix"
- "update"
都可以!

### Q: 两个地方的提交信息必须一样吗？
A: 不是必须，但建议一样，这样你知道同步的是什么内容。

### Q: 我可以不写提交信息吗？
A: 不行，Git要求必须写。但可以写很简单的，比如 "update"。

### Q: 如果我忘记同步了怎么办？
A: 没关系!随时可以补同步:
```powershell
.\sync-repos-v2.ps1 -message "补同步"
```

### Q: 每次都要同步吗？
A: 不是!同步是可选的。只有当你需要更新子仓库时才同步。
