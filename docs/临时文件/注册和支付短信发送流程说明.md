# 注册和支付短信发送流程说明

## 业务场景分析

系统有两种注册场景，短信发送时机和内容完全不同：

### 场景A：免费试用套餐（price = 0）

```
用户注册
  ↓
立即创建租户（status='active'）
  ↓
立即生成授权码
  ↓
立即创建管理员账号
  ↓
发送"账号开通通知"短信 ✅
  ↓
用户收到短信，包含：
  - 租户编码
  - 授权码
  - 管理员账号（您的手机号）
  - 初始密码
  - 套餐名称
  - 到期日期
  ↓
用户可以立即登录使用
```

**短信模板：账号开通通知**
```
尊敬的${tenantName}，您的云客CRM账号已开通！租户编码：${tenantCode}，授权码：${licenseKey}，
管理员账号为您的手机号，初始密码：${adminPassword}，套餐：${packageName}，到期日期：${expireDate}。
请妥善保管账号信息，登录后请立即修改密码。
```

---

### 场景B：付费套餐（price > 0）

```
用户注册
  ↓
创建租户（status='inactive'，license_status='pending'）
  ↓
不生成授权码 ❌
  ↓
不创建管理员账号 ❌
  ↓
不发送短信 ❌
  ↓
显示支付页面（微信/支付宝/对公转账）
  ↓
用户完成支付
  ↓
支付回调/管理员确认到账
  ↓
激活租户（status='active'，license_status='active'）
  ↓
生成授权码
  ↓
创建管理员账号
  ↓
发送"支付成功账号开通通知"短信 ✅
  ↓
用户收到短信，包含：
  - 订单号
  - 支付金额
  - 租户编码
  - 授权码
  - 管理员账号（您的手机号）
  - 初始密码
  - 套餐名称
  - 到期日期
  ↓
用户可以登录使用
```

**短信模板：支付成功账号开通通知**
```
尊敬的${tenantName}，您的订单已支付成功！订单号：${orderNo}，支付金额：${amount}元。
您的云客CRM账号已开通，租户编码：${tenantCode}，授权码：${licenseKey}，管理员账号为您的手机号，
初始密码：${adminPassword}，套餐：${packageName}，到期日期：${expireDate}。
请妥善保管账号信息，登录后请立即修改密码。
```

---

## 关键区别

| 项目 | 免费试用 | 付费套餐 |
|------|---------|---------|
| 租户创建时机 | 注册时立即创建 | 注册时创建（未激活） |
| 授权码生成时机 | 注册时立即生成 | 支付成功后生成 |
| 管理员账号创建 | 注册时立即创建 | 支付成功后创建 |
| 短信发送时机 | 注册成功后立即发送 | 支付成功后发送 |
| 短信模板 | 账号开通通知 | 支付成功账号开通通知 |
| 短信内容 | 账号信息 | 支付信息 + 账号信息 |
| 用户可登录时机 | 注册后立即可登录 | 支付成功后可登录 |

### 付费套餐流程

**文件：** `backend/src/services/PaymentService.ts`

**关键代码：**
```typescript
private async activateTenant(tenantId: string, packageId: string): Promise<string | null> {
  // 1. 生成授权码
  const licenseKey = this.generateLicenseKey()
  
  // 2. 更新租户状态为active
  await AppDataSource.query(
    `UPDATE tenants SET 
      license_key = ?, 
      license_status = 'active', 
      status = 'active',
      expire_date = ?, 
      max_users = ?,
      updated_at = NOW()
    WHERE id = ?`,
    [licenseKey, expireDateStr, pkg.max_users, tenantId]
  )
  
  // 3. 创建默认管理员账号
  const adminAccount = await createDefaultAdmin({
    tenantId: tenantId,
    phone: tenant.phone,
    realName: tenant.contact || '系统管理员'
  })
  
  // 4. 发送支付成功账号开通通知
  await this.sendActivationNotification(tenantId, licenseKey, {
    tenantCode: tenant.code,
    tenantName: tenant.name,
    adminPassword: adminAccount.password,
    packageName: pkg.name,
    expireDate: expireDateStr
  })
}
```

## 注意事项

1. **免费试用和付费套餐使用不同的短信模板**
   - 免费试用：账号开通通知（6个变量）
   - 付费套餐：支付成功账号开通通知（8个变量，多了订单号和金额）

2. **付费套餐注册时不发送短信**
   - 避免用户收到没有授权码和密码的无用短信
   - 等支付成功后再发送完整信息

3. **管理员账号创建时机**
   - 免费试用：注册时立即创建
   - 付费套餐：支付成功后创建

4. **授权码生成时机**
   - 免费试用：注册时立即生成
   - 付费套餐：支付成功后生成
