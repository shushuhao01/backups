## 一、功能概述

### 1.1 设计理念
授权码登录验证功能采用**类企业微信的企业ID验证流程**，在用户登录前先验证租户身份，实现多租户SaaS系统的安全隔离。

### 1.2 核心流程
```
用户访问登录页 
  → 检查是否需要授权验证
  → 输入租户编码/授权码
  → 后端验证租户身份
  → 验证通过后显示登录表单
  → 输入用户名密码登录
  → 登录成功进入系统
```

## 四、Backend后端实现分析

### 4.1 租户授权路由 (`backend/src/routes/tenantLicense.ts`)

#### ✅ 完整实现的API端点

**1. GET `/tenant-license/check-private`**
- 功能：检查私有部署激活状态
- 无需认证
- 返回：已激活租户信息或 `{ activated: false }`
- 用途：私有部署模式下，前端可跳过授权码输入

**2. POST `/tenant-license/verify`**
- 功能：验证授权码（首次激活）
- 参数：`{ licenseKey: string }`
- 验证逻辑：
  ```typescript
  // 1. 查询租户
  SELECT * FROM tenants WHERE license_key = ?
  
  // 2. 检查状态
  if (tenant.status === 'disabled') return 403
  if (tenant.license_status === 'suspended') return 403
  if (tenant.expire_date < now) return 403
  
  // 3. 首次激活
  if (tenant.license_status === 'pending') {
    UPDATE tenants SET 
      license_status = 'active',
      activated_at = NOW()
    
    // 发送管理员通知
    adminNotificationService.notify('tenant_login', {...})
  }
  
  // 4. 返回租户信息
  return { tenantId, tenantCode, tenantName, packageName, ... }
  ```

**3. POST `/tenant-license/verify-code`**
- 功能：通过租户编码识别租户（日常登录）
- 参数：`{ tenantCode: string }`
- 特殊处理：
  ```typescript
  if (tenant.license_status === 'pending') {
    return {
      success: false,
      message: '该租户尚未激活，请先使用授权码激活系统',
      needActivation: true  // 前端收到后自动切换到授权码模式
    }
  }
  ```

**4. GET `/tenant-license/info`**
- 功能：获取当前租户信息
- 需要认证：从 `req.tenantId` 获取
- 返回：完整租户信息（授权码脱敏）

**5. POST `/tenant-license/heartbeat`**
- 功能：心跳检测，检查授权是否有效
- 定期调用，防止授权过期后继续使用
- 更新 `last_verify_at` 字段

### 4.2 用户登录集成 (`backend/src/controllers/UserController.ts`)

#### ✅ 租户隔离登录

```typescript
login = catchAsync(async (req: Request, res: Response) => {
  const { username, password, tenantId } = req.body
  
  // 构建查询条件：按租户过滤
  const whereClause: any = { username }
  if (tenantId) {
    whereClause.tenantId = tenantId
  }
  
  // 查找用户（同租户下）
  const user = await this.userRepository.findOne({ where: whereClause })
  
  // ... 密码验证、IP验证等
  
  // 生成JWT令牌（包含tenantId）
  const tokenPayload = {
    userId: user.id,
    username: user.username,
    role: user.roleId || user.role,
    departmentId: user.departmentId,
    tenantId: user.tenantId || undefined  // 🔥 关键：JWT中包含租户ID
  }
  
  const tokens = JwtConfig.generateTokenPair(tokenPayload)
})
```

#### ✅ 租户隔离特性

**1. 用户名唯一性**
- 同租户下用户名唯一
- 不同租户可存在相同用户名

**2. JWT Payload包含租户ID**
```typescript
{
  userId: "user_xxx",
  username: "zhangsan",
  role: "sales_staff",
  tenantId: "tenant_xxx"  // 后续所有请求都带上
}
```

**3. 所有API自动过滤**
```typescript
private getTenantIdFromRequest(req: Request): string | null {
  return (req as any).tenantId || req.body?.tenantId || (req as any).user?.tenantId || null
}

private buildTenantWhere(req: Request, baseWhere: any = {}): any {
  const tenantId = this.getTenantIdFromRequest(req)
  if (tenantId) {
    return { ...baseWhere, tenantId }
  }
  return baseWhere
}
```

## 六、数据库表结构

### 6.1 tenants 表（租户表）

```sql
CREATE TABLE tenants (
  id VARCHAR(36) PRIMARY KEY,
  code VARCHAR(20) UNIQUE NOT NULL,           -- 租户编码 T20260XXXXX
  name VARCHAR(100) NOT NULL,                 -- 租户名称
  license_key VARCHAR(100) UNIQUE NOT NULL,   -- 授权码
  license_status ENUM('pending', 'active', 'expired', 'suspended'),
  status ENUM('active', 'disabled', 'expired'),
  package_id VARCHAR(36),                     -- 套餐ID
  max_users INT DEFAULT 10,                   -- 最大用户数
  user_count INT DEFAULT 0,                   -- 当前用户数
  expire_date DATETIME,                       -- 到期时间
  features JSON,                              -- 自定义功能
  activated_at DATETIME,                      -- 激活时间
  last_verify_at DATETIME,                    -- 最后验证时间
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
)
```

### 6.2 tenant_license_logs 表（授权日志）

```sql
CREATE TABLE tenant_license_logs (
  id VARCHAR(36) PRIMARY KEY,
  tenant_id VARCHAR(36),
  license_key VARCHAR(100),
  action VARCHAR(50),        -- verify, heartbeat
  result VARCHAR(20),        -- success, failed
  message TEXT,
  ip_address VARCHAR(50),
  user_agent TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
)
```

### 6.3 users 表（用户表）

```sql
CREATE TABLE users (
  id VARCHAR(36) PRIMARY KEY,
  tenant_id VARCHAR(36),     -- 🔥 租户ID（SaaS模式）

  password VARCHAR(255) NOT NULL,
  real_name VARCHAR(50),
  role VARCHAR(50),
  department_id VARCHAR(36),
  status ENUM('active', 'inactive', 'locked'),
  -- ... 其他字段
  UNIQUE KEY unique_username_per_tenant (tenant_id, username)  -- 同租户下用户名唯一
```
