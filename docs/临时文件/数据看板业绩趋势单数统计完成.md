# 数据看板业绩趋势单数统计功能完成

## 需求说明
在CRM数据看板模块的业绩趋势图表中，tooltip目前只显示金额（下单业绩和签收业绩），需要补充单数统计。

**期望效果**：
- 下单业绩：￥1380（3单）
- 签收业绩：￥880（2单）

## 实现方案

### 1. 后端API修改（backend/src/routes/dashboard.ts）

#### 1.1 添加单数统计数组
在 `/dashboard/charts` 接口中，新增两个数组用于存储单数：
```typescript
const orderCountData: number[] = [];  // 下单单数
const deliveredCountData: number[] = [];  // 签收单数
```

#### 1.2 统计各时间段的单数
在三个时间段（day/week/month）的循环中，都添加了单数统计：

**月视图（本月每天）**：
```typescript
orderCountData.push(validOrders.length);
deliveredCountData.push(deliveredOrders.length);
```

**周视图（最近7天）**：
```typescript
orderCountData.push(validOrders.length);
deliveredCountData.push(deliveredOrders.length);
```

**日视图（今日每小时）**：
```typescript
orderCountData.push(validOrders.length);
deliveredCountData.push(deliveredOrders.length);
```

#### 1.3 返回单数数据
修改API返回结构，在series中添加counts字段：
```typescript
data: {
  performance: {
    categories,
    series: [
      { name: '下单业绩', data: orderRevenueData, counts: orderCountData },
      { name: '签收业绩', data: deliveredRevenueData, counts: deliveredCountData }
    ]
  },
  orderStatus
}
```

### 2. 前端API接口更新（src/api/dashboard.ts）

#### 2.1 更新接口定义
在 `DashboardChartData` 接口中添加单数字段：
```typescript
export interface DashboardChartData {
  revenue: Array<{
    date: string
    amount: number
    deliveredAmount?: number
    orderCount?: number      // 🔥 下单单数
    deliveredCount?: number  // 🔥 签收单数
  }>
  // ...
}
```

#### 2.2 解析后端返回的单数数据
在 `getChartData` 函数中，从后端响应中提取单数：
```typescript
revenue: data.performance?.series?.[0]?.data?.map((amount: number, index: number) => ({
  date: data.performance?.categories?.[index] || `${index + 1}月`,
  amount,
  deliveredAmount: data.performance?.series?.[1]?.data?.[index] || 0,
  orderCount: data.performance?.series?.[0]?.counts?.[index] || 0,      // 🔥 下单单数
  deliveredCount: data.performance?.series?.[1]?.counts?.[index] || 0   // 🔥 签收单数
}))
```

### 3. 前端视图更新（src/views/Dashboard.vue）

#### 3.1 添加单数数据字段
在 `performanceChartData` 中添加单数数组：
```typescript
const performanceChartData = ref({
  xAxisData: [] as string[],
  orderData: [] as number[],
  signData: [] as number[],
  orderCountData: [] as number[],   // 🔥 下单单数
  signCountData: [] as number[],    // 🔥 签收单数
  title: '业绩趋势'
})
```

#### 3.2 更新tooltip显示格式
修改 `performanceChartOption` 的tooltip formatter，显示单数：
```typescript
formatter: function(params: Array<{axisValue: string, value: number, marker: string, seriesName: string, dataIndex: number}>) {
  let result = `${params[0].axisValue}<br/>`
  params.forEach((param) => {
    const value = `¥${param.value.toLocaleString()}`
    // 根据系列名称获取对应的单数
    let count = 0
    if (param.seriesName === '下单业绩') {
      count = performanceChartData.value.orderCountData[param.dataIndex] || 0
    } else if (param.seriesName === '签收业绩') {
      count = performanceChartData.value.signCountData[param.dataIndex] || 0
    }
    result += `${param.marker}${param.seriesName}: ${value}（${count}单）<br/>`
  })
  return result
}
```

#### 3.3 加载图表数据时填充单数
在 `loadRealChartData` 函数中，从API响应中提取单数数据：
```typescript
performanceChartData.value = {
  xAxisData: chartData.revenue.map((item: { date: string }) => item.date),
  orderData: chartData.revenue.map((item: { amount: number }) => item.amount),
  signData: chartData.revenue.map((item: { deliveredAmount?: number }) => item.deliveredAmount || 0),
  orderCountData: chartData.revenue.map((item: { orderCount?: number }) => item.orderCount || 0),
  signCountData: chartData.revenue.map((item: { deliveredCount?: number }) => item.deliveredCount || 0),
  title: getPerformanceTitle()
}
```

## 数据流程

1. **后端统计**：在生成业绩金额数据的同时，统计对应的订单数量
2. **API传输**：通过series[].counts字段传递单数数组
3. **前端解析**：将counts数据映射到orderCount和deliveredCount字段
4. **图表显示**：在tooltip中根据dataIndex获取对应的单数并显示

## 显示效果

鼠标悬停在业绩趋势图的数据点上时，tooltip会显示：
```
3月25日
● 下单业绩: ¥1,380（3单）
● 签收业绩: ¥880（2单）
```

## 技术要点

1. **数据同步**：单数数组和金额数组的索引一一对应，确保数据准确性
2. **向后兼容**：保留了原有的数据结构，新增字段为可选字段
3. **空值处理**：使用 `|| 0` 确保在数据缺失时显示0而不是undefined
4. **类型安全**：在TypeScript接口中明确定义了新增字段的类型
