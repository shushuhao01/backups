## 功能说明
Admin后台可以控制CRM前端的模块显示,禁用的模块在CRM前端菜单中自动隐藏。

```bash
cd ..

# Admin后台 (Terminal 7)
cd admin
npm run dev
```

### 2. 确认服务地址
- 后端API: http://localhost:3000
- CRM前端: http://localhost:5173
- Admin后台: http://localhost:5174

## 常见问题排查

### 问题1: CRM前端菜单没有变化
**可能原因:**
- 缓存未过期(5分钟缓存)
- 浏览器缓存

**解决方法:**
1. 硬刷新页面: Ctrl+F5 (Windows) 或 Cmd+Shift+R (Mac)
2. 清除浏览器缓存
3. 等待5分钟后刷新

### 问题2: 后端API返回错误
**可能原因:**
- 后端服务未启动
- 数据库连接失败
- ModuleConfig表不存在

**解决方法:**
1. 检查后端服务是否运行
2. 检查数据库连接
3. 运行数据库迁移脚本

### 问题3: Admin后台无法禁用模块
**可能原因:**
- 尝试禁用系统模块
- 权限不足

**解决方法:**
1. 确认不是系统管理模块
2. 使用admin账号登录
3. 检查后端日志

## 技术实现说明

### 后端
- API接口: `GET /api/v1/system/modules/status`
- 查询数据库: `module_configs`表,`is_enabled=1`
- 返回格式: `{ success: true, data: { enabledModules: ['order', 'customer', ...] } }`

### 前端
- 服务: `src/services/moduleStatusService.ts`
- 缓存时间: 5分钟
- 菜单过滤: `src/utils/menu.ts` - `getUserAccessibleMenus()`
- 组件: `src/components/DynamicMenu.vue`

## 相关文件
- `backend/src/routes/system.ts` - 后端API接口
