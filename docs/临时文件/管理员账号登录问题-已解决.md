# 管理员账号登录问题 - 已解决 ✅

## 问题回顾

用户反馈：创建租户后，使用返回的默认管理员账号和密码登录时，提示"账号密码错误"。

## 根本原因

1. **密码加密方式不一致**：
   - 系统原有用户使用 `bcrypt` 加密
   - 新创建的管理员账号最初使用 `pbkdf2` 加密
   - 登录验证只支持 `bcrypt.compare`，导致密码验证失败

2. **账号被锁定**：
   - 密码验证失败导致登录失败次数累积
   - 失败5次后账号自动锁定（status = 'locked'）

## 解决方案

### 1. 修改密码加密方式

将 `backend/src/utils/adminAccountHelper.ts` 中的密码加密从 pbkdf2 改为 bcrypt：

```typescript
// 修改前（pbkdf2）
export const hashPassword = (password: string): { hash: string; salt: string } => {
  const salt = crypto.randomBytes(16).toString('hex');
  const hash = crypto.pbkdf2Sync(password, salt, 1000, 64, 'sha512').toString('hex');
  return { hash, salt };
};

// 修改后（bcrypt）
export const hashPassword = async (password: string): Promise<string> => {
  const salt = await bcrypt.genSalt(12);
  const hash = await bcrypt.hash(password, salt);
  return hash;
};
```

### 2. 添加 salt 字段

为 users 表添加 salt 字段（虽然 bcrypt 不需要单独的 salt 字段，但为了兼容性保留）：

```sql
ALTER TABLE users ADD COLUMN salt VARCHAR(255) NULL AFTER password;
```

### 3. 解锁被锁定的账号

```sql
UPDATE users 
SET status = 'active', login_fail_count = 0, locked_at = NULL
WHERE status = 'locked';
```

## 用户操作指南

### 对于新创建的租户/私有客户

1. 在 Admin 后台创建租户或私有客户
2. 系统会返回完整的登录信息：
   - 登录地址
   - 租户编码（租户客户）
   - 授权码
   - 管理员账号（手机号）
   - 初始密码：Aa123456

3. 使用返回的账号密码登录 CRM 系统
4. 首次登录后建议立即修改密码

### 对于之前创建的账号（使用 pbkdf2 加密的）

如果你之前创建的账号无法登录，有两个选择：

#### 选项1：重新创建（推荐）
1. 在 Admin 后台删除旧的租户/私有客户
2. 重新创建，新账号将使用 bcrypt 加密
3. 使用新返回的账号密码登录

#### 选项2：手动重置密码
联系技术支持，手动重置密码为 bcrypt 加密
