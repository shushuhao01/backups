# 模块禁用功能问题分析

## 问题现象
在Admin后台禁用了3个模块（财务管理、售后管理、物流管理），但CRM主应用刷新后这些模块的菜单仍然显示。

## 根本原因

### 1. CRM前端有模块状态服务
文件：`src/services/moduleStatusService.ts`

CRM前端有一个模块状态服务，用于：
- 从后端获取启用的模块列表
- 缓存模块状态（5分钟缓存）
- 根据模块状态控制菜单显示

关键代码：
```typescript
async getEnabledModules(): Promise<string[]> {
  // 调用后端API: /system/modules/status
  const response = await request.get('/system/modules/status')
  
  if (response.success && response.data?.enabledModules) {
    this.enabledModules = new Set(response.data.enabledModules)
    return Array.from(this.enabledModules)
  }
  
  // 失败时返回所有模块（保证系统可用）
  const defaultModules = [
    'dashboard', 'customer', 'order', 'product', 'logistics',
    'performance', 'service', 'finance', 'data', 'serviceManagement', 'system'
  ]
  return defaultModules
}
```

### 2. 后端缺少对应的API接口
**问题核心**：后端`backend/src/routes/system.ts`中没有`/system/modules/status`接口！

因此：
1. CRM前端调用`/system/modules/status`失败
2. 失败后返回默认的所有模块列表
3. 所以无论Admin后台如何设置，CRM前端都显示所有模块

## 解决方案

### 需要在后端添加模块状态API接口

在`backend/src/routes/system.ts`中添加：

```typescript
/**
 * 获取启用的模块列表
 * @route GET /api/v1/system/modules/status
 * @access Private (All authenticated users)
 */
router.get('/modules/status', authenticateToken, async (_req: Request, res: Response) => {
  try {
    const moduleConfigRepository = AppDataSource.getRepository(ModuleConfig);
    
    // 获取所有启用的模块
    const enabledModules = await moduleConfigRepository.find({
      where: { is_enabled: true },
      select: ['code']
    });
    
    // 提取模块代码列表
    const moduleKeys = enabledModules.map(m => m.code);
    
    res.json({
      success: true,
      data: {
        enabledModules: moduleKeys
      }
    });
  } catch (error: any) {
    console.error('[System] 获取模块状态失败:', error);
    res.status(500).json({
      success: false,
      message: '获取模块状态失败',
      error: error.message
    });
  }
});
```

### 模块代码映射关系

Admin后台的模块代码需要映射到CRM前端的菜单ID：

| Admin模块代码 | CRM菜单ID | 说明 |
|--------------|-----------|------|
| order_management | order | 订单管理 |
| customer_management | customer | 客户管理 |
| finance_management | finance | 财务管理 |
| logistics_management | logistics | 物流管理 |
| aftersales_management | service | 售后管理 |
| call_management | serviceManagement | 通话管理 |
| data_management | data | 资料管理 |
| performance_management | performance | 业绩统计 |
| product_management | product | 商品管理 |
| system_management | system | 系统管理 |

### 实施步骤

1. **添加后端API接口**
   - 在`backend/src/routes/system.ts`添加`/modules/status`接口
   - 导入`ModuleConfig`实体
   - 查询启用的模块并返回代码列表

2. **可能需要调整模块代码**
   - 确保Admin后台的模块代码与CRM前端的菜单ID匹配
   - 或者在API中做代码转换映射

3. **测试验证**
   - 在Admin后台禁用某个模块
   - 刷新CRM前端（或等待5分钟缓存过期）
   - 验证对应菜单是否隐藏
