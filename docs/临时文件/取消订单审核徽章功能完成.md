# 取消订单审核徽章功能完成说明

## 功能概述
为订单管理的订单列表中的"取消订单审核"按钮添加待处理数量的徽章提示，当有待审核的取消订单申请时，在按钮右上角显示数字统计，处理后徽章消失。

## 实现内容

### 1. 后端API开发
**文件**: `backend/src/routes/orders.ts`

新增API端点：
```
GET /api/v1/orders/pending-cancel-count
```

功能：
- 获取待审核的取消订单数量（status = 'pending_cancel'）
- 返回格式：`{ success: true, code: 200, data: { count: number } }`
- 支持租户隔离

**租户隔离实现原理**:
```typescript
// 1. 使用 getTenantRepo 获取仓库
const orderRepository = getTenantRepo(Order);

// 2. getTenantRepo 通过 Proxy 拦截 count 方法
// 3. 自动注入租户过滤条件
const count = await orderRepository.count({
  where: { status: 'pending_cancel' }
  // 实际执行: WHERE tenant_id = ? AND status = 'pending_cancel'
});
```

**租户上下文获取**:
- 从 `authenticateToken` 中间件获取当前用户的 `tenant_id`
- 存储在 `AsyncLocalStorage` 中，整个请求周期可用
- 无需手动传递，自动注入到所有查询条件中

### 2. 前端API配置
**文件**: `src/api/config.ts`

添加API端点配置：
```typescript
PENDING_CANCEL_COUNT: '/orders/pending-cancel-count'
```

**文件**: `src/api/order.ts`

添加API调用方法：
```typescript
getPendingCancelCount: () =>
  api.get<{ count: number }>(API_ENDPOINTS.ORDERS.PENDING_CANCEL_COUNT)
```

### 3. 前端页面实现
**文件**: `src/views/Order/List.vue`

#### 3.1 数据定义
```typescript
const pendingCancelCount = ref(0) // 待审核的取消申请数量
```

#### 3.2 按钮徽章显示
使用 `el-badge` 组件包裹按钮：
```vue
<el-badge
  v-if="canViewCancelAudit"
  :value="pendingCancelCount"
  :hidden="pendingCancelCount === 0"
  :max="99"
  type="danger"
>
  <el-button
    @click="handleOpenCancelAudit"
    size="small"
    :icon="DocumentChecked"
    type="warning"
  >
    取消订单审核
  </el-button>
</el-badge>
```

特性：
- 当数量为0时，徽章自动隐藏
- 最大显示99，超过显示"99+"
- 红色徽章（type="danger"）突出显示

#### 3.3 数据加载方法
```typescript
const loadPendingCancelCount = async () => {
  try {
    const { orderApi } = await import('@/api/order')
    const response = await orderApi.getPendingCancelCount()
    
    if (response && response.data) {
      pendingCancelCount.value = response.data.count || 0
    }
  } catch (error) {
    console.error('获取待审核取消订单数量失败:', error)
    // 静默失败，不影响用户体验
  }
}
```

#### 3.4 触发更新的时机
1. **页面加载时** (`onMounted`)
   - 与其他初始化数据并行加载

2. **提交取消申请后** (`submitCancelRequest`)
   - 用户提交新的取消申请后，数量+1

3. **审核处理后** (`handleAuditApprove` / `handleAuditReject`)
   - 管理员审核通过或拒绝后，数量更新

## 用户体验

### 视觉效果
- 按钮右上角显示红色圆形徽章
- 数字清晰可见，最多显示两位数
- 无待审核时徽章自动隐藏，保持界面简洁

### 交互流程
1. 销售人员提交取消订单申请 → 徽章数字+1
2. 管理员点击"取消订单审核"按钮 → 打开审核弹窗
3. 管理员审核通过/拒绝 → 徽章数字-1
4. 所有申请处理完毕 → 徽章自动消失

## 技术特点

1. **性能优化**
   - 使用独立的轻量级API获取数量，不加载完整订单数据
   - 静默失败，不影响页面其他功能

2. **实时更新**
   - 在关键操作后自动更新数量
   - 无需手动刷新页面

3. **租户隔离 ✅**
   - 后端API使用 `getTenantRepo(Order)` 获取订单仓库
   - `getTenantRepo` 通过 Proxy 拦截 `count()` 方法
   - 自动注入 `tenant_id` 过滤条件：`WHERE tenant_id = ? AND status = 'pending_cancel'`
   - 每个租户只能看到自己的待审核数量
   - 租户上下文从认证中间件 `authenticateToken` 自动获取
   - **验证方式**: 运行 `node backend/test-pending-cancel-count-tenant-isolation.js`

4. **权限控制**
   - 复用现有的 `canViewCancelAudit` 权限判断
   - 只有有权限的用户才能看到按钮和徽章
