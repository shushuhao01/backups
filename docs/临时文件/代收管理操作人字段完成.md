# 代收管理操作人字段完成

## 修改时间
2026-03-24

## 需求说明
为代收管理的详情弹窗添加操作人和操作时间字段：
1. 改代收操作：显示改代收时间和操作人姓名
2. 返款操作：显示返款时间和操作人姓名

## ✅ 已完成的工作

### 1. 数据库修改

#### 本地数据库（crm_local）
✅ 已执行完成，添加了4个字段：
- `cod_cancelled_by` - 改代收操作人ID
- `cod_cancelled_by_name` - 改代收操作人姓名
- `cod_returned_by` - 返款操作人ID
- `cod_returned_by_name` - 返款操作人姓名

#### Schema.sql源文件
✅ 已更新完成，字段定义如下：
```sql
`cod_returned_at` DATETIME NULL COMMENT '返款时间',
`cod_returned_by` VARCHAR(36) NULL COMMENT '返款操作人ID',
`cod_returned_by_name` VARCHAR(50) NULL COMMENT '返款操作人姓名',
`cod_cancelled_at` DATETIME NULL COMMENT '取消代收时间',
`cod_cancelled_by` VARCHAR(36) NULL COMMENT '改代收操作人ID',
`cod_cancelled_by_name` VARCHAR(50) NULL COMMENT '改代收操作人姓名',
```

## 📋 生产环境部署

### SQL脚本
文件：`backend/database-migrations/production-add-cod-operator-fields.sql`

在宝塔phpMyAdmin中执行以下SQL：

```sql
-- 添加改代收操作人ID字段
ALTER TABLE `orders` 
ADD COLUMN `cod_cancelled_by` varchar(36) DEFAULT NULL COMMENT '改代收操作人ID' AFTER `cod_cancelled_at`;

-- 添加改代收操作人姓名字段
ALTER TABLE `orders` 
ADD COLUMN `cod_cancelled_by_name` varchar(50) DEFAULT NULL COMMENT '改代收操作人姓名' AFTER `cod_cancelled_by`;

-- 添加返款操作人ID字段
ALTER TABLE `orders` 
ADD COLUMN `cod_returned_by` varchar(36) DEFAULT NULL COMMENT '返款操作人ID' AFTER `cod_returned_at`;

-- 添加返款操作人姓名字段
ALTER TABLE `orders` 
ADD COLUMN `cod_returned_by_name` varchar(50) DEFAULT NULL COMMENT '返款操作人姓名' AFTER `cod_returned_by`;
```

### 验证SQL
```sql
SELECT 
  COLUMN_NAME AS '字段名',
  COLUMN_TYPE AS '类型',
  COLUMN_COMMENT AS '注释'
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_SCHEMA = DATABASE()
  AND TABLE_NAME = 'orders'
  AND COLUMN_NAME IN ('cod_cancelled_by', 'cod_cancelled_by_name', 'cod_returned_by', 'cod_returned_by_name')
ORDER BY ORDINAL_POSITION;
```

## 显示效果

### 详情弹窗字段顺序
1. 订单号
2. 客户姓名
3. 订单金额
4. 代收金额
5. 代收状态
6. 已返款金额
7. **改代收时间**（仅在已改代收状态时显示）
8. **改代收操作人**（仅在已改代收状态时显示）
9. **返款时间**（仅在已返款状态时显示）
10. **返款操作人**（仅在已返款状态时显示）
11. 销售人员
12. 物流单号
13. 发货时间
14. 下单时间
15. 最新物流

### 显示规则
- **改代收信息**：只在 `codStatus === 'cancelled'` 且 `codCancelledAt` 有值时显示
- **返款信息**：只在 `codStatus === 'returned'` 且 `codReturnedAt` 有值时显示
- **历史数据**：已有的改代收和返款记录没有操作人信息，显示为"-"
