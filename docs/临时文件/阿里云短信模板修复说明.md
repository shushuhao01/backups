# 阿里云短信模板修复说明

## 问题描述

在申请阿里云短信模板时，AI检测失败，提示：

```
变量内传入链接等
检测要素：特定规定-模板变量
失败原因：不支持链接等。IP地址、公众号、小程序、APP、二维码等作为变量传入，
请将这些内容体现在变量外，固定到模板内容中。
```

## 问题原因

1. **URL不能作为变量**: 阿里云不允许将 `${loginUrl}` 这样的URL作为变量传入
2. **手机号不能作为变量**: 阿里云不允许将 `${adminUsername}` (手机号)作为变量传入
3. **签名格式问题**: 模板内容不应包含【】签名符号，签名会自动添加

## 修复方案

### 业务流程说明

系统有两种注册场景：

**场景A：免费试用套餐（price = 0）**
1. 用户注册 → 立即创建租户（status='active'）
2. 立即生成授权码和管理员账号
3. 发送"账号开通通知"（包含授权码、密码等完整信息）

**场景B：付费套餐（price > 0）**
1. 用户注册 → 创建租户（status='inactive'，待支付）
2. 不生成授权码，不创建管理员账号
3. 不发送短信
4. 用户支付成功 → 激活租户，生成授权码，创建管理员账号
5. 发送"支付成功账号开通通知"（包含支付信息+授权码+密码等完整信息）

### 步骤4：提交审核
等待1-2个工作日审核

## 注意事项

1. **签名会自动添加**: 不要在模板内容中写【云客CRM】，系统会自动添加签名
2. **URL处理方案**: 
   - 方案A: 在短信中不提供登录地址，用户通过官网或收藏的地址登录
   - 方案B: 发送两条短信，第二条包含固定的登录地址文字（不作为变量）
   - 方案C: 通过邮件发送包含登录地址的详细信息
3. **手机号说明**: 使用"管理员账号为您的手机号"这样的固定文字，用户自然知道是哪个手机号
4. **字数限制**: 单条短信最多500字符，建议控制在300字符以内
