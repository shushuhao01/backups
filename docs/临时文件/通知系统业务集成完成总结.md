## 二、通知发送流程

### 2.1 注册流程

```
用户填写注册信息
    ↓
验证手机号和验证码
    ↓
创建租户记录
    ↓
免费套餐? ─── 是 ──→ 创建管理员账号
    │                    ↓
    │              发送注册成功通知 ✅
    │                    ↓
    │              返回注册成功
    │
    └─── 否 ──→ 返回待支付状态
```

### 2.2 支付流程

```
用户扫码支付
    ↓
支付平台回调
    ↓
验证签名
    ↓
更新订单状态
    ↓
激活租户账号
    ↓
生成授权码
    ↓
创建管理员账号
    ↓
发送支付成功通知 ✅
    ↓
返回成功响应
```

## 三、通知渠道

### 3.1 邮件通知

- 支持HTML格式
- 包含完整的账号信息
- 带有操作链接
- 优先发送给有邮箱的用户

- 纯文本格式

- 70字以内

### 3.3 自动选择

系统会根据用户提供的联系方式自动选择:
1. 优先使用邮箱(如果提供)
2. 其次使用手机号
3. 根据模板配置决定发送邮件还是短信

## 五、错误处理

### 5.1 不影响主流程

通知发送失败不会影响注册或支付流程:
```typescript
try {
  await notificationTemplateService.sendByTemplate(...)
  console.log('✓ 通知发送成功')
} catch (error) {
  console.error('发送通知失败:', error)
  // 不抛出错误，避免影响主流程
}
```

### 5.2 日志记录

所有通知发送都会记录日志:
- 发送时间
- 模板代码
- 变量数据
- 发送结果
- 错误信息(如果失败)

## 七、待集成的业务场景

### 7.1 授权到期提醒 (P1)

**需要**: 创建定时任务

```typescript
// 每天检测即将到期的授权
const expiringSoon = await AppDataSource.query(`
  SELECT * FROM tenants 
  WHERE expire_date BETWEEN CURDATE() AND DATE_ADD(CURDATE(), INTERVAL 7 DAY)
  AND status = 'active'
`);

for (const tenant of expiringSoon) {
  const remainDays = Math.ceil(
    (new Date(tenant.expire_date) - new Date()) / (24 * 60 * 60 * 1000)
  );
  
  await notificationTemplateService.sendByTemplate('license_expire_soon', {
    tenantName: tenant.name,
    licenseKey: tenant.license_key,
    expireDate: tenant.expire_date,
    remainDays
  });
}
```

## 十、监控和日志

### 10.1 发送日志

所有通知发送都会记录在 `notification_logs` 表中:
- 模板代码
- 发送时间
- 接收人
- 发送结果
- 错误信息

### 10.2 查看日志

```sql
-- 查看最近的通知发送记录
SELECT * FROM notification_logs 
ORDER BY created_at DESC 
LIMIT 100;

-- 查看失败的通知
SELECT * FROM notification_logs 
WHERE status = 'failed'
ORDER BY created_at DESC;

-- 统计发送成功率
SELECT 
  template_code,
  COUNT(*) as total,
  SUM(CASE WHEN status = 'success' THEN 1 ELSE 0 END) as success_count,
  ROUND(SUM(CASE WHEN status = 'success' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) as success_rate
FROM notification_logs
GROUP BY template_code;
```

## 十一、常见问题

### Q1: 为什么没有收到通知?

A: 检查以下几点:
1. 邮件/短信服务是否配置正确
2. 用户是否提供了邮箱或手机号
3. 查看 `notification_logs` 表的错误信息
4. 检查邮件是否进入垃圾箱

### Q2: 如何修改通知内容?

A: 
1. 登录Admin后台
2. 进入: 系统设置 → 通知模板
3. 找到对应模板
4. 修改邮件主题、内容或短信内容
5. 保存并测试

### Q3: 如何添加新的通知场景?

A: 
1. 在Admin后台创建新模板
2. 在业务代码中调用:
```typescript
await notificationTemplateService.sendByTemplate('模板代码', {
  变量1: '值1',
  变量2: '值2'
});
```

### Q4: 通知发送失败会影响业务吗?

A: 不会。通知发送失败只会记录日志,不会影响注册或支付等主流程。
