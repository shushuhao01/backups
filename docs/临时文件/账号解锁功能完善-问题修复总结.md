# 账号解锁功能完善 - 问题修复总结

## 问题描述

用户反馈：点击解锁后，账号还是登录不了，并且在客户详情页没有显示账号被锁定的状态。

## 根本原因分析

通过诊断工具发现了两个关键问题：

### 1. 密码加密格式错误
- **问题**：部分管理员账号的密码使用了 `pbkdf2` 加密（128字符），而不是系统要求的 `bcrypt` 加密（60字符）
- **影响**：即使账号解锁，由于密码格式不匹配，登录验证时 `bcrypt.compare()` 无法正确验证，导致登录失败
- **示例**：
  ```
  错误格式: 33295b406aa9ff195daa41722b4d2d6814602b234f4cb135618b670f7199b649... (128字符)
  正确格式: $2b$12$rFKY3gIx68pKsqw3SNBXsO4TE7klSDQoH0bE2wjgbkq0xfC7ViXG. (60字符)
  ```

### 2. 缺少账号锁定状态显示
- **问题**：客户详情页没有显示管理员账号的锁定状态
- **影响**：管理员无法直观看到账号是否被锁定，不知道需要解锁

## 解决方案

### 1. 增强解锁API - 自动检测并修复密码格式

#### 租户解锁API (`backend/src/routes/admin/tenants.ts`)
```typescript
router.post('/:id/unlock-admin', async (req: Request, res: Response) => {
  // 1. 查找被锁定的管理员账号
  const lockedUsers = await AppDataSource.query(
    `SELECT id, username, password FROM users
     WHERE tenant_id = ? AND status = 'locked' AND role = 'admin'`,
    [id]
  );

  // 2. 检查并修复密码格式
  for (const user of lockedUsers) {
    const isBcryptFormat = user.password && user.password.startsWith('$2') && user.password.length === 60;
    
    if (!isBcryptFormat) {
      // 密码格式错误，重新生成bcrypt密码
      const hashedPassword = await bcrypt.hash('Aa123456', 12);
      await AppDataSource.query(
        `UPDATE users SET password = ? WHERE id = ?`,
        [hashedPassword, user.id]
      );
      fixedPasswordCount++;
    }
  }

  // 3. 解锁账号
  await AppDataSource.query(
    `UPDATE users
     SET status = 'active', login_fail_count = 0, locked_at = NULL
     WHERE tenant_id = ? AND status = 'locked' AND role = 'admin'`,
    [id]
  );
});
```

**功能特点**：
- ✅ 自动检测密码格式（检查是否以 `$2` 开头且长度为60）
- ✅ 自动修复错误格式的密码（重置为 `Aa123456`）
- ✅ 返回修复信息，告知管理员密码已重置
- ✅ 记录详细日志

#### 私有客户解锁API (`backend/src/routes/admin/private-customers.ts`)
同样的逻辑应用到私有客户解锁功能。

### 2. 租户详情API增强 - 返回管理员账号状态

```typescript
router.get('/:id', async (req: Request, res: Response) => {
  // 查询租户基本信息
  const tenant = rows[0];

  // 查询该租户的管理员账号状态
  const adminUsers = await AppDataSource.query(
    `SELECT id, username, status, login_fail_count, locked_at 
     FROM users 
     WHERE tenant_id = ? AND role = 'admin'
     ORDER BY created_at DESC`,
    [id]
  );

  // 添加管理员账号信息到返回数据
  tenant.adminUsers = adminUsers;
  tenant.hasLockedAdmin = adminUsers.some((u: any) => u.status === 'locked');

  res.json({ success: true, data: tenant });
});
```

## 使用说明

### 1. 解锁账号
1. 进入租户客户列表或私有客户列表
2. 点击操作列的"更多"按钮
3. 选择"解锁账号"
4. 系统会自动：
   - 检测密码格式
   - 修复错误格式的密码（重置为 Aa123456）
   - 解锁账号
   - 重置登录失败次数

### 2. 查看账号状态
1. 进入租户客户详情页
2. 在"基本信息"卡片中查看"管理员账号"字段
3. 可以看到：
   - 账号用户名
   - 锁定状态（红色标签）或正常状态（绿色标签）
   - 登录失败次数

### 3. 密码重置提示
如果解锁时检测到密码格式错误，系统会：
- 自动重置密码为 `Aa123456`
- 在成功消息中提示："已解锁 X 个管理员账号，其中 X 个账号的密码已重置为 Aa123456"
- 请及时通知客户修改密码

## 预防措施

### 1. 确保创建账号时使用正确的加密方式
在 `backend/src/utils/adminAccountHelper.ts` 中：
```typescript
export async function hashPassword(password: string): Promise<string> {
  return await bcrypt.hash(password, 12); // 使用 bcrypt
}
```

### 2. 定期检查密码格式
可以运行诊断脚本检查：
```bash

```

### 3. 监控登录失败
