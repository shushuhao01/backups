# 日常开发同步流程 - 超简单版

## 🚀 你只需要记住这3步

### 第1步: 正常开发
```powershell
# 修改任何文件，和以前一样
# 比如修改了订单功能
code src/views/Order/List.vue
```

### 第2步: 提交到主仓库
```powershell
git add .
git commit -m "优化订单列表功能"
git push origin main
```

### 第3步: 一键同步到所有子仓库
```powershell
.\sync-repos-v2.ps1 -message "优化订单列表功能"
```

就这么简单! 🎉

---

## 📖 详细说明

### 脚本会自动做什么？

当你运行 `.\sync-repos-v2.ps1 -message "你的提交信息"` 时，脚本会：

1. **自动识别你修改了什么**
   - 如果修改了 `src/`、`backend/` 等CRM代码 → 推送到 `crm-system`
   - 如果修改了 `website/` → 推送到 `crm-website`
   - 如果修改了 `admin/` → 推送到 `crm-admin`

2. **自动排除不需要的目录**
   - 推送到 `crm-system` 时，自动排除 `website/`、`admin/`
   - 推送到 `crm-website` 时，只推送 `website/` 目录
   - 推送到 `crm-admin` 时，只推送 `admin/` 目录

3. **一次性推送到所有仓库**
   - 不需要你手动区分
   - 不需要你运行多次命令
   - 一条命令搞定所有同步

## 🎯 实际例子

### 例子1: 修改了CRM订单功能

```powershell
# 1. 修改代码
code src/views/Order/List.vue
code backend/src/routes/orders.ts

# 2. 提交
git add .
git commit -m "优化订单列表和API"
git push origin main

# 3. 同步（会自动推送到crm-system仓库）
.\sync-repos-v2.ps1 -message "优化订单列表和API"
```

**结果:**
- ✅ 主仓库(CRM)已更新
- ✅ crm-system仓库已更新（客户可以获取最新代码）
- ⏭️ crm-website仓库不受影响（因为没修改官网）
- ⏭️ crm-admin仓库不受影响（因为没修改Admin）

## 🔧 高级用法（可选）

### 只同步某个仓库

如果你只想同步某个特定仓库：

```powershell
# 只同步CRM系统
.\sync-repos-v2.ps1 -onlyCRM -message "更新CRM"

# 只同步官网
.\sync-repos-v2.ps1 -onlyWebsite -message "更新官网"

# 只同步Admin
.\sync-repos-v2.ps1 -onlyAdmin -message "更新Admin"
```

### 测试模式（不实际推送）

想看看会推送什么，但不实际推送：

```powershell
.\sync-repos-v2.ps1 -dryRun
```

### 跳过主仓库推送

如果你已经手动推送了主仓库，只想同步子仓库：

```powershell
.\sync-repos-v2.ps1 -skipMain -message "同步子仓库"
```

## ❓ 常见问题

### Q: 我每次都要运行同步命令吗？
A: 不是必须的！同步是可选的。
- 如果你只是本地测试，不需要同步
- 如果你想让客户获取最新代码，才需要同步
- 如果你只想提交到主仓库，也不需要同步

### Q: 同步会很慢吗？
A: 第一次会慢一点（几分钟），后续很快（几秒钟）。

### Q: 我忘记同步了怎么办？
A: 没关系！随时可以补同步：
```powershell
.\sync-repos-v2.ps1 -message "补同步之前的更新"
```

### Q: 同步失败了怎么办？
A: 检查网络连接，然后重新运行命令就行。

### Q: 脚本怎么知道推送到哪个仓库？
A: 脚本会：
- 检查你修改了哪些目录
- 自动推送到对应的仓库
- 自动排除不需要的文件
