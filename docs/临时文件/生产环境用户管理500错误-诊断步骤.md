## 📋 可能的原因分析

### 原因1: 表结构不匹配

**症状**: 后端代码查询的字段在表中不存在

**检查方法**: 
```sql
SHOW COLUMNS FROM `roles`;
```

**解决方法**: 
- 如果缺少字段,执行 `fix-roles-table-production.sql`

### 原因2: 数据损坏

**症状**: 表中有NULL值或异常数据

**检查方法**:
```sql
SELECT * FROM `roles` WHERE name IS NULL OR name = '';
```

**解决方法**: 
- 修复或删除异常数据

### 原因3: 字符集问题

**症状**: 中文乱码或编码错误

**检查方法**:
```sql
SHOW TABLE STATUS LIKE 'roles';
```

**解决方法**: 
- 修改表字符集为utf8mb4

### 原因4: 权限问题

**症状**: 数据库用户没有查询权限

**检查方法**:
```sql
SHOW GRANTS FOR CURRENT_USER();
```

**解决方法**: 
- 授予SELECT权限

### 原因5: 表锁定

**症状**: 表被锁定,无法查询

**检查方法**:
```sql
SHOW OPEN TABLES WHERE `Table` = 'roles';
```

**解决方法**: 
- 解锁表或重启MySQL

### 原因6: 后端代码更新

**症状**: 后端代码查询了新字段,但数据库未更新

**检查方法**: 
- 查看后端日志
- 对比开发环境和生产环境的表结构

**解决方法**: 
- 同步数据库结构
