# 同步命令快速参考卡 📋

## 🎯 最常用的命令（99%的情况用这个）

```powershell
.\sync-repos-v2.ps1 -message "你的提交信息"
```

**这一条命令会自动:**
- ✅ 推送到主仓库
- ✅ 同步到crm-system（CRM系统，客户交付用）
- ✅ 同步到crm-website（官网）
- ✅ 同步到crm-admin（Admin后台）
- ✅ 自动识别修改了什么
- ✅ 自动排除不需要的目录

---

## 📝 完整的3步流程

```powershell
# 第1步: 开发（和以前一样）
# 修改任何文件...

# 第2步: 提交到主仓库
git add .
git commit -m "更新功能"
git push origin main

# 第3步: 一键同步到所有子仓库
.\sync-repos-v2.ps1 -message "更新功能"
```

---

## 🔧 其他命令（偶尔用）

```powershell
# 只同步CRM系统（客户交付仓库）
.\sync-repos-v2.ps1 -onlyCRM -message "更新CRM"

# 只同步官网
.\sync-repos-v2.ps1 -onlyWebsite -message "更新官网"

# 只同步Admin
.\sync-repos-v2.ps1 -onlyAdmin -message "更新Admin"

# 测试模式（看看会推送什么，但不实际推送）
.\sync-repos-v2.ps1 -dryRun

# 跳过主仓库推送（已经手动推送过了）
.\sync-repos-v2.ps1 -skipMain -message "只同步子仓库"
```

## ❓ 常见问题

**Q: 每次都要同步吗？**  
A: 不是！同步是可选的。只有当你需要更新子仓库时才同步。

**Q: 同步会很慢吗？**  
A: 第一次慢（几分钟），后续快（几秒）。

**Q: 忘记同步了怎么办？**  
A: 随时可以补同步，运行命令就行。

**Q: 脚本怎么知道推送到哪个仓库？**  
A: 脚本会自动识别你修改了什么，推送到对应的仓库。
