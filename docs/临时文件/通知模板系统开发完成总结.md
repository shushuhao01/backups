## 三、使用方法

### 3.1 在业务代码中发送通知

```typescript
import { notificationTemplateService } from '@/services/NotificationTemplateService';

// 发送注册成功通知
await notificationTemplateService.sendByTemplate('tenant_register_success', {
  systemName: '云客CRM',
  tenantName: tenant.name,
  adminUsername: 'admin',
  adminPassword: 'admin123',
  packageName: packageInfo.name,
  expireDate: tenant.expireDate
});
```

### 3.2 网站地址自动注入

模板中可以使用以下变量,会自动替换为配置的地址:
- `{{crmUrl}}` → `https://crm.yunkes.com`
- `{{websiteUrl}}` → `https://yunkes.com`
- `{{apiUrl}}` → `https://api.yunkes.com`
- `{{adminUrl}}` → `https://admin.yunkes.com`

示例:
```html
<p><a href="{{crmUrl}}/login">立即登录</a></p>
```

渲染后:
```html
<p><a href="https://crm.yunkes.com/login">立即登录</a></p>
```

### 3.3 Admin后台管理

1. 登录Admin后台: `https://admin.yunkes.com`
2. 进入: 系统设置 → 通知模板
3. 可以查看、编辑、测试所有模板

## 六、技术亮点

### 6.1 统一的网站地址管理

通过 `backend/src/config/sites.ts` 统一管理所有系统地址,避免硬编码。

### 6.2 变量自动注入

模板渲染时自动注入网站地址变量,无需在业务代码中传递。

### 6.3 多渠道支持

同一个模板可以同时支持邮件和短信,根据配置自动发送。

### 6.4 系统模板保护

系统预置模板不允许删除,只能修改内容,保证核心功能稳定。

### 6.5 完整的测试功能

Admin后台提供预览和测试发送功能,方便调试模板内容。

## 七、注意事项

### 7.1 邮件和短信配置

使用通知模板前,需要先在Admin后台配置:
- 系统设置 → 基础配置 → 邮件配置
- 系统设置 → 基础配置 → 短信配置

### 7.2 变量命名规范

- 使用驼峰命名: `tenantName`, `orderNumber`
- 避免特殊字符
- 保持语义清晰

### 7.3 短信内容限制

- 建议70字以内
- 最多500字
- 避免使用特殊符号

### 7.4 系统模板修改

- 系统模板可以修改内容
- 不能修改模板代码
- 不能删除系统模板
