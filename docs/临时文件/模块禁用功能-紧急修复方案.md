# 模块禁用功能 - 紧急修复方案

## 问题诊断

根据控制台日志分析:

```
[ModuleStatus] 获取模块状态失败: Failed to resolve component resolution via Proxy(Object)
[ModuleStatus] 使用默认模块列表,保证系统可用
```

**根本原因**:
1. `getUserAccessibleMenus`函数在执行过程中抛出异常
2. 异常原因是组件解析错误,可能是Vue的响应式代理问题
3. `DynamicMenu.vue`的降级方案没有正确处理模块过滤

## 修复方案

### 方案1:修复menu.ts中的响应式问题(推荐)

问题可能出在`menu.ts`中对菜单项的处理。需要确保不直接修改响应式对象。

### 方案2:简化DynamicMenu的降级逻辑

修改`src/components/DynamicMenu.vue`,改进错误处理。

## 临时解决方案

由于问题比较复杂,建议先使用一个简化版本的菜单过滤逻辑:

### 步骤1:备份当前文件

```bash
cp src/utils/menu.ts src/utils/menu.ts.backup
```

### 步骤2:简化menu.ts

将`getUserAccessibleMenus`函数简化,移除可能导致问题的复杂逻辑。

### 步骤3:测试

重启前端服务,查看菜单是否正常显示。

## 详细排查步骤

1. 检查`menuConfig`是否是响应式对象
2. 检查`filterByModuleStatus`函数是否修改了原始对象
3. 检查是否有循环引用导致的代理问题

## 快速修复(不考虑模块控制)

如果需要立即恢复菜单显示,可以临时禁用模块过滤:

在`src/utils/menu.ts`的`getUserAccessibleMenus`函数中,注释掉模块过滤部分,直接进行权限过滤。
