# 仓库同步命令 - 超简单使用指南

## 你需要记住的唯一命令

```powershell
.\sync-repos-v2.ps1 -message "你的提交信息"
```

就这一个命令！

## 日常开发流程（3步）

### 第1步：正常开发
```powershell
# 修改任何文件，和以前一样
code src/views/Customer/List.vue
code website/src/views/Home.vue
code admin/src/views/Dashboard.vue
```

### 第2步：提交到主仓库
```powershell
git add .
git commit -m "添加新功能"
git push origin main
```

### 第3步：同步到各个独立仓库
```powershell
.\sync-repos-v2.ps1 -message "添加新功能"
```

完成！就这么简单。

## 常用命令

### 1. 同步所有仓库（最常用）
```powershell
.\sync-repos-v2.ps1 -message "更新功能"
```

### 2. 只同步官网
```powershell
.\sync-repos-v2.ps1 -onlyWebsite -message "更新官网"
```

### 3. 只同步Admin
```powershell
.\sync-repos-v2.ps1 -onlyAdmin -message "更新Admin"
```

### 4. 测试模式（不实际推送）
```powershell
.\sync-repos-v2.ps1 -dryRun
```

## 什么时候需要同步？

### 需要同步的情况：
- ✅ 修改了CRM代码，需要客户获取最新版本
- ✅ 修改了官网代码，需要更新官网仓库
- ✅ 修改了Admin代码，需要更新Admin仓库

### 不需要同步的情况：
- ❌ 只是本地测试，还没提交
- ❌ 只提交到主仓库，暂时不想更新独立仓库
- ❌ 修改的是文档或配置，不影响交付

## 常见问题

### Q: 我忘记同步了怎么办？
A: 没关系！随时可以补同步：
```powershell
.\sync-repos-v2.ps1 -message "补同步之前的更新"
```

### Q: 同步会很慢吗？
A: 第一次会慢一点（处理历史），后续很快（只同步增量）。

### Q: 同步失败了怎么办？
A: 检查网络，然后重新运行命令就行。

### Q: 我可以不同步吗？
A: 可以！同步是可选的。只有当你需要更新独立仓库时才同步。

## 快捷方式（可选）

如果你觉得每次输入命令太长，可以创建一个简短的别名：

```powershell
# 在PowerShell配置文件中添加
function sync { .\sync-repos-v2.ps1 -message $args[0] }

# 然后就可以这样用
sync "更新功能"
```
