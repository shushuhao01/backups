## 📝 日常开发流程

### 方式1: 使用同步脚本（推荐）

```powershell
# 1. 正常开发
# 修改任何文件...

# 2. 提交到主仓库
git add .
git commit -m "更新功能"
git push origin main

# 3. 一键同步到所有独立仓库
.\sync-repos-v2.ps1 -message "更新功能"
```

### 方式2: 只同步某个仓库

```powershell
# 只同步CRM系统（客户交付仓库）
.\sync-repos-v2.ps1 -onlyCRM -message "更新CRM"

# 只同步官网
.\sync-repos-v2.ps1 -onlyWebsite -message "更新官网"

# 只同步Admin
.\sync-repos-v2.ps1 -onlyAdmin -message "更新Admin"
```

### 方式3: 测试模式（不实际推送）

```powershell
# 测试同步流程，不实际推送
.\sync-repos-v2.ps1 -dryRun
```

## 📊 仓库同步说明

### 同步是手动的
- 同步不是自动的
- 你需要运行 `.\sync-repos-v2.ps1` 命令
- 只有当你需要更新独立仓库时才同步

### 什么时候需要同步？
- ✅ 修改了CRM代码，需要客户获取最新版本
- ✅ 修改了官网代码，需要更新官网仓库
- ✅ 修改了Admin代码，需要更新Admin仓库

### 什么时候不需要同步？
- ❌ 只是本地测试，还没提交
- ❌ 只提交到主仓库，暂时不想更新独立仓库
- ❌ 修改的是文档或配置，不影响交付


## ❓ 常见问题

### Q: 本地开发会受影响吗？
A: 完全不会！本地继续在同一目录开发。

### Q: 同步会很慢吗？
A: 第一次会慢一点，后续很快（只同步增量）。

### Q: 我可以不同步吗？
A: 可以！同步是可选的。只有当你需要更新独立仓库时才同步。

### Q: 客户能看到完整历史吗？
A: 能。crm-system仓库包含完整的Git历史。
