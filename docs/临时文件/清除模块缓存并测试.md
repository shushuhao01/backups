# 清除模块缓存并测试

## 问题
在Admin后台禁用了2个模块，但CRM前端还能看到这些模块。

## 原因
前端的`moduleStatusService`有5分钟缓存，需要清除缓存或等待缓存过期。

## 解决方案

### 方案1：清除浏览器缓存（推荐）
1. 打开CRM前端（http://localhost:5173）
2. 按 F12 打开开发者工具
3. 在控制台执行以下命令：
```javascript
// 清除模块状态缓存
localStorage.removeItem('module_status_cache');
sessionStorage.clear();

// 刷新页面
location.reload();
```

### 方案2：使用无痕模式
1. 关闭当前CRM前端页面
2. 使用无痕模式打开浏览器（Ctrl+Shift+N）
3. 访问 http://localhost:5173
4. 重新登录

### 方案3：等待缓存过期
- 缓存时间：5分钟
- 5分钟后自动刷新，无需任何操作

### 方案4：强制刷新（最简单）
1. 在CRM前端页面
2. 按 Ctrl+Shift+R（强制刷新，清除缓存）
3. 或者 Ctrl+F5

## 验证步骤

### 1. 检查数据库状态
```bash
node backend/check-modules-database.js
```

应该看到被禁用的模块状态为 `disabled`。

### 2. 检查API响应
在浏览器控制台查看：
```
[ModuleStatus] API响应完整对象: ...
[ModuleStatus] 提取的模块列表: [...]
```

被禁用的模块不应该出现在列表中。

### 3. 检查菜单显示
左侧菜单栏不应该显示被禁用的模块。

## 如果还是不生效

### 检查1：确认数据库状态
```bash
cd backend
node check-modules-database.js
```

### 检查2：查看控制台日志
刷新CRM前端，查看控制台是否有：
```
[ModuleStatus] 成功更新模块状态: ['dashboard', 'customer', 'order', ...]
```

### 检查3：手动清除缓存
在浏览器控制台执行：
```javascript
// 查看当前缓存
console.log('当前缓存:', localStorage.getItem('module_status_cache'));

// 清除所有缓存
localStorage.clear();
sessionStorage.clear();

// 刷新
location.reload();
```

## 注意事项
1. 模块状态缓存时间：5分钟
2. 每次在Admin后台修改模块状态后，需要清除CRM前端缓存
3. 或者等待5分钟让缓存自动过期
