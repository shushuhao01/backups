**分析时间**: 2026-03-21  
**需求来源**: 用户反馈

## 📋 需求概述

### 核心需求
1. **自动创建管理员账号** - 创建租户/私有客户时，使用手机号作为默认管理员账号
2. **统一信息展示** - 生成授权码后显示完整登录信息（账号、密码、登录地址、授权码）
3. **一键复制功能** - 方便管理员复制信息发送给客户
4. **官网注册同步** - 确保官网注册支付后返回的信息与Admin后台一致
5. **登录信息持久化** - 授权码/租户编码退出后不需要重新输入

## 📊 数据库表结构分析

### users 表（CRM用户表）

```sql
CREATE TABLE users (
  id VARCHAR(36) PRIMARY KEY,
  tenant_id VARCHAR(36),           -- 租户ID（数据隔离）
  username VARCHAR(50) UNIQUE,     -- 用户名（使用手机号）
  password VARCHAR(255),           -- 密码哈希
  salt VARCHAR(255),               -- 密码盐
  real_name VARCHAR(50),           -- 真实姓名
  phone VARCHAR(20),               -- 手机号
  email VARCHAR(100),              -- 邮箱
  role VARCHAR(20),                -- 角色（admin/manager/sales等）
  role_id VARCHAR(36),             -- 角色ID（关联roles表）
  status VARCHAR(20),              -- 状态（active/inactive）
  created_at DATETIME,
  updated_at DATETIME,
  INDEX idx_tenant_id (tenant_id),
  INDEX idx_username (username)
);
```

### tenants 表（SaaS租户表）

```sql
CREATE TABLE tenants (

  max_users INT,                   -- 最大用户数
  expire_date DATE,                -- 到期日期
  status VARCHAR(20),              -- 状态（active/inactive）
  created_at DATETIME,
  updated_at DATETIME
);
```

## 🛠️ 技术实现要点

### 1. 密码生成和加密

```typescript
// 统一的密码生成函数
const generateDefaultPassword = (): string => {
  return 'Aa123456'  // 符合复杂度要求：大写+小写+数字
}

// 统一的密码加密函数
const hashPassword = (password: string): { hash: string; salt: string } => {
  const salt = crypto.randomBytes(16).toString('hex')
  const hash = crypto.pbkdf2Sync(password, salt, 1000, 64, 'sha512').toString('hex')
  return { hash, salt }
}
```

### 2. 管理员账号创建

```typescript
// 统一的管理员创建函数
const createDefaultAdmin = async (params: {
  tenantId: string | null,  // SaaS租户ID，私有部署为null
  phone: string,            // 手机号（作为用户名）
  realName: string,         // 真实姓名
  email?: string            // 邮箱（可选）
}): Promise<{ username: string; password: string }> => {
  const { tenantId, phone, realName, email } = params
  
  // 检查用户名是否已存在
  const existing = await AppDataSource.query(
    'SELECT id FROM users WHERE username = ?', [phone]
  )
  if (existing.length > 0) {
    throw new Error('该手机号已被使用')
  }
  
  const userId = uuidv4()
  const password = generateDefaultPassword()
  const { hash, salt } = hashPassword(password)
  const now = new Date().toISOString().slice(0, 19).replace('T', ' ')
  
  await AppDataSource.query(
    `INSERT INTO users (
      id, tenant_id, username, password, salt, real_name, phone, email,
      role, status, created_at, updated_at
    ) VALUES (?, ?, ?, ?, ?, ?, ?, ?, 'admin', 'active', ?, ?)`,
    [userId, tenantId, phone, hash, salt, realName, phone, email, now, now]
  )
  
  return { username: phone, password }
}
```

## ⚠️ 注意事项

### 1. 数据安全

- 密码必须加密存储（使用 pbkdf2 + salt）
- 初始密码要求用户首次登录后修改
- 管理员账号创建失败不应影响租户创建流程

### 2. 用户体验

- 信息展示要清晰，方便复制
- 一键复制功能要可靠
- 登录持久化要稳定

### 3. 数据隔离

- 每个租户的管理员账号必须关联正确的 `tenant_id`
- 私有部署首次激活时需要创建本地租户记录
- 确保不同租户的数据完全隔离

### 4. 兼容性

- 要兼容现有的登录流程
- 要兼容现有的授权验证逻辑
- 要兼容现有的数据结构
