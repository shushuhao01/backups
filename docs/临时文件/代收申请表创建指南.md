# 代收申请表创建指南

## 问题说明

在执行 `production-add-cod-review-fields-simple.sql` 时报错，原因是生产环境中 `cod_cancel_applications` 表不存在。

## 解决方案

### 方案一：创建完整的表（推荐）

使用 `production-create-cod-application-simple.sql` 一次性创建包含所有字段的表。

#### 执行步骤

1. 登录宝塔面板
2. 进入 phpMyAdmin
3. 选择生产数据库
4. 点击"SQL"标签
5. 复制以下SQL并执行：

```sql
-- 创建代收取消申请表（包含审核字段）
CREATE TABLE `cod_cancel_applications` (
  `id` varchar(36) NOT NULL,
  `order_id` varchar(36) NOT NULL COMMENT '订单ID',
  `order_number` varchar(50) DEFAULT NULL COMMENT '订单号',
  `customer_id` varchar(36) DEFAULT NULL COMMENT '客户ID',
  `customer_name` varchar(100) DEFAULT NULL COMMENT '客户姓名',
  `original_cod_amount` decimal(10,2) DEFAULT 0.00 COMMENT '原代收金额',
  `new_cod_amount` decimal(10,2) DEFAULT 0.00 COMMENT '新代收金额',
  `reason` text COMMENT '申请原因',
  `status` varchar(20) DEFAULT 'pending' COMMENT '状态：pending-待审核, approved-已通过, rejected-已拒绝',
  `applicant_id` varchar(36) DEFAULT NULL COMMENT '申请人ID',
  `applicant_name` varchar(50) DEFAULT NULL COMMENT '申请人姓名',
  `applied_at` datetime DEFAULT NULL COMMENT '申请时间',
  `reviewer_id` varchar(36) DEFAULT NULL COMMENT '审核人ID',
  `reviewer_name` varchar(50) DEFAULT NULL COMMENT '审核人姓名（操作人）',
  `review_remark` text DEFAULT NULL COMMENT '审核备注',
  `reviewed_at` datetime DEFAULT NULL COMMENT '审核时间（处理时间）',
  `created_at` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `tenant_id` varchar(36) DEFAULT NULL COMMENT '租户ID',
  PRIMARY KEY (`id`),
  KEY `idx_order_id` (`order_id`),
  KEY `idx_status` (`status`),
  KEY `idx_tenant_id` (`tenant_id`),
  KEY `idx_applied_at` (`applied_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='代收取消申请表';
```

6. 点击"执行"
7. 如果提示"表已存在"，说明表已经创建过了，可以忽略

#### 验证

执行以下SQL验证表是否创建成功：

```sql
-- 查看表结构
SHOW CREATE TABLE `cod_cancel_applications`;

-- 查看字段列表
SELECT 
  COLUMN_NAME AS '字段名',
  COLUMN_TYPE AS '类型',
  COLUMN_COMMENT AS '注释'
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_SCHEMA = DATABASE()
  AND TABLE_NAME = 'cod_cancel_applications'
ORDER BY ORDINAL_POSITION;
```

### 方案二：分步执行（如果表已存在但缺少字段）

如果表已经存在，但缺少审核相关字段，可以使用原来的 `production-add-cod-review-fields-simple.sql`：

```sql
-- 添加 reviewer_id 字段
ALTER TABLE `cod_cancel_applications` 
ADD COLUMN `reviewer_id` varchar(36) DEFAULT NULL COMMENT '审核人ID' AFTER `status`;

-- 添加 reviewer_name 字段
ALTER TABLE `cod_cancel_applications` 
ADD COLUMN `reviewer_name` varchar(50) DEFAULT NULL COMMENT '审核人姓名（操作人）' AFTER `reviewer_id`;

-- 添加 review_remark 字段
ALTER TABLE `cod_cancel_applications` 
ADD COLUMN `review_remark` text DEFAULT NULL COMMENT '审核备注' AFTER `reviewer_name`;

-- 添加 reviewed_at 字段
ALTER TABLE `cod_cancel_applications` 
ADD COLUMN `reviewed_at` datetime DEFAULT NULL COMMENT '审核时间（处理时间）' AFTER `review_remark`;
```

## 功能说明

### 代收申请审核功能

这个表是用于**代收申请审核**功能的，与**代收管理**是两个不同的功能：

1. **代收管理**（CodCollection）
   - 路径：财务管理 > 代收管理
   - 功能：管理已发货订单的代收款项
   - 表：`orders` 表（使用订单表的代收相关字段）
   - 状态：✅ 已完成，包括标签彩色统计

2. **代收申请审核**（CodApplicationReview）
   - 路径：财务管理 > 代收申请审核
   - 功能：销售人员申请修改代收金额，财务审核
   - 表：`cod_cancel_applications` 表
   - 状态：⏳ 需要创建表

## 注意事项

1. **表不存在的情况**
   - 使用方案一，一次性创建完整的表
   - 这样可以避免分步执行时的错误

2. **表已存在的情况**
   - 如果执行方案一时提示"表已存在"，说明表已经创建过了
   - 可以使用方案二添加缺少的字段

3. **字段已存在的情况**
   - 如果执行ALTER TABLE时提示"字段已存在"，可以忽略
   - 不会影响现有数据
