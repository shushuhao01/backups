# 系统管理部自动创建功能 - 完成总结

## 功能概述

在创建租户（SaaS）或私有客户时，系统会自动创建"系统管理部"，并将默认管理员账号归属到该部门。系统管理部具有特殊保护机制，不可删除、不可禁用、名称和编码不可修改。

## 实现细节

### 1. 核心函数（backend/src/utils/adminAccountHelper.ts）

#### 1.1 系统管理部常量
```typescript
export const SYSTEM_DEPARTMENT = {
  NAME: '系统管理部',
  CODE: 'SYS_ADMIN',
  DESCRIPTION: '系统管理和维护部门'
};
```

#### 1.2 创建系统管理部
```typescript
export const createSystemDepartment = async (tenantId: string | null): Promise<string>
```
- 参数：`tenantId` - 租户ID（SaaS租户），私有部署为null
- 返回：部门ID
- 功能：
  - 检查是否已存在系统管理部（避免重复创建）
  - 创建系统管理部记录到departments表
  - 设置部门名称、编码、描述等信息

#### 1.3 创建默认管理员
```typescript
export const createDefaultAdmin = async (params: {
  tenantId: string | null;
  phone: string;
  realName: string;
  email?: string;
}): Promise<{ username: string; password: string; departmentId: string }>
```
- 功能：
  - 先调用 `createSystemDepartment()` 创建系统管理部
  - 创建管理员账号，使用手机号作为用户名
  - 将管理员归属到系统管理部（设置department_id和department_name）
  - 默认密码：Aa123456
  - 默认角色：admin

- 编辑限制：前端提示不可修改系统预设部门

#### 3.2 后端保护（backend/src/controllers/DepartmentController.ts）

##### 3.2.1 禁止修改名称和编码
```typescript
async updateDepartment(req: Request, res: Response) {
  const systemPresetDepartments = ['系统管理部'];
  if (systemPresetDepartments.includes(department.name)) {
    if (name && name !== department.name) {
      return res.status(400).json({ message: '系统预设部门名称不可修改' });
    }
    if (code && code !== department.code) {
      return res.status(400).json({ message: '系统预设部门编码不可修改' });
    }
  }
}
```

##### 3.2.2 禁止删除
```typescript
async deleteDepartment(req: Request, res: Response) {
  const systemPresetDepartments = ['系统管理部'];
  if (systemPresetDepartments.includes(department.name)) {
    return res.status(400).json({ message: '系统预设部门不可删除' });
  }
}
```

##### 3.2.3 禁止禁用
```typescript
async updateDepartmentStatus(req: Request, res: Response) {
  const nonDisableableDepartments = ['系统管理部'];
  if (status === 'inactive' && nonDisableableDepartments.includes(department.name)) {
    return res.status(400).json({ message: '系统预设部门不可禁用' });
  }
}
```

### 4. 数据库结构

#### 4.1 departments表
```sql
CREATE TABLE departments (
  id VARCHAR(50) PRIMARY KEY,
  tenant_id VARCHAR(36) NULL,           -- 租户ID（私有部署为NULL）
  name VARCHAR(100) NOT NULL,           -- 部门名称（系统管理部）
  code VARCHAR(50) NULL,                -- 部门编码（SYS_ADMIN）
  description TEXT NULL,                -- 部门描述
  parent_id VARCHAR(50) NULL,
  manager_id VARCHAR(50) NULL,
  level INT DEFAULT 1,
  sort_order INT DEFAULT 0,
  status VARCHAR(20) DEFAULT 'active',
  member_count INT DEFAULT 0,
  created_at DATETIME,
  updated_at DATETIME
);
```

#### 4.2 users表（相关字段）
```sql
CREATE TABLE users (
  id VARCHAR(50) PRIMARY KEY,
  tenant_id VARCHAR(36) NULL,           -- 租户ID（私有部署为NULL）
  username VARCHAR(50) NOT NULL,        -- 用户名（手机号）
  password VARCHAR(255) NOT NULL,       -- 密码（加密）
  real_name VARCHAR(100),               -- 真实姓名
  phone VARCHAR(20),                    -- 手机号
  email VARCHAR(100),                   -- 邮箱
  role VARCHAR(50) DEFAULT 'admin',     -- 角色
  role_id VARCHAR(50) DEFAULT 'admin',
  department_id VARCHAR(50),            -- 部门ID（系统管理部ID）
  department_name VARCHAR(100),         -- 部门名称（系统管理部）
  status VARCHAR(20) DEFAULT 'active',
  created_at DATETIME,
  updated_at DATETIME
);
```

## 注意事项

1. **租户隔离**：系统管理部通过tenant_id字段实现租户隔离，每个租户都有独立的系统管理部
2. **私有部署**：私有客户的tenant_id为NULL，系统管理部也为NULL
3. **默认密码**：Aa123456（符合复杂度要求：大写+小写+数字）
4. **手机号验证**：创建私有客户时必须提供手机号，且格式必须正确
5. **幂等性**：重复调用createSystemDepartment不会创建重复部门
6. **错误处理**：创建失败不影响租户/客户创建流程，会记录日志但不抛出错误
