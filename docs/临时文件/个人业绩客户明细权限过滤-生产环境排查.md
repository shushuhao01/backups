# 个人业绩客户明细权限过滤 - 生产环境排查指南

## 问题描述

个人业绩页面的客户明细列表，在开发环境正确过滤（只显示当前用户的客户），但在生产环境仍然显示所有客户。

## 问题原因分析

### 可能的原因

1. **后端代码未更新**
   - 生产环境的后端代码可能没有正确部署最新版本
   - 需要确认生产环境是否包含 `onlyMine` 参数的过滤逻辑

2. **Node.js进程缓存**
   - 即使代码已更新，Node.js进程可能仍在使用旧代码
   - 需要重启后端服务

3. **前端代码未更新**
   - 前端调用API时可能没有传递 `onlyMine=true` 参数
   - 需要确认前端是否正确传参

4. **浏览器缓存**
   - 浏览器可能缓存了旧的前端代码
   - 需要清除浏览器缓存或强制刷新

## 关键代码位置

### 后端代码

文件：`backend/src/routes/customers.ts`

关键代码段（第30-90行）：

```typescript
// 获取查询参数
const { onlyMine } = req.query;

// 获取当前用户信息
const currentUser = (req as any).currentUser || (req as any).user;
const userId = currentUser?.id;

// 如果传入 onlyMine=true，强制只查询当前用户的客户
if (onlyMine === 'true' || onlyMine === true) {
  console.log('[客户列表] 强制只查询当前用户的客户, userId:', userId);
  
  // 查询分享给当前用户的客户
  const shareRepository = AppDataSource.getRepository(CustomerShare);
  const sharedCustomers = await shareRepository.find({
    where: { sharedTo: userId, status: 'active' },
    select: ['customerId']
  });
  const sharedCustomerIds = sharedCustomers.map(s => s.customerId);

  // 构建查询条件
  if (sharedCustomerIds.length > 0) {
    queryBuilder.where(
      '(customer.createdBy = :userId OR customer.salesPersonId = :userId OR customer.id IN (:...sharedIds))',
      { userId, sharedIds: sharedCustomerIds }
    );
  } else {
    queryBuilder.where(
      '(customer.createdBy = :userId OR customer.salesPersonId = :userId)',
      { userId }
    );
  }
}
```

## 常见问题

### Q1: 为什么开发环境正常，生产环境不正常？

A: 可能的原因：
1. 生产环境代码未更新
2. 生产环境服务未重启
3. 生产环境使用了CDN缓存

### Q2: 如何确认生产环境代码是否最新？

A: 
1. SSH登录服务器
2. 运行 `git log -1` 查看最新提交
3. 对比本地代码的最新提交

### Q3: 重启服务后仍然不正常怎么办？

A:
1. 检查是否有多个后端进程在运行
2. 确认重启的是正确的进程
3. 查看后端日志确认是否有错误

### Q4: 如何验证API是否正确过滤？

A:
1. 使用测试脚本验证
2. 查看后端日志
3. 使用Postman或curl直接测试API
