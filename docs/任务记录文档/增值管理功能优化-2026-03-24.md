# 增值管理功能优化

**日期**: 2026-03-24  
**模块**: 财务管理 → 增值管理（ValueAddedManage）  
**任务类型**: 功能优化 + Bug修复  
**状态**: ✅ 已完成

---

## 任务概述

修复和优化增值管理页面的多个问题：
1. 批量修改无效/补单状态时未弹出原因选择对话框
2. 列表勾选后批量操作按钮前无已选数量提示
3. 批量搜索缺失结果无提示，无法知道哪些关键词未匹配到
4. 批量改有效状态未更新备注字段
5. 备注列自动换行显示两行 → 改为单行溢出省略+悬浮提示
6. **批量改无效/补单弹窗重新设计**：统一设置+每条订单独立预览编辑

---

## 修复详情

### 1. 批量修改无效/补单弹出原因对话框

**问题描述**: 
- 单行修改订单为"无效"状态时会弹出原因选择对话框 ✅
- 但**批量**修改为"无效"状态时，直接执行修改，没有弹出原因对话框 ❌
- 修改为"补单"状态（单行/批量）都没有弹出原因对话框 ❌

**修复方案**:
- 新增 `isStatusNeedReason(status)` 函数，判断状态是否需要弹出原因对话框
- 修改 `handleBatchValidStatus` 函数，当目标状态需要原因时，打开备注对话框
- 新增 `batchRemarkContext` 状态变量，存储批量操作的上下文

### 2. 勾选后显示已选数量

**问题描述**: 勾选多个订单后，用户不知道具体选了多少条。
**修复方案**: 在批量操作按钮组最前面添加 `已勾选 N 条` 提示。

### 3. 批量搜索缺失结果提示

**问题描述**: 批量输入多个手机号搜索时，如果某些手机号没有匹配到结果，用户无法知道少了哪几条。
**修复方案**: 搜索框右侧增加 `缺N` 标签，悬浮弹窗列出缺失关键词+一键复制。

### 4. 批量改有效状态更新备注

**问题描述**: 勾选多条订单后批量改为"有效"，备注字段未更新为"有效"。
**修复方案**: `handleBatchValidStatus` 在非弹窗分支中也传递 `remark: statusLabel`（如"有效"）。

### 5. 备注列单行溢出+悬浮提示

**问题描述**: 备注列自动换行显示两行，占用过多空间。
**修复方案**: 
- 使用 CSS `text-overflow: ellipsis` + `white-space: nowrap` 实现单行显示
- 所有有内容的备注绑定 `el-tooltip`，鼠标悬浮300ms后显示完整内容
- 列宽从 `min-width: 180` 增加到 `min-width: 250`

### 6. 🔥 批量原因对话框重新设计（核心改动）

**问题描述**: 批量改无效时弹窗只有一个统一原因输入，无法为每条订单设置不同原因，也无法预览受影响的订单。

**重新设计方案**:

**对话框布局（960px宽）**：
- **顶部 - 统一设置区域**（灰色背景卡片）：
  - 统一原因下拉选择 + 统一备注输入框 + "应用到全部"按钮
  - 点击"应用到全部"将统一设置同步填充到下方每条订单
- **中间 - 分割线**
- **底部 - 受影响订单预览**（可滚动表格，max-height: 360px）：
  - 列：姓名、订单号、单价、修改后状态（Tag显示目标状态）、原因（下拉选择）、备注（输入框）
  - 每条订单可独立修改原因和备注
  - 批量统一后单条仍可再调整

**新增状态变量**：
- `batchOrderItems`: 每条订单的独立原因/备注数据
- `unifiedReasonPreset`: 统一原因下拉值
- `unifiedRemarkText`: 统一备注输入值
- `batchTargetStatusLabel` / `batchTargetStatusType`: 目标状态的显示标签和类型

**提交逻辑**：
- 为每条订单生成最终备注（格式：`状态名：原因/备注`）
- 按备注内容分组，相同备注的订单一起批量API调用（优化请求数）
- 递增使用过的预设原因的使用次数

**单行模式不变**：单行操作（单个订单改无效/补单/恢复）仍使用原有的简单对话框。

### 具体修改点

**模板部分**:
- 备注对话框根据 `batchRemarkContext` 切换单行/批量模式
- 批量模式：统一设置区域 + el-divider + 可滚动预览表格
- 单行模式：保持原有无效/恢复/补单/通用四种类型
- 备注列：改为 `el-tooltip` 包裹 + CSS单行省略

**脚本部分**:
- 新增接口 `BatchOrderItem`：id, orderNumber, customerName, unitPrice, reasonPresetId, customRemark
- 新增状态：`batchOrderItems`, `unifiedReasonPreset`, `unifiedRemarkText`
- 新增计算属性：`batchTargetStatusLabel`, `batchTargetStatusType`
- 新增函数：`applyUnifiedToAll()`, `handleItemReasonChange()`
- 修改函数：`handleBatchValidStatus()`（初始化batchOrderItems）、`confirmRemark()`（批量分组提交）、`cancelRemarkDialog()`（清理状态）

**样式部分**:
- `.batch-unified-section` / `.batch-unified-header` / `.batch-unified-label` / `.batch-unified-row`
- `.batch-preview-section` / `.batch-preview-header` / `.batch-preview-label` / `.batch-preview-tip`
- `.batch-preview-table-wrapper` / `.batch-preview-table`
- `.remark-ellipsis` - 单行省略样式
- 修复：原本在 `</style>` 外的孤立CSS移入 `<style scoped>` 内
