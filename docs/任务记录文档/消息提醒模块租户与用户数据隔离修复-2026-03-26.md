# 消息提醒模块租户与用户数据隔离修复

**任务日期**：2026-03-26  
**任务类型**：Bug修复 — 消息提醒模块前端缓存数据隔离  
**影响范围**：消息铃铛（MessageBell）、通知存储（notification store）、用户登出流程  
**总阶段数**：3  
**当前进度**：全部 3 阶段已完成 ✅

---

## 一、问题描述

### 现象
在同一浏览器中：
1. 先登录A租户（中国移动-欧阳），产生系统消息提醒
2. 退出后登录B租户（欢乐颂-张女士），铃铛消息中心仍能看到欧阳的消息
3. 反之亦然，张女士的消息欧阳也能看到
4. 即使同一租户内不同成员之间也能互相看到彼此的消息

### 影响
- **数据安全**：租户间敏感业务信息泄露（订单、客户名称、金额等）
- **用户隐私**：同租户内不同成员看到彼此的个人消息
- **业务准确性**：消息未读计数不准确，显示非本人的消息

---

## 二、根本原因分析

### 后端 API 层（✅ 已有隔离，无需修改）

| API | tenant_id 隔离 | userId 隔离 |
|-----|:---:|:---:|
| `GET /system-messages` | ✅ | ✅ target_user_id |
| `POST /system-messages/send` | ✅ | ✅ |
| `PUT /system-messages/read-all` | ✅ | ✅ |
| `PUT /system-messages/:id/read` | - | ✅ |
| `DELETE /system-messages/:id` | ✅ | ✅ |
| `DELETE /system-messages/clear-all` | ✅ | ✅ |
| `GET /announcements/published` | ✅ | ✅ 角色/部门 |
| `PUT /announcements/:id/read` | - | ✅ |

### 前端缓存层（🔴 问题所在）

| # | 问题 | 文件 | 说明 |
|---|------|------|------|
| P1 | `notification-messages` localStorage Key 全局共享 | `stores/notification.ts` | 所有用户共享同一个Key，切换用户后读到上一个人的消息 |
| P2 | `hidden-announcements` localStorage Key 全局共享 | `components/MessageBell.vue` | 所有用户共享隐藏公告列表 |
| P3 | `notification-mock-cleaned-v2` 全局共享 | `stores/notification.ts` | 清理标记与用户无关 |
| P4 | 登出时未清理通知缓存 | `stores/user.ts` | `clearUserData()` 未清理消息相关 localStorage |
| P5 | API返回空消息时保留旧缓存 | `stores/notification.ts` | 新用户无消息时展示上一个用户的旧消息 |
| P6 | 合并策略保留上一用户的本地消息 | `stores/notification.ts` | `localOnlyMessages` 包含上一个用户的数据 |

---

## 三、修复方案

### 核心策略：三层防护

1. **localStorage Key 按用户+租户隔离** → 从根本上隔离存储
2. **登出时彻底清空消息缓存** → 防止敏感数据残留
3. **API加载时不合并旧数据** → 完全以服务端为准

---

## 四、修改文件清单

### 4.1 `src/stores/notification.ts` — 通知存储（核心修改）

**修改点：**

#### ① 新增用户隔离 Key 生成逻辑
```typescript
// 🔥 当前用户标识缓存
let currentStorageKey = 'notification-messages'

// 生成隔离的 localStorage Key
// 格式: notification-messages-{userId}-{tenantId}
const getStorageKey = (): string => {
  const userStr = localStorage.getItem('user') || localStorage.getItem('crm_current_user')
  if (userStr) {
    const user = JSON.parse(userStr)
    const userId = user.id || user.userId || ''
    const tenantId = user.tenantId || ''
    if (userId) {
      return `notification-messages-${userId}-${tenantId}`
    }
  }
  return 'notification-messages'
}

const refreshStorageKey = () => {
  currentStorageKey = getStorageKey()
}
```

#### ② 存储读写使用隔离 Key
```typescript
// 读取
const stored = localStorage.getItem(currentStorageKey)
// 保存
localStorage.setItem(currentStorageKey, JSON.stringify(msgs))
```

#### ③ 旧缓存自动迁移清理（v3）
```typescript
const checkAndCleanOldMockData = () => {
  const cleanedKey = 'notification-mock-cleaned-v3'
  if (localStorage.getItem(cleanedKey)) return

  // 清理旧的全局共享数据
  localStorage.removeItem('notification-messages')
  localStorage.removeItem('notification-mock-cleaned-v2')
  localStorage.removeItem('hidden-announcements')
  localStorage.setItem(cleanedKey, 'true')
}
```

#### ④ API 加载不再合并旧数据
```typescript
// 修改前（有数据泄露风险）：
const localOnlyMessages = messages.value.filter(m => !existingIds.has(m.id))
messages.value = [...newMessages, ...localOnlyMessages]

// 修改后（完全以服务端为准）：
messages.value = newMessages
```

#### ⑤ API 返回空消息时清空缓存
```typescript
// 修改前：保留旧缓存
if (apiMessages.length === 0) {
  return []  // 上一个用户的消息仍留在内存中
}

// 修改后：清空
if (apiMessages.length === 0) {
  messages.value = []
  saveMessagesToStorage(messages.value)
  return []
}
```

#### ⑥ 新增 `resetNotificationState()` 方法
```typescript
// 登出时调用，彻底清空内存和缓存
const resetNotificationState = () => {
  messages.value = []
  localStorage.removeItem(currentStorageKey)
  localStorage.removeItem('notification-messages')
  currentStorageKey = 'notification-messages'
}
```

### 4.3 `src/stores/user.ts` — 用户存储

**修改点：**

#### ① `clearUserData()` 中增加消息缓存清理
```typescript
// 清理消息通知缓存
try {
  localStorage.removeItem('notification-messages')
  const savedUserStr = localStorage.getItem('user') || ...
  if (savedUserStr) {
    const savedUser = JSON.parse(savedUserStr)
    const uid = savedUser.id || ''
    const tid = savedUser.tenantId || ''
    if (uid) {
      localStorage.removeItem(`notification-messages-${uid}-${tid}`)
      localStorage.removeItem(`hidden-announcements-${uid}-${tid}`)
    }
  }
} catch (_e) {
  localStorage.removeItem('notification-messages')
}
```

#### ② `logout()` 中增加 WebSocket 断开 + 通知状态重置
```typescript
const logout = async () => {
  // 断开 WebSocket + 重置通知状态
  try {
    const { useNotificationStore } = await import('@/stores/notification')
    const notificationStore = useNotificationStore()
    await notificationStore.disconnectWebSocket()
    notificationStore.resetNotificationState()
  } catch (_e) { }

  // 原有登出逻辑...
  clearUserData()
}
```

## 六、修复效果

| 场景 | 修复前 | 修复后 |
|------|--------|--------|
| A租户用户登出，B租户用户登入 | 看到A租户的消息 | 只看到B租户自己的消息 |
| 同租户成员A退出，成员B登入 | 看到成员A的消息 | 只看到成员B自己的消息 |
| API返回空消息（新用户） | 显示上一个用户的旧消息 | 显示空列表 |
| 用户隐藏了公告后切换用户 | 新用户也看不到被隐藏的公告 | 每个用户独立的隐藏列表 |
| 登出后 localStorage 残留 | 消息缓存残留在浏览器中 | 缓存已清理 |
