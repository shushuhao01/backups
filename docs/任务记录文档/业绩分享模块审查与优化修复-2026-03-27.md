## 一、问题清单

### 问题1：🔴 业绩守恒缺陷 — 分享比例设计导致原始下单人可能丢失全部业绩

**现象**：当用户A的订单分享给B和C，分享表单要求所有成员的比例总和 = 100%。如果A将50%分给B、50%分给C，A自己不在分享成员列表中，则：
- A的业绩被扣除100%（`remainingPercentage = 100 - 100 = 0`），A的订单业绩和单数变为0
- B获得50%业绩，C获得50%业绩

**根因**：分享表单中的"成员"仅代表接收方，原始下单人未被自动纳入分享成员列表。计算逻辑是 `原始业绩 - 总分享比例 = 下单人保留业绩`，当总分享比例=100%时下单人业绩归零。

**影响文件**：
- `src/views/Performance/Share.vue` - 分享表单
- `src/stores/performance.ts` - 业绩计算逻辑

---

### 问题2：🔴 个人业绩/团队业绩详情中无分享标记

**现象**：在个人业绩和团队业绩的订单明细表格中，已被分享的订单没有任何视觉标记。用户无法直观看出：
- 哪些订单被分享了
- 分享了多少金额/比例
- 与谁共享

**影响文件**：
- `src/views/Performance/Personal.vue` - 订单明细表格（第212-291行）
- `src/views/Performance/Team.vue` - 成员详情弹窗的订单表格（第287-340行）

---

### 问题3：🟡 后端API未考虑分享对业绩数据的影响

**现象**：后端 `/api/v1/performance/personal` 和 `/api/v1/performance/team` 接口仅基于原始订单金额计算业绩，未扣除分享部分和增加接收部分。虽然前端在计算属性中做了补偿处理，但前后端数据不一致。

**影响文件**：
- `backend/src/routes/performance.ts` — `GET /personal`（第366-497行）、`GET /team`（第503-690行）

---

### 问题4：🟡 租户隔离微小漏洞

**现象**：在分享列表查询的 `userId` 过滤条件中，EXISTS子查询访问 `performance_share_members` 表时未加租户隔离条件：
```sql
EXISTS (SELECT 1 FROM performance_share_members psm 
  WHERE psm.share_id = ps.id AND psm.user_id = ?)
-- 缺少：AND psm.tenant_id = ?
```

虽然外层查询已经过滤了 `ps.tenant_id`，但防御性不足。

**影响文件**：
- `backend/src/routes/performance.ts` — `GET /shares`（第44-46行）

---

### 问题5：🟡 数据范围控制函数 `super_admin` 角色遗漏

**现象**：在 `performance.ts` store 的 `applyDataScopeControl` 函数中，只检查了 `admin` 角色有全量数据权限，遗漏了 `super_admin` 角色：
```typescript
if (currentUser.role === 'admin') { // 缺少 super_admin
  return orders
}
```

**影响文件**：
- `src/stores/performance.ts` — `applyDataScopeControl`（第770行）
- `src/views/Performance/Share.vue` — `applyDataScopeControl`（第820行）

---

### 问题6：🟡 分享时间与订单时间维度不一致

**现象**：在 `personalPerformance` 计算中：
- 步骤1：按**订单创建时间**过滤，扣除分享比例
- 步骤2：按**分享创建时间**过滤，加上接收到的分享

如果1月的订单在2月才创建分享记录，则：
- 查看1月数据：订单全额显示（分享还不存在）
- 查看2月数据：分享金额被加入，但对应的订单扣除不在2月范围内

**影响文件**：
- `src/stores/performance.ts` — `personalPerformance` 计算属性（第337-353行）

## 二、修复方案

### 方案1：修复分享表单 — 业绩守恒机制（问题1）

**修改点**：
1. 在 `Share.vue` 的分享表单中，分享成员的比例总和由"必须=100%"改为"允许≤100%"
2. 原始下单人自动保留 `100% - 分享总比例` 的业绩（守恒）
3. 实时显示"下单人保留: X% (¥xxx)"信息
4. 当分享比例=100%时显示警告：⚠️ 原始下单人将不保留任何业绩和单数
5. 默认1个成员50%（下单人保留50%），而非原来的2个成员各50%
6. 后端同步验证规则改为≤100%且>0%

### 方案2：个人业绩/团队业绩添加分享标记（问题2）

**修改点**：
1. `Personal.vue` 订单明细表格：新增"分享状态"列，显示分享徽标和悬浮详情
2. `Team.vue` 成员详情弹窗：同样新增分享标记列
3. 鼠标悬浮显示 tooltip：分享比例、分享金额、共享成员

### 方案3：修复租户隔离和角色遗漏（问题4、5）

**修改点**：
1. 后端分享列表查询的EXISTS子查询添加租户隔离
2. 前端 `applyDataScopeControl` 补充 `super_admin` 角色检查

### 方案4：统一分享时间维度（问题6）

**修改点**：
1. 分享业绩的时间参考统一使用**订单创建时间**而非分享创建时间
