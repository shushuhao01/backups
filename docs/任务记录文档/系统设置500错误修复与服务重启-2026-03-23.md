## 二、根因分析

### 2.1 支付方式 500 错误
- `payment_method_options` 数据库表缺少 `tenant_id` 列（或表不存在）
- 实体定义了 `tenantId` 字段，但 `synchronize: false` 不会自动建表/加列
- 唯一索引 `idx_value(value)` 未包含 `tenant_id`，多租户冲突

### 2.2 商品设置 500 错误
- `system_configs` 表唯一索引 `idx_config_key_group(configKey, configGroup)` 未包含 `tenant_id`
- 不同租户尝试插入相同 `configKey+configGroup` 记录被拒绝
- JSON 类型配置序列化错误（`String(array)` → `"a,b"` 而非 `'["a","b"]'`）

### 2.3 服务启动失败（新发现）
**8个源代码文件编码损坏**，中文字符变为乱码，导致 TypeScript 字符串字面量解析失败：

## 五、租户数据隔离说明

订单设置（支付方式配置）和商品设置（优惠折扣）均为**租户级别隔离**数据：

| 配置类型 | 存储表 | 隔离方式 |
|---------|--------|---------|
| 支付方式配置 | `payment_method_options` | `tenant_id` + `getTenantRepo` Proxy |
| 商品优惠折扣设置 | `system_configs` (group=product_settings) | `tenant_id` + `getTenantRepo` Proxy |
| 订单流转设置 | `system_configs` (group=order_settings) | `tenant_id` + `getTenantRepo` Proxy |
| 安全设置等 | `system_configs` (各分组) | `tenant_id` + `getTenantRepo` Proxy |

每个租户的设置互不影响，通过 `getTenantRepo()` 自动注入 `tenant_id` 过滤和写入。
