## 问题分析与修复

### 问题1：支付方式预设数据需要自动初始化

**现象**：代码改了但预设支付方式仍然为空，因为迁移脚本未执行

**根因**：
- `schema.sql` 改了但只对新建数据库生效
- 迁移脚本 `fix-order-settings-tenant-isolation.sql` 需要手动执行
- 运行中的数据库 `payment_method_options` 表缺少 `is_system` 列和预设数据

**修复方案**：创建 `initOrderSettings.ts`，在后端启动时自动检查并修复数据库：
- 自动添加缺失的列（`tenant_id`、`is_system`）
- 自动修复唯一索引
- 自动初始化6种预设支付方式（微信支付、支付宝支付、银行转账、云闪付、货到付款、其他）
- 在 `database.ts` 的 `initializeDatabase()` 中调用

### 问题2：部门下单限制保存仍失败（500错误）

**根因**：与问题1相同 — 数据库结构修改只在 schema.sql 中，未应用到运行中的数据库

**修复**：`initOrderSettings.ts` 同时自动修复 `department_order_limits` 表：
- 检查并添加 `tenant_id` 列
- 检查并替换唯一索引 `(department_id)` → `(tenant_id, department_id)`

### 问题3：自定义字段列设置显示默认名称

**现象**：在系统设置中配置了自定义字段（如将 custom_field1 命名为"病史"），但在订单列表的"列设置"下拉中仍显示"自定义字段1"

**根因分析**：
1. `Order/List.vue` 中的 `tableColumns` computed 已有逻辑从 `fieldConfigStore` 获取正确名称 ✓
2. `DynamicTable.vue` 将 `tableColumns` 传给 `TableColumnSettings.vue`
3. **`TableColumnSettings.vue` 仅在 `onMounted` 时初始化列数据**，不监听 `props.columns` 变化
4. 异步加载 store 时，初始化已完成，之后 props 中的 label 更新不会同步到列设置面板

**修复**：在 `TableColumnSettings.vue` 中添加 `watch`，监听 `props.columns` 变化时同步更新 label：
```typescript
watch(() => props.columns, (newColumns) => {
  // 只更新 label，保持 visible 和 order 不变
  newColumns.forEach(newCol => {
    const existing = tableColumns.value.find(col => col.prop === newCol.prop)
    if (existing && existing.label !== newCol.label) {
      existing.label = newCol.label
    }
  })
})
```

**影响范围**：该修复同时生效于所有使用 `DynamicTable` 的页面：
- 订单列表（`Order/List.vue`）
- 物流发货（`Logistics/Shipping.vue`）
- 其他使用 `DynamicTable + TableColumnSettings` 组件的页面

**导出功能**：
- `export.ts` 已有逻辑从 `localStorage.getItem('crm_order_field_config')` 读取字段配置
- **问题**：`orderFieldConfig` store 从 API 加载后不写入 localStorage
- **修复**：在 store 的 `loadConfig()` 和 `saveConfig()` 成功后，同步写入 `localStorage`

## 启动/部署说明

后端重启后将自动执行以下操作（无需手动干预）：
1. 检查 `department_order_limits` 表，自动添加 `tenant_id` 列和联合唯一索引
2. 检查 `payment_method_options` 表，自动添加 `tenant_id`、`is_system` 列和唯一索引
3. 自动插入6种预设支付方式（如不存在）
4. 检查 `system_configs` 表，自动修复唯一索引为含 `tenant_id` 的联合唯一索引

**重启命令**：
```bash
# 在 backend 目录下
npm run dev:local
```
