# 增值管理配置管理-系统预设全租户共享修复

**日期**: 2026-03-26  
**模块**: 财务管理 > 增值管理 > 配置管理（状态配置弹窗）  
**类型**: 功能修复（租户隔离后系统预设数据不可见）

---

## 问题描述

### 现象
- 打开增值管理的"状态配置"弹窗，有效状态、结算状态、备注预设三个标签页均显示"暂无配置，请添加"
- 之前在未做租户隔离时这些系统预设内容是存在的

### 根因分析
1. **数据绑定到单一租户**: 租户隔离改造后，所有 6 条状态配置和 15 条备注预设数据被分配给了租户 `198eeeb5`（天河佳信），其他租户查询时因 `tenant_id` 过滤条件不匹配而看不到
2. **ensureSystemDefaultConfigs()** 使用 `getTenantRepo()` 创建预设，导致每次只为当前租户创建副本（设计上有缺陷）
3. **ensureSystemDefaultRemarkPresets()** 函数虽然已定义，但在 `GET /remark-presets` 路由中**从未被调用**
4. **value_added_remark_presets 表缺少 `is_system` 列**，无法区分系统预设和自定义预设

### 数据库实际状态（修复前）
```
value_added_status_configs: 6条记录, 全部 tenant_id = '198eeeb5-...'
value_added_remark_presets: 15条记录, 全部 tenant_id = '198eeeb5-...', 无 is_system 列
```

---

## 设计方案

### 核心原则：系统预设全局共享，租户自定义私有
- **系统预设**: `tenant_id = NULL`, `is_system = 1` → 全租户可见，不可删除
- **租户自定义**: `tenant_id = <tenantId>`, `is_system = 0` → 仅本租户可见，可删除
- **查询逻辑**: `WHERE tenant_id IS NULL OR tenant_id = ?`（返回系统 + 本租户的合集）

### 优势
- 数据库只存一份系统预设，不需要为每个租户重复创建
- 任何新租户首次访问即可看到系统预设
- 租户可以添加自己的自定义状态/备注，互不影响

| 文件路径 | 修改内容 |

| `src/api/valueAdded.ts` | RemarkPreset 接口添加 `is_system` 字段 |

### 后端修改详情 (`valueAdded.ts`)

#### 1. `ensureSystemDefaultConfigs()` 函数
- **修改前**: 使用 `getTenantRepo()` 创建预设 → 绑定到当前租户
- **修改后**: 使用原生 SQL 创建预设，`tenant_id = NULL`，全局共享
- 新增一次性迁移逻辑：`UPDATE ... SET tenant_id = NULL WHERE is_system = 1 AND tenant_id IS NOT NULL`

#### 2. `ensureSystemDefaultRemarkPresets()` 函数
- **修改前**: 创建预设绑定到当前租户，且从未被调用
- **修改后**: 使用原生 SQL 创建预设，`tenant_id = NULL`；自动添加 `is_system` 列；迁移旧数据

#### 3. `GET /status-configs` 路由
- **修改前**: 使用 `getTenantRepo()` 查询 → 只返回当前租户数据
- **修改后**: 使用原生 SQL 查询 `WHERE (tenant_id IS NULL OR tenant_id = ?)` → 返回系统 + 本租户数据

#### 4. `DELETE /status-configs/:id` 路由
- **修改前**: 使用 `getTenantRepo()` 查找配置
- **修改后**: 使用原生 SQL 查找，确保系统预设 (`is_system=1`) 不可删除

#### 5. `GET /remark-presets` 路由
- **修改前**: 不调用初始化函数，使用 `tenantSQL()` 过滤只返回本租户数据
- **修改后**: 首先调用 `ensureSystemDefaultRemarkPresets()`，然后查询 `WHERE (tenant_id IS NULL OR tenant_id = ?)` 返回系统 + 本租户数据

#### 6. `DELETE /remark-presets/:id` 路由
- **修改前**: 无系统预设保护
- **修改后**: 删除前检查 `is_system` 标志，系统预设返回 403 禁止删除

### 前端修改详情

#### `ValueAddedConfigDialog.vue`
- 无效原因预设和通用备注预设列表：系统预设显示 🔒 图标和"系统预设"标签
- 系统预设不显示删除按钮

#### `src/api/valueAdded.ts`

## 数据库变更

### 自动执行（首次API访问时）
1. `value_added_status_configs` 表：已有的 `is_system=1` 记录自动迁移为 `tenant_id=NULL`
2. `value_added_remark_presets` 表：自动添加 `is_system TINYINT DEFAULT 0` 列
3. `value_added_remark_presets` 表：匹配系统预设的旧数据自动迁移为 `tenant_id=NULL, is_system=1`

### 修复后数据库状态
```
value_added_status_configs: 6条系统预设, tenant_id = NULL, is_system = 1
value_added_remark_presets: 15条系统预设, tenant_id = NULL, is_system = 1
```
