### 1.1 增值管理路由（valueAdded.ts）- 严重问题

**问题说明**: valueAdded.ts 中几乎所有数据库操作均使用 `AppDataSource.getRepository()` 而非 `getTenantRepo()`，导致在SaaS多租户环境下完全没有租户数据隔离。任何租户的用户都能看到、修改、删除其他租户的增值订单、外包公司、价格档位、状态配置等数据。

**影响范围**: 
- 增值订单列表查询（GET /orders）
- 订单同步函数（syncOrdersToValueAddedOptimized、syncOrdersToValueAdded）
- 统计接口（GET /stats）
- 创建订单（POST /orders）
- 批量处理订单（PUT /orders/batch-process）
- 公司统计函数（updateCompanyStats）
- 外包公司CRUD（GET/POST/PUT/DELETE /companies）
- 结算报表（GET /settlement-report）
- 状态配置管理（GET/POST/DELETE /status-configs）
- 价格档位管理（GET/POST/PUT/DELETE /companies/:companyId/price-tiers）
- 设置默认公司（PUT /companies/:id/set-default）
- 公司排序（PUT /companies/sort）- 存在两处重复路由
- 批量修改订单公司（PUT /orders/batch-update-company）
- 修改单个订单公司（PUT /orders/:id/company）

**修复方式**:

| 修复项 | 修改前 | 修改后 |
|--------|--------|--------|
| 仓储获取 | `AppDataSource.getRepository(Entity)` | `getTenantRepo(Entity)` |

| 原始SQL插入 | 未写入 tenant_id | 获取 `TenantContextManager.getTenantId()` 并写入 |
| 导入 | 未导入租户工具 | 添加 `import { getTenantRepo, tenantSQL }` |

### 1.2 绩效管理路由（finance.ts）- leftJoin问题

**问题说明**: finance.ts 中绩效数据列表和绩效管理列表使用 `leftJoin` 关联 `Department` 和 `User` 表时，未添加租户过滤条件，可能在极端情况下匹配到其他租户的部门名称或用户名称。

**修复方式**: 在 leftJoin 的 ON 条件中添加 `AND dept/creator.tenant_id = :_joinTid`（仅在SaaS模式下生效）。

**修复的具体位置**:
1. GET /performance-data - 绩效数据列表（只读）的 leftJoin
2. GET /performance-manage - 绩效管理列表（可编辑）的 leftJoin

### 2.2 performance.ts 修复（2处）

#### 修复1：GET /shares 子查询添加租户过滤

**问题说明**: `performance_share_members` 的子查询只按 `share_id` 关联，未添加 `psm.tenant_id` 过滤条件。虽然外层查询已有 `ps.tenant_id` 过滤，但在防御纵深原则下，子查询也应独立过滤。

**修复前**:
```sql
(SELECT JSON_ARRAYAGG(...) FROM performance_share_members psm 
 WHERE psm.share_id = ps.id) as shareMembers
```

**修复后**:
```sql
(SELECT JSON_ARRAYAGG(...) FROM performance_share_members psm 
 WHERE psm.share_id = ps.id AND psm.tenant_id = ?) as shareMembers
```

**代码变更**:
- 新增 `const tSub = tenantSQL('psm.');`
- 子查询末尾追加 `${tSub.sql}`
- params 数组开头插入 `...tSub.params`（因为子查询参数在SQL中位于外层查询之前）

#### 修复2：GET /stats 用户统计 JOIN 添加 psm 表租户过滤

**问题说明**: 用户统计查询 `JOIN performance_share_members psm ON ps.id = psm.share_id` 只有 `ps.tenant_id` 过滤，缺少 `psm.tenant_id` 过滤。

**修复前**:
```typescript
const tStatsUser = tenantSQL('ps.');
// WHERE (psm.user_id = ? OR ps.created_by = ?)${tStatsUser.sql}
```

**修复后**:
```typescript
const tStatsUser = tenantSQL('ps.');
const tStatsUserMember = tenantSQL('psm.');
// WHERE (psm.user_id = ? OR ps.created_by = ?)${tStatsUser.sql}${tStatsUserMember.sql}
```

### 数据隔离保证

即使不同租户存在：
- ✅ **同一客户手机号** → 通过 `tenant_id` 隔离，各租户独立
- ✅ **相同用户名** → 通过 `tenant_id` 隔离，各租户独立
- ✅ **相同订单号** → 通过 `tenant_id` 隔离，各租户独立
- ✅ **相同公司名** → 通过 `tenant_id` 隔离，各租户独立
