## 三、阶段 1 修复内容（已完成 ✅）

### 3.1 修复 `departmentStore.loadDepartments` 报错

**修改文件**:
- `src/views/Logistics/List.vue` (行 1090)
- `src/views/Logistics/StatusUpdate.vue` (行 1260)
- `src/views/System/User.vue` (行 2536)
- `src/views/System/DepartmentDetail.vue` (行 501)

**修改内容**: `departmentStore.loadDepartments()` → `departmentStore.fetchDepartments()`

### 3.2 修复 Companies.vue API 500 错误

**修改文件**: `src/views/Logistics/Companies.vue`

**原因**: `loadData` 函数使用 `apiService.get()` 传参格式错误：
```js
// ❌ 错误：apiService.get 的第二参数直接作为 query params
apiService.get('/logistics/companies/list', { params: { page, pageSize } })
// 导致 URL: ?params[page]=1&params[pageSize]=10

// ✅ 修复：改用已导入的 logisticsApi，使用正确的 api.get 格式
logisticsApi.getCompanies({ page, pageSize })
```

同时修复了 `tableData` 的类型声明：`ref([])` → `ref<LogisticsCompany[]>([])`

### 3.3 修复物流公司管理端点租户隔离

**修改文件**: `backend/src/routes/logistics.ts`

将所有物流公司管理端点（6个）从 `AppDataSource!.getRepository(LogisticsCompany)` 改为 `getTenantRepo(LogisticsCompany)`：
- `GET /companies/list`
- `GET /companies/active`
- `POST /companies`
- `PUT /companies/:id`
- `PATCH /companies/:id/status`
- `DELETE /companies/:id`

### 3.4 修复实体 unique 约束

**修改文件**:
- `backend/src/entities/LogisticsCompany.ts`: 移除 `code` 列的 `unique: true`
- `backend/src/entities/LogisticsApiConfig.ts`: 移除 `companyCode` 列的 `unique: true`

### 4.9 修复调试端点安全漏洞

**端点**: `GET /debug/order-phone`

**问题**: 该端点在 `authenticateToken` 中间件之前定义，完全无需认证即可查看任意租户的订单数据
**修复**: 
- 将端点移到 `router.use(authenticateToken)` 之后，强制要求认证
- 使用 `getTenantRepo(Order)` 确保只能查看当前租户的数据
- `createQueryBuilder` 中 `.where()` 改为 `.andWhere()` 配合租户过滤

### 4.10 清理无用导入

移除不再使用的 `AppDataSource` 直接导入。整个 `logistics.ts` 文件中不再有任何直接使用 `AppDataSource!.getRepository()` 的代码。

## 五、阶段 3 修复内容（已完成 ✅）

### 5.1 修复 ensureDefaultLogisticsCompanies() 初始化逻辑缺陷

**问题**：原代码用 `if (count === 0)` 判断，当租户已有>=1条记录时不再初始化剩余公司。导致某些租户只看到1家物流公司（德邦）。

**修复**：改为逐个检查每个预设公司代码（SF/ZTO/YTO/STO/YD/JTSD/EMS/JD/DBL）是否存在，缺失则补充插入。

**修改文件**：`backend/src/routes/logistics.ts` — `ensureDefaultLogisticsCompanies()`

### 5.2 修复 ensureDefaultApiConfigs() 同样的逻辑缺陷

同上，从 `count === 0` 改为逐个 companyCode 检查。

**修改文件**：`backend/src/routes/logistics.ts` — `ensureDefaultApiConfigs()`

### 5.3 为所有现有租户补充完整物流公司和API配置数据

通过Node.js脚本（mysql2连接 charset:utf8mb4）为7个租户各插入9家物流公司和9条API配置。
