## 一、任务背景

改造CRM消息管理模块，实现：

1. **公告分类**：公司公告（租户内部发布）与系统公告（管理后台发布）分离
2. **系统公告**：管理后台发布，全员或指定公司/角色/部门收到（用于系统维护升级等通知）
3. **公司公告**：租户管理员发布，仅通知本公司成员（用于内部通知）
4. **颜色区分**：CRM前端通过颜色标签区分系统公告和公司公告
5. **第三方通知配置**：各租户独立维护自己的通知渠道配置
6. **超时提醒/消息清理**：按租户隔离
7. **管理后台**：新增消息管理子菜单，支持系统公告发布

## 四、设计说明

### 系统公告投递机制

系统公告不直接创建 `system_messages`，而是采用**拉取模式**：

1. 管理后台创建系统公告（`source='system'`，`tenantId=null`）
2. CRM 用户获取公告时，`getPublishedAnnouncements` 同时查询：
   - 本租户的公司公告（`tenant_id = 当前租户`）
   - 所有系统公告，再按 `target_tenants` 过滤是否面向当前租户
3. 前端按 `source` 字段显示不同颜色标签

### 公告类型区分

| 属性 | 公司公告 | 系统公告 |
|------|---------|---------|
| source | `company` | `system` |
| tenant_id | 创建者的租户ID | `null` |
| target_tenants | 不使用 | JSON数组（null=全部租户） |
| 创建入口 | CRM消息管理 | 管理后台 |
| 可见范围 | 仅本租户成员 | 指定租户/全部租户 |
| 颜色标识 | 蓝色 | 红/橙色 |

### 6.2 MessageCleanupService 租户隔离改造

**文件**: `backend/src/services/MessageCleanupService.ts`

**改造前问题**：
- `getConfig()` 使用 `AppDataSource.query()` 无租户过滤，读取的是所有租户混合配置
- `executeCleanup()` DELETE 操作无 `tenant_id` 条件，会删除所有租户的数据
- `manualCleanup()` 同样无租户过滤

**改造后架构**：
1. 新增 `executeForAllTenants()` 方法：查询所有有清理配置的租户，逐一执行
2. `getConfig(tenantId)` 改为按租户读取配置
3. `executeCleanup(config, tenantId)` 添加 `AND tenant_id = ?` 条件
4. `manualCleanup()` 通过 `TenantContextManager.getTenantId()` 获取当前租户
5. SaaS vs 私有模式分支：SaaS 遍历租户，私有模式全局执行

```typescript
// SaaS模式：遍历所有租户独立执行
private async executeForAllTenants() {
  const rows = await AppDataSource.query(
    `SELECT DISTINCT tenant_id FROM system_configs WHERE configKey = ? AND tenant_id IS NOT NULL`,
    [CONFIG_KEY]
  )
  for (const tenantId of tenantIds) {
    await TenantContextManager.run({ tenantId }, async () => {
      await this.executeCleanup(config, tenantId)
    })
  }
}
```

### 6.3 TimeoutReminderService 租户隔离改造

**文件**: `backend/src/services/TimeoutReminderService.ts`

**改造前问题**：
- 后台服务运行时无 HTTP 请求上下文 → `getTenantRepo()` 无 TenantContext → 返回无租户过滤的基础仓储
- 所有租户的订单/售后数据混在一起检测

**改造后架构**：
1. 新增 `runChecksForAllTenants()` 方法：查询所有活跃租户，逐一在 `TenantContextManager.run()` 中执行
2. 抽取 `runChecksOnce()` 方法：在当前租户上下文中执行一次完整检测
3. `runAllChecks()` 根据 SaaS/私有模式分支调度

```typescript
// SaaS模式：遍历所有活跃租户
private async runChecksForAllTenants() {
  const tenants = await dataSource.query(`SELECT id FROM tenants WHERE status = 'active'`)
  for (const tenant of tenants) {
    await TenantContextManager.run({ tenantId: tenant.id }, async () => {
      await this.runChecksOnce()
    })
  }
}
```
