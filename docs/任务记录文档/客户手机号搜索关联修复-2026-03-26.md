## 问题描述

客户信息创建后，如果客户新增了手机号（存储在 `other_phones` JSON数组字段），通过新手机号在客户列表、订单列表、售后、物流、财务等模块搜索时均查不到对应的客户和关联数据。

### 根本原因

客户实体有两个手机号字段：
- `phone`：主手机号（VARCHAR字段）
- `otherPhones` / `other_phones`：其他手机号（JSON数组字段）

系统所有搜索功能（共17处）仅搜索了主手机号 `phone` 字段，未搜索 `other_phones` 字段。

## 修复策略

### 搜索方式
- **模糊搜索场景**：使用 `CAST(customer.other_phones AS CHAR) LIKE '%keyword%'`
- **精确匹配场景**（check-exists）：使用 `JSON_CONTAINS(customer.other_phones, JSON_QUOTE(:phone))`
- **订单/售后关联搜索**：通过 `EXISTS (SELECT 1 FROM customers c WHERE c.id = xxx.customer_id AND c.tenant_id = xxx.tenant_id AND CAST(c.other_phones AS CHAR) LIKE :kw)` 子查询

### 租户数据隔离保障
- `getTenantRepo` 自动为所有查询添加 `tenant_id` 过滤
- 子查询中通过 `c.tenant_id = order.tenant_id` 确保跨表查询不泄露数据
- 即使不同租户存在相同手机号，查询结果也严格按租户隔离

## 阶段1修改详情（已完成 ✅）

### 修改文件：`backend/src/routes/customers.ts`

#### 1. 客户列表keyword搜索（行146）
**修改前**：
```sql
customer.name LIKE :keyword OR customer.phone LIKE :keyword OR customer.customerNo LIKE :keyword
```
**修改后**：
```sql
customer.name LIKE :keyword OR customer.phone LIKE :keyword OR customer.customerNo LIKE :keyword OR CAST(customer.other_phones AS CHAR) LIKE :keyword
```

#### 2. 客户列表phone筛选（行154）
**修改前**：
```sql
customer.phone LIKE :phone
```
**修改后**：
```sql
customer.phone LIKE :phone OR CAST(customer.other_phones AS CHAR) LIKE :phone
```

#### 3. 客户搜索API /search（行951）
**修改前**：
```sql
customer.name LIKE :keyword OR customer.phone LIKE :keyword OR customer.customerNo LIKE :keyword
```
**修改后**：
```sql
customer.name LIKE :keyword OR customer.phone LIKE :keyword OR customer.customerNo LIKE :keyword OR CAST(customer.other_phones AS CHAR) LIKE :keyword
```

#### 4. 客户存在检查 /check-exists（行868）
**修改前**：
```typescript
customerRepository.findOne({ where: { phone: phone as string } })
```
**修改后**：
```typescript
customerRepository.createQueryBuilder('customer')
  .where('customer.phone = :phone OR JSON_CONTAINS(customer.other_phones, JSON_QUOTE(:phone))', { phone })
  .getOne()
```

### 修改文件：`backend/src/routes/finance.ts`（共4处）

财务模块的批量搜索使用 `forEach` 循环构建 `orConditions`，每处都在原有的 `order.customerPhone LIKE :paramKey` 后增加 `EXISTS` 子查询。

**修改前**（每处循环体）：
```javascript
orConditions.push(`(order.orderNumber LIKE :${paramKey} OR order.customerName LIKE :${paramKey} OR order.customerPhone LIKE :${paramKey})`);
```

**修改后**：
```javascript
orConditions.push(`(order.orderNumber LIKE :${paramKey} OR order.customerName LIKE :${paramKey} OR order.customerPhone LIKE :${paramKey} OR EXISTS (SELECT 1 FROM customers c WHERE c.id = order.customer_id AND CAST(c.other_phones AS CHAR) LIKE :${paramKey}))`);
```
