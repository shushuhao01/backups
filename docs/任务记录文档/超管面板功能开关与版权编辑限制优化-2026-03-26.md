## 三、详细修改说明

### 3.1 超管面板功能开关控制

**问题**：Admin后台功能开关缺少对CRM超管面板菜单的控制，开发者无法决定是否向租户/私有客户展示此子菜单。

**修复**：
- `admin/src/views/modules/Config.vue` — 在SaaS和私有的"其他功能"分组各添加"超管面板"开关项
  - 图标：`<Setting />`
  - 描述：超级管理员权限配置面板
  - 绑定：`form.featureFlags.saas.superAdminPanel` / `form.featureFlags.private.superAdminPanel`
  - 默认值：`true`（默认启用）
- `src/components/DynamicMenu.vue` — `menuFlagMap` 添加映射：
  ```js
  'system-super-admin-panel': 'superAdminPanel'
  ```
  当Admin后台关闭该开关后，CRM端对应部署模式的超管面板菜单自动隐藏。

### 3.2 版权信息编辑限制（按部署模式）

**问题**：SaaS租户可以编辑版权信息（不应该），私有部署客户可以编辑版权文字和技术支持（不应该）。

**修复规则**：
| 字段 | SaaS租户 | 私有部署 | 说明 |
|------|---------|---------|------|
| 版权文字(copyrightText) | ❌ 不可编辑 | ❌ 不可编辑 | 由开发者统一配置 |
| ICP备案号(icpNumber) | ❌ 不可编辑 | ✅ 可编辑 | 私有部署需自行填写 |
| 公安备案号(policeNumber) | ❌ 不可编辑 | ✅ 可编辑 | 私有部署需自行填写 |
| 技术支持(techSupport) | ❌ 不可编辑 | ❌ 不可编辑 | 由开发者统一配置 |

**实现**：
- 新增计算属性 `isSaasMode`、`isPrivateMode`（基于 `getDeployMode()`）
- 新增 `copyrightTextDisabled`、`icpDisabled`、`policeDisabled`、`techSupportDisabled` 计算属性
- 保存逻辑(`handleSave`)同步更新，禁用字段不会被发送到后端
- 提示文字根据部署模式动态显示

### 3.3 系统Logo上传注释禁用

**问题**：系统Logo上传功能当前无实际使用场景。

**修复**：
- `src/components/System/BasicSettingsPanel.vue` — Logo上传表单项和JS函数(`beforeLogoUpload`、`handleLogoUpload`)使用HTML/JS注释禁用
- `admin/src/views/modules/Config.vue` — Logo上传表单项和JS函数(`handleLogoChange`)使用HTML/JS注释禁用
- `Plus` 图标导入从 BasicSettingsPanel 中移除（避免ESLint报错）
- CSS样式保留（便于将来恢复）

### 3.4 控制台加密默认启用 + Admin全局开关

**问题**：控制台加密默认关闭，新租户需手动开启；Admin后台缺少全局控制开关。

**修复**：

#### 默认值变更
- `SuperAdminPermissionPanel.vue` — `secureConsoleEnabled` 初始值和重置值从 `false` → `true`
- `backend/src/routes/system.ts` — 安全设置默认值 `secureConsoleEnabled` 从 `false` → `true`

#### Admin全局开关
- `admin Config.vue` — "启用状态"tab添加"安全配置"分隔线和"启用控制台加密"开关
  - 绑定：`form.enableConsoleEncryption`
  - 默认值：`true`
  - 提示：全局启用后所有租户的控制台将被加密，租户可自行关闭

#### 后端返回
- `backend/src/routes/admin/index.ts` — 公开API(`/admin/public/system-config`)响应中新增 `enableConsoleEncryption` 字段（三处：正常、无数据、异常回退）

#### CRM前端联动
- `src/stores/config.ts`:
  - 新增 `adminConsoleEncryption` ref（`boolean | null`）
  - `loadSystemConfigFromAPI()` 中读取Admin返回的 `enableConsoleEncryption`
  - 同步到 `localStorage('crm_admin_console_encryption')` 供secureLogger同步读取
- `src/utils/secureLogger.ts`:
  - `isSecureConsoleEnabled()` 增加Admin全局开关优先级检查
  - 优先级：Admin全局禁用 → 强制关闭（不看租户设置）
  - Admin全局启用或未配置 → 使用租户本地设置
- `SuperAdminPermissionPanel.vue`:
  - 新增 `adminConsoleEncryptionDisabled` 计算属性
  - 当Admin全局禁用时，开关变为disabled状态并显示"管理后台已全局禁用控制台加密"提示

### 3.5 权限配置卡片隐藏

**问题**：超管面板系统配置tab中的"权限配置"卡片（defaultPermissionLevel、enforcePasswordPolicy、requireTwoFactor、sessionTimeout）功能不明确。

**评估结论**：这些配置仅保存在localStorage，部分通过configStore保存，但与角色权限系统没有实际后端联动，属于占位功能。

**修复**：
- 使用 `v-if="false"` 隐藏权限配置卡片（保留代码供将来实现）
- 安全配置卡片从 `span=12` 改为 `span=24`（全宽显示）

### 3.6 多租户数据隔离审查

**审查结论**：✅ 数据隔离安全

| 审查项 | 结果 | 说明 |
|-------|------|------|
| `system_configs` 表 | ✅ 安全 | 在 `TENANT_ENTITIES` 集合中（tenantRepo.ts line 32） |
| `getConfigsByGroup` | ✅ 安全 | 使用 `getTenantRepo(SystemConfig)` |
| `saveConfigsByGroup` | ✅ 安全 | 使用 `getTenantRepo(SystemConfig)` |
| `console-security-config` 路由 | ✅ 安全 | `authenticateToken` 中间件注入租户上下文 |
| 用户登录 | ✅ 安全 | `UserController.login` 显式检查 `tenantId` |
| `customers` 表 | ✅ 安全 | 有 `tenant_id` 列，在 `TENANT_ENTITIES` 中 |
| `users` 表 | ✅ 安全 | 有 `tenant_id` 列，在 `TENANT_ENTITIES` 中 |
| 同手机号跨租户 | ✅ 隔离 | `getTenantRepo` Proxy自动注入 `tenant_id` 条件 |
| 同用户名跨租户 | ✅ 隔离 | 登录查询显式添加 `tenantId` where条件 |

**隔离机制说明**：
- `getTenantRepo()` 使用 Proxy 模式拦截所有查询方法（find、findOne、findBy等），自动注入 `tenant_id` 条件
- `TenantContextManager` 使用 `AsyncLocalStorage` 在请求级别隔离租户上下文
- `authenticateToken` 中间件从JWT中提取 `tenantId` 并设置到 `TenantContextManager`

## 四、注意事项

1. **Logo相关CSS保留**：虽然Logo上传功能已注释，但CSS样式保留，将来恢复时取消注释即可
2. **权限配置卡片保留代码**：使用 `v-if="false"` 而非删除，将来实现后端联动后可恢复
3. **Admin全局开关优先级**：Admin全局禁用 > 租户本地配置，但Admin全局启用时租户可自行关闭
4. **版权编辑限制基于前端环境变量**：通过 `getDeployMode()` 判断（优先级：localStorage > VITE_DEPLOY_MODE > 默认saas），安全控制同时在后端的 `saveConfigsByGroup` 中保障

## 五、测试建议

1. **功能开关测试**：在Admin后台关闭超管面板开关 → CRM端菜单消失
2. **版权编辑测试**：
