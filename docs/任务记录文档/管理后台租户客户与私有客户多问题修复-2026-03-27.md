## 问题清单

### 问题1: 租户客户列表只显示1条数据，总数为0 ✅ 阶段1已修复
- **现象**: 后台租户客户列表只显示第一条租户，统计卡片总数全部为0
- **根因**: `AppDataSource.query()` 直接返回行数组，但代码使用了 `const [list] = ...` 解构
- **修复**: 去掉数组解构
- **影响文件**: `backend/src/routes/admin/tenants.ts`

### 问题2: 租户客户详情页功能模块全灰 ✅ 阶段1已修复
- **现象**: 详情页的功能模块授权卡片中所有模块都显示灰色
- **根因**: features存储文本特性而非模块ID
- **修复**: 模块ID优先存入features，详情页增加模块解析逻辑
- **影响文件**: `backend/src/routes/admin/tenants.ts`

### 问题3: 用户数和存储空间限制控制 ✅ 阶段2已修复
- **现象**: 文件上传未执行租户存储空间限制检查
- **根因**: 
  1. `checkUserLimit` 中间件已正确接入用户创建路由 ✅
  2. `checkStorageLimit` 中间件**从未在任何上传路由中被调用** ❌
- **修复**: 为所有上传路由添加 `checkStorageLimit` 中间件
- **影响文件**: 
  - `backend/src/routes/system.ts` — 5个上传路由
  - `backend/src/routes/codApplication.ts` — 尾款凭证上传
  - `backend/src/routes/mobile.ts` — 通话录音上传
  - `backend/src/routes/profile.ts` — 用户头像上传

### 问题4: 私有客户详情页创建人显示UUID编码 ✅ 阶段1已修复
- **影响文件**: `backend/src/routes/admin/licenses.ts`, `admin/src/views/private-customers/Detail.vue`

### 问题5: 套餐管理只有4个但新建租户下拉有7个 ✅ 阶段1已修复
- **影响文件**: `admin/src/views/tenant-customers/List.vue`

### 问题6: [阶段2新增] 重新生成授权码和续期路由的解构bug ✅
- **现象**: `regenerate-license` 和 `renew` 路由使用 `const [rows] = ...` 解构，导致查不到租户
- **修复**: 去掉数组解构
- **影响文件**: `backend/src/routes/admin/tenants.ts`

### 问题7: [阶段2新增] CRM前端模块菜单不受租户授权模块限制 ✅
- **现象**: SaaS模式下，即使管理后台只给租户分配了部分模块，CRM前端仍显示所有全局启用的模块菜单
- **根因**: 后端 modules/status API 只查全局模块表，未与租户 features 交集；登录不返回租户模块
- **修复**: 后端API交集过滤 + 登录返回tenantModules + 前端双重过滤
- **影响文件**: `backend/src/routes/system.ts`, `backend/src/controllers/UserController.ts`, `src/stores/user.ts`, `src/services/moduleStatusService.ts`, `src/services/authApiService.ts`

### 3.2 边界场景检查 + 新发现Bug修复

#### 🔴 新发现：regenerate-license 路由 newLicenseKey 未声明

**问题**：`tenants.ts` 第305行使用 `newLicenseKey` 但该变量从未声明，调用时必定报 `ReferenceError`

#### 其他边界场景确认

| 场景 | 检查结果 |
|------|---------|
| 租户无features且无套餐 → 模块列表 | ✅ 安全：tenantModules为null时不过滤，使用全局配置 |
| features包含中文文本特性而非模块ID | ✅ 安全：hasModuleIds检查确保只接受有效模块ID |
| localStorage中tenantModules为空或解析失败 | ✅ 安全：有try-catch包裹，失败时不过滤 |
| 非SaaS模式（私有部署）| ✅ 安全：user.tenantId为空时不查询租户模块 |
| renew路由tenant不存在 | ⚠️ 低风险：oldExpireDate为undefined，日志中显示"undefined -> newDate"，不影响功能 |
| suspend/resume/delete路由无存在性检查 | ⚠️ 低风险：SQL操作影响0行但不报错 |
