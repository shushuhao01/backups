# 客户管理模块租户数据隔离修复

**任务日期**：2026-03-23  
**任务类型**：安全修复 — 租户数据隔离  
**影响范围**：客户管理模块（客户列表、新增、编辑、详情、分组、标签、分享、外呼、通话记录）

---

## 一、任务背景

CRM系统采用 SaaS 多租户架构，所有租户共享同一数据库，通过 `tenant_id` 列进行数据隔离。核心隔离工具包括：

- `getTenantRepo(Entity)` — Proxy代理仓储，自动注入 tenant_id
- `tenantSQL(prefix)` — 原始SQL的租户过滤片段
- `getCurrentTenantIdSafe()` — 获取当前请求的租户ID

但在客户管理模块中，多处**原始SQL查询**绕过了隔离机制，存在跨租户数据泄漏风险。

### 4.1 customers.ts — 1处修复

#### 修复 #1：客户跟进记录原始SQL添加 tenant_id 过滤

**位置**：`GET /api/v1/customers/:id/followups`

```typescript
// 修复前：
const followUps = await AppDataSource.query(`
  SELECT ... FROM follow_up_records
  WHERE customer_id = ?
  ORDER BY created_at DESC
`, [customerId]);

// 修复后：
const tFollowUp = tenantSQL('');
const followUps = await AppDataSource.query(`
  SELECT ... FROM follow_up_records
  WHERE customer_id = ?${tFollowUp.sql}
  ORDER BY created_at DESC
`, [customerId, ...tFollowUp.params]);
```

### 4.2 calls.ts — 14处修复

#### 新增导入
```typescript
import { getCurrentTenantIdSafe } from '../utils/tenantHelpers';
```

#### 修复模式说明

**SELECT/UPDATE/DELETE** — 使用 `tenantSQL('')` 追加 `AND tenant_id = ?`：
```typescript
const t = tenantSQL('');
await AppDataSource.query(
  `SELECT * FROM table WHERE id = ?${t.sql}`,
  [id, ...t.params]
);
```

**INSERT** — 使用 `getCurrentTenantIdSafe()` 获取 tenant_id 值写入：
```typescript
const tenantId = getCurrentTenantIdSafe() || null;
await AppDataSource.query(
  `INSERT INTO table (id, ..., tenant_id) VALUES (?, ..., ?)`,
  [id, ..., tenantId]
);
```

## 五、关键安全保障

### 5.1 同手机号不同租户的数据隔离

**场景**：租户A和租户B都有手机号为 `13571764646` 的客户。

**隔离机制**：
1. **客户列表**：`getTenantRepo(Customer).createQueryBuilder` 自动添加 `WHERE tenant_id = ?`，各租户只能看到自己的客户
2. **新增客户查重**：`customerRepository.findOne({ where: { phone } })` 通过 getTenantRepo 代理，自动限制在当前租户内查重。即租户A有该手机号，租户B仍可新建同手机号客户
3. **客户详情/编辑**：通过 `getTenantRepo.findOne({ where: { id } })` 查询，自动添加 tenant_id 条件
4. **客户分享**：`/shareable-users` 使用 `getTenantRepo(User)` 只返回同租户的用户
5. **通话/跟进记录**：本次修复后，原始SQL查询也已添加 tenant_id 过滤

### 5.2 数据隔离层级

```
请求层  →  JWT Token (含 tenantId)
    ↓
中间件  →  tenantAuth 提取 tenantId → AsyncLocalStorage
    ↓
路由层  →  getTenantRepo / tenantSQL 自动注入 tenant_id
    ↓
数据库  →  WHERE tenant_id = ? 确保隔离
```
