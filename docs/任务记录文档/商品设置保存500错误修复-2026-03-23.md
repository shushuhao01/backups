# 商品设置保存500错误修复报告

**任务日期**：2026-03-23  
**任务类型**：Bug修复  
**影响范围**：系统管理 > 系统设置 > 商品设置（优惠折扣配置等）  
**严重程度**：高（所有系统设置保存功能受影响）

---

## 一、问题描述

### 1.1 错误现象
在CRM系统「系统管理 > 系统设置 > 商品设置」标签页中，保存优惠折扣设置时报500错误：

```
[API] 响应错误:
Object { message: "Request failed with status code 500", name: "AxiosError", code: "ERR_BAD_RESPONSE" }
[商品设置] API调用失败，已保存到本地: Request failed with status code 500
```

### 1.2 影响范围
此问题影响所有使用 `saveConfigsByGroup` 函数的系统设置保存操作，包括：
- 商品设置（product_settings）
- 安全设置（security_settings）
- 通话设置（call_settings）
- 存储设置（storage_settings）
- 订单设置（order_settings）
- 备份设置（backup_settings）
- 通知设置（notification_settings）
- 等所有基于 `system_configs` 表的配置

---

## 二、根因分析

### 2.1 核心原因：唯一索引不兼容多租户

`system_configs` 表有一个唯一索引 `idx_config_key_group`，约束在 `(configKey, configGroup)` 上：

```sql
UNIQUE KEY idx_config_key_group (configKey, configGroup)
```

**但在 SaaS 多租户模式下**，不同租户应该可以有各自的 `(configKey, configGroup)` 配置。

实际情况：
- 数据库中旧租户 `198eeeb5-...` 已有 `product_settings` 组的配置记录
- 当前租户 `9a454086-...` 通过 `getTenantRepo` 查询时按 `tenant_id` 过滤，查不到记录
- 代码逻辑判断"不存在则创建"，尝试 INSERT 新记录
- **唯一索引 `(configKey, configGroup)` 拒绝了插入**，因为旧租户已有同 key+group 的数据
- TypeORM 抛出 SQL 异常 → 500 Internal Server Error

### 2.2 辅助问题：JSON 类型配置值序列化错误

`saveConfigsByGroup` 函数对所有类型的配置值统一使用 `String()` 转换：

```typescript
config.configValue = String(settings[item.key]);
```

对于 `json` 类型的配置（如 `priceModificationRoles: ["admin", "manager"]`），`String(["admin", "manager"])` 结果为 `"admin,manager"`，而非正确的 `'["admin","manager"]'`。导致读取时 `JSON.parse("admin,manager")` 失败。

---

## 三、修复方案

### 3.1 修复数据库唯一索引

将单一唯一索引改为多租户兼容的联合唯一索引：

```sql
-- 删除旧的不兼容唯一索引
ALTER TABLE system_configs DROP INDEX idx_config_key_group;

-- 创建非唯一的普通索引（保持查询性能）
ALTER TABLE system_configs ADD INDEX idx_config_key_group (configKey, configGroup);

-- 创建新的多租户兼容唯一索引
ALTER TABLE system_configs ADD UNIQUE INDEX uk_tenant_config (tenant_id, configKey, configGroup);
```

### 3.2 修复 JSON 值序列化

**文件**：`backend/src/routes/system.ts` — `saveConfigsByGroup` 函数

```diff
+ // 对 json 类型使用 JSON.stringify，其他类型使用 String()
+ const serializedValue = item.type === 'json' && typeof settings[item.key] === 'object'
+   ? JSON.stringify(settings[item.key])
+   : String(settings[item.key]);
+
  if (config) {
-   config.configValue = String(settings[item.key]);
+   config.configValue = serializedValue;
  } else {
    config = configRepository.create({
-     configValue: String(settings[item.key]),
+     configValue: serializedValue,
    });
  }
```
