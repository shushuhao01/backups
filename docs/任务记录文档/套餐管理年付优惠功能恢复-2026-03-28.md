## 任务背景

在之前的开发中，已经开发过年付优惠配置功能（年付折扣、赠送月数、固定年价等），数据库字段和后端实体都已存在，但管理后台UI、后端API的读写操作、以及官网前端的动态展示被遗漏或回退，导致该功能不可用。

## ⚠️ 关键根因发现

**表名不一致问题：**
- TypeORM Entity `Package.ts` 原映射表名为 `packages`
- 所有手写SQL路由（admin/packages.ts、public/packages.ts等）使用的表名为 `tenant_packages`  
- 迁移脚本 `add-yearly-discount-config.sql` 原操作的是 `packages` 表（错误表名）
- 实际数据库中套餐数据存储在 `tenant_packages` 表中

**首次修复导致的问题：**
- 将公开API的SELECT从 `SELECT *` 改为指定字段列表（包含yearly字段）
- 虽然 `tenant_packages` 实际已有yearly字段，但SQL改动后如果字段名拼写有任何差异就会导致500报错
- 官网定价页因此显示为空白（API返回失败）

## 年付优惠功能说明

### 三种优惠方式

| 方式 | 字段 | 说明 | 示例 |
|------|------|------|------|
| **赠送月数** | `yearly_bonus_months` | 年付12个月+赠送N个月 | 送2个月 → 付10个月价格 |
| **折扣优惠** | `yearly_discount_rate` | 年付总价打折（百分比） | 16.67% → 约8.3折 |
| **固定年价** | `yearly_price` | 直接设定年付价格 | 年付价格=990元 |

### 计算逻辑优先级

1. 优先使用 `yearly_price`（固定年价）
2. 其次使用 `yearly_bonus_months`（赠送月数）：年付总价 = 月价 × (12 - 赠送月数)
3. 再次使用 `yearly_discount_rate`（折扣率）：年付总价 = 月价 × 12 × (1 - 折扣率/100)
4. 兜底：默认8折（月价 × 0.8 × 12）

### 数据库字段

```sql
yearly_discount_rate DECIMAL(5,2) DEFAULT 0.00  -- 年付折扣率（0-100）
yearly_bonus_months INT DEFAULT 0                -- 年付赠送月数
yearly_price DECIMAL(10,2) DEFAULT NULL          -- 年付固定价格
```

## 影响评估

- **向后兼容**：所有计算函数的第二个参数为可选，不传则使用原有逻辑
- **兜底机制**：API未加载时使用硬编码兜底价格
- **自动迁移**：admin路由首次加载时自动检测并添加yearly字段到tenant_packages（如果不存在）
- **安全SELECT**：公开API使用SELECT *，不会因字段不存在而报错
- **不影响现有套餐**：只有配置了年付优惠的套餐才会显示优惠信息
- **不影响支付流程**：价格计算只在展示层，实际支付金额由后端计算
- **不影响其他模块**：仅修改套餐管理相关文件
