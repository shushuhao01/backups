## 二、发现的问题清单

### 问题1：Profile路由simpleAuth缺少租户上下文设置（严重 - 租户隔离漏洞）
- **位置**: `backend/src/routes/profile.ts` → `simpleAuth` 中间件
- **问题**: `simpleAuth` 只做JWT验证并设置 `req.user`，但**不设置 TenantContext**
- **影响**: `getTenantRepo(User)` 无法获取租户ID，SaaS模式下所有profile操作（获取信息、更新信息、修改密码）都没有租户隔离，可能跨租户访问/修改数据
- **对比**: `authenticateToken` 中间件（`middleware/auth.ts:72-76`）正确设置了 `TenantContextManager.setContext()`
- **严重度**: 🔴 严重

### 问题2：User Store的User接口缺少关键字段
- **位置**: `src/stores/user.ts` → `User` 接口
- **问题**: `PersonalSettingsModal.vue` 的fallback逻辑（行454-460）尝试访问 `user.username`、`user.phone`、`user.realName`、`user.roleName`，但 `User` 接口中没有这些字段定义
- **影响**: TypeScript类型检查不完整，IDE无法正确提示
- **严重度**: 🟡 中等

### 问题3：loginWithApi未保存username和phone到currentUser
- **位置**: `src/stores/user.ts` → `loginWithApi` 方法（行568-588）
- **问题**: 设置 `currentUser.value` 时未包含 `username`、`phone`、`realName` 字段
- **影响**: 当 `profileApiService.getProfile()` 失败时，PersonalSettingsModal的fallback逻辑从 `userStore.user` 获取数据会缺少用户名、手机号等关键字段
- **严重度**: 🟡 中等

### 问题4：passwordService TypeScript类型错误
- **位置**: `src/services/passwordService.ts:181`
- **问题**: `apiError` 类型为 `unknown`，但直接访问 `apiError.response?.data?.message` 和 `apiError.message`
- **影响**: TypeScript编译警告，运行时可能抛出类型错误
- **严重度**: 🟢 低

### 问题5：Profile密码修改路由缺少新旧密码相同检查
- **位置**: `backend/src/routes/profile.ts` → `PUT /password`
- **问题**: 未检查新密码是否与当前密码相同（`auth.ts`的handler有此检查）
- **影响**: 用户可以将密码"修改"为原来的密码，逻辑不一致
- **严重度**: 🟢 低

## 三、修复方案与执行

### 修复1：Profile路由simpleAuth添加租户上下文 ✅
**文件**: `backend/src/routes/profile.ts`

**修改内容**:
1. 新增导入 `TenantContextManager`
2. 在 `simpleAuth` 中间件中，JWT验证成功后从payload提取tenantId，设置到TenantContext

```typescript
// 新增导入
import { TenantContextManager } from '../utils/tenantContext';

// simpleAuth中新增（payload验证成功后）
if (payload.tenantId) {
  req.tenantId = payload.tenantId;
  TenantContextManager.setContext({ tenantId: payload.tenantId, userId: payload.userId });
}
```

**效果**: 所有Profile路由（GET/PUT /profile, PUT /profile/password, POST /profile/avatar）的数据库操作都会经过租户隔离过滤

### 修复2：User接口添加缺失字段 ✅
**文件**: `src/stores/user.ts`

**新增字段**:
```typescript
username?: string    // 用户名（登录名）
phone?: string       // 手机号
realName?: string    // 真实姓名
roleName?: string    // 角色显示名称
position?: string    // 职位
```

### 修复3：loginWithApi保存完整用户信息 ✅
**文件**: `src/stores/user.ts`

**修改内容**: 在 `currentUser.value` 赋值时新增三个字段：
```typescript
username: userData.username,
phone: userData.phone || '',
realName: userData.realName || userData.name,
```

### 修复4：passwordService类型安全修复 ✅
**文件**: `src/services/passwordService.ts`

**修改内容**: 将 `apiError` 安全转换为 `any` 后再访问属性：
```typescript
const err = apiError as any
const errorMessage = err?.response?.data?.message || err?.message || '密码修改失败'
```

### 修复5：Profile密码修改添加新旧密码相同检查 ✅
**文件**: `backend/src/routes/profile.ts`

**新增检查**:
```typescript
const isSamePassword = await bcrypt.compare(newPassword, user.password);
if (isSamePassword) {
  return res.status(400).json({
    success: false,
    message: '新密码不能与当前密码相同',
    code: 'SAME_PASSWORD'
  });
}
```

## 五、密码修改完整流程验证

### 正常流程
1. 用户点击右上角头像 → 选择"个人设置"
2. 弹出 `PersonalSettingsModal` → 点击"密码修改"标签
3. 填写当前密码、新密码、确认新密码
4. 前端验证（密码强度、确认一致性、新旧不同）
5. 调用 `passwordService.changePassword()` → `apiService.put('/profile/password', data)`
6. 后端 `PUT /api/v1/profile/password` 接收请求
7. `simpleAuth` 验证JWT并设置租户上下文 ✅（修复后）
8. `getUserRepository()` 返回带租户隔离的仓储
9. 根据JWT中的userId（+tenantId）查找用户
10. `bcrypt.compare()` 验证当前密码
11. 检查新密码不等于当前密码 ✅（修复后）
12. `bcrypt.hash()` 加密新密码
13. 保存到数据库
14. 返回成功响应

### 强制修改密码流程
1. 用户登录后，系统检测到 `forcePasswordChange = true`
2. 弹出 `PasswordChangeModal`（不可关闭）
3. 同样的密码修改流程

## 六、租户数据隔离验证

### 修复前的问题
- Profile路由的 `simpleAuth` 不设置 `TenantContext`
- `getTenantRepo(User)` 返回的Proxy无法获取tenantId
- SaaS模式下，用户A（租户1）的profile操作不会被限制在租户1的数据范围内

### 修复后的保障
- `simpleAuth` 从JWT payload提取 `tenantId` 并设置到 `TenantContextManager`
- `getTenantRepo(User)` 通过 `getCurrentTenantId()` 获取到正确的tenantId
- 所有 `findOne`、`save` 操作自动注入 `tenant_id` 过滤条件
- 确保用户只能查看和修改自己租户下的数据

## 七、注意事项

1. **密码加密**: 后端使用 `bcrypt` 加密，salt rounds = 12
2. **两个密码修改端点**: `/api/v1/profile/password`（profile路由）和 `/api/v1/auth/password`（auth路由），前端主要使用profile路由
3. **User实体toJSON**: `User.toJSON()` 方法会自动过滤password字段，防止密码泄露到前端
4. **前端密码策略**: 由 `passwordService.getPasswordPolicy()` 从系统配置动态获取（长度、复杂度等）
