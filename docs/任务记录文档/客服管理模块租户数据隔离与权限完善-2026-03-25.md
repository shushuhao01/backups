## 问题1：租户数据隔离（严重）

### 🔴 根本原因
**`customerServicePermissions.ts` 路由文件完全没有使用任何认证中间件！**
- 没有 `authenticateToken` → JWT 未解析 → `req.tenantId` 为空
- `TenantContextManager.setContext()` 从未被调用
- `getTenantRepo()` 获取不到 tenantId → 返回原始仓储 → **查出所有租户数据**

### 修复方案
```typescript
// 添加到路由文件顶部
import { authenticateToken, requireManagerOrAdmin } from '../middleware/auth';

router.use(authenticateToken);       // 解析JWT → 设置 req.tenantId → 调用 TenantContextManager.setContext()
router.use(requireManagerOrAdmin);   // 仅管理员可操作
```

### 其他修复

#### 1. 实体唯一约束修正
- **问题**：`user_id` 列设置了全局 `unique: true`，多租户环境下冲突
- **修复**：去掉全局唯一，改为 `(tenant_id, user_id)` 复合唯一
- **schema.sql**：同步更新建表语句，添加 `tenant_id` 列

#### 2. 可选成员接口加固
- 排除 `super_admin` 和 `admin` 角色（管理员不应被添加为客服）
- 通过 `getTenantRepo(User)` 自动过滤本租户用户

#### 3. 自动角色切换
- 创建客服：自动将用户角色更改为 `customer_service`（排除超管和管理员）
- 删除客服：自动将角色从 `customer_service` 恢复为 `user`
- 批量配置：创建分支也添加自动角色更改

## 生产环境 SQL（兼容 phpMyAdmin）

如果生产数据库中该表已存在，执行以下迁移SQL：

```sql
-- 客服权限表：添加 tenant_id 列（如果不存在）
ALTER TABLE `customer_service_permissions`
  ADD COLUMN `tenant_id` VARCHAR(36) NULL COMMENT '租户ID' AFTER `id`;

-- 删除 user_id 的全局唯一索引
ALTER TABLE `customer_service_permissions`
  DROP INDEX `idx_user_id`;

-- 添加 tenant_id 索引
ALTER TABLE `customer_service_permissions`
  ADD INDEX `idx_tenant_id` (`tenant_id`);

-- 添加 (tenant_id, user_id) 复合唯一索引
ALTER TABLE `customer_service_permissions`
  ADD UNIQUE INDEX `idx_tenant_user` (`tenant_id`, `user_id`);
```

如果生产数据库中该表不存在，执行完整建表SQL：

```sql
CREATE TABLE IF NOT EXISTS `customer_service_permissions` (
  `id` VARCHAR(50) PRIMARY KEY COMMENT '记录ID',
  `tenant_id` VARCHAR(36) NULL COMMENT '租户ID',
  `user_id` VARCHAR(50) NOT NULL COMMENT '用户ID',
  `customer_service_type` VARCHAR(30) NOT NULL DEFAULT 'general' COMMENT '客服类型: after_sales售后, audit审核, logistics物流, product商品, finance财务, general通用',
  `data_scope` VARCHAR(20) NOT NULL DEFAULT 'self' COMMENT '数据范围: all全部, department部门, self个人, custom自定义',
  `department_ids` JSON NULL COMMENT '可访问的部门ID列表',
  `custom_permissions` JSON NULL COMMENT '自定义权限列表',
  `permission_template` VARCHAR(50) NULL COMMENT '使用的权限模板ID',
  `status` ENUM('active', 'inactive') DEFAULT 'active' COMMENT '状态',
  `remark` TEXT NULL COMMENT '备注',
  `created_by` VARCHAR(50) NULL COMMENT '创建人ID',
  `created_by_name` VARCHAR(50) NULL COMMENT '创建人姓名',
  `updated_by` VARCHAR(50) NULL COMMENT '更新人ID',
  `updated_by_name` VARCHAR(50) NULL COMMENT '更新人姓名',
  `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  UNIQUE INDEX `idx_tenant_user` (`tenant_id`, `user_id`),
  INDEX `idx_tenant_id` (`tenant_id`),
  INDEX `idx_service_type` (`customer_service_type`),
  INDEX `idx_data_scope` (`data_scope`),
  INDEX `idx_status` (`status`),
  INDEX `idx_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='客服权限配置表';
```
