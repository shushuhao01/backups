## 一、任务背景

CRM系统采用 SaaS 多租户架构，所有租户共享同一数据库，通过 `tenant_id` 列进行数据隔离。核心隔离工具包括：

- `getTenantRepo(Entity)` — Proxy代理仓储，自动注入 tenant_id
- `tenantRawSQL(prefix)` — 原始SQL的租户过滤片段
- `getCurrentTenantIdSafe()` — 获取当前请求的租户ID

系统管理模块中发现多处租户数据隔离问题，需要分阶段修复。

## 三、阶段1 — 部门管理租户隔离修复

### 3.1 严重问题修复

#### 问题1：getDepartmentMembers 的 orWhere 绕过租户隔离 ⚠️ 严重

**文件**：`backend/src/controllers/DepartmentController.ts`  
**位置**：`getDepartmentMembers` 方法（约第569行）  
**问题描述**：

原代码使用 `.where().orWhere()` 查询部门成员：

```typescript
// 修复前（存在泄漏）
const users = await this.userRepository
  .createQueryBuilder('user')
  .where('user.departmentId = :id', { id })
  .orWhere('user.departmentName = :name', { name: departmentName })
  .getMany();
```

`getTenantRepo` Proxy 拦截 `createQueryBuilder` 时通过 `andWhere` 注入 `tenant_id` 条件，但 `.where()` 被拦截后会重新追加 `tenant_id`。然而 `.orWhere()` **不被拦截**，导致最终 SQL 为：

```sql
WHERE (user.tenant_id = ? AND user.departmentId = ?) OR user.departmentName = ?
```

由于 SQL 运算符优先级 (`AND` > `OR`)，`departmentName` 条件**完全绕过了租户过滤**，可能泄漏其他租户的用户数据。

**修复方案**：将 OR 条件包裹在括号内，作为 `.where()` 的参数：

```typescript
// 修复后（租户过滤始终生效）
const users = await this.userRepository
  .createQueryBuilder('user')
  .where('(user.departmentId = :id OR user.departmentName = :name)', { id, name: departmentName })
  .getMany();
```

修复后 SQL：`WHERE (user.departmentId = ? OR user.departmentName = ?) AND user.tenant_id = ?`
