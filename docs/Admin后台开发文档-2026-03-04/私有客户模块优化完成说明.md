## 核心特性

### 1. 客户与授权分离
- 客户是主体，授权是属性
- 一个客户可以有多个授权（历史授权、续期授权）
- 清晰的数据模型

### 2. 完整的授权历史
- 记录客户的所有授权
- 包括创建时间、到期时间、状态等
- 支持查看授权详情

### 3. 详细的操作日志
- 记录所有授权相关操作
- 包括创建、激活、续期、吊销等
- 便于审计和追溯

### 4. 灵活的搜索筛选
- 支持关键词搜索
- 支持行业筛选
- 支持状态筛选
- 支持授权状态筛选

### 5. 数据保护机制
- 有有效授权的客户无法删除
- 防止误删除重要数据
