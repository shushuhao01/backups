### 步骤6: 模拟支付成功（手动测试）

由于开发环境无法真实支付，需要手动更新数据库来模拟支付成功：

#### 6.1 查询订单号

在支付二维码页面记录订单号，例如：`PAY1772697489638D0Z9ZL`

#### 6.2 手动更新订单状态

```bash
cd backend
node -e "const mysql = require('mysql2/promise'); require('dotenv').config({ path: '.env.local' }); (async () => { const conn = await mysql.createConnection({ host: process.env.DB_HOST, port: process.env.DB_PORT, user: process.env.DB_USERNAME, password: process.env.DB_PASSWORD, database: process.env.DB_DATABASE }); await conn.query('UPDATE payment_orders SET status = ?, paid_at = NOW() WHERE order_no = ?', ['paid', 'PAY1772697489638D0Z9ZL']); await conn.query('UPDATE tenants SET status = ?, license_status = ?, activated_at = NOW(), expire_date = DATE_ADD(CURDATE(), INTERVAL 30 DAY) WHERE id = (SELECT tenant_id FROM payment_orders WHERE order_no = ?)', ['active', 'active', 'PAY1772697489638D0Z9ZL']); console.log('✓ 支付状态已更新'); await conn.end(); })()"
```

**注意**: 将命令中的订单号替换为实际订单号

#### 6.3 等待自动跳转

**预期效果**:
- ✅ 3秒内自动检测到支付成功
- ✅ 自动跳转到支付成功页面

## 🐛 常见问题

### 1. 页面无法访问

**问题**: 访问 http://localhost:8081/ 显示无法连接

**解决**:
```bash
# 检查官网前端服务是否运行
# 重启官网服务
cd website
npm run dev
```

### 2. 套餐列表为空

**问题**: 注册页面不显示套餐

**解决**:
```bash
# 检查后端API是否运行
curl http://localhost:3000/api/v1/public/packages

# 检查数据库packages表是否有数据
```

### 3. 验证码发送失败

**问题**: 点击"获取验证码"无响应

**解决**:
- 开发环境验证码固定为 `123456`
- 检查后端API `/api/v1/public/register/send-code` 是否正常

### 4. 注册提交失败

**问题**: 点击"下一步"报错

**解决**:
- 检查所有必填项是否填写
- 验证码是否正确（开发环境：123456）
- 查看浏览器控制台错误信息
- 查看后端日志

### 5. 支付二维码不显示

**问题**: 创建支付订单后无二维码

**解决**:
- 开发环境显示"开发模式"提示是正常的
- 检查支付配置是否启用
- 查看后端日志
