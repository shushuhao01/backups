# 私有客户模块 - 问题修复说明

## 问题描述

在访问私有客户列表页面时，出现以下错误：

1. **前端错误**: `No metadata for "PrivateCustomer" was found`
2. **后端错误**: `Request failed with status code 500`
3. **数据库错误**: `Table 'crm_local.private_customers' doesn't exist`

## 问题原因

1. `PrivateCustomer`实体类没有在TypeORM中注册
2. 数据库表`private_customers`没有创建
3. `licenses`表缺少必要的字段（`customer_type`, `private_customer_id`, `tenant_id`）

## 修复步骤

### 1. 注册PrivateCustomer实体类 ✅

**文件**: `backend/src/config/database.ts`

**修改内容**:
```typescript
// 添加导入
import { PrivateCustomer } from '../entities/PrivateCustomer';

// 在entities数组中添加
const entities = [
  // ... 其他实体
  Tenant,
  TenantSettings,
  TenantLog,
  PrivateCustomer  // 新增
];
```

### 2. 执行数据库迁移 ✅

**迁移脚本**: `backend/database-migrations/create-private-customers-table.sql`

**执行命令**:
```bash
node backend/execute-private-customers-migration.js
```

**创建的表和字段**:
- 创建`private_customers`表
- 在`licenses`表中添加`customer_type`字段
- 在`licenses`表中添加`private_customer_id`字段
- 在`licenses`表中添加`tenant_id`字段
- 添加相关索引

### 3. 重启服务

修复完成后，需要重启后端服务让修改生效。

## 注意事项

1. **数据库迁移只需执行一次**，如果已经执行过，不需要重复执行
2. **后端服务必须重启**才能加载新注册的实体类
3. **前端服务不需要重启**，热更新会自动生效
4. 如果遇到端口占用，请先关闭已有的服务进程

## 常见问题

### Q1: 后端启动失败，提示数据库连接错误
**A**: 检查`.env.local`文件中的数据库配置是否正确

### Q2: 前端访问API返回404
**A**: 确认后端服务已启动，并且Admin后台功能已启用（`ENABLE_ADMIN=true`）

### Q3: 创建客户时提示"No metadata for PrivateCustomer"
**A**: 确认已重启后端服务，让实体类注册生效

### Q4: 数据库迁移执行失败
**A**: 检查数据库连接配置，确认有足够的权限执行DDL语句
