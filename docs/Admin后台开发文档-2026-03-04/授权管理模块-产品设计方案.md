# 授权管理模块 - 产品设计方案

## 设计原则

基于SaaS平台和私有部署的双重需求，采用**混合方案**：
- 独立的授权管理模块作为主要管理入口
- 在租户/客户详情中嵌入授权信息，方便快速查看和操作

## 数据库设计优化

### licenses表新增字段

```sql
-- 客户类型和关联
ALTER TABLE licenses ADD COLUMN customer_type VARCHAR(20) DEFAULT 'private' 
  COMMENT '客户类型: saas-SaaS租户, private-私有客户';
ALTER TABLE licenses ADD COLUMN tenant_id VARCHAR(36) 
  COMMENT '关联的租户ID（如果是SaaS租户）';
ALTER TABLE licenses ADD COLUMN private_customer_id VARCHAR(36) 
  COMMENT '关联的私有客户ID（如果是私有客户）';

-- 添加索引
ALTER TABLE licenses ADD INDEX idx_customer_type (customer_type);
ALTER TABLE licenses ADD INDEX idx_tenant_id (tenant_id);
ALTER TABLE licenses ADD INDEX idx_private_customer_id (private_customer_id);
```

### 创建私有客户表（如果还没有）

```sql
CREATE TABLE IF NOT EXISTS private_customers (
  id VARCHAR(36) PRIMARY KEY,
  customer_name VARCHAR(200) NOT NULL COMMENT '客户名称',
  contact_person VARCHAR(100) COMMENT '联系人',
  contact_phone VARCHAR(50) COMMENT '联系电话',
  contact_email VARCHAR(100) COMMENT '联系邮箱',
  company_address VARCHAR(500) COMMENT '公司地址',
  industry VARCHAR(100) COMMENT '所属行业',
  company_size VARCHAR(50) COMMENT '公司规模',
  deployment_type VARCHAR(50) DEFAULT 'on-premise' COMMENT '部署类型: on-premise-本地部署, cloud-云部署',
  status VARCHAR(20) DEFAULT 'active' COMMENT '状态: active-正常, inactive-停用',
  notes TEXT COMMENT '备注',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_customer_name (customer_name),
  INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='私有部署客户表';
```

## API设计优化

### 新增/修改的API

1. **获取租户的授权列表**
   ```
   GET /api/v1/admin/tenants/:tenantId/licenses
   ```

2. **获取私有客户的授权列表**
   ```
   GET /api/v1/admin/private-customers/:customerId/licenses
   ```

3. **为租户生成授权**
   ```
   POST /api/v1/admin/tenants/:tenantId/licenses
   ```

4. **为私有客户生成授权**
   ```
   POST /api/v1/admin/private-customers/:customerId/licenses
   ```

5. **授权列表增加客户类型筛选**
   ```
   GET /api/v1/admin/licenses?customerType=saas|private
   ```

## 用户操作流程

### 场景1: 为新租户生成授权

**流程A（从授权管理入口）**:
1. 进入"授权管理" → "生成授权"
2. 选择客户类型：SaaS租户
3. 从下拉列表选择租户
4. 配置授权参数
5. 生成授权

**流程B（从租户管理入口）**:
1. 进入"租户管理" → 选择租户 → "租户详情"
2. 在授权信息卡片中点击"生成新授权"
3. 配置授权参数
4. 生成授权

### 场景2: 查看租户的所有授权

**流程A（从授权管理入口）**:
1. 进入"授权管理" → "授权列表"
2. 在搜索框输入租户名称
3. 或使用筛选条件：客户类型=SaaS租户

**流程B（从租户管理入口）**:
1. 进入"租户管理" → 选择租户 → "租户详情"
2. 查看授权信息卡片（显示最近5条）
3. 点击"管理授权"跳转到授权列表（自动筛选该租户）

### 场景3: 续期授权

**流程A（从授权管理入口）**:
1. 进入"授权管理" → "授权列表"
2. 找到要续期的授权
3. 点击"续期"按钮

**流程B（从租户管理入口）**:
1. 进入"租户管理" → 选择租户 → "租户详情"
2. 在授权信息卡片中点击"续期授权"

## 总结

**推荐方案**: 混合方案

**核心思路**:
- 授权管理作为独立模块，提供完整的管理功能
- 在租户/客户详情中嵌入授权信息，提供快速查看和操作
- 两个入口互相关联，灵活切换

**优势**:
- ✅ 既能统一管理，又能快速操作
- ✅ 信息集中展示，减少页面跳转
- ✅ 适应不同的使用场景
- ✅ 扩展性好，易于维护
