## 🔧 关于SDK集成

### 什么是SDK集成？

SDK（Software Development Kit）是软件开发工具包，用于调用第三方服务的API。

### 微信支付SDK

**作用**:
- 调用微信支付API创建订单
- 生成支付二维码
- 验证微信服务器的回调签名
- 解密回调数据

**实现方式**:
```typescript
// 1. 创建支付
const result = await wechatPayService.createNativePay({
  orderNo: 'PAY123456',
  amount: 99.00,
  description: '基础版套餐'
})
// 返回: { qrCode: 'weixin://...', payUrl: '...' }

// 2. 处理回调
const result = await wechatPayService.handleCallback(callbackData)
// 自动验证签名、解密数据、更新订单、激活租户
```

**为什么需要SDK？**
- 微信支付API需要复杂的签名算法
- 回调数据需要解密
- 需要验证回调的真实性
- 需要处理各种错误情况

### 支付宝SDK

**作用**:
- 调用支付宝API创建订单
- 生成支付二维码
- 验证支付宝服务器的回调签名

**实现方式**:
```typescript
// 1. 创建支付
const result = await alipayService.createQRPay({
  orderNo: 'PAY123456',
  amount: 99.00,
  subject: '基础版套餐'
})
// 返回: { qrCode: 'https://qr.alipay.com/...', payUrl: '...' }

// 2. 处理回调
const result = await alipayService.handleCallback(params)
// 自动验证签名、更新订单、激活租户
```

**为什么需要SDK？**
- 支付宝API需要RSA2签名
- 需要验证回调签名
- 需要处理各种错误情况

### 对公转账为什么不需要SDK？

**原因**:
1. 不需要调用第三方API（只是返回银行账户信息）
2. 不需要生成二维码
3. 不需要处理自动回调（人工审核）
4. 不需要复杂的签名算法

**实现方式**:
```typescript
// 只需要返回银行账户信息
return {
  bankInfo: {
    accountName: '公司名称',
    bankName: '中国工商银行',
    accountNumber: '6222 **** **** 1234',
    remarkTemplate: '订单号：PAY123456',
    // ... 其他信息
  }
}
```

## 📊 完整支付流程对比

### 微信/支付宝流程

```
创建订单 → 生成二维码 → 用户扫码 → 支付成功
    ↓
微信/支付宝服务器回调 → 验证签名 → 更新订单
    ↓
激活租户 → 生成授权码 → 完成
```

**特点**: 全自动，无需人工干预

### 对公转账流程

```
创建订单 → 返回银行账户 → 用户转账 → 上传凭证
    ↓
管理员审核 → 审核通过 → 更新订单
    ↓
激活租户 → 生成授权码 → 完成
```

**特点**: 需要人工审核，适合大额交易
