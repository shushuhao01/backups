# 私有客户模块优化方案

## 对比分析

### 租户客户 vs 私有客户

| 特性 | 租户客户（SaaS） | 私有客户（私有部署） |
|------|-----------------|---------------------|
| 部署方式 | 云端多租户共享 | 客户服务器独立部署 |
| 授权方式 | 授权码登录系统 | 授权码激活系统 |
| 资源管理 | 用户数、存储空间实时监控 | 无需监控（客户自己管理） |
| 套餐概念 | 有套餐（基础版、专业版、企业版） | 无套餐（按功能模块授权） |
| 数据管理 | 可导出/导入数据 | 不需要（数据在客户服务器） |
| 用户列表 | 可查看租户下所有用户 | 不需要（客户自己管理） |
| 操作日志 | 需要详细日志 | 需要授权操作日志 |

## 私有客户列表页优化

### 当前问题
- 列表显示的是授权信息，而不是客户信息
- 缺少客户的基本信息展示
- 一个客户可能有多个授权（历史授权、续期授权）

### 优化方案

#### 列表字段设计
```
客户名称 | 联系人 | 联系电话 | 行业 | 当前授权状态 | 授权类型 | 到期时间 | 创建时间 | 操作
```

#### 详细字段说明

1. **客户名称** - 公司名称
2. **联系人** - 主要联系人姓名
3. **联系电话** - 联系电话
4. **行业** - 所属行业（可选）
5. **当前授权状态** - 显示当前有效授权的状态
   - 🟢 有效（绿色）
   - 🟡 即将过期（黄色，30天内）
   - 🔴 已过期（红色）
   - ⚪ 未激活（灰色）
   - 🚫 已吊销（红色）
6. **授权类型** - 试用版/年度版/永久版
7. **到期时间** - 当前授权的到期时间
8. **创建时间** - 客户创建时间

#### 操作按钮
- **详情** - 查看客户详情
- **编辑** - 编辑客户信息
- **更多** ▼
  - 生成新授权
  - 续期授权
  - 查看授权历史
  - 删除客户

### 详细设计

#### 1. 基本信息卡片
```typescript
{
  customerName: string      // 客户名称
  contactPerson: string     // 联系人
  contactPhone: string      // 联系电话
  contactEmail: string      // 联系邮箱
  companyAddress: string    // 公司地址
  industry: string          // 所属行业
  companySize: string       // 公司规模（10人以下、10-50人、50-200人、200人以上）
  deploymentType: string    // 部署类型（本地部署、云部署）
  status: string            // 客户状态（active-正常、inactive-停用）
  notes: string             // 备注
  createdAt: string         // 创建时间
  updatedAt: string         // 更新时间
}
```

#### 2. 当前授权信息卡片
显示客户当前有效的授权信息（最新的、未过期的授权）

```typescript
{
  licenseKey: string        // 授权码
  licenseType: string       // 授权类型（trial、annual、perpetual）
  status: string            // 状态（pending、active、expired、revoked）
  maxUsers: number          // 最大用户数
  maxStorageGb: number      // 最大存储空间（GB）
  features: string[]        // 功能模块列表
  expiresAt: string         // 到期时间
  activatedAt: string       // 激活时间
  machineId: string         // 激活机器码
}
```

**操作按钮**:
- **续期** - 延长授权到期时间
- **吊销** - 吊销当前授权（仅激活状态可用）
- **重新激活** - 重新激活已吊销的授权

#### 3. 授权历史记录
显示该客户的所有授权记录（包括历史授权、续期授权）

**表格字段**:
- 授权码（部分隐藏）
- 授权类型（试用/年度/永久）
- 状态（待激活/有效/已过期/已吊销）
- 创建时间
- 到期时间
- 激活时间
- 操作（查看详情、续期、吊销）

#### 4. 授权操作日志
记录所有授权相关的操作

**日志类型**:
- `create` - 创建授权
- `activate` - 激活授权
- `renew` - 续期授权
- `revoke` - 吊销授权
- `reactivate` - 重新激活
- `update` - 更新授权配置

**表格字段**:
- 时间
- 操作类型（带颜色标签）
- 操作人（管理员）
- 操作详情（描述）
- IP地址

#### 5. 部署信息（可选，未来扩展）
如果私有部署系统会定期回传信息，可以显示：
- 服务器信息（操作系统、CPU、内存）
- 系统版本
- 最后连接时间
- 健康状态

## 数据库设计

### private_customers 表
```sql
CREATE TABLE IF NOT EXISTS private_customers (
  id VARCHAR(36) PRIMARY KEY,
  customer_name VARCHAR(200) NOT NULL COMMENT '客户名称',
  contact_person VARCHAR(100) COMMENT '联系人',
  contact_phone VARCHAR(50) COMMENT '联系电话',
  contact_email VARCHAR(100) COMMENT '联系邮箱',
  company_address VARCHAR(500) COMMENT '公司地址',
  industry VARCHAR(100) COMMENT '所属行业',
  company_size VARCHAR(50) COMMENT '公司规模',
  deployment_type VARCHAR(50) DEFAULT 'on-premise' COMMENT '部署类型: on-premise-本地部署, cloud-云部署',
  status VARCHAR(20) DEFAULT 'active' COMMENT '状态: active-正常, inactive-停用',
  notes TEXT COMMENT '备注',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_customer_name (customer_name),
  INDEX idx_status (status),
  INDEX idx_industry (industry)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='私有部署客户表';
```

### licenses 表优化

## API设计

### 私有客户管理API

```typescript
// 获取私有客户列表
GET /api/v1/admin/private-customers
Query: {
  page: number
  pageSize: number
  keyword?: string          // 搜索关键词（客户名称、联系人）
  industry?: string         // 行业筛选
  status?: string           // 状态筛选
  licenseStatus?: string    // 授权状态筛选
}
Response: {
  list: PrivateCustomer[]
  total: number
}

// 创建私有客户（同时生成授权）
POST /api/v1/admin/private-customers
Body: {
  // 客户信息
  customerName: string
  contactPerson: string
  contactPhone: string
  contactEmail: string
  companyAddress?: string
  industry?: string
  companySize?: string
  deploymentType?: string
  notes?: string
  
  // 授权配置
  licenseType: 'trial' | 'annual' | 'perpetual'
  maxUsers: number
  maxStorageGb: number
  features: string[]
  expiresAt?: string        // 非永久版必填
}
Response: {
  customer: PrivateCustomer
  license: License
}

// 获取私有客户详情
GET /api/v1/admin/private-customers/:id
Response: {
  customer: PrivateCustomer
  currentLicense: License   // 当前有效授权
  licenseHistory: License[] // 授权历史（最近10条）
  logs: Log[]               // 操作日志（最近20条）
}

// 更新私有客户信息
PUT /api/v1/admin/private-customers/:id
Body: {
  customerName?: string
  contactPerson?: string
  contactPhone?: string
  contactEmail?: string
  companyAddress?: string
  industry?: string
  companySize?: string
  notes?: string
}

// 删除私有客户
DELETE /api/v1/admin/private-customers/:id

// 获取客户的所有授权
GET /api/v1/admin/private-customers/:id/licenses
Query: {
  page: number
  pageSize: number
}

// 为客户生成新授权
POST /api/v1/admin/private-customers/:id/licenses
Body: {
  licenseType: string
  maxUsers: number
  maxStorageGb: number
  features: string[]
  expiresAt?: string
  notes?: string
}

// 续期授权
POST /api/v1/admin/private-customers/:customerId/licenses/:licenseId/renew
Body: {
  expiresAt: string
}

// 吊销授权
POST /api/v1/admin/private-customers/:customerId/licenses/:licenseId/revoke
Body: {
  reason: string
}

// 重新激活授权
POST /api/v1/admin/private-customers/:customerId/licenses/:licenseId/reactivate

// 获取授权操作日志
GET /api/v1/admin/private-customers/:customerId/licenses/:licenseId/logs
Query: {
  page: number
  pageSize: number
}
```
