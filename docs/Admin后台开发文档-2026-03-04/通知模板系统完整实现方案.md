## 一、系统概述

通知模板系统是一个统一的消息通知管理平台,支持邮件和短信两种渠道,通过模板化管理实现各种业务场景的自动化通知。

### 1.1 核心功能

- ✅ 模板管理(CRUD)
- ✅ 变量替换和渲染
- ✅ 多渠道发送(邮件+短信)
- ✅ 业务场景集成
- ✅ 发送日志记录
- ✅ Admin后台管理界面

### 1.2 支持的业务场景

| 分类 | 场景 | 邮件 | 短信 |
|------|------|------|------|
| 租户注册 | 注册成功 | ✅ | ✅ |
| 支付 | 支付成功 | ✅ | ✅ |
| 支付 | 待支付提醒 | ✅ | ✅ |
| 支付 | 退款成功 | ✅ | ✅ |
| 授权码 | 授权码生成 | ✅ | ❌ |
| 授权码 | 即将到期 | ✅ | ✅ |
| 授权码 | 已到期 | ✅ | ✅ |
| 租户状态 | 账号激活 | ✅ | ✅ |
| 租户状态 | 账号暂停 | ✅ | ✅ |
| 租户状态 | 账号恢复 | ✅ | ✅ |
| 续费 | 续费成功 | ✅ | ✅ |
| 升级 | 套餐升级 | ✅ | ✅ |

## 二、数据库设计

### 2.1 通知模板表

```sql
CREATE TABLE notification_templates (
  id VARCHAR(36) PRIMARY KEY,
  template_code VARCHAR(100) UNIQUE,    -- 模板代码
  template_name VARCHAR(200),           -- 模板名称
  template_type VARCHAR(50),            -- email/sms/both
  category VARCHAR(50),                 -- 业务分类
  scene VARCHAR(100),                   -- 使用场景
  
  email_subject VARCHAR(200),           -- 邮件主题
  email_content TEXT,                   -- 邮件内容(HTML)
  
  sms_content VARCHAR(500),             -- 短信内容
  sms_template_code VARCHAR(100),       -- 短信服务商模板代码
  
  variables JSON,                       -- 可用变量
  variable_description TEXT,            -- 变量说明
  
  is_enabled TINYINT(1),               -- 是否启用
  is_system TINYINT(1),                -- 是否系统模板
  priority VARCHAR(20),                 -- 优先级
  
  send_email TINYINT(1),               -- 是否发送邮件
  send_sms TINYINT(1),                 -- 是否发送短信
  
  created_at DATETIME,
  updated_at DATETIME
);
```

### 2.2 系统默认模板

已预置13个系统模板,涵盖所有核心业务场景。

## 三、后端实现

### 3.1 核心服务

**NotificationTemplateService** (`backend/src/services/NotificationTemplateService.ts`)

主要方法:
```typescript
// 根据模板代码发送通知
sendByTemplate(templateCode, variables, options)

// 渲染模板
renderTemplate(template, variables)

// 模板管理
getAllTemplates()
getTemplate(templateCode)
createTemplate(data)
updateTemplate(templateCode, data)
deleteTemplate(templateCode)

// 测试模板
testTemplate(templateCode, variables)
```

### 3.2 API接口设计

```
GET    /api/v1/admin/notification-templates          # 获取所有模板
GET    /api/v1/admin/notification-templates/:code    # 获取单个模板
POST   /api/v1/admin/notification-templates          # 创建模板
PUT    /api/v1/admin/notification-templates/:code    # 更新模板
DELETE /api/v1/admin/notification-templates/:code    # 删除模板
POST   /api/v1/admin/notification-templates/:code/test  # 测试模板
POST   /api/v1/admin/notification-templates/:code/send  # 发送通知
```

### 3.3 业务集成示例

#### 注册成功后发送通知

```typescript
// backend/src/controllers/public/RegisterController.ts

// 注册成功后
await notificationTemplateService.sendByTemplate('tenant_register_success', {
  systemName: '云客CRM',
  tenantName: tenant.name,
  adminUsername: 'admin',
  adminPassword: 'admin123',
  packageName: packageInfo.name,
  expireDate: tenant.expireDate,
  loginUrl: `${process.env.WEBSITE_URL}/login`
})
```

#### 支付成功后发送通知

```typescript
// backend/src/services/WechatPayService.ts

  packageName: packageInfo.name,

  payTime: new Date().toLocaleString('zh-CN'),
  serviceStartDate: tenant.activatedAt,
  serviceEndDate: tenant.expireDate,

```

#### 授权即将到期提醒

```typescript
// backend/src/services/LicenseService.ts
// 定时任务检测即将到期的授权
const expiringSoon = await licenseRepo.find({
  where: {
    expireDate: Between(
      new Date(),
      new Date(Date.now() + 7 * 24 * 60 * 60 * 1000)
    )
  }
})

for (const license of expiringSoon) {
  await notificationTemplateService.sendByTemplate('license_expire_soon', {
    tenantName: license.tenantName,
    licenseKey: license.licenseKey,
    expireDate: license.expireDate,
    remainDays: Math.ceil((license.expireDate - Date.now()) / (24 * 60 * 60 * 1000)),
    renewUrl: `${process.env.WEBSITE_URL}/renew`
  })
}
```

## 五、使用指南

### 5.1 配置流程

1. **配置邮件和短信服务**
   - Admin后台 → 系统设置 → 邮件配置
   - Admin后台 → 系统设置 → 短信配置

2. **查看和编辑模板**
   - Admin后台 → 系统设置 → 通知模板
   - 选择要编辑的模板
   - 修改邮件主题、内容或短信内容
   - 保存

3. **测试模板**
   - 点击"测试"按钮
   - 输入测试变量值
   - 点击"预览"查看渲染效果
   - 点击"发送测试"实际发送

### 5.2 业务集成

在业务代码中调用:

```typescript
import { notificationTemplateService } from '@/services/NotificationTemplateService'

// 发送通知
await notificationTemplateService.sendByTemplate('模板代码', {
  变量1: '值1',
  变量2: '值2'
}, {
  priority: 'high',
  actionUrl: '/some/url'
})
```

### 5.3 变量说明

每个模板都有预定义的变量,在模板内容中使用 `{{变量名}}` 格式引用。

示例:
```
邮件主题: 欢迎注册{{systemName}}
邮件内容: 尊敬的 <strong>{{tenantName}}</strong>，您好！
```

渲染后:
```
邮件主题: 欢迎注册云客CRM
邮件内容: 尊敬的 <strong>张三公司</strong>，您好！
```

## 八、常见问题

### Q1: 如何添加新的通知场景?

A: 
1. 在Admin后台创建新模板
2. 定义模板代码和变量
3. 编写邮件和短信内容
4. 在业务代码中调用 `sendByTemplate`

### Q2: 如何自定义邮件样式?

A: 
在邮件内容中使用HTML和内联CSS:
```html
<div style="background:#f5f5f5;padding:20px;">
  <h2 style="color:#409eff;">标题</h2>
  <p>内容</p>
</div>
```

### Q3: 短信内容有长度限制吗?

A: 
是的,短信内容限制500字符,建议控制在70字以内以避免拆分成多条。

### Q4: 如何处理发送失败?

A: 
系统会记录发送日志,可以在Admin后台查看失败原因,并支持手动重发。
