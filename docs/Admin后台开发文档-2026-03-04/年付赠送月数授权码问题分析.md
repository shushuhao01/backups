# 年付赠送月数授权码问题分析

## 问题描述

用户提问：**"后台管理套餐这里给了赠送几个月，那么给到用户的，用户支付以后拿到的授权码，是有原来的时期加这几个月没有？"**

## 当前问题

经过代码审查，发现**当前系统没有正确处理年付赠送月数**：

### 1. 缺失的数据流

```
用户选择年付 → 官网显示"送3个月" → 用户支付
                                    ↓
                            ❌ 计费周期信息丢失
                                    ↓
                        生成授权码时只使用 duration_days
                                    ↓
                        ❌ 没有加上赠送的月数
```

### 2. 当前代码问题

#### 问题1: 注册API不接收计费周期
**文件**: `backend/src/routes/public/register.ts`

```typescript
// 当前代码：只接收 packageCode，不接收 billingCycle
const { companyName, contactName, phone, code, email, packageCode } = req.body
```

#### 问题2: 支付订单不保存计费周期
**文件**: `backend/src/routes/public/payment.ts`

```typescript
// 当前代码：创建订单时不保存 billingCycle
const { packageId, packageName, amount, payType, tenantId, tenantName,
        contactName, contactPhone, contactEmail } = req.body
```

#### 问题3: 授权码生成不考虑赠送月数
**文件**: `backend/src/services/PaymentService.ts`

```typescript
// 当前代码：只使用 duration_days，不考虑年付赠送月数
const packages = await AppDataSource.query(
  'SELECT duration_days, max_users FROM tenant_packages WHERE id = ?', [packageId]
)

const expireDate = new Date()
expireDate.setDate(expireDate.getDate() + packages[0].duration_days)
// ❌ 没有加上 yearly_bonus_months
```

## 业务逻辑分析

### 正确的业务逻辑应该是：

#### 月付套餐
- 用户支付: ¥299
- 授权有效期: 30天（1个月）
- 计算方式: `duration_days` (通常是30天)

#### 年付套餐（送3个月）
- 用户支付: ¥1990
- 授权有效期: 450天（15个月 = 12个月 + 3个月赠送）
- 计算方式: `(12 + yearly_bonus_months) × 30天` = 15 × 30 = 450天

### 数据库现状

#### packages 表字段
```sql
- duration_days: 30 (月付基础天数)
- yearly_bonus_months: 3 (年付赠送月数)
- yearly_price: 1990 (年付价格)
- billing_cycle: 'monthly' (套餐默认计费周期)
```

#### payment_orders 表（需要添加字段）
```sql
❌ 缺少 billing_cycle 字段
❌ 缺少 bonus_months 字段
```

#### tenants 表
```sql
- expire_date: 到期时间
✅ 这个字段应该根据计费周期和赠送月数正确计算
```

## 修复方案

### 方案1: 完整修复（推荐）

#### 步骤1: 数据库迁移
添加 `payment_orders` 表字段：
```sql
ALTER TABLE payment_orders 
ADD COLUMN billing_cycle VARCHAR(20) DEFAULT 'monthly' COMMENT '计费周期：monthly-月付，yearly-年付';

ALTER TABLE payment_orders 
ADD COLUMN bonus_months INT DEFAULT 0 COMMENT '赠送月数';
```

#### 步骤2: 修改注册API
接收并保存计费周期：
```typescript
const { companyName, contactName, phone, code, email, packageCode, billingCycle } = req.body
```

#### 步骤3: 修改支付订单创建
保存计费周期和赠送月数：
```typescript
const { packageId, packageName, amount, payType, billingCycle, tenantId, ... } = req.body

// 查询套餐的赠送月数
const packages = await AppDataSource.query(
  'SELECT yearly_bonus_months FROM packages WHERE id = ?', [packageId]
)
const bonusMonths = billingCycle === 'yearly' ? packages[0].yearly_bonus_months : 0

// 保存到订单
await AppDataSource.query(
  'INSERT INTO payment_orders (..., billing_cycle, bonus_months) VALUES (..., ?, ?)',
  [..., billingCycle, bonusMonths]
)
```

#### 步骤4: 修改授权码生成逻辑
根据计费周期和赠送月数计算有效期：
```typescript
// 获取订单信息（包含计费周期和赠送月数）
const orders = await AppDataSource.query(
  'SELECT tenant_id, package_id, billing_cycle, bonus_months FROM payment_orders WHERE order_no = ?',
  [orderNo]
)

// 获取套餐基础天数
const packages = await AppDataSource.query(
  'SELECT duration_days FROM packages WHERE id = ?', [packageId]
)

// 计算有效期
let totalDays = packages[0].duration_days // 基础天数（通常30天）

if (orders[0].billing_cycle === 'yearly') {
  // 年付：12个月 + 赠送月数
  totalDays = (12 + orders[0].bonus_months) * 30
}

const expireDate = new Date()
expireDate.setDate(expireDate.getDate() + totalDays)
```

### 方案2: 简化修复（临时方案）

如果不想修改数据库，可以在生成授权码时动态查询：

```typescript
// 从订单金额判断是否年付
const orders = await AppDataSource.query(
  'SELECT tenant_id, package_id, amount FROM payment_orders WHERE order_no = ?',
  [orderNo]
)

// 查询套餐信息
const packages = await AppDataSource.query(
  'SELECT price, yearly_price, yearly_bonus_months, duration_days FROM packages WHERE id = ?',
  [packageId]
)

// 判断是否年付
const isYearly = orders[0].amount >= (packages[0].yearly_price || packages[0].price * 10)

// 计算有效期
let totalDays = packages[0].duration_days
if (isYearly) {
  totalDays = (12 + packages[0].yearly_bonus_months) * 30
}
```

**缺点**: 不够准确，如果年付价格调整可能判断错误
