# 授权管理模块测试指南

## 📋 测试准备

### 1. 启动后端服务
```bash
cd backend
npm run dev
```

### 2. 启动前端服务
```bash
cd admin
npm run dev
```

### 3. 访问Admin后台
```
http://localhost:5173
```

## 🧪 后端API测试

### 方式1：使用测试脚本
```bash
cd backend
node test-licenses-api.js
```

### 方式2：使用Postman/Thunder Client

#### 1. 管理员登录
```http
POST http://localhost:3000/api/v1/admin/auth/login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123"
}
```

**响应**:
```json
{
  "success": true,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": "xxx",
      "username": "admin"
    }
  }
}
```

#### 2. 创建授权
```http
POST http://localhost:3000/api/v1/admin/licenses
Authorization: Bearer <token>
Content-Type: application/json

{
  "customerName": "测试公司A",
  "customerContact": "张三",
  "customerPhone": "13800138000",
  "customerEmail": "test@example.com",
  "licenseType": "trial",
  "maxUsers": 20,
  "maxStorageGb": 10,
  "features": ["订单管理", "客户管理"],
  "expiresAt": "2026-04-04",
  "notes": "测试授权"
}
```

#### 3. 获取授权列表
```http
GET http://localhost:3000/api/v1/admin/licenses?page=1&pageSize=20
Authorization: Bearer <token>
```

#### 4. 获取授权详情
```http
GET http://localhost:3000/api/v1/admin/licenses/:id
Authorization: Bearer <token>
```

#### 5. 激活授权
```http
POST http://localhost:3000/api/v1/admin/licenses/:id/activate
Authorization: Bearer <token>
Content-Type: application/json

{
  "machineId": "MACHINE-001"
}
```

#### 6. 验证授权
```http
POST http://localhost:3000/api/v1/admin/licenses/verify
Content-Type: application/json

{
  "licenseKey": "XXXX-XXXX-XXXX-XXXX-XXXX",
  "machineId": "MACHINE-001"
}
```

#### 7. 续期授权
```http
POST http://localhost:3000/api/v1/admin/licenses/:id/renew
Authorization: Bearer <token>
Content-Type: application/json

{
  "expiresAt": "2027-04-04"
}
```

#### 8. 停用授权
```http
POST http://localhost:3000/api/v1/admin/licenses/:id/deactivate
Authorization: Bearer <token>
Content-Type: application/json

{
  "reason": "客户要求停用"
}
```

#### 9. 获取授权日志
```http
GET http://localhost:3000/api/v1/admin/licenses/:id/logs?page=1&pageSize=10
Authorization: Bearer <token>
```

#### 10. 获取授权统计
```http
GET http://localhost:3000/api/v1/admin/licenses/statistics
Authorization: Bearer <token>
```

#### 11. 更新授权
```http
PUT http://localhost:3000/api/v1/admin/licenses/:id
Authorization: Bearer <token>
Content-Type: application/json

{
  "maxUsers": 50,
  "notes": "更新用户数"
}
```

#### 12. 删除授权
```http
DELETE http://localhost:3000/api/v1/admin/licenses/:id
Authorization: Bearer <token>
```

## 🐛 常见问题

### 1. 后端服务无法启动
**问题**: 运行 `npm run dev` 后报错

**解决方案**:
- 检查Node.js版本（需要 >= 14）
- 检查数据库连接配置
- 检查端口是否被占用
- 运行 `npm install` 重新安装依赖

### 2. 前端无法访问后端API
**问题**: 前端调用API时报CORS错误

**解决方案**:
- 检查后端CORS配置
- 检查API地址配置（.env文件）
- 检查后端服务是否正常运行

### 3. 授权列表为空
**问题**: 授权列表页面显示为空

**解决方案**:
- 检查数据库中是否有数据
- 检查API是否正常返回
- 检查前端是否正确处理响应
- 查看浏览器控制台错误信息

### 4. 授权码无法复制
**问题**: 点击复制按钮没有反应

**解决方案**:
- 检查浏览器是否支持Clipboard API
- 检查是否使用HTTPS（某些浏览器要求）
- 尝试手动选择复制
