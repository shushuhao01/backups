# 对公转账功能完整实现说明

## 📅 完成时间
2026-03-05

## 🎯 功能概述

对公转账是除微信支付和支付宝之外的第三种支付方式，适用于企业客户通过银行转账的方式完成支付。

与微信/支付宝的自动回调不同，对公转账需要：
1. 用户手动转账到指定银行账户
2. 用户上传转账凭证
3. 管理员人工审核确认
4. 审核通过后激活租户

## ✅ 已实现功能

### 1. 创建对公转账订单

**API**: `POST /api/v1/public/payment/create`

**请求参数**:
```json
{
  "packageId": "basic",
  "packageName": "基础版",
  "amount": 99.00,
  "payType": "bank",  // 对公转账
  "billingCycle": "monthly",
  "tenantId": "xxx",
  "tenantName": "测试企业",
  "contactName": "张三",
  "contactPhone": "13800138000",
  "contactEmail": "test@example.com"
}
```

**响应**:
```json
{
  "code": 0,
  "message": "创建支付订单成功",
  "data": {
    "orderId": "xxx",
    "orderNo": "PAY1234567890",
    "payType": "bank",
    "bankInfo": {
      "accountName": "XX科技有限公司",
      "bankName": "中国工商银行",
      "accountNumber": "6222 **** **** 1234",
      "bankAddress": "北京市朝阳区支行",
      "bankCode": "102100099996",
      "swiftCode": "ICBKCNBJ",
      "remarkTemplate": "订单号：PAY1234567890",
      "arrivalTime": "1-3个工作日",
      "notice": "请在转账备注中填写订单号，转账后请上传凭证"
    },
    "expireTime": "2026-03-05 15:30:00"
  }
}
```

**订单状态**: `pending_transfer`（等待转账）

---

### 2. 上传转账凭证

**API**: `POST /api/v1/public/payment/upload-proof/:orderNo`

**请求参数**:
```json
{
  "proofUrl": "https://example.com/uploads/proof/xxx.jpg",
  "transferTime": "2026-03-05 14:30:00",
  "transferAmount": 99.00,
  "transferAccount": "张三",
  "remark": "已转账，请查收"
}
```

**响应**:
```json
{
  "code": 0,
  "message": "凭证上传成功，等待审核"
}
```

**订单状态**: `pending_review`（等待审核）

---

### 3. 管理员审核（通过）

**API**: `POST /api/v1/admin/payment/orders/:id/approve`

**请求参数**:
```json
{
  "remark": "转账已到账，审核通过"
}
```

**响应**:
```json
{
  "success": true,
  "message": "审核通过，订单已完成"
}
```

**执行操作**:
1. 更新订单状态为 `paid`（已支付）
2. 记录支付时间
3. 生成授权码
4. 激活租户
5. 创建授权记录

**订单状态**: `paid`（已支付）

---

### 4. 管理员审核（拒绝）

**API**: `POST /api/v1/admin/payment/orders/:id/reject`

**请求参数**:
```json
{
  "reason": "转账金额不符，请重新转账"
}
```

**响应**:
```json
{
  "success": true,
  "message": "已拒绝该转账"
}
```

**订单状态**: `review_rejected`（审核拒绝）

## 🔧 配置指南

### 在Admin后台配置对公转账

1. 登录Admin后台
2. 进入"支付管理" → "支付配置"
3. 选择"对公转账"标签
4. 填写以下信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 账户名称 | 收款账户名称 | XX科技有限公司 |
| 开户银行 | 银行名称 | 中国工商银行 |
| 银行账号 | 收款账号 | 6222 0000 0000 1234 |
| 开户行地址 | 具体支行 | 北京市朝阳区支行 |
| 联行号 | 银行联行号 | 102100099996 |
| SWIFT代码 | 国际汇款代码 | ICBKCNBJ |
| 转账备注模板 | 备注格式 | 订单号：{orderNo} |
| 到账时间 | 预计到账时间 | 1-3个工作日 |
| 转账说明 | 注意事项 | 请在转账备注中填写订单号 |
| 自动确认 | 是否自动确认 | 否（需人工审核） |

5. 点击"保存配置"
6. 启用对公转账

## 📝 订单状态说明

### 对公转账订单状态流转

```
pending_transfer（等待转账）
    ↓
pending_review（等待审核）
    ↓
    ├─→ paid（已支付）- 审核通过
    └─→ review_rejected（审核拒绝）- 审核拒绝
```

### 状态详细说明

| 状态 | 说明 | 用户操作 | 管理员操作 |
|------|------|----------|-----------|
| pending_transfer | 订单已创建，等待用户转账 | 完成转账，上传凭证 | - |
| pending_review | 凭证已上传，等待审核 | 等待审核结果 | 审核通过/拒绝 |
| paid | 审核通过，支付完成 | 查看授权码，登录CRM | - |
| review_rejected | 审核拒绝 | 重新转账或联系客服 | - |
| closed | 订单已关闭 | - | - |

## 🔍 与微信/支付宝的区别

| 特性 | 微信/支付宝 | 对公转账 |
|------|------------|---------|
| 支付方式 | 扫码支付 | 银行转账 |
| 支付确认 | 自动回调 | 人工审核 |
| 到账时间 | 即时 | 1-3个工作日 |
| 凭证上传 | 不需要 | 需要 |
| 激活方式 | 自动激活 | 审核通过后激活 |
| 适用场景 | 个人/小企业 | 大企业/政府单位 |
| SDK集成 | 需要 | 不需要 |

## 🎯 关于SDK集成

### 微信支付SDK集成

**文件**: `backend/src/services/WechatPayService.ts`

**需要的配置**:
- AppID（微信开放平台应用ID）
- 商户号（微信支付商户号）
- API密钥V3（32位字符串）
- 证书序列号（从商户证书中获取）

**SDK功能**:
- 创建Native支付（生成二维码）
- 生成签名（HMAC-SHA256）
- 验证回调签名
- 解密回调数据（AES-256-GCM）
- 查询订单状态

**使用方式**:
```typescript
// 创建支付
const result = await wechatPayService.createNativePay({
  orderNo: 'PAY123456',
  amount: 99.00,
  description: '基础版套餐'
})

// 处理回调
const result = await wechatPayService.handleCallback(callbackData)
```

### 支付宝SDK集成

**文件**: `backend/src/services/AlipayService.ts`

**需要的配置**:
- AppID（支付宝开放平台应用ID）
- 应用私钥（RSA2私钥）
- 支付宝公钥（RSA2公钥）
- 签名类型（RSA2）

**SDK功能**:
- 创建当面付（生成二维码）
- 生成签名（RSA2）
- 验证回调签名
- 查询订单状态

**使用方式**:
```typescript
// 创建支付
const result = await alipayService.createQRPay({
  orderNo: 'PAY123456',
  amount: 99.00,
  subject: '基础版套餐'
})

// 处理回调
const result = await alipayService.handleCallback(params)
```

### 对公转账"SDK"

对公转账不需要真正的SDK，因为：
1. 不需要调用第三方API
2. 不需要生成二维码
3. 不需要处理自动回调
4. 只需要返回银行账户信息

**实现方式**:
```typescript
// 获取银行配置
const bankConfig = await getBankConfig()

// 返回银行账户信息
return {
  bankInfo: {
    accountName: config.accountName,
    bankName: config.bankName,
    accountNumber: config.accountNumber,
    // ... 其他信息
  }
}
```
