# Admin后台数据库迁移 - 快速参考

## 📦 迁移内容

**12个表** + **5个字段补充**

### 表清单
- admin_users (Admin用户)
- admin_operation_logs (Admin日志)
- tenants (租户)
- tenant_settings (租户设置)
- tenant_logs (租户日志)
- packages (套餐)
- licenses (授权)
- license_logs (授权日志)
- private_customers (私有客户)
- versions (版本)
- changelogs (更新日志)
- payment_orders (支付订单-字段补充)

## 🚀 快速执行 (宝塔面板)

### 方式一: 一键执行 (推荐)

1. **备份数据库** (必须!)
2. **进入数据库管理** → 选择数据库 → 点击"管理"
3. **点击"SQL"标签**
4. **复制粘贴**: `backend/database-migrations/production-admin-baota.sql`
5. **点击"执行"**
6. **验证**: 执行下面的验证SQL

### 方式二: 分步执行 (如果方式一报错)

如果一键执行遇到语法错误,请使用分步执行:
- 参考文档: `backend/database-migrations/宝塔执行指南-分步版.md`
- 每次执行一个CREATE TABLE语句
- 共12个表,分7个步骤执行

## ✅ 验证SQL

```sql
-- 应该看到12个表
SELECT TABLE_NAME, TABLE_ROWS
FROM information_schema.TABLES 
WHERE TABLE_SCHEMA = DATABASE() 
  AND TABLE_NAME IN (
    'admin_users', 'admin_operation_logs',
    'tenants', 'tenant_settings', 'tenant_logs',
    'packages', 'licenses', 'license_logs',
    'private_customers', 'versions', 'changelogs',
    'payment_orders'
  )
ORDER BY TABLE_NAME;
```
