# 租户套餐限制功能 - 快速参考

## 🎯 功能概述

租户套餐限制功能确保每个租户严格遵守其套餐配置，包括：
- ⏰ **过期时间控制** - 过期后无法访问系统
- 👥 **用户数限制** - 限制可创建的用户数量
- 💾 **存储空间限制** - 限制可上传的文件总大小

## 📋 数据库字段

### tenants 表

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| expire_date | DATE | 过期时间 | NULL |
| max_users | INT | 最大用户数 | 10 |
| max_storage_gb | INT | 最大存储空间(GB) | 5 |
| user_count | INT | 当前用户数 | 0 |
| used_storage_mb | DECIMAL(10,2) | 已使用存储(MB) | 0 |

## 🔧 核心中间件

### 1. tenantAuth - 租户认证

**位置**: `backend/src/middleware/tenantAuth.ts`

**功能**:
- 验证租户状态（active/inactive）
- 检查授权状态（active/pending/suspended/expired）
- 检查过期时间

**应用**: 所有需要租户认证的路由

### 2. checkUserLimit - 用户数限制

**位置**: `backend/src/middleware/checkTenantLimits.ts`

**功能**:
- 检查当前用户数是否达到 max_users
- 超限时返回403错误
- 接近限制（90%）时记录警告

**应用**: 用户创建路由

### 3. checkStorageLimit - 存储空间限制

**位置**: `backend/src/middleware/checkTenantLimits.ts`

**功能**:
- 检查存储空间是否充足
- 超限时返回403错误
- 接近限制（90%）时记录警告

**应用**: 文件上传路由

## 📝 使用示例

### 创建用户路由

```typescript
router.post('/', 
  authenticateToken,      // JWT认证
  tenantAuth,             // 租户认证（含过期检查）
  requireManagerOrAdmin,  // 权限检查
  checkUserLimit,         // 用户数限制检查
  validate(schema),       // 参数验证
  controller.createUser   // 业务逻辑
);
```

### 文件上传路由

```typescript
router.post('/upload-avatar', 
  authenticateToken,      // JWT认证
  tenantAuth,             // 租户认证（含过期检查）
  checkStorageLimit,      // 存储空间限制检查
  upload.single('image'), // 文件上传
  handleImageUpload       // 业务逻辑
);
```

## 🔍 故障排查

### 问题：限制不生效

**检查清单**:
1. ✅ 确认是SaaS模式：`deployConfig.isSaaS()` 返回 true
2. ✅ 确认中间件已应用：检查路由定义
3. ✅ 确认数据库字段存在：运行 `test-tenant-limits-manual.js`
4. ✅ 确认后端已重启：重启后端服务

### 问题：统计数据不准确

**解决方案**:
1. 查询实际数据：
```sql
SELECT 
  t.id, t.name, t.user_count,
  (SELECT COUNT(*) FROM users WHERE tenant_id = t.id) as actual_count
FROM tenants t;
```

2. 修复统计数据：
```sql
UPDATE tenants t
SET user_count = (SELECT COUNT(*) FROM users WHERE tenant_id = t.id);
```
