# 年付赠送月数功能完成总结

## 问题背景

用户提问：**"后台管理套餐这里给了赠送几个月，那么给到用户的，用户支付以后拿到的授权码，是有原来的时期加这几个月没有？"**

经过代码审查，发现系统存在严重的业务逻辑bug：
- ❌ 用户选择年付套餐（如"送3个月"），支付成功后生成的授权码，并没有加上赠送的月数
- ❌ 用户支付¥1990年付（送3个月），但只获得30天（1个月）的授权
- ❌ 应该获得450天（15个月 = 12+3）的授权

#### 修改2: PaymentService - 创建订单时查询赠送月数
**文件**: `backend/src/services/PaymentService.ts`

```typescript
async createOrder(params: CreateOrderParams) {
  const billingCycle = params.billingCycle || 'monthly'
  
  // 查询套餐的赠送月数
  let bonusMonths = 0
  if (billingCycle === 'yearly') {
    const packages = await AppDataSource.query(
      'SELECT yearly_bonus_months FROM packages WHERE id = ?',
      [params.packageId]
    )
    if (packages && packages.length > 0) {
      bonusMonths = packages[0].yearly_bonus_months || 0
    }
  }

  // 创建订单记录（保存billing_cycle和bonus_months）
  await AppDataSource.query(
    `INSERT INTO payment_orders
     (..., billing_cycle, bonus_months, ...)
     VALUES (..., ?, ?, ...)`,
    [..., billingCycle, bonusMonths, ...]
  )
}
```

#### 修改3: PaymentService - 授权码生成时计算正确有效期
**文件**: `backend/src/services/PaymentService.ts`

```typescript
private async activateTenant(tenantId: string, packageId: string) {
  // 获取订单信息（包含计费周期和赠送月数）
  const orders = await AppDataSource.query(
    'SELECT billing_cycle, bonus_months FROM payment_orders WHERE tenant_id = ? ...',
    [tenantId, packageId]
  )

  // 获取套餐信息
  const packages = await AppDataSource.query(
    'SELECT duration_days, max_users FROM packages WHERE id = ?',
    [packageId]
  )

  const billingCycle = orders[0].billing_cycle
  const bonusMonths = orders[0].bonus_months || 0
  const baseDays = packages[0].duration_days || 30

  // 计算有效期
  let totalDays = baseDays
  if (billingCycle === 'yearly') {
    // 年付：12个月 + 赠送月数
    totalDays = (12 + bonusMonths) * 30
    console.log(`年付套餐: 12个月 + ${bonusMonths}个月赠送 = ${totalDays}天`)
  }

  const expireDate = new Date()
  expireDate.setDate(expireDate.getDate() + totalDays)

  // 生成授权码并设置到期时间
  await AppDataSource.query(
    `UPDATE tenants SET license_key = ?, expire_date = ?, ... WHERE id = ?`,
    [licenseKey, expireDate, ...]
  )
}
```

## 数据流对比

### 修复前的数据流
```
用户选择年付 → 官网显示"送3个月" → 用户支付¥1990
                                      ↓
                              ❌ 计费周期信息丢失
                                      ↓
                          生成授权码时只使用 duration_days (30天)
                                      ↓
                              ❌ 没有加上赠送的3个月
                                      ↓
                          用户只获得30天授权（应该是450天）
```

### 修复后的数据流
```
用户选择年付 → 官网显示"送3个月" → 用户支付¥1990
                                      ↓
                          前端传递 billingCycle='yearly'
                                      ↓
                          后端查询 yearly_bonus_months=3
                                      ↓
                          保存到订单: billing_cycle='yearly', bonus_months=3
                                      ↓
                          支付成功后激活租户
                                      ↓
                          计算有效期: (12 + 3) × 30 = 450天
                                      ↓
                          ✅ 用户获得450天授权（15个月）
```

## 数据修复建议

如果生产环境已有用户支付了年付但只获得了1个月授权，需要：

1. 查询所有年付订单：
```sql
SELECT 
  o.order_no, o.tenant_id, o.amount, o.created_at,
  t.name, t.license_key, t.expire_date,
  p.name as package_name, p.yearly_price, p.yearly_bonus_months
FROM payment_orders o
JOIN tenants t ON o.tenant_id = t.id
JOIN packages p ON o.package_id = p.id
WHERE o.status = 'paid'
  AND o.amount >= p.yearly_price
  AND t.expire_date < DATE_ADD(NOW(), INTERVAL 12 MONTH)
ORDER BY o.created_at DESC;
```

2. 手动延长授权有效期：
```sql
UPDATE tenants
SET expire_date = DATE_ADD(expire_date, INTERVAL 11 MONTH)  -- 补上11个月
WHERE id = '受影响的租户ID';
```

3. 发送通知给受影响用户
