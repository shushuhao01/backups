### 2. 实体类开发

#### ApiConfig实体 (`backend/src/entities/ApiConfig.ts`)
**字段**:
- id - 主键
- name - API名称
- code - API代码（唯一）
- description - API描述
- apiKey - API密钥（唯一）
- apiSecret - API密钥（加密）
- status - 状态（active/inactive）
- rateLimit - 速率限制
- allowedIps - 允许的IP（JSON数组）
- expiresAt - 过期时间
- lastUsedAt - 最后使用时间

**辅助方法**:
- `getAllowedIpList()` - 获取IP白名单
- `setAllowedIpList(ips)` - 设置IP白名单
- `isIpAllowed(ip)` - 检查IP是否允许
- `isExpired()` - 检查是否过期
- `isActive()` - 检查是否激活

#### ApiCallLog实体 (`backend/src/entities/ApiCallLog.ts`)
**字段**:
- id - 主键
- apiConfigId - API配置ID
- apiKey - API密钥
- endpoint - 调用端点
- method - 请求方法
- requestParams - 请求参数
- responseStatus - 响应状态码
- responseTime - 响应时间（ms）
- ipAddress - IP地址
- userAgent - User Agent
- errorMessage - 错误信息

**辅助方法**:
- `isSuccess()` - 判断是否成功
- `getRequestParamsObject()` - 获取请求参数对象
- `setRequestParamsObject(params)` - 设置请求参数

### 3. Service层开发

#### ApiConfigService (`backend/src/services/ApiConfigService.ts`)

**核心方法**:
1. `getApiConfigs(query)` - 获取API配置列表
   - 支持分页
   - 支持按状态筛选
   - 支持按代码/名称搜索

2. `getApiConfigById(id)` - 获取API配置详情

3. `getApiConfigByKey(apiKey)` - 根据API Key获取配置

4. `createApiConfig(data)` - 创建API配置
   - 自动生成API Key和Secret
   - 格式：`ak_` + 32位随机字符串
   - 格式：`sk_` + 64位随机字符串

5. `updateApiConfig(id, data)` - 更新API配置

6. `deleteApiConfig(id)` - 删除API配置

7. `regenerateKey(id)` - 重新生成API密钥

8. `getApiCallLogs(query)` - 获取API调用日志
   - 支持分页
   - 支持按API配置ID筛选
   - 支持按端点筛选
   - 支持按时间范围筛选

9. `logApiCall(data)` - 记录API调用

10. `updateLastUsedAt(id)` - 更新最后使用时间

11. `getApiStatistics(apiConfigId)` - 获取API统计信息
    - 总调用次数
    - 成功次数
    - 成功率
    - 平均响应时间
    - 错误次数

### 5. 路由配置

#### 路由文件 (`backend/src/routes/admin/api-configs.ts`)

```typescript
GET    /api/v1/admin/api-configs                    // 获取API配置列表
GET    /api/v1/admin/api-configs/logs               // 获取所有API调用日志
GET    /api/v1/admin/api-configs/:id/statistics     // 获取API统计信息
GET    /api/v1/admin/api-configs/:id/logs           // 获取API调用日志
GET    /api/v1/admin/api-configs/:id                // 获取API配置详情
POST   /api/v1/admin/api-configs                    // 创建API配置
PUT    /api/v1/admin/api-configs/:id                // 更新API配置
DELETE /api/v1/admin/api-configs/:id                // 删除API配置
POST   /api/v1/admin/api-configs/:id/regenerate-key // 重新生成密钥
```

#### 路由注册
在 `backend/src/routes/admin/index.ts` 中注册：
```typescript

router.use('/api-configs', apiConfigsRouter);
```

### 6. 中间件开发

#### API密钥认证中间件 (`backend/src/middleware/apiKeyAuth.ts`)

**功能**:
- 从请求头获取API密钥（`X-API-Key`）
- 验证API密钥是否有效
- 检查API配置状态（是否激活、是否过期）
- 检查IP白名单
- 更新最后使用时间
- 将API配置信息附加到请求对象

**使用方式**:
```typescript
import { apiKeyAuth } from '../middleware/apiKeyAuth';
router.use(apiKeyAuth);
```

#### API速率限制中间件 (`backend/src/middleware/apiRateLimit.ts`)

**功能**:
- 基于内存的速率限制
- 按API Key限制调用频率
- 1小时为一个周期
- 超过限制返回429状态码
- 设置响应头：
  - `X-RateLimit-Limit` - 速率限制
  - `X-RateLimit-Remaining` - 剩余次数
  - `X-RateLimit-Reset` - 重置时间

**自动清理**:
- 每小时自动清理过期的速率限制记录

**使用方式**:
```typescript
import { apiRateLimit } from '../middleware/apiRateLimit';
router.use(apiKeyAuth, apiRateLimit);
```
