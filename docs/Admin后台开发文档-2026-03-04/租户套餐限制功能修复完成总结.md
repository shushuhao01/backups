### 3. 用户统计自动更新 ✅

**文件**: `backend/src/controllers/UserController.ts`

**修改**: 在创建和删除用户后更新统计

```typescript

import { updateTenantUserCount, decrementTenantUserCount } from '../middleware/checkTenantLimits';

// 创建用户后更新统计
const savedUser = await this.userRepository.save(user);

// 更新租户用户数统计（仅SaaS模式）
const tenantId = (req as any).tenantId;
if (tenantId) {
  await updateTenantUserCount(tenantId, 1);
}

// 删除用户后减少统计
await this.userRepository.remove(user);

// 减少租户用户数统计（仅SaaS模式）
const tenantId = (req as any).tenantId;
if (tenantId) {
  await decrementTenantUserCount(tenantId, 1);
}
```

**效果**:
- 创建用户成功后，tenants.user_count 自动 +1
- 删除用户后，tenants.user_count 自动 -1
- 统计数据实时准确

## 功能说明

### 1. 过期时间控制

- **检查时机**: 每次API请求时（tenantAuth中间件）
- **检查逻辑**: `tenant.expireDate < 当前时间`
- **拒绝响应**: 403 Forbidden - "租户已过期，请联系管理员续费"
- **影响范围**: 所有需要认证的API

### 2. 用户数限制

- **检查时机**: 创建用户前（checkUserLimit中间件）
- **检查逻辑**: `当前用户数 >= max_users`
- **拒绝响应**: 403 Forbidden - "用户数已达上限，当前 X/Y 个用户"
- **统计更新**: 
  - 创建用户成功后：user_count +1
  - 删除用户后：user_count -1

### 3. 存储空间限制

- **检查时机**: 上传文件前（checkStorageLimit中间件）
- **检查逻辑**: `used_storage_mb + 文件大小 > max_storage_gb * 1024`
- **拒绝响应**: 403 Forbidden - "存储空间不足，需要 XMB，剩余 YMB"
- **统计更新**: 
  - 上传文件成功后：used_storage_mb += 文件大小
  - 删除文件后：used_storage_mb -= 文件大小

### 4. 警告机制

当资源使用率达到90%以上时，系统会记录警告日志：

```
[租户资源警告] 租户 XXX(tenant_id) 用户数接近上限: 9/10
[租户资源警告] 租户 XXX(tenant_id) 存储空间接近上限: 9216MB/10240MB
```

## 技术细节

### 中间件执行顺序

```
authenticateToken (验证JWT)
  ↓
tenantAuth (验证租户状态、过期时间)
  ↓
checkUserLimit / checkStorageLimit (检查资源限制)
  ↓
业务逻辑 (创建用户/上传文件)
  ↓
updateTenantUserCount / updateTenantStorage (更新统计)
```

### 仅SaaS模式生效

所有限制检查都包含模式判断：

```typescript
// 只在SaaS模式下检查
if (!deployConfig.isSaaS()) {
  next()
  return
}
```

私有部署模式不受这些限制影响。

### 错误响应格式

```json
{
  "success": false,
  "message": "用户数已达上限，当前 10/10 个用户",
  "code": "USER_LIMIT_EXCEEDED",
  "data": {
    "currentCount": 10,
    "maxUsers": 10,
    "usagePercent": 100
  }
}
```
