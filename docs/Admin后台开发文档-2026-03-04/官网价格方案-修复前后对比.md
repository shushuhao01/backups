# 官网价格方案页面 - 修复前后对比

## 问题定位

用户反馈：管理后台套餐管理显示正确，但官网价格方案页面切换到年付后：
1. ❌ 没有显示赠送月数
2. ❌ 年付价格不准确（固定8折，未使用数据库配置）
3. ❌ 节省金额计算不准确
4. ❌ 管理后台修改年付配置后，官网不同步

## 修复前后对比

### 基础版套餐示例

#### 数据库配置（Admin后台设置）
- 月付价格: ¥299
- 赠送月数: 3个月
- 年付价格: ¥1990
- 节省金额: ¥1598

#### 修复前（官网显示）
```
月付模式:
  ¥299/月

年付模式:
  ¥239/月                    ← 错误！使用固定8折计算（299 × 0.8）
  小绿字: 年付 ¥2868，省 ¥720  ← 错误！未使用数据库配置
  没有显示赠送月数             ← 缺失！
```

**问题分析:**
- 年付价格: 299 × 0.8 × 12 = ¥2868（硬编码8折）
- 节省金额: 299 × 12 - 2868 = ¥720（基于错误的年付价格）
- 完全忽略了数据库中配置的"送3个月"和"年付¥1990"

#### 修复后（官网显示）
```
月付模式:
  ¥299/月

年付模式:
  ¥166/月                              ← 正确！1990 ÷ 12 = 166
  小绿字: 年付 ¥1990（送3个月），省 ¥1598  ← 正确！使用数据库配置
```

**计算逻辑:**
- 年付价格: 直接使用数据库配置 ¥1990
- 折算月价: 1990 ÷ 12 = ¥166/月
- 节省金额: 299 × 12 - 1990 = ¥1598
- 赠送月数: 显示"送3个月"

## 核心修复点

### 1. 年付价格计算逻辑

**修复前（硬编码）:**
```typescript
// 固定8折
export function getYearlyPrice(monthlyPrice: number): number {
  return Math.round(monthlyPrice * 0.8)
}
```

**修复后（动态配置）:**
```typescript
// 使用数据库配置
export function getYearlyPrice(pkg: Package): number {
  // 优先使用配置的年付价格
  if (pkg.yearly_price) {
    return pkg.yearly_price
  }
  // 否则根据赠送月数计算
  return pkg.price * (12 - pkg.yearly_bonus_months)
}
```

### 2. 节省金额计算逻辑

**修复前:**
```typescript
// 基于错误的年付价格计算
export function getYearlySaving(monthlyPrice: number): number {
  return monthlyPrice * 12 - getYearlyTotal(monthlyPrice)
}
// 结果: 299 × 12 - 2868 = ¥720（错误）
```

**修复后:**
```typescript
// 基于实际年付价格计算
export function getYearlySaving(pkg: Package): number {
  return pkg.price * 12 - getYearlyPrice(pkg)
}
// 结果: 299 × 12 - 1990 = ¥1598（正确）
```

### 3. 赠送月数显示

**修复前:**
```vue
<p class="price-note" v-if="isYearly">
  年付 ¥{{ getYearlyTotal(pkg.price) }}，省 ¥{{ getYearlySaving(pkg.price) }}
</p>
<!-- 没有显示赠送月数 -->
```

**修复后:**
```vue
<p class="price-note" v-if="isYearly">
  年付 ¥{{ getYearlyTotal(pkg) }}
  <template v-if="pkg.yearly_bonus_months > 0">
    （送{{ pkg.yearly_bonus_months }}个月）
  </template>
  ，省 ¥{{ getYearlySaving(pkg) }}
</p>
<!-- 动态显示赠送月数 -->
```
