# 支付流程打通状态说明

## 问题
在Admin后台的支付配置中配置好微信支付和支付宝后，是否可以启用并与官网对接，当租户注册提交后，是否可以直接调用和扫码支付对应的金额？

## 回答：基本框架已完成，但需要补充关键功能

### 当前实现状态

#### ✅ 已完成的部分

1. **数据库表结构** ✅
   - `payment_configs` 表：存储支付配置（微信、支付宝、对公转账）
   - `payment_orders` 表：存储支付订单
   - `payment_logs` 表：记录支付日志

2. **Admin后台配置界面** ✅
   - 支付配置页面：`admin/src/views/payment/Config.vue`
   - 支持配置微信支付、支付宝、对公转账
   - 配置数据AES-256-CBC加密存储
   - 敏感信息脱敏显示

3. **配置保存API** ✅
   - `POST /admin/payment/config/wechat` - 保存微信支付配置
   - `POST /admin/payment/config/alipay` - 保存支付宝配置
   - `POST /admin/payment/config/bank` - 保存对公转账配置
   - `GET /admin/payment/config` - 获取所有配置

4. **官网注册流程** ✅
   - 注册页面：`website/src/views/Register.vue`
   - 支持选择套餐（SaaS/私有部署）
   - 支持月付/年付切换
   - 支持选择支付方式（微信/支付宝）

5. **支付服务基础框架** ✅
   - `PaymentService` - 支付订单管理
   - `WechatPayService` - 微信支付服务（框架）
   - `AlipayService` - 支付宝服务（框架）

#### ❌ 缺失的关键功能

1. **创建支付订单API** ❌
   - 官网调用的 `POST /api/v1/public/payment/create` 接口未实现
   - 需要调用微信/支付宝API生成支付二维码
   - 需要返回二维码URL或base64数据

2. **查询支付状态API** ❌
   - 官网调用的 `GET /api/v1/public/payment/query/:orderNo` 接口未实现
   - 需要查询数据库订单状态
   - 需要返回租户信息和授权码

3. **微信支付SDK集成** ❌
   - 未集成微信支付官方SDK
   - 未实现Native支付（扫码支付）
   - 未实现签名生成和验证

4. **支付宝SDK集成** ❌
   - 未集成支付宝官方SDK
   - 未实现当面付（扫码支付）
   - 未实现签名生成和验证

5. **支付回调处理** ⚠️
   - 回调接口已定义但未完整实现
   - 需要验证签名
   - 需要更新订单状态
   - 需要生成授权码

6. **租户激活流程** ❌
   - 支付成功后需要激活租户
   - 需要生成授权码
   - 需要设置过期时间

### 2. 实现公共支付控制器

创建 `backend/src/controllers/public/PaymentController.ts`：

```typescript
import { Request, Response } from 'express'
import { AppDataSource } from '../../config/database'
import { v4 as uuidv4 } from 'uuid'
import { wechatPayService } from '../../services/WechatPayService'
import { alipayService } from '../../services/AlipayService'

export class PublicPaymentController {
  /**
   * 创建支付订单
   */
  async createOrder(req: Request, res: Response) {
    try {
      const {
        packageId, packageName, amount, payType, billingCycle,
        tenantId, tenantName, contactName, contactPhone, contactEmail
      } = req.body

      // 生成订单号
      const orderNo = `PAY${Date.now()}${Math.random().toString(36).substr(2, 6).toUpperCase()}`

      // 创建订单
      const orderId = uuidv4()
      await AppDataSource.query(
        `INSERT INTO payment_orders (
          id, order_no, customer_type, tenant_id, tenant_name,
          package_id, package_name, amount, billing_cycle, pay_type,
          status, contact_name, contact_phone, contact_email,
          expire_time, created_at
        ) VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, NOW())`,
        [
          orderId, orderNo, 'tenant', tenantId, tenantName,
          packageId, packageName, amount, billingCycle, payType,
          'pending', contactName, contactPhone, contactEmail,
          new Date(Date.now() + 30 * 60 * 1000) // 30分钟过期
        ]
      )

      // 调用支付API生成二维码
      let paymentResult
      if (payType === 'wechat') {
        paymentResult = await wechatPayService.createNativePay({
          orderNo,
          amount,
          description: packageName
        })
      } else if (payType === 'alipay') {
        paymentResult = await alipayService.createQRPay({
          orderNo,
          amount,
          subject: packageName
        })
      } else {
        return res.status(400).json({ code: 1, message: '不支持的支付方式' })
      }

      // 更新订单二维码
      await AppDataSource.query(
        'UPDATE payment_orders SET qr_code = ?, pay_url = ? WHERE id = ?',
        [paymentResult.qrCode, paymentResult.payUrl, orderId]
      )

      res.json({
        code: 0,
        data: {
          orderId,
          orderNo,
          qrCode: paymentResult.qrCode,
          payUrl: paymentResult.payUrl
        }
      })
    } catch (error: any) {
      console.error('创建支付订单失败:', error)
      res.status(500).json({ code: 1, message: error.message })
    }
  }

  /**
   * 查询支付状态
   */
  async queryOrder(req: Request, res: Response) {
    try {
      const { orderNo } = req.params

      // 查询订单
      const orders = await AppDataSource.query(
        `SELECT o.*, t.code as tenant_code, l.license_key
         FROM payment_orders o
         LEFT JOIN tenants t ON o.tenant_id = t.id
         LEFT JOIN licenses l ON o.tenant_id = l.tenant_id
         WHERE o.order_no = ?`,
        [orderNo]
      )

      if (orders.length === 0) {
        return res.status(404).json({ code: 1, message: '订单不存在' })
      }

      const order = orders[0]

      res.json({
        code: 0,
        data: {
          status: order.status,
          tenantCode: order.tenant_code,
          licenseKey: order.license_key
        }
      })
    } catch (error: any) {
      console.error('查询支付状态失败:', error)
      res.status(500).json({ code: 1, message: error.message })
    }
  }
}

export const publicPaymentController = new PublicPaymentController()
```
