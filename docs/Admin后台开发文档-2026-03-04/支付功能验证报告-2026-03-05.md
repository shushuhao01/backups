## 开发模式 vs 生产模式

### 开发模式

**触发条件**: 未配置完整的API密钥

**特点**:
- 返回模拟二维码（MOCK_WECHAT_xxx / MOCK_ALIPAY_xxx）
- 不调用真实支付API
- 跳过签名验证
- 方便本地开发测试

**使用场景**:
- 本地开发
- 功能测试
- UI调试

### 生产模式

**触发条件**: 配置完整的API密钥

**特点**:
- 调用真实支付API
- 生成真实支付二维码
- 严格验证签名
- 需要公网可访问的回调地址

**使用场景**:
- 生产环境
- 真实支付
- 正式上线

## 配置指南

### 1. 微信支付配置

在Admin后台 → 支付管理 → 支付配置 → 微信支付：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| AppID | 微信开放平台应用ID | wx1234567890abcdef |
| 商户号 | 微信支付商户号 | 1234567890 |
| API密钥V3 | 微信支付API密钥V3 | 32位字符串 |
| 证书序列号 | 商户证书序列号 | 从证书中获取 |
| 回调地址 | 支付成功回调地址 | https://your-domain.com/api/v1/admin/payment/notify/wechat |

### 2. 支付宝配置

在Admin后台 → 支付管理 → 支付配置 → 支付宝：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| AppID | 支付宝开放平台应用ID | 2021001234567890 |
| 应用私钥 | RSA2私钥 | MIIEvQIBADANBgkqhkiG9w0... |
| 支付宝公钥 | 支付宝RSA2公钥 | MIIBIjANBgkqhkiG9w0BAQE... |
| 签名类型 | 签名算法 | RSA2 |
| 回调地址 | 支付成功回调地址 | https://your-domain.com/api/v1/admin/payment/notify/alipay |

### 3. 回调地址配置

**重要**: 回调地址必须是公网可访问的HTTPS地址

#### 开发环境

使用内网穿透工具（如ngrok）：

```bash
# 安装ngrok
# 下载: https://ngrok.com/download

# 启动内网穿透
ngrok http 3000

# 获得公网地址，例如: https://abc123.ngrok.io

# 配置回调地址:
# 微信: https://abc123.ngrok.io/api/v1/admin/payment/notify/wechat
# 支付宝: https://abc123.ngrok.io/api/v1/admin/payment/notify/alipay
```

#### 生产环境

```
微信: https://your-domain.com/api/v1/admin/payment/notify/wechat
支付宝: https://your-domain.com/api/v1/admin/payment/notify/alipay
```
