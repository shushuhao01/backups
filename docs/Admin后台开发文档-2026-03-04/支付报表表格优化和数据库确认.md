# 支付报表表格优化和数据库确认

## 完成时间
2026-03-05

## 问题描述

用户反馈两个问题：
1. 支付报表的详细数据表格没有横向铺满页面
2. 需要确认数据库是否支持支付配置（微信、支付宝、对公转账）

## 解决方案

### 1. 表格宽度优化

#### 问题分析
原表格使用固定宽度（`width`），导致列宽固定，无法自适应页面宽度，右侧留有大量空白。

#### 优化方案
将所有列的 `width` 改为 `min-width`，让表格列可以自适应扩展：

```vue
<!-- 优化前 -->
<el-table-column prop="date" label="日期" width="120" fixed="left" />
<el-table-column prop="amount" label="收入金额" width="120" align="right" />

<!-- 优化后 -->
<el-table-column prop="date" label="日期" min-width="120" fixed="left" />
<el-table-column prop="amount" label="收入金额" min-width="140" align="right" />
```

#### 优化效果
- 表格会自动铺满容器宽度
- 各列按比例分配剩余空间
- 保持最小宽度，避免内容挤压
- 金额列增加到140px，显示更舒适

### 2. 数据库支持确认

#### payment_configs表结构

已在数据库迁移脚本中定义，支持三种支付方式：

```sql
CREATE TABLE IF NOT EXISTS `payment_configs` (
  `id` VARCHAR(36) PRIMARY KEY COMMENT '配置ID',
  `pay_type` VARCHAR(20) NOT NULL UNIQUE COMMENT '支付方式: wechat/alipay/bank',
  `enabled` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否启用',
  `config_data` TEXT COMMENT '配置数据(加密存储)',
  `notify_url` VARCHAR(500) COMMENT '回调地址',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX `idx_pay_type` (`pay_type`),
  INDEX `idx_enabled` (`enabled`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='支付配置表';
```

#### 支持的支付方式

##### 1. 微信支付 (wechat)
配置字段（存储在config_data中，AES-256-CBC加密）：
- `appId`: 应用ID
- `mchId`: 商户号
- `apiKey`: API密钥
- `certPath`: 证书路径
- `keyPath`: 密钥路径

##### 2. 支付宝 (alipay)
配置字段：
- `appId`: 应用ID
- `privateKey`: 应用私钥
- `publicKey`: 支付宝公钥
- `gatewayUrl`: 网关地址

##### 3. 对公转账 (bank)
配置字段：
- `accountName`: 收款单位名称
- `bankName`: 开户银行
- `accountNumber`: 银行账号
- `bankAddress`: 开户行地址
- `bankCode`: 联行号
- `swiftCode`: SWIFT代码
- `transferNote`: 转账备注要求
- `arrivalTime`: 到账时间说明
- `tips`: 温馨提示
- `autoConfirm`: 自动确认开关

#### 数据安全
- 所有敏感配置数据使用AES-256-CBC加密存储
- 银行账号支持脱敏显示（中间位数用****替换）
- 配置数据仅管理员可见

### 2. 配置数据加密
```typescript
import crypto from 'crypto';

// 加密
function encrypt(text: string, key: string): string {
  const iv = crypto.randomBytes(16);
  const cipher = crypto.createCipheriv('aes-256-cbc', Buffer.from(key), iv);
  let encrypted = cipher.update(text, 'utf8', 'hex');
  encrypted += cipher.final('hex');
  return iv.toString('hex') + ':' + encrypted;
}

// 解密
function decrypt(text: string, key: string): string {
  const parts = text.split(':');
  const iv = Buffer.from(parts[0], 'hex');
  const encrypted = parts[1];
  const decipher = crypto.createDecipheriv('aes-256-cbc', Buffer.from(key), iv);
  let decrypted = decipher.update(encrypted, 'hex', 'utf8');
  decrypted += decipher.final('utf8');
  return decrypted;
}
```

### 3. 银行账号脱敏
```typescript
function maskBankAccount(account: string): string {
  if (account.length <= 8) return account;
  const start = account.slice(0, 4);
  const end = account.slice(-4);
  const middle = '*'.repeat(account.length - 8);
  return start + middle + end;
}

// 示例: 6222021234567890 → 6222************7890
```
