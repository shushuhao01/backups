# 套餐管理500错误修复

## 问题描述

Admin后台访问套餐管理页面时出现500错误:

```
Unknown column 'packages.yearly_discount_rate' in 'field list'
```

## 原因分析

数据库中缺少新添加的年付配置字段:
- `yearly_discount_rate` - 年付折扣率
- `yearly_bonus_months` - 年付赠送月数  
- `yearly_price` - 年付价格

这些字段在代码中已经添加,但数据库迁移脚本还没有执行。

## 解决方案

### 1. 执行数据库迁移

**文件:** `backend/execute-yearly-config-migration.js`

```bash
cd backend
node execute-yearly-config-migration.js
```

### 2. 迁移内容

1. **添加字段**
```sql
ALTER TABLE packages 
ADD COLUMN yearly_discount_rate DECIMAL(5, 2) DEFAULT 0.00 COMMENT '年付折扣率',
ADD COLUMN yearly_bonus_months INT DEFAULT 0 COMMENT '年付赠送月数',
ADD COLUMN yearly_price DECIMAL(10, 2) DEFAULT NULL COMMENT '年付价格';
```

2. **更新SaaS套餐配置**
```sql
UPDATE packages 
SET 
  yearly_discount_rate = 16.67,
  yearly_bonus_months = 2,
  yearly_price = price * 10
WHERE type = 'saas' AND billing_cycle = 'monthly' AND price > 0;
```

3. **修正免费试用套餐有效期**
```sql
UPDATE packages 
SET duration_days = 7
WHERE (code LIKE '%TRIAL%' OR is_trial = 1) AND price = 0;
```

## 注意事项

1. **数据库密码**
   - 迁移脚本使用`.env`文件中的数据库配置
   - 确保数据库用户名和密码正确

2. **字段默认值**
   - `yearly_discount_rate`: 默认0.00
   - `yearly_bonus_months`: 默认0
   - `yearly_price`: 默认NULL（自动计算）

3. **仅SaaS月付套餐**
   - 只有SaaS类型且计费周期为monthly的套餐才配置年付
   - 免费试用套餐不配置年付
   - 私有部署版是一次性付费,不需要年付
