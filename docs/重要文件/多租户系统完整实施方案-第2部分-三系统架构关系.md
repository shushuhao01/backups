# 多租户系统完整实施方案 - 第2部分：三系统架构关系

## 🏗️ 三个系统的架构关系

CRM项目包含三个独立但相互关联的系统：

1. **CRM主系统** - 核心业务系统
2. **Admin后台** - 平台管理系统
3. **官网** - 展示和销售系统

## 二、数据流向

### 1. 私有部署流程

```
步骤1: 购买
用户 → 联系销售 → 购买授权码

步骤2: Admin后台操作
销售 → Admin后台 → 创建私有客户 → 生成授权码 → 发送给客户

步骤3: 部署
客户 → 下载CRM系统 → 部署到自己服务器 → 配置数据库

步骤4: 激活
客户 → 打开CRM系统 → 输入授权码 → 激活成功 → 开始使用

数据库状态:
- 客户使用独立数据库
- 所有表的 tenant_id = NULL
- 不需要租户隔离
```

### 2. SaaS租户流程

```
步骤1: 注册
用户 → 官网 → 注册账号 → 选择套餐 → 支付

步骤2: Admin后台自动处理
官网支付成功 → 调用Admin后台API → 创建租户记录 → 生成授权码

步骤3: 初始化
Admin后台 → 初始化租户数据 → 创建默认管理员 → 创建默认角色

步骤4: 登录使用
用户 → 收到授权码和登录信息 → 登录CRM系统 → 开始使用

数据库状态:
- 所有租户共享同一个数据库
- 每个租户有唯一的 tenant_id (UUID)
- 所有查询自动添加 tenant_id 过滤
```

### 3. 版本更新流程

```
开发完成新版本
    ↓
Admin后台发布版本
    ├─→ 私有部署客户
    │   ├─→ 检查更新（调用Admin后台API）
    │   ├─→ 下载新版本
    │   └─→ 手动更新
    │
    └─→ SaaS租户
        ├─→ Admin后台推送更新通知
        ├─→ 自动更新（或定时更新）
        └─→ 用户无感知
```

## 三、控制关系

### Admin后台对CRM系统的控制

#### 1. 功能模块控制
```typescript
// Admin后台设置
{
  tenant_id: 'uuid-a',
  enabled_modules: [
    'customer_management',  // 客户管理
    'order_management',     // 订单管理
    'product_management',   // 商品管理
    'finance_management',   // 财务管理
    // 'logistics_management' // 物流管理（未启用）
  ]
}

// CRM系统检查
if (!isModuleEnabled('logistics_management')) {
  // 隐藏物流管理菜单
  // 禁用物流相关功能
}
```

#### 2. 版本控制
```typescript
// Admin后台发布版本
{
  version: '1.8.2',
  release_date: '2026-03-15',
  force_update: true,  // 强制更新
  min_version: '1.8.0' // 最低兼容版本
}

// CRM系统检查更新
const currentVersion = '1.8.0'
const latestVersion = await checkUpdate()
if (latestVersion.force_update && currentVersion < latestVersion.min_version) {
  // 强制更新，禁止使用
  showUpdateDialog({ force: true })
}
```

#### 3. 授权控制
```typescript
// Admin后台生成授权码
{
  license_key: 'XXXX-XXXX-XXXX-XXXX',
  tenant_id: 'uuid-a',
  expire_date: '2027-03-15',
  max_users: 50,
  max_storage_gb: 100
}

// CRM系统验证授权
const license = await verifyLicense(license_key)
if (license.expired) {
  // 授权已过期，禁止使用
  showExpiredDialog()
}
if (currentUsers >= license.max_users) {
  // 用户数已达上限，禁止添加新用户
  showUserLimitDialog()
}
```

#### 4. 资源限制
```typescript
// Admin后台设置限制
{
  tenant_id: 'uuid-a',
  max_users: 50,           // 最大用户数
  max_storage_gb: 100,     // 最大存储空间
  max_orders_per_month: 10000  // 每月最大订单数
}

// CRM系统检查限制
if (currentUsers >= maxUsers) {
  throw new Error('用户数已达上限')
}
if (storageUsed >= maxStorage) {
  throw new Error('存储空间已满')
}
```

#### 5. 状态控制
```typescript
// Admin后台控制租户状态
{
  tenant_id: 'uuid-a',
  status: 'suspended'  // active, inactive, suspended, expired
}

// CRM系统检查状态
const tenant = await getTenantInfo()
if (tenant.status === 'suspended') {
  // 租户已暂停，禁止使用
  showSuspendedDialog()
}
```

### Admin后台对官网的控制

#### 1. 套餐管理
```typescript
// Admin后台定义套餐
{
  id: 'package-basic',
  name: '基础版',
  price: 999,
  features: {
    max_users: 10,
    max_storage_gb: 10,
    enabled_modules: ['customer', 'order', 'product']
  }
}

// 官网展示套餐
const packages = await getPackages()
packages.forEach(pkg => {
  renderPackageCard(pkg)
})
```

#### 2. 支付管理
```typescript
// 官网创建订单
const order = await createOrder({
  package_id: 'package-basic',
  user_phone: '13800138000',
  payment_method: 'wechat'
})

// Admin后台处理支付
onPaymentSuccess(order_id => {
  // 创建租户
  const tenant = await createTenant({
    phone: order.user_phone,
    package_id: order.package_id
  })
  
  // 生成授权码
  const license = await generateLicense(tenant.id)
  
  // 发送给用户
  await sendSMS(order.user_phone, license.key)
})
```

#### 3. 用户管理
```typescript
// 官网用户注册
const user = await register({
  phone: '13800138000',
  code: '123456',
  package_id: 'package-basic'
})

// Admin后台创建租户
const tenant = await createTenant({
  phone: user.phone,
  package_id: user.package_id,
  status: 'trial'  // 试用状态
})

// 初始化租户数据
await initTenantData(tenant.id)
```

### 官网对CRM系统的关系

#### 1. 用户入口
- 官网是用户注册和购买的入口
- 用户在官网完成注册和支付
- Admin后台创建租户并生成授权码
- 用户使用授权码登录CRM系统

#### 2. 信息展示
- 官网展示CRM系统的功能和价格
- 官网展示不同套餐的功能对比
- 官网提供帮助文档和使用指南

#### 3. 支付链接
- 官网处理用户支付
- 支付成功后调用Admin后台API
- Admin后台创建租户并初始化数据
- 用户可以立即登录CRM系统

## 四、API接口设计

### Admin后台提供的API

#### 1. 租户管理API
```typescript
// 创建租户
POST /api/v1/admin/tenants
{
  tenant_name: '测试公司',
  contact_phone: '13800138000',
  package_id: 'package-basic'
}

// 获取租户信息
GET /api/v1/admin/tenants/:tenant_id

// 更新租户状态
PUT /api/v1/admin/tenants/:tenant_id/status
{
  status: 'active' | 'inactive' | 'suspended' | 'expired'
}
```

#### 2. 授权管理API
```typescript
// 生成授权码
POST /api/v1/admin/licenses
{
  tenant_id: 'uuid-a',
  expire_date: '2027-03-15',
  max_users: 50
}

// 验证授权码（CRM系统调用）
POST /api/v1/license/verify
{
  license_key: 'XXXX-XXXX-XXXX-XXXX'
}

// 撤销授权
DELETE /api/v1/admin/licenses/:license_id
```

#### 3. 版本管理API
```typescript
// 发布版本
POST /api/v1/admin/versions
{
  version: '1.8.2',
  release_notes: '修复Bug，优化性能',
  force_update: true
}

// 检查更新（CRM系统调用）
GET /api/v1/version/check
{
  current_version: '1.8.0'
}
```

#### 4. 套餐管理API
```typescript
// 获取套餐列表（官网调用）
GET /api/v1/packages

// 创建订单（官网调用）
POST /api/v1/payment/orders
{
  package_id: 'package-basic',
  user_phone: '13800138000',
  payment_method: 'wechat'
}
```
