## 一、系统架构概览

### 1.1 四个子系统

| 子系统 | 目录 | 权限模型 | 用户类型 | 认证方式 |
|--------|------|----------|----------|----------|
| **CRM前端** | `src/` | 5种角色 + 功能权限 + 数据范围 | 租户内用户（User） | JWT Token |
| **管理后台Admin** | `admin/src/` | 单一 `isAdmin` 校验 | 平台管理员（AdminUser） | 独立JWT（`isAdmin:true`） |
| **官网Website** | `website/src/` | 无权限控制（全公开） | 访客 | 无 |
| **后端API** | `backend/src/` | JWT + 角色中间件 + 租户隔离 | 两套用户体系 | `authenticateToken` / `adminAuthMiddleware` |

### 1.2 两套独立的认证体系

```
CRM用户体系                          Admin用户体系
┌──────────────────────┐            ┌──────────────────────┐
│ 实体: User           │            │ 实体: AdminUser      │
│ 表:   users          │            │ 表:   admin_users    │
│ 认证: authenticateToken│           │ 认证: adminAuthMiddleware│
│ JWT:  { userId,      │            │ JWT:  { isAdmin:true,│
│        role,         │            │        adminId }     │
│        tenantId }    │            │                      │
│ 路由: /api/v1/*      │            │ 路由: /api/admin/*   │
└──────────────────────┘            └──────────────────────┘
```

### 1.3 数据库角色定义（`roles` 表）

> 文件: `backend/src/entities/Role.ts`

| 字段 | 类型 | 说明 |

## 三、管理后台（Admin）权限分析

> 目录: `admin/src/`  
> 认证: `backend/src/middleware/adminAuth.ts`

### 3.1 认证机制

```typescript
// adminAuth.ts 第32-37行
const decoded = jwt.verify(token, process.env.JWT_SECRET || 'admin-secret-key')
if (!decoded.isAdmin) {
  return res.status(403).json({ message: '无权访问管理后台' })
}
```

- JWT中包含 `isAdmin: true` 标记
- **没有细粒度角色/权限控制**
- 所有登录后的管理员可访问所有后台功能

### 3.2 后台功能模块

| 模块 | 路由 | 权限要求 |
|------|------|----------|
| 仪表盘 | `/` | 仅需登录 |
| 私有客户管理 | `/private-customers/*` | 仅需登录 |
| 租户客户管理 | `/tenant-customers/*` | 仅需登录 |
| 模块服务 | `/modules/*` | 仅需登录 |
| 支付管理 | `/payment/*` | 仅需登录 |
| 版本发布 | `/versions/*` | 仅需登录 |
| 接口管理 | `/api/*` | 仅需登录 |
| 系统设置 | `/settings/*` | 仅需登录 |

### 3.3 后台路由守卫

```typescript
// admin/src/router/index.ts 第216-226行
router.beforeEach((to, _from, next) => {
  const userStore = useUserStore()
  if (to.meta.requiresAuth !== false && !userStore.isLoggedIn) {
    next('/login')
  } else if (to.path === '/login' && userStore.isLoggedIn) {
    next('/')
  } else {
    next()
  }
})
```

**结论：Admin后台只检查是否登录，不检查任何角色或权限。所有管理员权限完全相同。**

## 六、租户数据隔离 - 全面审计

### 6.1 租户隔离机制概述

| 项目 | 说明 |
|------|------|
| **隔离策略** | 共享数据库 + `tenant_id` 字段 |
| **激活条件** | `DEPLOY_MODE=saas` 环境变量（`deployConfig.isSaaS()`） |
| **核心工具** | `getTenantRepo()` Proxy 自动注入 |
| **辅助函数** | `tenantRawSQL()`, `addTenantFilter()`, `withTenant()`, `setTenantOnEntity()` |
| **上下文管理** | `TenantContextManager`（AsyncLocalStorage）|
| **JWT携带** | `payload.tenantId` 在 `authenticateToken` 中注入 |

### 6.2 隔离工具详解

> 文件: `backend/src/utils/tenantHelpers.ts`

| 工具函数 | 用途 | 适用场景 |
|----------|------|----------|
| `getCurrentTenantIdSafe()` | 获取当前租户ID | 通用 |
| `withTenant(where)` | 为 find/findOne 添加 tenant_id | TypeORM Repository |
| `addTenantFilter(qb, alias)` | 为 QueryBuilder 添加过滤 | TypeORM QueryBuilder |
| `tenantRawSQL(prefix?)` | 返回 ` AND tenant_id = ?` SQL片段 | 原始SQL查询 |
| `tenantWhereSQL(prefix?)` | 返回 `WHERE tenant_id = ?` SQL片段 | 原始SQL（首个条件） |
| `setTenantOnEntity(entity)` | 保存前设置 tenant_id | 实体写入 |

> 文件: `backend/src/utils/tenantRepo.ts`

`getTenantRepo(Entity)` 返回一个 Proxy 包装的 Repository，自动拦截：
- `find` / `findOne` / `findOneBy` / `findBy` / `findAndCount` / `count` → 自动追加 `tenant_id` WHERE 条件
- `save` / `create` → 自动设置 `tenant_id`
- `createQueryBuilder` → 自动追加 `WHERE alias.tenant_id = ?`
- `update` / `delete` → 自动追加 `tenant_id` 条件
