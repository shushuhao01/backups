# CRM项目仓库分离方案

## 背景

当前项目采用单仓库（Monorepo）结构，包含主CRM系统、官网、Admin后台和移动端APP。由于商业模式涉及私有部署销售，需要将仓库分离以保护商业机密和简化客户交付。

## 问题分析

### 当前风险
1. **安全风险**: 客户获取代码时会看到官网和Admin管理后台
2. **商业机密**: 定价策略、授权机制、营销策略可能泄露
3. **部署复杂**: 客户需要手动排除不相关代码
4. **版本管理**: 三个系统版本耦合，不利于独立迭代

## 推荐方案：完全分离

### 仓库结构

```
GitHub Organization: YourCompany
├── crm-system          (私有 - 交付给客户)
├── crm-website         (私有 - 仅内部使用)
├── crm-admin           (私有 - 仅内部使用)
└── crm-app             (私有 - 可选择性交付)
```

## 实施步骤

### 第一步：创建新仓库

```bash
# 1. 在GitHub创建4个新的私有仓库
# - crm-system
# - crm-website
# - crm-admin
# - crm-app

# 2. 在本地创建新目录
mkdir crm-repos
cd crm-repos
```

### 第二步：分离CRM主系统

```bash
# 1. Clone当前仓库
git clone https://github.com/YourOrg/CRM.git crm-system-temp
cd crm-system-temp

# 2. 删除不需要的目录
rm -rf website/
rm -rf admin/
rm -rf crmAPP/

# 3. 清理不相关的文档
cd docs
rm -rf 官网项目分析报告.md
rm -rf Admin后台管理系统深度分析报告.md
cd ..

# 4. 更新README.md
# 编辑README.md，移除官网和Admin相关内容

# 5. 推送到新仓库
git remote remove origin
git remote add origin https://github.com/YourOrg/crm-system.git
git push -u origin main
```

### 第三步：分离官网

```bash
# 1. 重新clone原仓库
cd ../

cd crm-website-temp

# 2. 只保留website目录
mv website/* ./
rm -rf website/
rm -rf admin/
rm -rf crmAPP/
rm -rf src/
rm -rf backend/
rm -rf database/
# ... 删除其他CRM相关目录

# 3. 更新README.md

# 4. 推送到新仓库
```


## 客户交付流程

### 方案1：直接Git Clone（推荐）

```bash
# 客户执行
git clone https://github.com/YourOrg/crm-system.git
cd crm-system
npm install
# 按照README.md部署
```

**优点**:
- ✅ 简单直接
- ✅ 客户可以获取更新
- ✅ 可以使用Git管理

**注意**:
- 需要给客户分配只读权限
- 使用GitHub的Deploy Keys或Personal Access Token

### 方案2：打包交付

```bash
# 在服务器上执行
cd crm-system
./scripts/package.sh

# 生成 crm-system-v1.8.0.tar.gz
# 发送给客户
```

**优点**:
- ✅ 不暴露Git历史
- ✅ 可以移除敏感信息
- ✅ 更安全

**缺点**:
- ❌ 更新麻烦
- ❌ 需要手动打包

### 方案3：私有NPM包（高级）

```bash
# 发布到私有NPM仓库
npm publish --registry=https://your-npm-registry.com
```

## 共享代码管理

如果三个系统有共享代码（如类型定义、工具函数），可以：

### 方案1：创建共享包

```

├── src/

│   ├── utils/
│   └── constants/
```

## 安全建议

### 1. 仓库权限
- crm-system: 客户只读权限
- crm-website: 仅内部团队
- crm-admin: 仅核心团队

### 2. 敏感信息
- 所有仓库使用 .env.example
- 不提交真实配置
- 使用环境变量

### 3. 授权管理
- crm-system 包含授权验证客户端
- crm-admin 包含授权生成和管理
- 授权密钥不在代码中

### 4. 代码混淆
- 生产构建时混淆代码
- 移除source map
- 压缩和优化
