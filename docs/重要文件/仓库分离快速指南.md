# CRM仓库分离 - 5分钟快速指南

## 🎯 目标

保持本地在同一个目录开发，但GitHub上分离成多个独立仓库，保护商业机密。

## 📋 准备工作（5分钟）

### 第一步：在GitHub创建4个私有仓库

登录GitHub，创建以下仓库（**不要**初始化README）：

1. `crm-system` - CRM主系统（交付给客户）
2. `crm-website` - 官网（仅内部）
3. `crm-admin` - 管理后台（仅内部）
4. `crm-app` - 移动端（可选）

### 第二步：配置远程仓库

在项目根目录运行：

```powershell
.\setup-remotes.ps1
```

按提示输入你的GitHub用户名或组织名。

### 第三步：首次同步

```powershell
.\sync-repos.ps1 -message "Initial sync"
```

等待几分钟，所有代码会自动同步到各个仓库。

## ✅ 完成！

现在你的GitHub上有4个独立仓库了！

## 📝 日常使用

### 开发流程（和以前一样）

```powershell
# 1. 正常开发
code src/views/Customer/List.vue
code website/src/views/Home.vue
code admin/src/views/Dashboard.vue

# 2. 提交代码
git add .
git commit -m "添加新功能"
git push origin main

# 3. 同步到各个仓库（一键完成）
.\sync-repos.ps1 -message "添加新功能"
```

### 常用命令

```powershell
# 基本同步
.\sync-repos.ps1 -message "你的提交信息"

# 只同步CRM系统
.\sync-repos.ps1 -onlyCRM

# 只同步官网
.\sync-repos.ps1 -onlyWebsite

# 只同步Admin
.\sync-repos.ps1 -onlyAdmin

# 跳过主仓库推送
.\sync-repos.ps1 -skipMain
```

## 🎁 客户交付

客户只需要：

```bash
git clone https://github.com/你的用户名/crm-system.git
```

他们**看不到**website和admin的代码！

## 🔧 故障排除

### 问题1：提示"远程仓库未配置"

**解决**：重新运行配置脚本
```powershell
.\setup-remotes.ps1
```

### 问题2：同步失败

**解决**：检查GitHub仓库是否创建，网络是否正常

### 问题3：权限错误

**解决**：确保你有仓库的写入权限
