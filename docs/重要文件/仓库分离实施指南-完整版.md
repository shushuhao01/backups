## 常见问题

### Q1: 同步失败怎么办？

**A**: 检查以下几点：

1. **网络问题**
   ```powershell
   # 测试GitHub连接
   ping github.com
   ```

2. **权限问题**
   ```powershell
   # 检查Git凭据
   git config --global user.name
   git config --global user.email
   ```

3. **仓库未创建**
   - 确认GitHub上已创建4个仓库
   - 确认仓库名称正确

4. **远程仓库未配置**
   ```powershell
   # 重新配置
   .\setup-remotes.ps1
   ```

### Q2: 如何撤销同步？

**A**: 同步是推送操作，无法撤销，但可以：

1. **删除远程仓库**
   - 在GitHub上删除仓库
   - 重新创建并同步

2. **强制推送新版本**
   ```powershell
   .\sync-repos-v2.ps1 -message "重新同步"
   ```

### Q3: 本地开发会受影响吗？

**A**: 不会！

- ✅ 本地代码完全不变
- ✅ 继续在同一个目录开发
- ✅ 只是GitHub上分离了仓库
- ✅ 同步脚本不会修改本地文件

### Q4: 原来的CRM仓库怎么办？

**A**: 可以保留或删除：

**保留**（推荐）：
- 作为备份
- 继续作为主开发仓库
- 不影响新的分离仓库

**删除**：
- 如果确定不需要了
- 在GitHub上删除即可

### Q5: 如何更新客户的系统？

**A**: 客户在他们的服务器上执行：

```bash
cd /www/wwwroot/crm-system
git pull origin main
npm install
npm run build
pm2 restart crm-api
```

### Q6: 同步很慢怎么办？

**A**: 

1. **使用代理**
   ```powershell
   git config --global http.proxy http://127.0.0.1:7890
   ```

2. **只同步需要的仓库**
   ```powershell
   .\sync-repos-v2.ps1 -onlyCRM
   ```

3. **跳过主仓库**
   ```powershell
   .\sync-repos-v2.ps1 -skipMain
   ```

## 故障排除

### 错误1: "fatal: refusing to merge unrelated histories"

**原因**: 远程仓库已有内容

**解决**:
```powershell
# 删除远程仓库内容，重新创建空仓库
# 或者使用强制推送
git push -f crm-system main
```

### 错误2: "error: failed to push some refs"

**原因**: 权限问题或网络问题

**解决**:
```powershell
# 1. 检查GitHub登录状态
# 2. 检查仓库权限
# 3. 检查网络连接
# 4. 重新推送
.\sync-repos-v2.ps1 -message "重试"
```

### 错误3: "remote: Repository not found"

**原因**: 仓库未创建或名称错误

**解决**:
```powershell
# 1. 检查GitHub上是否已创建仓库
# 2. 检查仓库名称是否正确
# 3. 重新配置远程仓库
.\setup-remotes.ps1
```
