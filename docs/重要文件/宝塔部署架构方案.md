# CRM系统宝塔部署架构方案

## 核心问题

GitHub仓库分离后，在宝塔服务器上如何部署？是多个站点还是一个站点？能否共用数据库和后端服务？

## 推荐方案：单服务器多站点架构

### 架构概览

```
宝塔服务器 (1台服务器)
├── MySQL数据库 (1个，共用)
│   └── crm_production (数据库名)
│
├── 后端API服务 (1个，共用)
│   ├── 域名: api.yourdomain.com
│   ├── 端口: 3000
│   └── 代码: crm-system/backend/
│
├── CRM主系统前端 (站点1 - 给客户)
│   ├── 域名: crm.客户域名.com
│   ├── 端口: 80/443
│   └── 代码: crm-system/dist/
│
├── 官网 (站点2 - 你自己的)
│   ├── 域名: www.yourdomain.com
│   ├── 端口: 80/443
│   └── 代码: crm-website/dist/
│
└── Admin后台 (站点3 - 你自己的)
    ├── 域名: admin.yourdomain.com
    ├── 端口: 80/443
    └── 代码: crm-admin/dist/
```

## 详细部署方案

### 方案A：你自己的服务器（推荐）⭐⭐⭐⭐⭐

**适用场景**：你自己运营CRM、官网、Admin后台

#### 服务器配置

```
服务器: 1台宝塔服务器
CPU: 4核
内存: 8GB
硬盘: 100GB SSD
```

#### 站点配置

**1. 后端API服务（Node.js）**
```
站点名称: CRM-API
域名: api.yourdomain.com
运行目录: /www/wwwroot/crm-api
端口: 3000
PM2管理: 是
数据库: crm_production
```

**2. CRM主系统前端**
```
站点名称: CRM-System
域名: crm.yourdomain.com
运行目录: /www/wwwroot/crm-system
端口: 80/443 (Nginx)
反向代理: api.yourdomain.com:3000
```

**3. 官网**
```
站点名称: Website
域名: www.yourdomain.com
运行目录: /www/wwwroot/website
端口: 80/443 (Nginx)
反向代理: api.yourdomain.com:3000
```

**4. Admin后台**
```
站点名称: Admin
域名: admin.yourdomain.com
运行目录: /www/wwwroot/admin
端口: 80/443 (Nginx)
反向代理: api.yourdomain.com:3000
```

#### 数据库配置

```
数据库名: crm_production
用户名: crm_user
密码: [强密码]
权限: 所有权限
共用: 是（所有站点共用同一个数据库）
```

#### 优点
- ✅ 只需要1台服务器
- ✅ 1个数据库，数据统一
- ✅ 1个后端服务，维护简单
- ✅ 成本低
- ✅ 管理方便

#### 缺点
- ❌ 所有服务在同一台服务器，风险集中
- ❌ 流量大时可能需要扩容

### 方案C：混合部署（最灵活）⭐⭐⭐⭐

**适用场景**：你运营官网和Admin，客户私有部署CRM

#### 你的服务器

```
服务器1: 你的宝塔服务器
├── 官网 (www.yourdomain.com)
├── Admin后台 (admin.yourdomain.com)
├── 后端API (api.yourdomain.com)
└── 数据库 (你的运营数据)
```

#### 客户服务器（多个）

```
客户A服务器:
├── CRM系统 (crm.客户A.com)
├── 后端API (api.客户A.com)
└── 数据库 (客户A的数据)

客户B服务器:
├── CRM系统 (crm.客户B.com)
├── 后端API (api.客户B.com)
└── 数据库 (客户B的数据)
```

## 共用数据库和服务器的详细说明

### 可以共用的情况

#### 1. 你自己运营（方案A）

**共用数据库**：✅ 可以
```
所有站点连接同一个数据库：
- CRM系统 → crm_production
- 官网 → crm_production
- Admin后台 → crm_production
```

**共用后端服务**：✅ 可以
```
所有前端站点调用同一个API：
- CRM前端 → api.yourdomain.com:3000
- 官网前端 → api.yourdomain.com:3000
- Admin前端 → api.yourdomain.com:3000
```

**共用服务器**：✅ 可以
```
所有站点部署在同一台服务器：
- 节省成本
- 管理方便
- 数据统一
```

#### 2. 多租户SaaS模式

**共用数据库**：✅ 可以（但需要租户隔离）
```sql
-- 所有表添加 tenant_id 字段
CREATE TABLE customers (
  id VARCHAR(36) PRIMARY KEY,
  tenant_id VARCHAR(36) NOT NULL,  -- 租户ID
  name VARCHAR(100),
  ...
  INDEX idx_tenant (tenant_id)
);

-- 查询时必须带租户ID
SELECT * FROM customers WHERE tenant_id = '客户A的ID';
```

**共用后端服务**：✅ 可以
```javascript
// 后端API自动识别租户
app.use((req, res, next) => {
  const domain = req.hostname; // crm.客户A.com
  req.tenantId = getTenantIdByDomain(domain);
  next();
});
```

### 不能共用的情况

#### 客户私有部署（方案B）

**共用数据库**：❌ 不可以
```
原因：
- 客户要求数据隔离
- 安全性要求
- 客户在自己的服务器上
```

**共用后端服务**：❌ 不可以
```
原因：
- 客户在自己的服务器上
- 需要独立运行
- 数据不能混在一起
```

**共用服务器**：❌ 不可以
```
原因：
- 客户购买的是私有部署
- 必须在客户自己的服务器上
```

## 宝塔面板具体操作步骤

### 场景1：你自己运营（单服务器多站点）

#### 第一步：创建数据库

1. 登录宝塔面板
2. 点击"数据库" → "添加数据库"
3. 填写信息：
   ```
   数据库名: crm_production
   用户名: crm_user
   密码: [生成强密码]
   访问权限: 本地服务器
   ```
4. 点击"提交"

#### 第二步：创建后端API站点

1. 点击"网站" → "Node项目"
2. 添加Node项目：
   ```
   项目名称: CRM-API
   项目路径: /www/wwwroot/crm-api
   运行端口: 3000
   启动文件: dist/app.js
   域名: api.yourdomain.com
   ```
3. 上传代码：
   ```bash
   cd /www/wwwroot/crm-api
   git clone https://github.com/shushuhao01/crm-system.git .
   cd backend
   npm install
   npm run build
   ```
4. 配置环境变量：
   ```bash
   # 创建 .env 文件
   DB_HOST=localhost
   DB_PORT=3306
   DB_USER=crm_user
   DB_PASSWORD=[数据库密码]
   DB_NAME=crm_production
   PORT=3000
   ```
5. 启动服务（使用PM2）

#### 第三步：创建CRM前端站点

1. 点击"网站" → "添加站点"
2. 填写信息：
   ```
   域名: crm.yourdomain.com
   根目录: /www/wwwroot/crm-system
   PHP版本: 纯静态
   ```
3. 上传代码：
   ```bash
   cd /www/wwwroot/crm-system
   git clone https://github.com/shushuhao01/crm-system.git .
   npm install
   npm run build
   # 将 dist/ 目录内容复制到站点根目录
   ```
4. 配置反向代理：
   - 点击站点设置 → "反向代理"
   - 添加代理：
     ```
     代理名称: API
     目标URL: http://127.0.0.1:3000
     发送域名: $host
     ```

#### 第四步：创建官网站点

1. 点击"网站" → "添加站点"
2. 填写信息：
   ```
   域名: www.yourdomain.com
   根目录: /www/wwwroot/website
   PHP版本: 纯静态
   ```
3. 上传代码：
   ```bash
   cd /www/wwwroot/website
   git clone https://github.com/shushuhao01/crm-website.git .
   npm install
   npm run build
   ```
4. 配置反向代理（同上）

#### 第五步：创建Admin后台站点

1. 点击"网站" → "添加站点"
2. 填写信息：
   ```
   域名: admin.yourdomain.com
   根目录: /www/wwwroot/admin
   PHP版本: 纯静态
   ```
3. 上传代码：
   ```bash
   cd /www/wwwroot/admin
   git clone https://github.com/shushuhao01/crm-admin.git .
   npm install
   npm run build
   ```
4. 配置反向代理（同上）

#### 第六步：配置SSL证书

为每个域名配置SSL：
1. 点击站点设置 → "SSL"
2. 选择"Let's Encrypt"
3. 勾选域名
4. 点击"申请"

## 常见问题

### Q1: 多个站点能共用一个后端服务吗？

**A**: 可以！

```javascript
// 后端API可以同时服务多个前端
// CRM前端、官网、Admin都调用同一个API
app.get('/api/customers', (req, res) => {
  // 处理请求
});
```

### Q2: 多个站点能共用一个数据库吗？

**A**: 可以！

```
所有站点连接同一个数据库：
- CRM系统读写 customers 表
- 官网读取 products 表
- Admin管理所有表
```

### Q3: 如何保证数据安全？

**A**: 
1. 使用不同的域名
2. 配置CORS限制
3. 使用JWT认证
4. Admin后台增加IP白名单

### Q4: 客户私有部署后，我还能访问他们的数据吗？

**A**: 不能！客户的数据在他们自己的服务器上，完全隔离。

### Q5: 如何更新客户的系统？

**A**: 
```bash
# 客户在他们的服务器上执行
cd /www/wwwroot/crm-system
git pull origin main
npm install
npm run build
pm2 restart crm-api
```
