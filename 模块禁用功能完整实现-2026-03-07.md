# 模块禁用功能完整实现

## 完成时间
2026-03-07

## 问题回顾
在Admin后台禁用了模块,但CRM主应用刷新后这些模块的菜单仍然显示。

## 根本原因
1. CRM前端有`moduleStatusService`服务,会调用后端API `/system/modules/status`获取启用的模块列表
2. 后端缺少这个API接口
3. 当API调用失败时,CRM前端返回所有模块的默认列表,导致无论Admin后台如何设置,CRM前端都显示所有菜单

## 解决方案

### 1. 添加后端API接口

在`backend/src/routes/system.ts`中添加了`/modules/status`接口:

```typescript
/**
 * @route GET /api/v1/system/modules/status
 * @desc 获取启用的模块列表（供CRM前端控制菜单显示）
 * @access Private (All authenticated users)
 */
router.get('/modules/status', authenticateToken, async (_req: Request, res: Response) => {
  try {
    const moduleConfigRepository = AppDataSource.getRepository(ModuleConfig);
    
    // 获取所有启用的模块
    const enabledModules = await moduleConfigRepository.find({
      where: { is_enabled: true },
      select: ['code']
    });
    
    // 模块代码映射：Admin后台模块代码 -> CRM前端菜单ID
    const moduleMapping: Record<string, string> = {
      'order_management': 'order',
      'customer_management': 'customer',
      'finance_management': 'finance',
      'logistics_management': 'logistics',
      'aftersales_management': 'service',
      'call_management': 'serviceManagement',
      'data_management': 'data',
      'performance_management': 'performance',
      'product_management': 'product',
      'system_management': 'system'
    };
    
    // 转换为CRM前端的菜单ID
    const menuIds = enabledModules
      .map(m => moduleMapping[m.code])
      .filter(id => id !== undefined);
    
    // 始终包含dashboard（数据看板）
    if (!menuIds.includes('dashboard')) {
      menuIds.unshift('dashboard');
    }
    
    res.json({
      success: true,
      data: {
        enabledModules: menuIds
      }
    });
  } catch (error: any) {
    console.error('[System] 获取模块状态失败:', error);
    res.status(500).json({
      success: false,
      message: '获取模块状态失败',
      error: error.message
    });
  }
});
```

### 2. 模块代码映射关系

| Admin后台模块代码 | CRM前端菜单ID | 模块名称 |
|------------------|--------------|---------|
| order_management | order | 订单管理 |
| customer_management | customer | 客户管理 |
| finance_management | finance | 财务管理 |
| logistics_management | logistics | 物流管理 |
| aftersales_management | service | 售后管理 |
| call_management | serviceManagement | 通话管理 |
| data_management | data | 资料管理 |
| performance_management | performance | 业绩统计 |
| product_management | product | 商品管理 |
| system_management | system | 系统管理 |

### 3. 特殊处理
- **数据看板(dashboard)**：始终显示,不受模块控制
- **系统管理(system)**：标记为系统模块,不可禁用

## 工作流程

### Admin后台操作
1. 管理员登录Admin后台
2. 进入"模块服务管理"页面
3. 禁用某个模块(如财务管理)
4. 数据库中`module_configs`表的`is_enabled`字段更新为0

### CRM前端获取
1. CRM前端的`moduleStatusService`调用`/api/v1/system/modules/status`
2. 后端查询数据库,获取所有`is_enabled=1`的模块
3. 将模块代码转换为菜单ID
4. 返回启用的菜单ID列表
5. CRM前端根据列表显示/隐藏菜单

### 缓存机制
- CRM前端缓存模块状态5分钟
- 5分钟后自动刷新
- 可以手动调用`moduleStatusService.refresh()`强制刷新

## 使用说明

### Admin后台管理员
1. 登录Admin后台: http://localhost:5174
2. 进入"出租客户管理" -> "模块服务管理"
3. 点击模块卡片的开关按钮启用/禁用模块
4. 系统模块(系统管理)不可禁用

### CRM用户
1. 登录CRM系统: http://localhost:5173
2. 刷新页面或等待5分钟缓存过期
3. 被禁用的模块菜单将自动隐藏
4. 已有的功能页面如果直接访问URL会显示404或无权限

## 注意事项

1. **缓存延迟**：CRM前端有5分钟缓存,禁用模块后可能需要等待或刷新页面
2. **数据看板**：始终显示,不受模块控制
3. **系统模块**：系统管理模块不可禁用
4. **权限控制**：模块禁用只是隐藏菜单,后端API仍需要权限控制
5. **已打开页面**：用户已经打开的页面不会自动关闭,需要刷新
