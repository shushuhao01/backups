# 生产环境紧急回退方案

## 问题分析
- e740cc0 版本时还没有 PaymentController.ts 文件，所以能正常构建
- 后续版本添加了 Admin 后台支付功能，引入了 PaymentController.ts
- 该文件存在 TypeScript 类型错误，导致构建失败

## 解决方案

### 方案1：回退到稳定版本（最快）

```bash
cd /www/wwwroot/abc789.cn

# 回退到e740cc0（最后一个稳定版本）
git reset --hard e740cc0

# 重新构建
cd backend
npm run build

# 重启服务
pm2 restart abc789.cn-backend
```

### 方案2：拉取最新修复（推荐）

```bash
cd /www/wwwroot/abc789.cn

# 强制拉取最新代码（包含修复）
git fetch origin
git reset --hard origin/main

# 重新构建
cd backend
npm run build

# 重启服务
pm2 restart abc789.cn-backend
```

### 方案3：手动修复当前版本

如果方案2失败，按照之前的《生产环境紧急修复指南.md》手动修复 PaymentController.ts 文件。

## 推荐执行顺序

1. **先尝试方案2**（拉取最新修复）
2. 如果方案2失败，使用**方案1**（回退到稳定版本）
3. 如果需要Admin后台功能，再使用**方案3**（手动修复）

## 验证

修复后检查：
```bash
pm2 status
pm2 logs abc789.cn-backend --lines 20
```

看到服务正常运行即可。
