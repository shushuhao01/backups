# 🏗️ 云客CRM系统 — 全项目深度分析报告

> 分析日期：2026年3月8日
> 分析范围：Backend后端 + CRM主系统 + Admin管理后台 + Website官网 + crmAPP移动端

## 一、项目全景

这是一个 **SaaS + 私有部署双模式** 的完整CRM产品体系，由5个子项目组成：

| 子项目 | 技术栈 | 代码规模 | 端口 | 定位 |
|--------|--------|---------|------|------|
| **Backend** | Node.js + Express + TypeORM + MySQL | ~60,000行 / 239个TS文件 | 3000 | 统一后端API |
| **CRM前端** | Vue 3 + TS + Element Plus + Pinia | ~9.5MB / 402个文件 | 5173 | 主业务系统 |
| **Admin后台** | Vue 3 + TS + Element Plus + Axios | ~27个视图文件 | 5174 | 平台管理后台 |
| **Website官网** | Vue 3 + TS + 纯手写CSS | ~6,800行 / 14个文件 | 8080 | 产品展示+注册 |
| **crmAPP** | uni-app + Vue 3 + TS | 14个页面 + 3个服务 | — | 电销手机助手 |

### 3.2 架构模式
- **整体架构**：单体应用（Monolith），但具备 SaaS 多租户能力的雏形
- **分层模式**：松散的 Controller → Route → Service → Repository 分层
- **部署模式**：支持双模式 — 私有部署和SaaS云端版，通过环境变量切换
- **API风格**：RESTful，统一前缀 `/api/v1`

### 3.6 认证和权限系统

1. **CRM用户认证**：JWT Bearer Token，Access Token(7天) + Refresh Token(30天)，登录失败5次锁定
2. **Admin后台认证**：独立JWT体系，Token中含 isAdmin 标志
3. **API密钥认证**：用于第三方API调用，支持IP白名单/速率限制/过期
4. **权限模型**：superadmin > admin > manager/department_manager > 普通员工

### 3.7 多租户实现（未完成）

- 采用 **共享数据库 + tenant_id字段** 方案
- 已实现：BaseRepository（泛型仓储）、AsyncLocalStorage上下文、tenantAuth中间件框架
- **未完成**：中间件核心验证被注释掉，大量实体缺少tenant_id字段，业务代码未使用BaseRepository
