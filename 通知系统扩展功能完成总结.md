## 一、本次新增功能

### 1.1 支付提醒服务 ✅

**文件**: `backend/src/services/PaymentReminderService.ts`

**功能**:
1. 检测24小时前创建但未支付的订单
2. 自动发送支付提醒通知
3. 自动关闭48小时超时订单
4. 防重复发送机制

**业务规则**:
- 订单创建后24小时未支付 → 发送提醒
- 订单创建后48小时未支付 → 自动关闭
- 每个订单每天最多发送1次提醒

**通知模板**: `payment_pending`

**代码示例**:
```typescript
// 发送支付提醒
await notificationTemplateService.sendByTemplate('payment_pending', {
  tenantName: order.tenant_name,
  orderNumber: order.order_no,
  packageName: order.package_name,
  amount: order.amount.toFixed(2),
  createTime: createTime
}, {
  to: order.email || order.phone,
  priority: 'normal',
  relatedId: order.id,
  relatedType: 'payment_order'
});
```

### 1.2 定时任务调度更新 ✅

**文件**: `backend/src/services/SchedulerService.ts`

**新增任务**:
- 支付提醒检查任务 - 每天早上10点执行

**任务列表**:

| license-expiration-check | 每天9:00 | 授权到期检查 |

## 二、功能架构

### 2.1 支付提醒流程

```
定时任务(每天10:00)
    ↓
查询24小时前创建的待支付订单
    ↓
检查今天是否已发送提醒
    ↓
否 → 发送支付提醒通知
    ↓
记录发送日志
    ↓
查询48小时前创建的待支付订单
    ↓
自动关闭超时订单
```

### 2.2 防重复机制

```typescript
// 检查今天是否已发送过提醒
const logs = await AppDataSource.query(`
  SELECT id FROM notification_logs
  WHERE template_code = 'payment_pending'
    AND related_id = ?
    AND DATE(created_at) = ?
`, [order.id, today]);

if (logs.length > 0) {
  console.log('今天已发送过提醒,跳过');
  return false;
}
```

| 任务执行时间 | 日志分析 | <30秒 |

### 7.2 监控SQL

```sql
-- 待支付订单统计
SELECT 
  COUNT(*) as total,
  COUNT(CASE WHEN created_at < DATE_SUB(NOW(), INTERVAL 24 HOUR) THEN 1 END) as over_24h,
  COUNT(CASE WHEN created_at < DATE_SUB(NOW(), INTERVAL 48 HOUR) THEN 1 END) as over_48h
FROM payment_orders
WHERE status = 'pending';

-- 今日支付提醒统计
SELECT 
  COUNT(*) as total,
  SUM(CASE WHEN status = 'success' THEN 1 ELSE 0 END) as success,
  SUM(CASE WHEN status = 'failed' THEN 1 ELSE 0 END) as failed
FROM notification_logs
WHERE template_code = 'payment_pending'
  AND DATE(created_at) = CURDATE();

-- 超时订单关闭统计
SELECT 
  DATE(updated_at) as date,
  COUNT(*) as closed_count
FROM payment_orders
WHERE status = 'closed'
  AND updated_at >= DATE_SUB(NOW(), INTERVAL 7 DAY)
GROUP BY DATE(updated_at)
ORDER BY date DESC;
```

## 十、常见问题

### Q1: 如何修改提醒时间?

A: 修改 `SchedulerService.ts` 中的 cron 表达式:
```typescript
// 改为每天下午2点执行
'0 14 * * *'
```

### Q2: 如何修改订单超时时间?

A: 修改 `PaymentReminderService.ts` 中的查询条件:
```typescript
// 改为72小时
WHERE po.created_at < DATE_SUB(NOW(), INTERVAL 72 HOUR)
```

### Q3: 如何测试定时任务?

A: 三种方式:
1. 运行测试脚本: `node backend/test-payment-reminder.js`
2. 手动触发API: `POST /api/v1/admin/scheduler/tasks/payment-reminder-check/trigger`
3. 修改cron表达式为更频繁的时间(如每分钟)

### Q4: 如何查看发送记录?
