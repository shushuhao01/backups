# 生产环境紧急回退操作指南

## 问题描述

GitHub 仓库已经回退到 c2e9b1a 版本，但是生产服务器上的代码没有同步回退，导致：
- 服务器上仍然有 Admin 后台的代码文件
- 构建时出现 TypeScript 编译错误
- 服务无法正常启动

## 解决方案

### 方案一：使用自动脚本（推荐）

1. 上传脚本到服务器：
```bash
# 将 生产环境紧急回退脚本.sh 上传到服务器
scp 生产环境紧急回退脚本.sh root@your-server:/root/
```

2. 执行脚本：
```bash
ssh root@your-server
cd /root
chmod +x 生产环境紧急回退脚本.sh
./生产环境紧急回退脚本.sh
```

### 方案二：手动执行（如果脚本有问题）

登录宝塔面板或 SSH，执行以下命令：

```bash
# 1. 进入后端目录
cd /www/wwwroot/abc789.cn/backend

# 2. 查看当前版本
git log --oneline -1

# 3. 停止服务
pm2 stop crm-backend-api

# 4. 强制拉取远程代码（这是关键！）
git fetch origin
git reset --hard origin/main

# 5. 验证版本
git log --oneline -1
# 应该显示: c2e9b1a (HEAD -> main, origin/main) fix: CRM系统优化 - 修复4个关键问题

# 6. 检查问题文件是否还存在
ls -la src/routes/public/payment.ts
# 应该显示: No such file or directory

ls -la src/controllers/admin/PaymentController.ts
# 应该显示: No such file or directory

# 7. 清除构建缓存
rm -rf dist
rm -rf node_modules/.cache

# 8. 重新构建
npm run build

# 9. 如果构建成功，启动服务
pm2 start crm-backend-api
pm2 save

# 10. 查看服务状态
pm2 status
pm2 logs crm-backend-api --lines 50
```

## 关键命令解释

### `git reset --hard origin/main`
这个命令会：
- 强制将本地代码重置为远程 origin/main 的状态
- 丢弃所有本地修改
- 确保本地和远程完全一致

### 为什么之前 `git pull` 不够？
- `git pull` 只会拉取新的提交，不会删除本地多余的文件
- 如果本地有未跟踪的文件（untracked files），`git pull` 不会删除它们
- `git reset --hard` 会强制重置到指定状态

## 验证要点

构建成功后，应该：

1. ✅ 没有 TypeScript 编译错误
2. ✅ 没有 `src/routes/public/payment.ts` 文件
3. ✅ 没有 `src/controllers/admin/PaymentController.ts` 文件
4. ✅ 没有 `src/services/PaymentService.ts` 文件
5. ✅ CRM 系统功能正常

## 如果还是失败

如果执行 `git reset --hard origin/main` 后还是有问题文件，说明：

1. GitHub 仓库可能没有真正回退成功
2. 需要在本地再次确认并强制推送

在本地执行：
```bash
# 确认本地版本
git log --oneline -1

# 如果不是 c2e9b1a，再次回退
git reset --hard c2e9b1a

# 强制推送
git push origin main --force

# 然后在服务器上再次执行
git fetch origin
git reset --hard origin/main
```

## 备份说明

脚本会自动备份当前状态到：
```
/www/backup/backend-YYYYMMDD-HHMMSS/
```

如果回退后有问题，可以从备份恢复。
