## 🔧 后端服务问题诊断

### 错误信息
```
Error: Access denied for user 'abc789'@'localhost' (using password: YES)
code: 'ER_ACCESS_DENIED_ERROR'
errno: 1045
sqlState: '28000'
```

### 可能原因

1. **数据库用户不存在或密码错误**
   - 用户名: `abc789`
   - 数据库: `crm_local`

2. **MySQL服务未启动**

3. **数据库权限不足**

---

## 🚀 解决方案

### 方案1: 检查MySQL服务

```powershell
# 检查MySQL服务状态
Get-Service -Name MySQL* | Select-Object Name, Status

# 如果未运行，启动MySQL服务
Start-Service -Name MySQL80  # 根据实际服务名调整
```

### 方案2: 创建数据库用户

```sql
-- 连接到MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE IF NOT EXISTS crm_local CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户并授权
CREATE USER IF NOT EXISTS 'abc789'@'localhost' IDENTIFIED BY 'YtZWJPF2bpsCscHX';
GRANT ALL PRIVILEGES ON crm_local.* TO 'abc789'@'localhost';
FLUSH PRIVILEGES;

-- 验证
SHOW GRANTS FOR 'abc789'@'localhost';
```

### 方案3: 使用root用户（临时方案）

修改 `backend/.env.local`:
```env
DB_USERNAME=root
DB_PASSWORD=你的root密码
```

### 方案4: 测试数据库连接

```powershell
# 在backend目录运行
node -e "const mysql = require('mysql2/promise'); require('dotenv').config({ path: '.env.local' }); (async () => { try { const conn = await mysql.createConnection({ host: process.env.DB_HOST, port: process.env.DB_PORT, user: process.env.DB_USERNAME, password: process.env.DB_PASSWORD, database: process.env.DB_DATABASE }); console.log('✅ 数据库连接成功'); await conn.end(); } catch (error) { console.error('❌ 数据库连接失败:', error.message); } })()"
```
