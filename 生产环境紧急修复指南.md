# 生产环境紧急修复指南 - PaymentController TypeScript错误

## 问题描述
PaymentController.ts 中的 updateOrderStatus 方法调用参数类型不完整，导致TypeScript编译失败。

## 修复步骤

### 方法1：直接在生产服务器上修改（推荐）

1. **SSH登录生产服务器**
```bash
ssh root@your-server-ip
```

2. **进入项目目录**
```bash
cd /www/wwwroot/abc789.cn/backend
```

3. **备份原文件**
```bash
cp src/controllers/admin/PaymentController.ts src/controllers/admin/PaymentController.ts.backup
```

4. **编辑文件**
```bash
vi src/controllers/admin/PaymentController.ts
```

5. **修复第531-537行（approveTransfer方法）**

找到这段代码（大约在第531行）：
```typescript
// 更新订单状态为已支付
await paymentService.updateOrderStatus(id, 'paid', {
  paidAt: new Date()
});
```

替换为：
```typescript
// 更新订单状态为已支付
await paymentService.updateOrderStatus(id, 'paid', {
  paidAt: new Date(),
  tradeNo: undefined,
  refundAmount: undefined,
  refundAt: undefined,
  refundReason: undefined
});
```

6. **修复第583-589行（rejectTransfer方法）**

找到这段代码（大约在第583行）：
```typescript
// 更新订单状态为审核拒绝
await paymentService.updateOrderStatus(id, 'review_rejected', {});
```

替换为：
```typescript
// 更新订单状态为审核拒绝
await paymentService.updateOrderStatus(id, 'review_rejected', {
  tradeNo: undefined,
  paidAt: undefined,
  refundAmount: undefined,
  refundAt: undefined,
  refundReason: undefined
});
```

7. **保存并退出vi**
```
按 ESC
输入 :wq
按 Enter
```

8. **重新构建**
```bash
npm run build
```

9. **重启服务**
```bash
pm2 restart abc789.cn-backend
```
