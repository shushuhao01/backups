# 管理员账号登录问题 - 完整解决方案

## 问题描述

创建租户/私有客户后，系统返回了默认管理员账号和密码，但是登录时提示"账号密码错误"。

## 根本原因

1. **密码加密方式不一致**：
   - 系统原有用户使用 `bcrypt` 加密
   - 新创建的管理员账号最初使用 `pbkdf2` 加密
   - 登录验证只支持 `bcrypt.compare`，导致 pbkdf2 加密的密码无法验证

2. **账号被锁定**：
   - 由于密码验证失败，登录失败次数累积
   - 失败5次后账号自动锁定（status = 'locked'）

3. **用户创建失败**：
   - 修改为 bcrypt 后，由于 `hashPassword` 改为异步函数
   - 但调用时没有使用 `await`，导致用户创建失败但没有抛出异常

## 解决方案

### 步骤1：修复 adminAccountHelper.ts

已将密码加密方式从 pbkdf2 改为 bcrypt，与系统一致：

```typescript
// 旧代码（pbkdf2）
export const hashPassword = (password: string): { hash: string; salt: string } => {
  const salt = crypto.randomBytes(16).toString('hex');
  const hash = crypto.pbkdf2Sync(password, salt, 1000, 64, 'sha512').toString('hex');
  return { hash, salt };
};

// 新代码（bcrypt）
export const hashPassword = async (password: string): Promise<string> => {
  const salt = await bcrypt.genSalt(12);
  const hash = await bcrypt.hash(password, salt);
  return hash;
};
```

### 步骤2：修复租户创建路由

需要在调用 `createDefaultAdmin` 时使用 `await`：

```typescript
// 修复前
const adminAccount = await createDefaultAdmin({...});

// 修复后（确保 hashPassword 被正确 await）
const adminAccount = await createDefaultAdmin({...});
```

### 步骤3：解锁被锁定的账号

```sql
UPDATE users 
SET status = 'active', login_fail_count = 0, locked_at = NULL
WHERE status = 'locked';
```

### 步骤4：重置旧用户密码（使用 pbkdf2 的用户）

对于已经使用 pbkdf2 加密的用户，需要重置密码为 bcrypt 加密：

```sql
-- 查找使用 pbkdf2 的用户（有 salt 字段且不为空）
SELECT id, username, phone, real_name 
FROM users 
WHERE salt IS NOT NULL AND salt != '';

-- 需要手动重置这些用户的密码
```
